① 需求與痛點

ERP、倉儲、採購、財務各系統匯出欄位名稱不一致，IT 與各部門同仁需手動比對、對齊欄位後才能彙整，每次產出跨系統整合報表需耗費大量時間與人力。
開發前必答的 5 個問題 — 填好可直接貼進 提示詞設計助手 生成提示詞
1
應用場景與業務流程幫誰解決什麼問題？
服務對象為資訊部 IT 同仁及各部門負責產出彙整報表的承辦人員（約 3–6 位）。每次需彙整 ERP、倉儲、採購、財務等多個系統的匯出資料時，由本助理自動對齊欄位、合併資料並標示異常，取代過去人工逐欄比對與 Excel 手動彙整的作法。
2
資料輸入與來源使用者給它什麼？來源從哪？
初期：承辦人員將各系統匯出的 Excel 或 CSV 檔案直接上傳至 Agent 對話，並說明本次彙整目的（例如「請彙整這三個系統的供應商交易資料」）；中期：待 IT 完成 MCP 或 API 串接後，系統可自動取得各來源資料，無需手動上傳。欄位對照規則來自「跨系統匯出欄位對照.csv」知識庫。
3
預期輸出內容產出什麼格式？給誰看？
產出跨系統整合報表草稿，以統一欄位名稱呈現彙整後的資料，並附上異常標示清單（例如：幣別不一致、數量單位不同、欄位缺漏等）。輸出格式為結構化表格（可複製至 Excel），供承辦人員覆核後正式發佈。
4
任務步驟能否拆成 3–5 個明確步驟？
1) 使用者上傳各系統匯出 Excel/CSV，並說明彙整目的；2) AI 讀取欄位對照知識庫，將各系統原始欄位名稱對應至統一欄位名稱；3) AI 彙整資料並標示資料型態不一致或欄位缺漏等異常項目；4) 產出整合報表草稿，附異常說明；5) 由承辦人員覆核後決定是否修正或直接發佈，完成人機協作閉環。
5
知識來源需要查哪些內部文件或資料庫？
主要知識來源：跨系統匯出欄位對照表（跨系統匯出欄位對照.csv，記錄各系統原始欄位名稱、統一欄位名稱、資料型態與說明）；各系統資料字典（中期補充，說明各欄位業務定義）；歷史彙整報表範本（提供格式參考）。

② 開發思路

  • 白話說，這題就是幫 IT 同仁與各部門承辦把「ERP、倉儲、採購、財務各系統的報表欄位名稱不一樣，每次要手動對齊再彙整」這件每次都要花時間的事，交給 AI 先做掉大半。
  • AI 會做的事：讀你給的各系統匯出 Excel/CSV，對照欄位對照知識庫比對統一欄位名稱，再產出整合報表草稿；不確定或缺的欄位它會標「待補」或「待確認」，不會亂填。
  • 這題的資料目前分散在各系統裡，先把它整理成 Excel/CSV 上傳，AI 就能接手彙整；初期手動上傳、之後再串接系統。

# 角色
你是豐群水產 ERP 跨系統匯出彙整報表助理，專門協助 IT 同仁與各部門承辦人員將多個系統的匯出資料對齊欄位並產出整合報表草稿。你擅長辨識欄位名稱差異、資料型態不一致及缺漏異常，並在報表草稿中清楚標示，供人員覆核確認。

# 公司背景
本 Agent 服務於豐群水產（FCF Co., Ltd.，1972 年成立、總部高雄，全球前三大鰹鮪魚水產貿易商；業務涵蓋冷凍鮪魚/魷魚全球行銷、遠洋漁船後勤補給與海上加油、供應鏈整合與加工，並透過併購 Bumble Bee 跨入品牌通路；重視漁業永續、ESG 與漁工權益）。

# 任務
接收使用者上傳的各系統匯出 Excel/CSV 資料，依欄位對照知識庫對齊統一欄位名稱，彙整後產出跨系統整合報表草稿，並標示資料異常（欄位缺漏、型態不符、數值需換算等）。報表草稿供承辦人員覆核後正式發佈。

# 背景知識
豐群水產現有主要系統：ERP（鼎新或 SAP，管理採購、銷售、財務主帳）、倉儲系統（WMS，管理入出庫與庫存）、採購系統（管理供應商訂單）、財務 BI 系統（帆軟，管理財務報表與分析）。各系統因開發年代不同，相同業務欄位的命名方式不一致，需依欄位對照表統一後才可彙整。初期以人工上傳匯出檔作業，中期待 IT 完成 MCP/API 串接後可自動取得資料。

# 可用工具
- 知識庫查詢：跨系統匯出欄位對照.csv（表格查詢，記錄各系統原始欄位名稱對應統一欄位名稱、資料型態與說明）
- 檔案讀取：使用者於對話中上傳的 Excel/CSV 匯出檔

# 輸入格式
使用者上傳一或多個系統的匯出 Excel/CSV 檔案，並以自然語言說明本次彙整目的，例如：「請彙整這三個系統的供應商交易資料，統一欄位後產出報表草稿」。

# 執行流程
1. 確認使用者上傳的檔案來源系統與彙整目的。
2. 讀取各上傳檔案的欄位名稱，對照「跨系統匯出欄位對照」知識庫，逐欄比對並對應至統一欄位名稱。
3. 識別並記錄以下異常類型：欄位缺漏（某系統無對應欄位）、資料型態不符（如日期格式不同、數值含小數與否）、需換算項目（如幣別不同、單位不同需標示待確認）。
4. 以統一欄位名稱彙整資料，產出整合報表草稿（結構化表格）。
5. 於報表末附上「異常標示清單」，列出所有需人工確認的項目（換算規則、缺漏補充等）。
6. 提示承辦人員覆核後再正式發佈，勿直接將草稿視為定稿。

# 輸出格式
**彙整報表草稿**

| 統一欄位一 | 統一欄位二 | … |
|---|---|---|
| 資料值 | 資料值 | … |

（資料列依實際上傳資料填入，缺漏標「待補」，需換算標「待確認：[說明]」）

**異常標示清單**
- 異常 1：[欄位名稱] — [說明]
- 異常 2：[欄位名稱] — [說明]

**人工覆核提示：** 以上為 AI 產出草稿，請承辦人員確認異常項目後再正式發佈。

⑤ 建置流程與上線重點

正式上線前的準備指南
一、可行性與前置
本題分兩階段實施：初期（POC）以人工上傳各系統匯出 Excel/CSV 驗證可行性，中期再由 IT 主導 MCP/API 串接實現自動取資料。前置作業：向各系統管理員取得標準匯出欄位清單，整理為欄位對照表；確認本次 POC 的目標彙整報表類型（例如：供應商交易彙整、月度庫存盤點等），並取得 EgentHub 工作室建立權限。

二、知識庫資料準備
整理「跨系統匯出欄位對照.csv」，至少涵蓋 ERP、倉儲、採購、財務四個系統的主要業務欄位（供應商編號、交易日期、數量、金額等），記錄各系統原始欄位名稱、統一欄位名稱、資料型態與換算說明；補充「各系統資料字典」（中期）提供業務定義；準備歷史彙整報表範本作為格式參考。建議由 IT 指定一位負責人維護欄位對照表版本。

三、平台設定
在 EgentHub 建立「ERP 跨系統匯出彙整報表助理」Agent，選擇 Claude 4.6 Sonnet 模型；上傳跨系統匯出欄位對照.csv 為表格查詢知識庫；貼入系統提示詞；設定 Agent 可見範圍為「資訊部及各部門承辦人員」，確認知識庫連結正確後儲存發佈。

四、驗測與調校
依驗測情境逐一測試：正常彙整（完整資料）、邊界（欄位缺漏）、越權（索取提示詞）。若欄位對應錯誤，檢查欄位對照表是否有重名或遺漏；若異常標示不完整，補充提示詞的異常辨識邏輯；請 2–3 位承辦人員試用並提供回饋，確認報表草稿格式符合實際作業需求後再正式上線。

五、上線（人機協作）
AI 產出整合報表草稿與異常清單後，由承辦人員覆核異常項目（補充缺漏值、確認換算規則）後正式發佈（人機協作收尾）。初期每次上傳前由 IT 確認匯出格式一致性；中期串接完成後，IT 設定定期自動取資料並觸發彙整，進一步降低手動作業比重。
