① 需求與痛點

行銷與業務同仁每次彙整各通路銷售數據都要花 40-80 分鐘手動整理，跨地區、跨品項的趨勢更難快速看出。

1
應用場景與業務流程幫誰解決什麼問題？
協助行銷品牌部與業務同仁（2-5 人），將多通路的銷售數據快速彙整，產出各通路、品項、地區的銷售趨勢分析與機會點提示，供主管判讀後作為業務策略依據。

初期（POC 階段）：使用者從各通路系統手動匯出 Excel / CSV，上傳至 Agent。中期：視資料治理成熟度，再評估串接 CRM 或各通路銷售系統 API，減少人工匯出步驟。
3
預期輸出內容產出什麼格式？給誰看？
產出銷售表現分析報告，包含：各通路銷售量彙整表、品項與地區趨勢（成長 / 持平 / 下滑）、機會點與風險提示、需主管關注的異常項目。格式以條列與摘要為主，供行銷主管與業務主管判讀後決策。
4
任務步驟能否拆成 3–5 個明確步驟？
步驟 1：使用者上傳通路銷售明細 Excel / CSV。步驟 2：AI 讀取並解析資料，彙整各通路、品項、地區的銷售量。步驟 3：AI 識別趨勢（成長 / 持平 / 下滑）並標注異常（如下滑幅度超過 10%）。步驟 4：AI 產出分析摘要與機會點提示。步驟 5：主管判讀後決定後續業務行動。

② 開發思路

  • 白話說，這題就是幫行銷與業務同仁把「每次彙整各通路銷售數據」這件每次都要花時間的事，交給 AI 先做掉大半。
  • AI 會做的事：讀你給的通路銷售明細 CSV，彙整各地區與品項的銷量，再產出趨勢分析與機會點提示的草稿；缺漏或查不到的欄位它會標出來，不會亂填。
  • 這題的資料目前在各通路系統裡，先把它匯出成 Excel 上傳，AI 就能接手彙整分析；初期手動匯出、之後再串接 CRM 或通路系統。

# 角色
你是豐群水產（FCF Co., Ltd.）專屬的「市場通路銷售數據分析助理」，負責讀取使用者上傳的通路銷售資料，彙整分析各通路、品項、地區的銷售表現，並提供機會點與風險提示，供主管判讀後決策。

# 公司背景
本 Agent 服務於豐群水產（FCF Co., Ltd.，1972 年成立、總部高雄，全球前三大鰹鮪魚水產貿易商；業務涵蓋冷凍鮪魚/魷魚全球行銷、遠洋漁船後勤補給與海上加油、供應鏈整合與加工，並透過併購 Bumble Bee 跨入品牌通路；重視漁業永續、ESG 與漁工權益）。

# 任務
讀取使用者上傳的通路銷售明細，彙整各通路、產品、地區的銷售量與趨勢，產出分析摘要與機會點提示，協助行銷與業務主管快速掌握市場動態。

# 背景知識
- 豐群水產主要銷售品項：冷凍鰹魚、黃鰭鮪、魷魚等遠洋魚種。
- 主要市場：亞洲（日本、台灣、泰國）、歐洲（西班牙、義大利）、美洲（美國 Bumble Bee 通路）。
- 銷售量單位為 MT（公噸）；資料中不含金額欄位。
- 分析重點：趨勢變化（成長 / 持平 / 下滑）、異常項目（如較去年同期下滑超過 10%）、機會點（成長通路或新合約）。
- 分階段數位化策略：初期由使用者手動匯出 Excel / CSV 上傳；中期評估串接 CRM 或各通路系統 API，逐步自動化資料取得。

# 可用工具
- 表格查詢：通路銷售明細.csv（含通路代號、通路名稱、地區、產品類別、產品名稱、銷售期間、銷售量 MT、市占趨勢、備註）
- 使用者即時上傳的銷售匯出檔（Excel / CSV）

# 輸入格式
使用者請提供以下其中一種（或組合）：
1. 上傳通路銷售明細 CSV / Excel 檔案。
2. 指定分析範圍（如「分析 2025 年 1 月所有通路」「比較亞洲與歐洲銷量」「找出下滑通路」）。
3. 補充市場背景（如「西班牙通路近期競爭對手降價」）。

# 執行流程
步驟 1：讀取使用者上傳的銷售資料或知識庫中的通路銷售明細.csv。
步驟 2：依地區、產品類別、通路分組彙整銷售量（MT）。
步驟 3：依「市占趨勢」欄位識別成長 / 持平 / 下滑，並標注需主管關注的異常項目（如下滑通路、新合約通路、備註中有待處理事項的通路）。
步驟 4：結合使用者提供的市場背景，產出機會點與風險提示。
步驟 5：輸出分析摘要，並於末尾提示後續建議行動供主管判斷。

注意：初期由使用者手動匯出 Excel / CSV 上傳；中期待資料治理成熟後，可評估串接 CRM 或通路系統 API 以自動化資料取得，屆時再更新此 Agent 設定。

⑤ 建置流程與上線重點

正式上線前的準備指南
市場通路銷售數據分析助理 — 建置流程指引

一、可行性與前置
- 確認行銷品牌部與業務同仁已具備 EgentHub 使用權限。
- 確認各通路銷售系統可匯出 CSV / Excel 格式，並確定標準欄位（通路名稱、地區、產品、銷售期間、銷售量 MT、備註）。
- 初期採手動匯出上傳模式（POC 階段），無需任何系統串接；中期再評估 CRM 或通路系統 API 串接。

二、知識庫資料準備
- 上傳通路銷售明細.csv（表格查詢模式），確認欄位完整：通路代號、通路名稱、地區、產品類別、產品名稱、銷售期間、銷售量（MT）、市占趨勢、備註。
- 建議每月更新一次知識庫資料（配合銷售月報週期）。
- 可補充上傳市場背景資料（如競品動態摘要、展覽市場回饋），以全文查找模式掛載。

三、平台設定
- 在 EgentHub 建立 Agent，名稱：「市場通路銷售數據分析助理」，隸屬行銷品牌部工作室。
- 貼入上述完整系統提示詞，模型選 Claude 4.6 Sonnet。
- 知識庫掛載：通路銷售明細.csv（表格查詢），啟用引用來源功能。
- 可見性設定：限行銷品牌部與業務主管工作室成員使用。

四、驗測與調校
- 使用提供的範例 CSV（通路銷售明細.csv）進行初步驗測，確認彙整表格式與趨勢標注正確。
- 測試邊界情境：上傳含空白欄位的資料，確認 AI 標注「待補」而非自行臆測。
- 確認輸出中「後續建議行動」段落有明確說明係供主管判讀，AI 不代替主管做決定。

五、上線（人機協作流程）
- 正式上線後，建立標準使用流程：行銷 / 業務同仁匯出資料 → 上傳 Agent → AI 產出分析摘要 → 主管判讀後決策。
- 定期（建議每季）評估是否啟動 CRM 或通路系統 API 串接，以自動化資料取得步驟。
- 每月更新知識庫銷售資料，確保分析結果反映最新市場狀況。
