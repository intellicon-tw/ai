喬訊電子 AI Agent 開發實務工作坊

由各部門任務盤點轉化的 POC 解題示範（多部門教學網）

共 5 個部門合計 11 款 Agent POC點任一題 → 痛點→思路→提示詞→知識庫→建置→驗測

本教學網依喬訊 0518 訪談與 42 項任務盤點，挑出 6 個帶頭部門的旗艦 POC 做成解題示範。點進各部門可看到每一題的：需求與痛點 → 開發思路 → 完整提示詞 → 知識庫設計 → 建置流程 → 驗測情境。所有提示詞毫無保留，目標是讓喬訊種子團隊能自主開發。

A

業務部

A 級3 題 POC部門總覽 →

業務部是喬訊最適合帶頭的部門之一：全球外銷、英文詢價與報價量大、重複性高、多人共做，完全符合 Turbo 法則。盤點場景具體、輸入輸出清楚，可直接開發。建議補上近半年實際英文回覆樣本、最新報價價格表（ERP 匯出）與完整料號規格庫，Agent 就能立即見效。

  • 1.1國際客戶英文詢價回覆助理內容生成型（英文回覆草稿）
  • 1.2報價單建立助理標準化作業型（報價單生成）
  • 1.3連接器選型推薦助理知識問答型（選型推薦）
A

進出口・報關

A 級2 題 POC部門總覽 →

進出口/報關是喬訊文件密集度最高的環節之一：全球出貨，Invoice/PL/報關文件量大、報關單 PDF 逐欄人工建檔最容易出錯。這類『讀文件→擷取→建檔』正是 AI 最擅長的補強場景。建議提供真實報關單樣張（去識別）、常用 HS code 與稅率對照、出貨主檔欄位定義。

  • 2.1進口報關單 PDF 辨識與建檔助理審查比對型（OCR 辨識＋稅則比對）
  • 2.2出口文件生成助理（Invoice/PL）標準化作業型（出口文件生成）
A

品保部

A 級2 題 POC部門總覽 →

品保是連接器廠的價值核心，COA/檢驗報告與客訴 8D 都是高頻、高重複、需一致性的文件工作，非常適合做 Agent。盤點具體、可直接開發。建議補齊各料號完整檢測項目與允收基準、近 2-3 年實際 8D 報告（去識別），可靠度測試數據格式統一後，Agent 能再升級到數據解讀。

  • 3.1檢驗報告 COA 生成助理標準化作業型（COA 生成＋判定）
  • 3.2客訴 8D 草稿助理內容生成型（8D 報告草稿）
A

研發工程部

A 級3 題 POC部門總覽 →

研發工程部承載喬訊最高價值的知識：圖面規格擷取、新舊圖面比對能立即省時，打樣/試模經驗庫更是 ROF 旗艦傳承題。視覺類題目盤點清楚、可直接開發。建議提供脫敏客戶圖面樣張、ECN 變更紀錄、以及由資深工程師持續萃取的試模問題案例。

  • 4.1客戶圖面規格擷取助理審查比對型（視覺規格擷取）
  • 4.2新舊圖面差異比對助理審查比對型（視覺差異比對）
  • 4.3打樣問題知識助理（經驗傳承）知識問答型（ROF 經驗傳承）
A

生管 PMC

A 級1 題 POC部門總覽 →

生管 PMC 是副總『跨系統資料分析』願景的起點：ERP 訂單與 BOM 核對目前約 4hr/次，正是補強 ERP 沒覆蓋的高價值場景。盤點具體、可直接開發。初期以 ERP 匯出 Excel 過渡即可見效；建議提供訂單表/BOM 表欄位定義，中期經 MCP/SQL Server 串接後可升級為自動預警。

  • 5.1訂單與 BOM 核對助理審查比對型（跨表核對）
