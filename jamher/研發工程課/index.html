占賀五金 AI Agent 開發實務工作坊｜研發工程課

由部門任務盤點轉化的 POC 解題示範（每題一頁：開發思路→提示詞→知識庫→建置→驗測）

共 6 款 Agent POC部門 研發工程課來源 部門任務盤點

這些題對應研發工程課的真實工作——從製程工令做 Control Plan / PFMEA、工程圖辨識轉檢驗表、新舊圖差異比對、螺帽重量試算，到把資深技師經驗結構化傳承。點進任一題看完整開發流程。

#01主打題

Control Plan 管制計畫產出

文件生成型（工令→管制文件）

工程師把製程工令逐項查 759 筆 key word 對照表、帶抽樣頻率與量具、再排版成 CP，每份要花 1 天以上、每月 20-40 份、3-5 位工程師輪流做。

 #02

PFMEA 製程失效模式分析產出

文件生成型（工令→失效分析）

工程師依製程流程逐項帶嚴重度/發生度/探測度、算 RPN、排改善順序，每份要花 1 天以上、每月 15-30 份、3-5 位工程師。

 #03

工程圖尺寸辨識轉檢驗管制表

圖面辨識型（工程圖→檢驗表）

工程師看客戶工程圖、逐一抄尺寸公差再對量具排成檢驗表，每張要花 30-60 分鐘、每月 60-120 張、3-5 位工程師。

 #04

螺帽重量與材料用量試算

計算試算型（規格→重量/用量）

工程/業務依規格材質手算單顆重量、千顆重量與線材用量供報價備料，每件 10-15 分鐘、每月 100-200 件、3-5 位工程/業務。

 #05

新舊工程圖差異比對

圖面審查型（新舊圖→差異清單）

工程師人工比對客戶新舊版圖面找尺寸/公差/註記變更點，每組 30-60 分鐘、每月 40-80 組、3-5 位工程師，漏看變更易造成後段量產風險。

 #06

資深技師經驗傳承知識庫（模具/攻牙 know-how）

知識問答型（情境→經驗建議）

資深技師的成型模具、攻牙參數與異常排除經驗多在腦中，新人遇問題只能反覆請教，傳承斷層風險高（長期戰略專案，3-5 位工程/技師）。
