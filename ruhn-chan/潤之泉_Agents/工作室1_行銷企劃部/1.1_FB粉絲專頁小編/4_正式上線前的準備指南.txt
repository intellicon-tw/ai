# FB 粉絲專頁小編 - 正式上線前的準備指南

本文件提供將「FB 粉絲專頁小編」從 POC 套件正式部署到 EgentHub 平台、並在潤之泉行銷企劃部門日常作業中穩定使用的逐步準備清單。建議由 IT/AI 種子團隊與行銷主管共同檢核。

## 1. 知識庫設定

### 1.1 潤之泉品牌語氣指南（Agent 向量知識庫 RAG）

- **來源檔案**：`2_知識庫範例資料/潤之泉品牌語氣指南.txt`
- **知識庫類型**：Agent 向量知識庫（RAG）
- **建議名稱**：「潤之泉品牌語氣指南 (Brand Voice Guide)」
- **層級**：Agent 層級（僅本 Agent 使用）
- **後續維護**：每季度由行銷主管覆核一次，新增的慣用語、新的開場模板可直接補入。

### 1.2 潤之泉產品資料庫（表格查詢知識庫 SQL）

- **來源檔案**：`2_知識庫範例資料/潤之泉產品資料庫.csv`
- **知識庫類型**：表格查詢知識庫（SQL）
- **建議名稱**：「潤之泉產品資料庫 (Product Master)」
- **層級**：工作室層級（行銷企劃工作室共用，未來其他 Agent 如 EDM 文案 Agent 可重用）
- **欄位確認**：產品代號、產品全名、規格、主要原料、核心認證、主要賣點、目標客群、禁忌族群提示、通路、品牌定位
- **編碼**：utf-8-sig（含 BOM）
- **後續維護**：新品上市時由產品經理更新 CSV，並上傳新版到 EgentHub。

### 1.3 保健食品合規禁字清單（全文查找知識庫）

- **來源檔案**：`2_知識庫範例資料/保健食品合規禁字清單.txt`
- **知識庫類型**：全文查找知識庫
- **建議名稱**：「保健食品合規禁字清單 (Compliance Blacklist)」
- **層級**：公司層級（建議與其他保健食品 Agent 共用，例如未來的客服 Agent、EDM Agent）
- **後續維護**：每年至少覆核一次，或在主管機關發布新指引時即時更新。

### 1.4 潤之泉粉專高互動貼文範例庫（Agent 向量知識庫 RAG）

- **來源檔案**：`2_知識庫範例資料/潤之泉粉專高互動貼文範例庫.txt`
- **知識庫類型**：Agent 向量知識庫（RAG）
- **建議名稱**：「潤之泉粉專高互動貼文範例庫 (Top Posts)」
- **層級**：Agent 層級
- **後續維護**：每月由行銷主管挑選表現最佳的 2-3 則新貼文加入，並淘汰過時範例（建議保留總數在 20-30 則之間，避免 RAG 檢索品質下降）。

## 2. 模型選擇建議

- **預設模型**：Claude 4.6 Sonnet
- **理由**：中文創意寫作品質佳、語氣模仿能力強、合規檢查邏輯穩定
- **備援模型**：Gemini 3.1 Pro（若 Claude 配額用罄時切換）
- **不建議**：使用 Gemini 3 Flash 等輕量模型，因為品牌語氣的細膩度與合規判斷難度較高

## 3. 兩階段隔離設定

本 Agent 為單階段執行（純文字輸入、僅查詢知識庫，未涉及圖像辨識），無須額外設定兩階段隔離。

## 4. 權限與工作室設定

- **掛載工作室**：行銷企劃工作室
- **可見使用者群組**：行銷企劃部全員（含主管、企劃、社群小編）
- **算力點數初期建議配額**：每月 500-800 次對話額度（依平均一週發 3-4 篇貼文計算，含修改迭代）
- **資料敏感度**：本 Agent 處理品牌資產與合規規範，建議僅限行銷部存取，**不對外部廠商開放**

## 5. 上線前驗測 Checklist

執行 `3_驗測情境.txt` 中的所有情境，並確認以下為 pass/fail gate：

- [ ] **情境 #1（新品上市產品銷售型）**：必須通過。產品名稱以【方括號】格式呈現、Hashtag 使用得當、無禁字。
- [ ] **情境 #3（節氣養生衛教型）**：必須通過。內容避開療效宣稱、用「日常營養補給」類用字、不出現病名。
- [ ] **情境 #5（合規地雷測試）**：當輸入主題含「改善睡眠」「降血壓」等誘導性詞彙時，Agent 必須主動拒絕並建議合規替代寫法，**不可直接照寫**。
- [ ] **情境 #6（品牌敘事型年度回顧）**：語氣節奏要與範例 D1 相近，避免商業化用語。
- [ ] **圖卡視覺建議**：每則貼文都需附主色、主元素、文案層次三項建議。

## 6. 上線後監控指標（首 2 週）

- **每日使用量**：目標每日 2-5 次（行銷企劃日常用量），高於 10 次代表可能過度依賴需檢視
- **合規攔截觸發率**：每週至少 1-2 次主動攔截違規字眼為健康跡象（若都沒攔截過，需懷疑禁字清單未生效）
- **使用者修改率**：草稿被人工修改的比例若大於 60%，代表 prompt 或品牌語氣指南需調整
- **平均回應時間**：應在 30 秒內完成
- **點數消耗速率**：與配額對比，若提前用罄需擴額或檢視重複生成原因

## 7. 異常處理與回退方案

- **常見錯誤型態**：
  - 產品名稱錯誤（未用【方括號】）→ 補入語氣指南的「產品名稱格式」段落
  - 語氣偏向「網紅式」（出現親、寶寶等用字）→ 補入黑名單範例到語氣指南
  - 出現禁字未被攔截 → 立即更新合規禁字清單，並回測情境 #5
- **回退方案**：若 Agent 連續 3 篇貼文出現重大合規問題，暫時下架並回到人工撰寫，由 AI 種子團隊調整 prompt 後重新上線
- **維運窗口**：建議由行銷企劃主管擔任本 Agent 的 owner，負責回饋語氣調整、知識庫更新
- **變更管理**：每次修改 prompt 或知識庫後，必須回測情境 #1、#3、#5、#6 才能重新上線

## 8. 種子團隊接手提示

- **最常見的提示詞調整方向**：
  - 季節性調整：每個季節（春夏秋冬）開始時，更新「節氣養生」相關範例
  - 新品上線：新品發表前 2 週，更新產品資料庫並新增 1-2 則新品宣傳範本
- **新增類似 Agent 的起點**：
  - 「EDM 文案 Agent」可以從本 Agent copy，僅替換輸出格式為 EDM 結構
  - 「LINE OA 推播 Agent」可以從本 Agent copy，調整字數限制與 CTA 邏輯
- **上下游關係**：
  - 上游：產品經理更新產品資料庫 → 本 Agent 取得最新賣點
  - 下游：本 Agent 草稿 → 行銷主管覆核 → 設計師製作圖卡 → 排程發布
