# 媒體新聞稿草擬 - 正式上線前的準備指南

本文件提供將「媒體新聞稿草擬」從 POC 套件正式部署到 EgentHub 平台、並在潤之泉日常作業中穩定使用的逐步準備清單。

## 1. 知識庫設定

- **潤之泉品牌語氣指南.txt**：TXT 格式，Agent 向量知識庫或全文查找知識庫
- **潤之泉產品資料庫.csv**：CSV 格式，表格查詢知識庫
- **保健食品合規禁字清單.txt**：TXT 格式，Agent 向量知識庫或全文查找知識庫

各檔案上傳到 EgentHub 後，建議由種子團隊每季覆核一次內容。CSV 一律維持 utf-8-sig 編碼，文中系統匯出後請另存為 UTF-8 with BOM。

## 2. 模型選擇建議

- **預設模型**：Claude 4.6 Sonnet（預設）
- **備援模型**：Gemini 3.1 Pro
- **不建議**：Claude 3.5 Sonnet（已棄用）、Gemini 1.5 Pro（已棄用）

## 3. 兩階段隔離設定

本 Agent 為單階段執行，無須額外設定

## 4. 權限與工作室設定

- **掛載工作室**：行銷企劃工作室
- **可見使用者群組**：行銷主管、創辦人
- **算力點數初期建議配額**：每月 100-200 次
- **資料敏感度**：本 Agent 涉及內部業務資料，建議僅限指定部門存取，**不對外部廠商開放**

## 5. 上線前驗測 Checklist

執行 `3_驗測情境.txt` 中的所有情境，並確認以下為 pass/fail gate：

- [ ] **情境 #1（標準路徑）**：必須通過。輸出格式正確、無捏造資料。
- [ ] **情境 #2（異常處理）**：必須觸發警示或主動詢問。
- [ ] **合規檢查（若適用）**：本 Agent 涉及對外文案時，必須通過合規檢查。

## 6. 上線後監控指標（首 2 週）

- 每日使用量：依部門人員數推估目標區間
- 使用者修改率：若大於 60%，代表 prompt 或知識庫需調整
- 平均回應時間：應在 30 秒內
- 點數消耗速率：與配額對比

## 7. 異常處理與回退方案

- 連續 3 次出現重大錯誤，暫時下架並回到人工流程
- 由種子團隊調整 prompt 後重新上線
- 每次修改後必須回測情境 #1、#2

## 8. 種子團隊接手提示

本 Agent 屬於知識庫驅動型，後續維護重點在知識庫更新（建議季度檢視一次）。新增類似 Agent 可從本 Agent 為起點 copy 後調整角色與輸出格式。
