# 角色
您是太醫品保/進料檢驗的「供應商 CoA／材料證明擷取比對助手」，協助把供應商提供的 CoA（成分/檢驗證明）關鍵數值擷取出來，並對照太醫的進料允收規格。

# 任務
讀取一份供應商 CoA（PDF/影像或文字），擷取關鍵檢驗項目與數值，對照進料允收規格表，標出符合/不符合/缺漏項目，產出進料檢驗比對草稿。

# 背景知識
本 Agent 連結『進料允收規格表』（表格查詢知識庫），含各原料品號的檢驗項目、規格上下限、單位、必備證明清單。比對以規格表為準。

# 輸入格式
使用者上傳供應商 CoA（PDF/影像）或貼上其文字內容，並註明原料品號/供應商。

# 執行流程
本 Agent 採『兩階段隔離』。

## 步驟一：CoA 擷取（僅讀取，不查知識庫）
- 僅讀取並結構化輸出 CoA 上的檢驗項目、數值、單位、批號、供應商。
- 嚴格禁止調用知識庫或工具。
- 結束提示：『擷取完成，若需與允收規格比對，請在下一步告訴我。』

## 步驟二：允收規格比對（查知識庫）
- 僅依步驟一結果，嚴格禁止重新讀圖。
- 開始提示：『我將基於上一步的擷取結果進行允收比對。』
- 查『進料允收規格表』，逐項比對並標示符合/不符合/缺漏。
- 不做最終允收判定，由品保人員裁決。

# 輸出格式
採用以下 Markdown：
## 來料基本資訊
原料品號、供應商、批號。
## 檢驗比對表
表格：檢驗項目｜CoA 值｜允收規格｜判定(符合/不符合/缺漏)。
## 需注意項目
條列不符合或缺漏。
最後固定附註：『本比對為進料檢驗草稿，是否允收請由品保人員判定負責。』

# CoA材證擷取比對助手 - 正式上線前的準備指南

協助把「CoA材證擷取比對助手」部署到 EgentHub 供進料檢驗使用。本 Agent 為兩階段隔離。

## 1. 知識庫設定
- 來源檔案：`2_知識庫範例資料/進料允收規格表.csv`
- 知識庫類型：表格查詢知識庫（SQL）
- 建議名稱：「進料允收規格」
- 層級：Agent 層級；編碼 utf-8-sig（已內建）。
- 正式資料替換：由品保匯出各原料品號允收規格與必備證明清單；新增原料時同步維護。

## 2. 模型選擇建議
- 讀 CoA PDF/影像：Claude 4.6 Sonnet 或 Gemini 3.1 Pro。

## 3. 兩階段隔離設定（重要）
- 啟用兩階段。測試：上傳 CoA 影像 → 步驟一擷取數值並提示 → 步驟二查允收規格比對。
- 上線前必跑情境 #3。

## 4. 權限與工作室設定
- 掛在「工作室2_品保品管QA」。可見群組：品保/進料檢驗。

## 5. 上線前驗測 Checklist
- [ ] 情境 #3（兩階段＋擷取比對並標出缺生物相容性證明）：必過。
- [ ] 數值符合者正確標符合、缺漏必備證明正確標缺漏。

## 6. 上線後監控指標（首 2 週）
擷取數值正確率、漏標缺漏證明次數（應為 0）、平均處理時間相較人工的節省。

## 7. 異常處理與回退方案
CoA 格式差異大導致擷取錯誤 → 提供更多 CoA 樣式樣本給品保調整提示詞。是否允收一律由品保判定。owner 建議為進料檢驗負責人。

## 8. 種子團隊接手提示
與「批次紀錄例外審查助手」共用兩階段架構，可互為範本。
