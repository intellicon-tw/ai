# 客戶訂單智能解析助手 - 正式上線前的準備指南

本文件提供將「客戶訂單智能解析助手」從 POC 套件正式部署到 EgentHub 平台、並在惠旺工業業務部 5 位業助日常打單作業中穩定使用的逐步準備清單。建議由 IT 與業務主管（柏瑋）共同檢核。

## 1. 知識庫設定

### 1.1 客戶品號主檔（表格查詢知識庫）

- **來源檔案**：`3_知識庫範例資料/客戶品號主檔.csv`
- **知識庫類型**：表格查詢知識庫（SQL）
- **建議名稱**：「客戶品號主檔 (Customer SKU Master)」
- **層級**：Agent 層級
- **欄位確認**：客戶代碼、客戶名稱、客戶品號、惠旺品號、品名、單位、客戶幣別、單價、最後更新日
- **正式資料替換**：上線前由業務窗口從鼎新 Workflow ERP 匯出最新主檔（建議每月更新一次，或客戶報價異動後即時更新），維持 utf-8-sig 編碼

## 2. 模型選擇建議

- **主要推理**：Claude 4.6 Sonnet — 對訂單 PDF 圖像辨識、表格抽取、多語客戶（英文 / 德文 / 日文）混合解析表現最穩。
- **若需大量輕量解析**（純文字 Email）：可降配至 Gemini 3 Flash 節省點數，但須先在驗測情境上重跑全套確認準確度。

## 3. 兩階段隔離設定

本 Agent 為**條件式兩階段**：

- 純文字 Email / Excel 貼上 → 直接走步驟二
- 上傳 PDF 或圖片 → 走步驟一（視覺解析）→ 步驟二（知識庫比對 + 輸出）

上線前必須以「測試 #1（純文字 Email）」與「測試 #2（含急件混合單）」確認文字路徑正常，再額外上傳 1-2 張真實客戶 PDF 訂單測試圖像路徑。EgentHub Agent 設定頁的「兩階段隔離」開關務必開啟。

## 4. 權限與工作室設定

- 工作室：業務部
- 可見群組：5 位業助 + 業務主管（柏瑋）+ 業務副主管
- 算力點數初期建議：依每日 16 張訂單 × 22 工作天估算月用量，第一個月設保守上限並觀察
- 資料敏感度：客戶報價屬機密，Agent 知識庫僅授權業務部，禁止跨部門可見

## 5. 上線前驗測 Checklist

執行 `2_驗測情境.txt` 中所有情境，並確認以下為 pass/fail gate：

- [ ] **情境 #1（標準 Email 訂單）**：必須通過。Bostitch 訂單兩品項皆正確帶入單價、無待修正項。
- [ ] **情境 #2（急件混合單）**：必須正確區分一般單與急件，鍵值規則切換正確。
- [ ] **情境 #3（異常防呆）**：必須停下未知品號與 99999 數量，列入待修正清單。

任一情境失敗即不可正式上線。

## 6. 上線後監控指標（首 2 週）

- 每日成功解析訂單數 / 待人工修正數的比例（目標：> 85% 自動完成）
- 客戶品號比對失敗次數（提示主檔需要補件）
- 業助平均單張訂單處理時間（基準：目前約 60 分鐘 → 目標 5–10 分鐘）
- 主管 audit 抽查 10% 訂單的正確率

## 7. 異常處理與回退方案

- 若連續 3 張訂單出現品號比對錯誤，先檢查客戶品號主檔是否過期、業務是否新增了未建檔的客戶品號。
- 若 PDF 解析信心度持續偏低，先確認上傳檔案非低解析掃描，必要時改為 Excel 輸入。
- 維運窗口：建議由業務主管指派一位資深業助擔任 Agent owner，每週彙整異常案例。
- 回退：完全失效時暫時回到目前 RPA 流程，不影響日常作業。

## 8. 種子團隊接手提示

本 Agent 最常見的調整方向為：(1) 客戶品號主檔欄位增減、(2) 防呆閥值（信心度、數量上限）、(3) 新客戶上線時的訂單格式變異。建議客戶以本 Agent 為起點，後續延伸「採購單自動建立」「出貨通知自動產出」等同類解析應用。
