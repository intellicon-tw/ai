# 智能呆滯料防漏助手 - 正式上線前的準備指南

本文件提供將「智能呆滯料防漏助手」正式部署到 EgentHub、供倉管與生管團隊每月庫存體檢使用的逐步準備清單。

## 1. 知識庫設定

### 1.1 庫存餘額表（表格查詢知識庫）

- **來源檔案**：`3_知識庫範例資料/庫存餘額表.csv`
- **建議名稱**：「庫存餘額表 (Inventory Balance)」

### 1.2 庫存異動明細（表格查詢知識庫）

- **來源檔案**：`3_知識庫範例資料/庫存異動明細.csv`
- **建議名稱**：「庫存異動明細 (Inventory Transactions)」
- **正式資料替換**：上線前由 IT 從 ERP 匯出近 24 個月完整異動明細，建議每月同步

## 2. 模型選擇建議

Claude 4.6 Sonnet。

## 3. 兩階段隔離設定

本 Agent 為單階段執行，無須額外設定。

## 4. 權限與工作室設定

- 工作室：生產管理
- 可見群組：倉管、生管、財務（庫存價值影響財報）
- 資料敏感度：呆滯料金額屬內部敏感資訊

## 5. 上線前驗測 Checklist

- [ ] **情境 #4（假性活水料識別）**：必須抓出 P-TRIGGER-T50 並標示「⚠ 假性活水」
- [ ] **情境 #5（依去化天數排序）**：停產料須排在最前
- [ ] **情境 #6（停產料識別）**：必須抓出零異動料並建議處置

## 6. 上線後監控指標（首 2 週）

- 每月呆滯料體檢使用次數
- 高風險料號從上月→本月的變動情況
- 倉管同仁主動回報的「真實 vs Agent 判定」差異案例

## 7. 異常處理與回退方案

- 若耗用率計算失準，先檢查庫存異動明細是否完整。
- 維運窗口：倉管主管擔任 owner，每月與生管、財務共同 review 高風險清單。
- 回退：失效時回到既有呆滯料報表流程。

## 8. 種子團隊接手提示

延伸應用：「呆滯料去化專案追蹤 Agent」「新機型備料安全水位建議 Agent」。本 Agent 揭露的「假性活水」邏輯是惠旺多年累積的隱性破口，建議高層重視。
