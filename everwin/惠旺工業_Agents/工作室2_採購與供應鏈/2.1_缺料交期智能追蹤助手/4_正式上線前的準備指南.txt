# 缺料交期智能追蹤助手 - 正式上線前的準備指南

本文件提供將「缺料交期智能追蹤助手」正式部署到 EgentHub、供採購、生管、業務三方協作使用的逐步準備清單。

## 1. 知識庫設定

### 1.1 採購單缺料追蹤（表格查詢知識庫）

- **來源檔案**：`3_知識庫範例資料/採購單缺料追蹤.csv`
- **知識庫類型**：表格查詢知識庫
- **建議名稱**：「採購單缺料追蹤 (PO Tracking)」
- **正式資料替換**：建議每日由採購助理上傳更新，第二階段可導入 MCP 自動同步看板資料庫

### 1.2 在手訂單明細（表格查詢知識庫）

- 同 1.3 Agent 的 1.2 設定。為避免資料分散，可在工作室層級建一份共用知識庫並於多個 Agent 引用。

## 2. 模型選擇建議

Claude 4.6 Sonnet — 涉及多表 join 與意圖分歧（查詢 / 更新 / 跟催清單），需穩定的推理能力。

## 3. 兩階段隔離設定

本 Agent 為單階段執行，無須額外設定。

## 4. 權限與工作室設定

- 工作室：採購與供應鏈
- 可見群組：採購、生管、業務（皆可查詢；更新交期僅採購可寫回）
- 資料敏感度：供應商交期屬內部營運資訊，禁止外部可見

## 5. 上線前驗測 Checklist

- [ ] **情境 #1（業務查詢含出貨影響評估）**：必須通過
- [ ] **情境 #2（廠商回覆解析與寫回確認）**：必須通過，且必須等使用者確認後才提示寫回
- [ ] **情境 #3（自動產出跟催清單）**：必須通過

## 6. 上線後監控指標（首 2 週）

- 業務查詢次數（目標：取代既有 Email/口頭詢問）
- 採購跟催清單使用頻率
- 跨部門 Email 跟催信件數（基準 vs 上線後，預期顯著下降）

## 7. 異常處理與回退方案

- 若 Agent 對「廠商回覆」解析錯誤，先確認回覆格式是否異於範本，必要時請廠商使用統一格式回信。
- 維運窗口：採購主管擔任 owner。
- 回退：失效時回到 Email 跟催，並保留現有看板。

## 8. 種子團隊接手提示

延伸應用：「廠商交期 SLA 評鑑 Agent」「自動跟催 Email 草稿 Agent」皆為自然延伸。本 Agent 是惠旺打開「資訊透明化」門檻的關鍵 POC，建議高層重點關注。
