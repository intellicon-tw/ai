# 缺料交期智能追蹤助手 - 驗測情境

本檔案分為兩部分：
- **Part A — 測試資料集**：3 筆可直接貼到 EgentHub 對話框做手動驗測的自然語言輸入。
- **Part B — 驗測情境**：3 個情境的預期輸出與驗證重點，作為手動或半自動回歸驗測依據。

---

## Part A — 測試資料集

### 測試 #1：業務查單一訂單

```
訂單單號 A20260120 的缺料目前交期更新為何？是否會影響最終出貨？
```

### 測試 #2：採購更新廠商回覆

```
PO20260105 旭東金屬最新回覆，T50 彈簧組 5000 PCS 預計 2026-04-22 可全數出貨，請更新看板。
```

### 測試 #3：產跟催清單

```
幫我列出今天有哪些品號 / 訂單已經達『需跟催』標準的清單。
```

## Part B — 驗測情境

### 情境 #1：業務查詢含出貨影響評估

**輸入描述**

> 業務同仁詢問訂單 #A20260120 的缺料交期。

**預期輸出**

> 回傳採購單 PO20260105 旭東金屬 T50 彈簧組，原交期 2026-03-15、最新交期 2026-04-10、需求交期 2026-05-20，標示「✓ 進度安全」並說明剩餘緩衝天數。

**驗證重點**

> 是否正確關聯到對應採購單、是否準確判斷出貨影響、緩衝天數計算是否正確。

### 情境 #2：廠商回覆解析與寫回確認

**輸入描述**

> 採購人員餵入旭東金屬的廠商回覆。

**預期輸出**

> Agent 解析出採購單號 PO20260105、品號 P-SPRING-T50、新交期 2026-04-22，產出更新指令並請使用者確認後再寫回。

**驗證重點**

> 解析準確度、是否避免直接寫入而先請求確認、新交期格式是否標準化。

### 情境 #3：自動產出跟催清單

**輸入描述**

> 採購要求今日跟催清單。

**預期輸出**

> 依供應商分組輸出，至少包含 V003 昌邦工業 P-MOTOR-DX5（異常天數 35 天）與 V004 精準零組 P-GAS-IM250（異常 15 天）。

**驗證重點**

> 閥值規則是否正確套用、是否漏掉已逾期但仍在進行中的項目。
