# BOM建立助手 - 驗測情境

本檔案分為兩部分：
- **Part A — 測試資料集**：2 筆可直接貼到 EgentHub 對話框做手動驗測的自然語言輸入。
- **Part B — 驗測情境**：2 個情境的預期輸出與驗證重點，作為手動或半自動回歸驗測依據。

---

## Part A — 測試資料集

### 測試 #1：純文字新成品 BOM

```
請幫我建立新成品 EW-T55-18GA (T55 進階版氣動釘槍) 的 BOM:
1. T55 機身 (鋁合金鑄造) x 1 PCS
2. T55 槍管 (鋼材精密加工) x 1 PCS
3. T55 彈簧組 (升級版 SUS316) x 1 SET
4. T50 板機 x 1 PCS (沿用舊型)
5. M4 螺絲組 x 10 PCS
```

### 測試 #2：既有成品小改

```
以 EW-DX5 為基礎建立 EW-DX5-Plus：
- 機身改用 DX5-Plus 強化版 (新料號)
- 馬達改 DX5-Plus 強化馬達 (新料號)
- 電池維持 P-BATTERY-DX5
- 新增散熱風扇 x 1
請比對與 EW-DX5 的差異。
```

---

## Part B — 驗測情境

### 情境 #1：新料號偵測與既有零件帶入

**輸入描述**

> 研發提交 EW-T55-18GA 的純文字 BOM。

**預期輸出**

> 新料號（T55 機身、T55 槍管、T55 彈簧組）標示「需新建」並要求補件規格；既有料（T50 板機、M4 螺絲組）成功帶入零件主檔資訊。

**驗證重點**

> 是否準確區分新/舊料、是否正確帶入既有單位成本、是否要求新料號的補件規格。

### 情境 #2：既有成品改版差異比對

**輸入描述**

> 研發以 EW-DX5 為基礎建立 EW-DX5-Plus。

**預期輸出**

> 輸出 EW-DX5-Plus 新 BOM，並以差異表列出：機身（變更）、馬達（變更）、電池（沿用）、散熱風扇（新增）。

**驗證重點**

> 差異比對是否清楚標示「變更 / 沿用 / 新增 / 刪除」、是否提供成本差異估算。
