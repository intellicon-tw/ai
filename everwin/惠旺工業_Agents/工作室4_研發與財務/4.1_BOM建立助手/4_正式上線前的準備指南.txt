# BOM 建立助手 - 正式上線前的準備指南

本文件提供將「BOM 建立助手」正式部署到 EgentHub、供研發與生管團隊在新機型開發階段使用的逐步準備清單。

## 1. 知識庫設定

### 1.1 零件主檔（表格查詢知識庫）

- **來源檔案**：`3_知識庫範例資料/零件主檔.csv`
- **建議名稱**：「零件主檔 (Component Master)」
- **正式資料替換**：上線前由 PDM 系統匯出最新版本

### 1.2 BOM 展開表（表格查詢知識庫）

- **來源檔案**：`3_知識庫範例資料/BOM展開表.csv`
- **建議名稱**：「BOM 展開表 (Existing BOMs)」— 作為相似成品比對的參考來源
- **正式資料替換**：與 3.1 Agent 共用

## 2. 模型選擇建議

- **主要推理**：Claude 4.6 Sonnet — 對工程圖、文字 BOM 的解析能力最穩。
- **若涉及大量手繪或低品質掃描圖**：可考慮 Gemini 3.1 Pro，視 POC 比較結果而定。

## 3. 兩階段隔離設定

本 Agent 為**條件式兩階段**：

- 純文字 BOM 列表 → 直接走步驟二
- 上傳工程圖 PDF / 圖片 → 走步驟一（視覺解析）→ 步驟二（料號比對 + 輸出）

EgentHub Agent 設定頁的「兩階段隔離」開關必須開啟。上線前以「測試 #1（純文字）」與「測試 #2（既有成品小改）」測試文字路徑，並至少上傳 1 張真實工程圖照片測試圖像路徑。

## 4. 權限與工作室設定

- 工作室：研發與財務
- 可見群組：研發工程師、生管 BOM 建檔人員、研發主管
- 資料敏感度：新機型 BOM 屬最高機密，務必限定研發群組

## 5. 上線前驗測 Checklist

- [ ] **情境 #1（新料號偵測與既有零件帶入）**：必須通過
- [ ] **情境 #2（既有成品改版差異比對）**：必須清楚標示變更 / 沿用 / 新增

## 6. 上線後監控指標（首 2 週）

- 每月 BOM 建立次數
- 新料號偵測率（過去未建檔的料號被 Agent 標出後成功補件的比例）
- 研發工程師對「相似成品差異比對」的滿意度

## 7. 異常處理與回退方案

- 若工程圖辨識失敗，先確認影像解析度（建議 300 dpi 以上），必要時改為文字輸入。
- 維運窗口：研發主管指派 owner。
- 回退：失效時回到既有 PDM 手動建檔流程。

## 8. 種子團隊接手提示

延伸應用：「材料替代建議 Agent」「成本預估 Agent」（依新 BOM 估算單位材料成本）。本 Agent 是 R&D 端 AI 化的起點，建議研發團隊持續貢獻新機型資料以提升 Agent 精度。
