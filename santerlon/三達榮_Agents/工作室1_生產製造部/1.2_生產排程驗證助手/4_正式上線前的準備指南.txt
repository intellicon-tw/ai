# 生產排程驗證助手 - 正式上線前的準備指南

本文件提供將「生產排程驗證助手」從 POC 套件正式部署到 EgentHub 平台、並在三達榮公司生管團隊日常作業中穩定使用的逐步準備清單。建議由 IT/AI 種子團隊與生管主管（廠長、生管主管）共同檢核。

---

## 1. 知識庫設定

本 Agent 共需設定 5 個「表格查詢知識庫」(SQL)。請依下列順序逐一建立。

### 1.1 schedule_gantt 排程甘特（主要驗證對象）

- **來源檔案**：`2_知識庫範例資料/schedule_gantt.csv`
- **知識庫類型**：表格查詢知識庫（SQL）
- **建議名稱**：「排程甘特 (Schedule Gantt)」
- **層級**：Agent 層級（僅本 Agent 使用）
- **欄位確認**：廠別、組別、序號、訂單號、款號、款號名稱、顏色、訂單數量、下單日期、客戶交期、變更交期、物料齊套日期、標準日產能、生產數量、計畫開始日期、計畫結束日期
- **資料量**：本範例 960 筆，正式上線後依實際排程批次浮動
- **正式資料替換**：
  - 短期：由生管人員每週手動匯出最新甘特 csv，覆蓋原檔
  - 中期：透過 EgentHub MCP 介接生管系統，每日同步

### 1.2 schedule_outsource 外發排程

- **來源檔案**：`2_知識庫範例資料/schedule_outsource.csv`
- **知識庫類型**：表格查詢知識庫（SQL）
- **建議名稱**：「外發排程 (Outsource Schedule)」
- **層級**：Agent 層級
- **欄位確認**：外發廠、序號、訂單號、款號、款號名稱、顏色、訂單數量、下單日期、客戶交期、變更交期、物料齊套日期、標準日產能、生產數量、計畫開始日期、計畫結束日期
- **資料品質提醒**：本範例資料的部分批次「款號名稱」與「顏色」欄位內容疑似順序顛倒（款號名稱欄記錄顏色字串，顏色欄記錄款號名稱字串）。**建議生管在上線前修正資料**，或於提示詞「背景知識」段保留欄位錯位提醒（已內建）

### 1.3 orders_master 訂單主檔

- **來源檔案**：`2_知識庫範例資料/orders_master.csv`
- **知識庫類型**：表格查詢知識庫（SQL）
- **建議名稱**：「業務訂單主檔 (Orders Master)」
- **層級**：Agent 層級
- **欄位確認**：訂單號、訂單序號、訂單版本、客戶簡稱、客戶簡碼、款號、款號名稱、COLORWAY、客戶款號代碼、訂單數量、加產數量、客戶訂單號、款號尺寸、下單日期、客戶要求交期、建立日期、生效日期
- **正式資料替換**：由業務窗口從 ERP 匯出，建議每日同步

### 1.4 schedule_draft 排程底稿

- **來源檔案**：`2_知識庫範例資料/schedule_draft.csv`
- **知識庫類型**：表格查詢知識庫（SQL）
- **建議名稱**：「排程底稿 (Schedule Draft)」
- **層級**：Agent 層級
- **欄位確認**：指定生產線、訂單號、客戶簡碼、款號型體、款號名稱、顏色、款號代碼、下單日期、客戶要求交期、訂單數量、加產數量、總數量、排程時間、預計出成品日期、已出口的數量、未出口數量
- **用途**：本表主要供 R8（未指定產線）驗證使用，亦可作為 R1（交期逾期）的對照來源

### 1.5 holidays 例假日清單

- **來源檔案**：`2_知識庫範例資料/holidays.csv`
- **知識庫類型**：表格查詢知識庫（SQL）
- **建議名稱**：「例假日清單 (Holidays)」
- **層級**：工作室層級（生產製造部多個 Agent 可共用）
- **欄位確認**：日期、假期名稱
- **正式資料替換**：每年 12 月由人資或行政更新次年假期表。請特別注意：本範例中包含勞動節、解放日&統一日、補假、國慶日等屬於東莞工廠在地假期，與台灣總部假期不同，請確認對應廠別

## 2. 模型選擇建議

- **建議預設模型**：Claude 4.6 Sonnet
- **理由**：本 Agent 需同時處理 5 張表 JOIN 查詢、8 條規則邏輯判斷、日期區間運算（含展開到每日加總），對推理深度與一致性要求極高，建議使用 Claude 4.6 Sonnet。
- **次選模型**：Gemini 3.1 Pro。對於以「指定組別、單日」這類窄範圍查詢的問題，可改用 Gemini 3 Flash 降低成本，但全庫掃描務必使用 Claude 4.6 Sonnet。
- **避免使用**：Claude 3.5 Sonnet 以前、Gemini 1.5 Pro 以前等已停止維護的舊版模型。

---

## 3. 兩階段隔離設定

本 Agent 為單階段執行（自然語言輸入 + 表格查詢與規則驗證），無須在 EgentHub 後台額外啟用兩階段隔離設定。

---

## 4. 工具設定

### Current Time 工具

- **啟用方式**：於 EgentHub Agent 設定頁的「工具」區塊啟用「Current Time」（系統內建工具）
- **用途**：支援「今天」「本週」「下週」「本月」等相對時間語意解析
- **驗證**：上線前以「掃描本週排程」「檢查下週的批次」等指令測試，確認 Agent 能正確推算對應的日期區間
- **時區**：請確認系統時區設為東莞時區（UTC+8），與三達榮台灣總部相同

---

## 5. 權限與工作室設定

- **工作室歸屬**：建議掛在「工作室1_生產製造部」之下。
- **使用者範圍**：
  - 必開：廠長、生管主管、生管人員、排程人員
  - 選開：業務主管（查訂單交期風險時使用）、採購主管（觀察物料齊套日影響時使用）
  - 暫不開放：一般作業員（資料粒度與權限考量）
- **算力點數初期建議**：依預估每日 20-40 次驗證估算，第一個月配 1.5 萬點觀察消耗。全庫掃描單次點數較高，需特別關注。
- **資料敏感度**：
  - 排程資料涉及訂單交期、客戶資訊、產能配置，屬高敏感度營業機密
  - 強烈建議啟用 EgentHub 的「日誌追蹤」與「分級權限」
  - 若未來考慮對外發廠開放查詢（讓外發廠看自己批次的問題清單），務必先做資料隔離設定，僅開放該外發廠相關紀錄

## 6. 上線前驗測 Checklist

執行 `3_驗測情境.txt` 中的全部 8 個情境，並確認以下為 pass/fail gate（必須通過才能正式上線）：

- [ ] **情境 #1（全庫掃描，標準路徑）**：必須通過。同時涵蓋 schedule_gantt 與 schedule_outsource，排序正確、統計正確、無捏造資料。
- [ ] **情境 #3（例假日 R6 驗證）**：必須通過。能正確查 holidays 表，計畫起訖區間覆蓋假期時準確命中。
- [ ] **情境 #5（模糊指令防呆）**：必須通過。觸發範圍確認對話而非直接掃全庫。
- [ ] **情境 #7（產能超載 R5 驗證）**：必須通過。能將批次按日展開、同日加總、與標準日產能比對。
- [ ] **情境 #8（無問題回覆）**：必須通過。僅輸出單一句子，不夾帶空表格或推理說明。

情境 #2、#4、#6 為「建議通過」，若初版回覆品質尚未穩定，可在上線後持續優化。

---

## 7. 上線後監控指標（首 2 週）

請生管主管或種子團隊成員每週檢視一次以下指標：

- **每日使用量**：使用次數、使用者人數、人均查詢次數
- **常見問句聚類**：實際使用者最常掃描哪一類範圍？是否有未被覆蓋的查詢需求？
- **誤判率**：使用者主動回報「這筆沒問題卻被列出」「這筆有問題卻沒列出」的次數，每週超過 5 次需檢視提示詞與資料品質。
- **回應時間**：全庫掃描預期 30-60 秒；單組單週查詢預期 10 秒內。超過此範圍需檢查模型負載或知識庫索引。
- **算力點數消耗速率**：全庫掃描單次點數較高，需評估是否限制單日使用次數。

## 8. 異常處理與回退方案

### 常見錯誤型態與排查步驟

- **遺漏外發廠批次**：檢查 schedule_outsource 知識庫是否正確掛載、欄位是否完整。
- **R5 產能超載漏判**：通常因「批次按日展開」邏輯未正確執行所致，建議重跑情境 #7，並在提示詞中加強「展開邏輯」說明。
- **R6 例假日漏判**：檢查 holidays 表是否更新、日期格式是否一致（建議統一 YYYY-MM-DD）。
- **「款號名稱」與「顏色」欄位錯位導致誤判**：請務必在 schedule_outsource 上線前修正資料，或在資料端做欄位校正後再上 Agent。
- **回覆冗長或夾帶推理**：違反問答指引，需後台微調提示詞或重新訓練種子團隊。

### 何時應暫時下架 Agent 並回到人工流程

- 連續 3 天以上發現重大誤判（漏報或誤報率合計 > 10%）
- 任一張知識庫資料遺失或污染
- 全庫掃描連續超時無法回覆

### Agent 維運窗口

- **業務負責人**：建議由生管主管擔任，負責決定規則調整與範圍擴展
- **技術維運**：由 IT 種子團隊成員擔任，負責每週／每日資料更新、提示詞微調、點數監控
- **變更管理**：規則邏輯（R1～R8）任何修改，必須重跑情境 #1、#3、#5、#7、#8 全部 5 個必過 gate；新增規則時建議新增對應驗測情境

## 9. 種子團隊接手提示

### 本 Agent 最常見的提示詞調整方向

- **新增驗證規則 R9～R12**：常見後續需求包含「客戶交期前留足出貨緩衝（例如 R9：計畫結束日 + 5 日 ≤ 客戶交期）」「跨組撞模具」「同款號多組重複排程」等，可在「規則驗證」段落擴充，並同步補測試情境。
- **規則嚴重度分級**：未來可在輸出表格加入「嚴重度」欄位（高／中／低），讓使用者快速聚焦關鍵問題。
- **異動建議自動化**：目前「建議處理方向」是 Agent 自行生成，未來可建立「處理建議對照表」作為知識庫，讓建議更標準化。

### 客戶若要新增類似 Agent，可從本 Agent 為起點 copy

- 「生產效率分析助手」（同部門姊妹 Agent，已交付）：兩個 Agent 共用部分資料來源邏輯，可互為延伸應用。
- 「物料齊套狀況查詢助手」（採購／生管交界）：聚焦 R3、R4 物料規則，可作為本 Agent 的精簡版。

### 與其他 Agent 的資料依賴關係

- **上游資料**：依賴生管系統匯出的甘特表與外發表、業務系統的訂單主檔、人資公布的例假日清單。
- **下游應用**：本 Agent 的問題清單可餵給「業務客戶訂單追蹤整合助手」，讓業務窗口看到客戶訂單對應的排程風險。
- **互補 Agent**：與「生產效率分析助手」結合可形成完整的生產可視化與排程決策閉環。
