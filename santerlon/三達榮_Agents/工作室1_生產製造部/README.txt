# 工作室1_生產製造部

本工作室為三達榮公司「生產製造部」的 AI Agent 開發包，第一階段共規劃 5 個 Agent，涵蓋生管「看現況 → 驗排程 → 預警物料 → 追交期 → 找空檔」完整決策閉環，目前全部完成 POC 開發。

## Agent 清單

| 編號 | Agent 名稱 | 知識庫類型 | 主要使用者 | 戰略目標 | 兩階段 | 狀態 |
|---|---|---|---|---|---|---|
| 1.1 | 生產效率分析助手 | 表格查詢知識庫 ×2 | 生管、廠長 | 看現況 | 否 | 已交付 |
| 1.2 | 生產排程驗證助手 | 表格查詢知識庫 ×5 | 生管、排程 | 驗排程 | 否 | 已交付 |
| 1.3 | 物料齊套風險預警助手 | 表格查詢知識庫 ×3 | 生管、採購 | 預警物料 | 否 | 已交付 |
| 1.4 | 客戶訂單交期達成追蹤助手 | 表格查詢知識庫 ×4 | 業務、客服、生管 | 追交期 | 否 | 已交付 |
| 1.5 | 產線負荷分析助手 | 表格查詢知識庫 ×3 | 生管、業務 | 找空檔 | 否 | 已交付 |

## 各 Agent 子目錄

- `1.1_生產效率分析助手/` — 1_prompt + 2_知識庫範例資料/（2026 年 3 月生產日報表 787 筆 + 生產組別與製程主檔 25 筆）+ 3_驗測情境（6 個）+ 4_正式上線前的準備指南
- `1.2_生產排程驗證助手/` — 1_prompt + 2_知識庫範例資料/（schedule_gantt 960 筆、schedule_outsource 280 筆、orders_master 1292 筆、schedule_draft 1393 筆、holidays 14 筆）+ 3_驗測情境（8 個）+ 4_正式上線前的準備指南
- `1.3_物料齊套風險預警助手/` — 1_prompt + 2_知識庫範例資料/（共用 schedule_gantt + schedule_outsource + holidays）+ 3_驗測情境（6 個）+ 4_正式上線前的準備指南
- `1.4_客戶訂單交期達成追蹤助手/` — 1_prompt + 2_知識庫範例資料/（schedule_gantt + schedule_outsource + orders_master + schedule_draft）+ 3_驗測情境（6 個）+ 4_正式上線前的準備指南
- `1.5_產線負荷分析助手/` — 1_prompt + 2_知識庫範例資料/（schedule_gantt + schedule_outsource + holidays）+ 3_驗測情境（6 個）+ 4_正式上線前的準備指南

## 知識庫資料表共用建議

EgentHub 上線時建議將下列知識庫掛在「工作室層級」共用，避免重複維護：

| 知識庫名稱 | 來源 | 共用 Agent |
|---|---|---|
| 排程甘特 (schedule_gantt) | 生管系統 | 1.2、1.3、1.4、1.5 |
| 外發排程 (schedule_outsource) | 生管系統 | 1.2、1.3、1.4、1.5 |
| 業務訂單主檔 (orders_master) | 業務 ERP | 1.2、1.4 |
| 排程底稿 (schedule_draft) | 生管系統 | 1.2、1.4 |
| 例假日清單 (holidays) | 人資 | 1.2、1.3、1.5 |

1.1 的「生產日報表」與「生產組別主檔」屬於 Agent 層級專屬。

## 5 隻 Agent 的決策閉環

```
[1.1 看現況] 各組今天做了多少？
     ↓
[1.2 驗排程] 排程有沒有違規？8 條規則
     ↓
[1.3 預警物料] 未來幾天哪些物料來不及？
     ↓
[1.4 追交期] 客戶訂單交期有沒有風險？
     ↓
[1.5 找空檔] 哪幾組還可以接新單？
```

這 5 隻 Agent 從生管主管視角出發，可以單獨使用，也可組合使用形成完整的生產決策面板。
