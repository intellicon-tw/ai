# 產線負荷分析助手 - 正式上線前的準備指南

本文件提供將「產線負荷分析助手」從 POC 套件正式部署到 EgentHub 平台、並在三達榮公司生管團隊與業務團隊日常作業中穩定使用的逐步準備清單。

---

## 1. 知識庫設定

本 Agent 共需設定 3 個「表格查詢知識庫」(SQL)。

### 1.1 schedule_gantt 排程甘特

- **來源檔案**：`2_知識庫範例資料/schedule_gantt.csv`
- **建議名稱**：「排程甘特」
- **層級**：工作室層級（與 1.2、1.3、1.4 共用）
- **編碼**：utf-8-sig

### 1.2 schedule_outsource 外發排程

- **來源檔案**：`2_知識庫範例資料/schedule_outsource.csv`
- **層級**：工作室層級
- **資料品質提醒**：「款號名稱」與「顏色」欄位疑似錯位，上線前修正

### 1.3 holidays 例假日清單

- **來源檔案**：`2_知識庫範例資料/holidays.csv`
- **層級**：工作室層級

---

## 2. 模型選擇建議

- **建議預設模型**：Claude 4.6 Sonnet
- **理由**：本 Agent 需要做工作日計算、加權平均、負荷率推算與排序，建議使用 Claude 4.6 Sonnet。
- **次選**：Gemini 3.1 Pro。

---

## 3. 兩階段隔離設定

本 Agent 為單階段執行，無須啟用兩階段隔離。

---

## 4. 工具設定

- **啟用 Current Time**：必須啟用。
- **時區**：UTC+8。

---

## 5. 權限與工作室設定

- **工作室歸屬**：工作室1_生產製造部
- **使用者範圍**：
  - 必開：生管主管、生管人員、業務主管（接單前確認產能）
  - 選開：廠長、各組組長（觀察自己組的負荷）、董事長辦公室（觀察整體稼動率）
- **算力點數初期建議**：依預估每日 20-30 次查詢估算，第一個月配 6,000 點

---

## 6. 上線前驗測 Checklist

執行 `3_驗測情境.txt` 中的 6 個情境，以下為 pass/fail gate：

- [ ] **情境 #1（找空檔）**：必須通過。工作日計算正確、負荷率公式正確、排序由低到高。
- [ ] **情境 #4（外發廠對比）**：必須通過。資料來源僅 schedule_outsource、缺資料的外發廠需明確標示。
- [ ] **情境 #5（自有廠整體）**：必須通過。24 個組別完整涵蓋、加權平均正確。
- [ ] **情境 #6（負向測試）**：必須通過。不可捏造資料。

---

## 7. 上線後監控指標（首 2 週）

- **每日使用量**
- **常見查詢類型**（找空檔 / 單組負荷 / 廠別總覽 / 整體對比）
- **負荷率準確性**：與生管實際盤點對照
- **回應時間**：預期 10-30 秒（依查詢範圍大小）

---

## 8. 異常處理與回退方案

### 常見錯誤型態

- **工作日數錯誤**：檢查 holidays 表是否更新。
- **批次跨區間導致數量計算錯誤**：需在提示詞中明確指定「跨區間批次採全額計入並備註」。
- **外發廠標準日產能空缺**：需提示使用者「該外發廠無標準產能資料，僅顯示排定生產量」。

### 何時應暫時下架

- 連續一週發現負荷率明顯失準（誤差 > 10%）。
- 標準日產能資料與實際差距過大（需先校正主檔）。

### Agent 維運窗口

- **業務負責人**：生管主管
- **技術維運**：IT 種子團隊
- **變更管理**：負荷率判定門檻調整需重跑情境 #1、#4、#5、#6 四個必過 gate

---

## 9. 種子團隊接手提示

### 最常見的提示詞調整方向

- **動態標準日產能**：未來可建立「組別產能歷史資料表」，依不同款式給不同標準日產能。
- **預測未來負荷**：結合 1.4 客戶訂單交期追蹤助手，將未排程訂單也考慮進來，做前瞻性負荷預測。
- **接單建議**：當業務問「我這張單還能接嗎」，由本 Agent 結合空檔與訂單交期評估後給出建議。

### 與其他 Agent 的資料依賴關係

- **上游**：依賴生管甘特與外發排程資料品質
- **互補**：與 1.2 排程驗證、1.3 物料預警、1.4 交期追蹤共同形成完整的生管決策套件
- **下游**：本 Agent 的「有空檔可接單」結果可餵給業務，作為接單參考
