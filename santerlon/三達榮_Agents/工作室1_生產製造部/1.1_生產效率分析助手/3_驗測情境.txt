# 生產效率分析助手 - 驗測情境

本檔案分為兩部分：
- **Part A — 測試資料集**：可直接貼到 EgentHub 對話框做手動驗測的自然語言輸入。
- **Part B — 驗測情境**：每個情境的預期輸出與驗證重點，作為手動或半自動回歸驗測依據。

---

## Part A — 測試資料集

### 測試 #1：日產量統計（產出統計類）

```
請幫我看一下 2026 年 3 月 4 日當天，所有生產組別的當日產出，由高到低排列。
```

### 測試 #2：訂單完成進度（訂單進度類）

```
VS2510042 這張訂單目前完成多少？大概什麼時候可以做完？
```

### 測試 #3：單組趨勢分析（趨勢分析類）

```
請分析針 BH4 組 3 月份的每日產出趨勢，並標出產量高低點。
```

### 測試 #4：跨組對比（跨組／跨製程對比類）

```
3 月份各製程（針車、成型、後整）的總產出與佔比是多少？哪個製程貢獻最大？
```

### 測試 #5：異常識別

```
3 月份有哪些組別的產出異常偏低或長期沒投產的？請列出來給我看。
```

### 測試 #6：超出資料範圍的查詢（防呆）

```
請告訴我針 XY3 組 3 月份的 OEE 與不良率多少。
```

## Part B — 驗測情境

### 情境 #1：日產量統計（標準路徑）

**輸入描述**

> 使用者詢問 2026-03-04 當天所有組別的當日產出，要求由高到低排列。

**預期輸出**

> 採用「產出統計類」表格格式，呈現 2026-03-04 當天各部門的當日產出加總，依產量由高到低排序。應正確排除訂單號為「1」的配件雜項紀錄（如打釘組A、針十组的配件項）。表格欄位應包含部門、當日產出數量，並備註「配件雜項已排除」。

**驗證重點**

> 1. 是否正確篩選日期為 2026-03-04 的紀錄。
> 2. 同一組別當日有多筆紀錄（例如針 BH1 組同日有 2 筆、針 BH3 組同日有 5 筆）時，是否正確加總而非僅顯示一筆。
> 3. 配件雜項（訂單號=1、訂單總數=999999）是否被排除。
> 4. 排序是否確實由高到低。
> 5. 不應出現任何捏造的組別或數字。

### 情境 #2：訂單完成進度（含預估完工日）

**輸入描述**

> 使用者詢問 VS2510042 訂單目前完成進度與預估完工日。

**預期輸出**

> 採用「訂單進度類」表格格式，需呈現：
> - 訂單號 VS2510042、款號（CIC017C43Z4 與 Z5 兩個顏色）、款號名稱（COSTCO 2026 40罐EVA壓膜折疊冰袋）
> - 訂單總數（依資料應為 Z4 訂單 127120 件、Z5 訂單 83920 件，需分別呈現或合併呈現並註明）
> - 截至資料最後日期（2026-03-31）的累積生產數量
> - 完成率百分比
> - 近 5 個工作日平均日產
> - 預估剩餘工日與預估完工日（依當前產出速度推估）
> - 逐日產出明細表

**驗證重點**

> 1. 是否正確識別該訂單有兩個顏色變體（Z4 與 Z5），並分別計算進度。
> 2. 累積生產數量是否為該訂單號所有日期、所有組別（針 XY3、XY4、XY5、XY6 都在做）的當日生產數量加總。
> 3. 完成率計算公式是否正確（累積 / 訂單總數 × 100%）。
> 4. 預估完工日是否合理（不應出現過去的日期或數十年後的日期）。
> 5. 若已 100% 完成，需明確標示「已完工」而非繼續預估。

### 情境 #3：單組趨勢分析

**輸入描述**

> 使用者要求分析針 BH4 組 3 月份的每日產出趨勢。

**預期輸出**

> 採用「趨勢分析類」表格格式，呈現針 BH4 組 2026-03-02 至 2026-03-31 期間，每日的當日產出量、與平均的差異、趨勢標記（如「高峰」「低谷」「持平」）。應包含全期間平均、最高與最低值及對應日期。資料顯示針 BH4 組主要做 8 吋方形午餐袋（CIW029C8Z0），日產量約 1500-1800 件區間。

**驗證重點**

> 1. 是否完整覆蓋整個 3 月份（即使中間某日無紀錄，也應註明）。
> 2. 全期間平均的計算是否合理。
> 3. 高低點是否確實標示對應的日期。
> 4. 「與平均差異」是否合理（百分比或絕對值皆可，但要一致）。
> 5. 備註應提供趨勢摘要（如「整體穩定」「月底略升」等定性描述）。

### 情境 #4：跨製程對比

**輸入描述**

> 使用者詢問 3 月份各製程的總產出與佔比，並要求指出貢獻最大的製程。

**預期輸出**

> 採用「跨組／跨製程對比類」表格格式，呈現 2026-03-02 至 2026-03-31 期間，「針車」「成型」「後整」「外協加工（空白製程）」四個類別的總產出、日均產出與佔比。應指出「針車」為主力製程（佔比應超過 50%）。

**驗證重點**

> 1. 是否將「製程」欄位為空白者（外協組）獨立計算為「外協加工」類別，而不是漏算。
> 2. 各製程的佔比加總是否為 100%。
> 3. 是否正確排除配件雜項。
> 4. 結論性陳述（「針車為主力」）需有數字支持，不應憑空。

### 情境 #5：異常識別

**輸入描述**

> 使用者要求列出 3 月份產出異常偏低或長期未投產的組別。

**預期輸出**

> 採用「異常識別類」表格格式，列出符合以下任一條件的組別／訂單：
> - 該組別當月平均產出明顯低於同類組別（例如針七組、針四組部分日期產量偏低）
> - 連續 3 日以上產出為 0 或缺紀錄
> - 配件雜項組（針十组、打釘組A）應於備註中說明「為配件生產組，產量為 0 屬正常」
>
> 表格應列出組別名稱、異常類型、具體狀況描述、建議關注方向。

**驗證重點**

> 1. 是否避免將「資料中本來就只有產配件」的組別（針十组、打釘組A）誤判為異常。
> 2. 是否能識別出「連續多日為 0」與「單日突降」的差別。
> 3. 建議關注方向是否具操作性（例如「請查當週是否有調機或人員調動」），而非空泛。
> 4. 備註是否強調「資料層面的異常徵兆，仍需現場主管確認」。

### 情境 #6：超出資料範圍的查詢（誠實揭露限制）

**輸入描述**

> 使用者詢問針 XY3 組的 OEE 與不良率，但目前資料中並無機台運轉時間與品檢紀錄。

**預期輸出**

> 採用「資料範圍提醒」格式，明確回覆：
> - 目前資料無法計算 OEE（缺機台運轉時間、計畫工時、標準節拍）
> - 目前資料無法計算不良率（缺品檢／不良數紀錄）
> - 建議於第二階段補充哪些資料表與欄位，補齊後即可支援該分析

**驗證重點**

> 1. 是否誠實告知無法計算，而非編造數字。
> 2. 是否具體說明「缺什麼資料」與「補齊後可做什麼」。
> 3. 語氣是否委婉且具建設性，不會讓使用者覺得被打槍。
> 4. 是否避免「請參考相關文件」這種模糊回應。
