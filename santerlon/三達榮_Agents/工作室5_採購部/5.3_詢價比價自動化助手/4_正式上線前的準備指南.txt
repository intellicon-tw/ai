# 詢價比價自動化助手 - 正式上線前的準備指南

本文件提供將「詢價比價自動化助手」從 POC 套件正式部署到 EgentHub 平台、並在三達榮公司採購日常作業中穩定使用的逐步準備清單。建議由 IT/AI 種子團隊與業務負責人共同檢核。

---

## 1. 知識庫設定

本 Agent 共需設定 2 個知識庫。

### 1. quotation_request_history.csv

- **檔案路徑**：`2_知識庫範例資料/quotation_request_history.csv`
- **知識庫類型**：表格查詢知識庫 (SQL)
- **建議名稱**：「過往詢價比價紀錄（示範資料，需替換）」
- **層級**：工作室層級
- **編碼**：utf-8-sig（CSV）／ UTF-8（TXT）
- **資料屬性**：**本檔為示範資料**，上線前須由各部門窗口替換為三達榮真實資料；保留欄位結構與檔名一致。

### 2. material_master.csv

- **檔案路徑**：`2_知識庫範例資料/material_master.csv`
- **知識庫類型**：表格查詢知識庫 (SQL)
- **建議名稱**：「物料主檔（與 3.3、5.1 共用）」
- **層級**：工作室層級
- **編碼**：utf-8-sig（CSV）／ UTF-8（TXT）
- **資料屬性**：**本檔為示範資料**，上線前須由各部門窗口替換為三達榮真實資料；保留欄位結構與檔名一致。


> **關於示範資料的重要提醒**：本 POC 套件中標示為「示範資料」的檔案，是智慧方案顧問團隊為了讓客戶快速看到 Agent 運作效果而預先生成的範例。在 EgentHub 正式上線前，必須由三達榮對應部門窗口（業務／品管／採購／HR 等）將真實資料替換進去，保留欄位結構與檔名一致即可，Agent 提示詞無須修改。**未替換真實資料前，Agent 僅能用於展示與驗測，不可作為正式業務決策依據。**

---

## 2. 模型選擇建議

- **建議預設模型**：Claude 4.6 Sonnet
- **理由**：數字運算與判斷
- **避免使用**：Claude 3.5 Sonnet 以前、Gemini 1.5 Pro 以前等已停止維護的舊版模型。

---

## 3. 兩階段隔離設定

本 Agent 為單階段執行，無須在 EgentHub 後台額外啟用兩階段隔離設定。

---

## 4. 工具設定

本 Agent 無須啟用外部工具，僅依賴表格查詢知識庫即可運作。

---

## 5. 權限與工作室設定

- **工作室歸屬**：工作室5_採購部
- **主要使用者**：採購
- **算力點數初期建議**：依預估每日使用次數估算，建議第一個月配 5,000-10,000 點觀察消耗。
- **資料敏感度**：本 Agent 涉及的資料可能包含客戶資訊、訂單金額、員工資料等商業機密，建議啟用 EgentHub 日誌追蹤功能，所有對話可追溯稽核；不建議開放給外部供應商。

---

## 6. 上線前驗測 Checklist

執行 `3_驗測情境.txt` 中的全部 2 個情境，並確認以下為 pass/fail gate（必須通過才能正式上線）：

- [ ] **情境 #1（三家比價）**：推薦有依據。
- [ ] **情境 #2（異常高價警示）**：警示明確。

---

## 7. 上線後監控指標（首 2 週）

- **每日使用量**：使用次數、使用者人數、人均查詢次數
- **常見問句聚類**：實際使用者最常問哪幾類問題？是否與 POC 設計的能力範圍吻合？
- **異常觸發次數**：使用者主動回報「答錯」「答不出來」的次數
- **回應時間**：依複雜度，預期 5-30 秒內回覆
- **算力點數消耗速率**：依此推估月成本

---

## 8. 異常處理與回退方案

### 常見錯誤型態

- **找不到資料但實際應有**：先確認知識庫是否更新到當期；再確認查詢條件中的關鍵字是否與資料一致（注意全形半形、簡繁體差異）
- **數字計算錯誤**：可能模型在處理大量資料時推理失準，建議使用者改用更具體的查詢條件
- **回覆冗長或夾帶推理**：違反問答指引「省略推理、僅保留結果」，需在後台微調提示詞

### 何時應暫時下架 Agent 並回到人工流程

- 連續 3 天以上發生重大錯誤
- 知識庫資料來源遺失或污染
- 使用者大量反映回覆品質下降

### Agent 維運窗口

- **業務負責人**：建議由 採購 主管擔任，負責決定查詢需求變更
- **技術維運**：由 IT 種子團隊成員擔任，負責資料更新、提示詞微調、點數監控
- **變更管理**：每次提示詞修改，至少需重跑情境 #1 與最後一個負向測試，全部通過後才上線

---

## 9. 種子團隊接手提示

### 本 Agent 最常見的提示詞調整方向

當客戶實際使用後，常見的調整方向包含：
- 新增分析維度或查詢條件
- 調整輸出表格欄位
- 改善「無資料」「資料不足」的回覆語氣
- 隨業務變化更新規則或門檻

### 客戶若要新增類似 Agent，可從本 Agent 為起點 copy

本 Agent 可作為「ROI 類型應用」的範本，種子團隊在規劃新 Agent 時，可優先參考本 Agent 的：
- 7 段式 Prompt 結構
- 輸出格式設計
- 驗測情境思路

### 與其他 Agent 的資料依賴關係

- **上游**：依賴各部門資料系統的匯出檔案品質與更新頻率
- **互補**：本 Agent 與工作室內其他 Agent 共用部分知識庫，建議統一資料維護
- **下游**：本 Agent 的輸出可作為其他 Agent 或業務決策的輸入
