# 客戶訂單輸入（題號 #33）知識庫範例

## 基本資訊
- 部門：業務部
- 提案人：巫倩雯
- 架構：D ‧ 流程自動化
- 建議知識庫類型：無（純流程）
- 出席時段：XT 11:00-12:00
- 共通專題：手工重複 ‧ 套用

## 輸入（客戶提供）
將客戶訂單直接上傳AI，讓AI產出所需資料

## 預期輸出（客戶提供）
產出客戶名稱/訂單單號/客戶品號/翔名品號/數量/單價，包含翔名品號是對應1料或N料，可以直接上拋ERP或產出excel供上拋，業助只需核對即可。

## 處理邏輯（客戶提供）
AI辨別以下資訊,
 -客戶別
 -對應翔名品號(利用客用品號帶出，並且分辦是否為集成接單)
 -數量
 -單價
 -客戶預交日

## 客戶提供的資料來源
1.ERP客戶品號資料建立明細
 2.生管提供集成接單明細(定期更新)

## EgentHub 知識庫上傳指引

### 建議的知識庫檔案
- 客戶訂單輸入_系統 MCP（對應系統 API 串接）／類型：MCP 工具
- 操作 SOP.pdf（標準作業流程）／類型：向量檢索
- 主檔資料.sql（相關資料表）／類型：SQL 表格

### 上傳步驟
1. 登入 EgentHub 工作室
2. 進入「知識庫管理」
3. 依上述清單上傳對應檔案
4. 設定每個檔案的處理類型（向量／SQL／全文）
5. 在 Agent 設定中綁定本知識庫

### 注意事項
- L0 級機密圖檔（客戶 CAD、製程參數）請走地端 only 路由
- SQL 表格類請先整理為 CSV / XLSX 格式
- 全文文件建議 PDF / DOCX
- 向量檢索適合長文件、案例庫、SOP
