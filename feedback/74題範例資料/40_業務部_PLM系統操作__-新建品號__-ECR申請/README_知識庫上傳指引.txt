# PLM系統操作  -新建品號  -ECR申請（題號 #40）知識庫範例

## 基本資訊
- 部門：業務部
- 提案人：王雅芝
- 架構：D ‧ 流程自動化
- 建議知識庫類型：SQL+全文 混合
- 出席時段：XT 11:00-12:00
- 共通專題：手工重複 ‧ 套用

## 輸入（客戶提供）
1.初步審核申請人的表格是否正確
 2.依照圖面請AI辨別是否有相似產品及差異點(確認是否真的需要新增)

## 預期輸出（客戶提供）
1.若有提供圖面比對現有品號圖面資料庫,確認是否有類似產品,是否尺寸公差相近
 2.檢查提供的圖面資訊是否和申請表一致(品名、規格、材質)
 3.檢查品名、規格、貨號是否有透漏原廠設備資訊
 4.若有欄位未填寫或是有誤,由AI先反饋給申請人修改,非DCC一筆一筆人工比對

## 處理邏輯（客戶提供）
依照申請人提供的excel及上傳的圖面,逐筆確認必要欄位的正確性

## EgentHub 知識庫上傳指引

### 建議的知識庫檔案
- ERP / PLM 系統 MCP（訂單、品號、ECR 簽核 API）／類型：MCP 工具
- 操作 SOP.pdf（標準操作步驟與安全規範）／類型：向量檢索
- 主檔資料.sql（客戶、品號、料號對照）／類型：SQL 表格

### 上傳步驟
1. 登入 EgentHub 工作室
2. 進入「知識庫管理」
3. 依上述清單上傳對應檔案
4. 設定每個檔案的處理類型（向量／SQL／全文）
5. 在 Agent 設定中綁定本知識庫

### 注意事項
- L0 級機密圖檔（客戶 CAD、製程參數）請走地端 only 路由
- SQL 表格類請先整理為 CSV / XLSX 格式
- 全文文件建議 PDF / DOCX
- 向量檢索適合長文件、案例庫、SOP
