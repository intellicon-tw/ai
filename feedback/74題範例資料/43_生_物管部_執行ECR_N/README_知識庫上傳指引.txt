# 執行ECR/N（題號 #43）知識庫範例

## 基本資訊
- 部門：生/物管部
- 提案人：彭明威
- 架構：D ‧ 流程自動化
- 建議知識庫類型：全文+SQL 混合
- 出席時段：XT 11:00-12:00
- 共通專題：手工重複 ‧ 套用

## 輸入（客戶提供）
1. 核對ECR變更內容與PLM、圖面、EEP三方間的差異性
 
 2. 於PLM中執行ECR發起變更通知
 
 3. 於PLM中執行ECN變更, 完成後自動檢查ERP變更後內容
 
 4. 編寫ECN後通知mail

## 預期輸出（客戶提供）
1. 三方比對資訊供人員檢查
 
 2. 完成ECR變更通知至簽核前檢查步驟
 
 3. 依照ECR變更通知內容執行PLM實際變更
 
 4. 依照範例填入當次變更事項mail

## 處理邏輯（客戶提供）
1. 數據檢索: a. ECR變更項目
  b. 新版圖面內資訊比對(如工程已將圖面進版, 則圖面檔在ECR內可查閱)
  c. PLM現行版次該品號相關資訊
  d. 進入 ERP 查找料號相關資訊, 及在庫、在途訂單、工單、採購單數量
 2. 關聯檢查: 自動確認是否關聯圖面、途程、MIP 自主檢查表, 若有缺失, 須在比對畫面上標記警告
 3. 系統變更: 更新 PLM 內的品名、規格、BOM 內容
  a. 自動檢核更新內容是否符合編碼原則 (料號申請SOP.doc)
  b. 自動將變更後的品名、規格、BOM 內容填入 PLM 指定暫存欄位，並由人員（物管）執行最終確認後一鍵寫入
 4. Mail 發布: 依標準範本自動彙整變更項、物管意見與 ERP 畫面生成 Mail 草稿，待確認後寄出

## EgentHub 知識庫上傳指引

### 建議的知識庫檔案
- ERP / PLM 系統 MCP（訂單、品號、ECR 簽核 API）／類型：MCP 工具
- 操作 SOP.pdf（標準操作步驟與安全規範）／類型：向量檢索
- 主檔資料.sql（客戶、品號、料號對照）／類型：SQL 表格

### 上傳步驟
1. 登入 EgentHub 工作室
2. 進入「知識庫管理」
3. 依上述清單上傳對應檔案
4. 設定每個檔案的處理類型（向量／SQL／全文）
5. 在 Agent 設定中綁定本知識庫

### 注意事項
- L0 級機密圖檔（客戶 CAD、製程參數）請走地端 only 路由
- SQL 表格類請先整理為 CSV / XLSX 格式
- 全文文件建議 PDF / DOCX
- 向量檢索適合長文件、案例庫、SOP
