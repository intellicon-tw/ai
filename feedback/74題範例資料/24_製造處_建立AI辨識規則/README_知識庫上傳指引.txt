## 輸入（客戶提供）
人眼能夠依據前後的文字(圖片)關係鏈，
 自然辨識出此是哪種(資訊、刀具、製程)類型，應該怎麼分類。
 若使用Excel VBA去搜尋，辨識規則建立不易。
 AI辨識文字，缺乏專業領域知識(業界無統一標準)，無法有效辨識文字的意義。

## 預期輸出（客戶提供）
藉由投餵大量資料，先讓AI初步辨識出能正確辨識的資訊 ( 建立記憶 )，
 產生矛盾或不知其意的資訊，統整反問人類，
 人類再給予規則教導與如何分辨 ( 建立規則 )。
 並建立 ( 規則或提示詞 )，讓AI永久記得，
 讓每次的辨識失敗教訓，都記得去尋找以前的紀錄。 ( 記憶失敗經驗 )

## 處理邏輯（客戶提供）
來源的困難處:
 一․刀具品名的命名 : 功能、用途、課別、被加工材、刀具材質…皆混在一起命名
 二․刀具規格的命名 : 一樣的意思,卻有不同的命名方式
 三․各課的製程單，格式不一、命名方式不一。 同樣產品品號的製程單，分類太多種:
  製程別 (A面、B面)、 一對多 (4A面、6B面)、機種版 (銑床版、I800搖籃版、自動化版)、 材質版 (圓料、方料)
 四․SQL資料庫，每一個資料表與屬性代碼的值，為翔名自訂的專屬格位，若非特定人員，不知其數值，代表的是甚麼意思?
 辨識的困難處:
 一․如:鎢鋼端銑刀鍍矽鈦金屬-鎢、鉬 / 金屬加工,鎢鋼端銑刀(鈦金屬用刀/立銑刀)
  金屬加工,鎢鋼端銑刀 / 石墨加工,鎢鋼面銑刀片 / 鎢鋼鑽頭(銑刀柄)無鍍層金屬-鋁 / 鎢鋼成型銑刀無鍍層金屬
 二․如:0.3mm*9*3*2T*45L*120°(ADRL-0030) /TKF12R150-T-16DRPR1725 / DR1.0鑽頭 / DR1.0鑽頭 / D10-R1圓鼻刀
  如:同樣刀具直徑的表示方式: 10.0mm (mm代表) / D10(D開頭代表) / θ10(希臘文Big5十六進位) / Ø10(Unicode十六進位)
 三․不同人手動建立的製程單: 檔名沒規則性(EXCEL檔)，打錯字,多了空格,奇怪的符號，格式不一,俗稱命名，帶有圖片(圖片上有註解文字)
 四․專門的文書人員，會使用ERP查詢,但不知其數值是SQL的哪個資料表的哪個屬性代碼，亦不知怎麼跨關聯性查詢。
  即使是MIS人員，雖知其邏輯原理，但不知其數值代表何意義(非專業領域)。
 範例:本月刀具費爆增，欲查詢原因。
  1.至SQL撈取刀具領用紀錄,找出刀具品號關聯性。
  2.查詢該刀具品號的購入價格，確認是否因原物料漲價而單價增漲?
  3.查詢刀具是使用在哪個產品品號，是否因接了大單而使用量爆增?
  4.查詢領取單位或人員，是否因過度領取造成浪費?
  5.查詢此產品的往年紀錄，所使用的刀具數量或價格，是否僅此次爆增而異常?
  6.將所查詢到的數據，整理統整，併依序排列成報表，讓主管一目了然。
