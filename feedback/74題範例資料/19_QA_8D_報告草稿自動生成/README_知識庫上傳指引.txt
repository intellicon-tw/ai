## 輸入（客戶提供）
工程師已查到客訴原因、Root Cause、改善對策後，仍需要花很多時間整理成 8D 格式，且容易出現英文不一致、內容不完整、D2/D4/D5/D6邏輯不連貫的問題。

## 預期輸出（客戶提供）
AI 自動產出 8D 報告草稿
 包含 D1~D8、問題描述、圍堵措施、Root Cause、改善對策、驗證結果、預防再發措施，以及中英文內容。

## 處理邏輯（客戶提供）
1. 匯入客訴資料
 2. LLM整理資訊
 3. 套用8D模板
 4. 生成D1~D8草稿
 5.工程師確認修改

## 客戶提供的資料來源
1. 8D Template
 2. 歷史8D
 3. 客戶格式
 4. 佐證資料
 5. 檢驗紀錄
 6. Lot資料
 7. SOP/WI

## EgentHub 知識庫上傳指引

### 建議的知識庫檔案
- 歷史 8D 報告庫.txt（完整 8D 報告（D1-D8）、客訴照片）／類型：向量檢索
- QN 資料庫.xlsx（Quality Notification 紀錄）／類型：SQL 表格
- Root Cause 資料庫.txt（根因分析、CAPA 紀錄）／類型：向量檢索
- FA Report 庫.pdf（Failure Analysis 報告）／類型：向量檢索

### 上傳步驟
1. 登入 EgentHub 工作室
2. 進入「知識庫管理」
3. 依上述清單上傳對應檔案
4. 設定每個檔案的處理類型（向量／SQL／全文）
5. 在 Agent 設定中綁定本知識庫

### 注意事項
- L0 級機密圖檔（客戶 CAD、製程參數）請走地端 only 路由
- SQL 表格類請先整理為 CSV / XLSX 格式
- 全文文件建議 PDF / DOCX
- 向量檢索適合長文件、案例庫、SOP
