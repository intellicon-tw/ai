# 歸檔檢查小助手（題號 #53）知識庫範例

## 基本資訊
- 部門：機構設計課
- 提案人：葉于萱
- 架構：D ‧ 流程自動化
- 建議知識庫類型：SQL 表格
- 出席時段：S7 14:20-15:00
- 共通專題：— ‧ 獨立題

## 輸入（客戶提供）
輸入料號即自動檢查檔案是否製作歸檔(MIP,PMP,PDF,PDF過時保留)

## 預期輸出（客戶提供）
輸入料號>檢查>列表Y/N>列N的品項一鍵發信提醒

## 處理邏輯（客戶提供）
1.輸入料號
 2.*如只輸入product code*超過一個的品號，提供下拉式選項
 3.反查所有上下階半品
 4.檢查是否有歸檔MIP,PMP,PDF(檔案限1個，需完全符合名字)
 5.檢查MIP原廠料號版次是否與系統一致
 6.檢查檔案位置是否正確
 7.檢查是否有將所有舊檔案(含舊料號)過時保留
 8.列表Y/N
 9.列N的品項-發信鈕提醒繪圖人員

## 客戶提供的資料來源
1.product code的邏輯(PLM)
 2.反查上下階半品/原廠料號版次的邏輯(PLM)
 3.檔案位置檢查的邏輯(共用槽)
 4.過時保留檢查的邏輯(共用槽)
 5.新舊料號對照表
 6.列表Y/N的格式
 7.發信格式

## EgentHub 知識庫上傳指引

### 建議的知識庫檔案
- 歸檔檢查小助手_系統 MCP（對應系統 API 串接）／類型：MCP 工具
- 操作 SOP.pdf（標準作業流程）／類型：向量檢索
- 主檔資料.sql（相關資料表）／類型：SQL 表格

### 上傳步驟
1. 登入 EgentHub 工作室
2. 進入「知識庫管理」
3. 依上述清單上傳對應檔案
4. 設定每個檔案的處理類型（向量／SQL／全文）
5. 在 Agent 設定中綁定本知識庫

### 注意事項
- L0 級機密圖檔（客戶 CAD、製程參數）請走地端 only 路由
- SQL 表格類請先整理為 CSV / XLSX 格式
- 全文文件建議 PDF / DOCX
- 向量檢索適合長文件、案例庫、SOP
