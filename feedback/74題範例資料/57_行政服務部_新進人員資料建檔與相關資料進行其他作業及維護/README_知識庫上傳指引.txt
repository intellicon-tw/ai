# 新進人員資料建檔與相關資料進行其他作業及維護（題號 #57）知識庫範例

## 基本資訊
- 部門：行政服務部
- 提案人：王湘渝
- 架構：D ‧ 流程自動化
- 建議知識庫類型：SQL+全文 混合
- 出席時段：XT 11:00-12:00
- 共通專題：手工重複 ‧ 套用

## 輸入（客戶提供）
建置網頁版人事基本資料表，並確保資料傳輸安全，AI協助整理學歷、工作經歷與個人資料，減少重複人工輸入作業。

## 預期輸出（客戶提供）
自動整理新進人員學歷、工作經歷與個人基本資料，提供HRM建檔、核薪與考核表填寫參考，減少重複輸入時間。

## 處理邏輯（客戶提供）
1.求職者於網站上直接填寫人事基本資料表
 2.AI辨識並整理學歷、工作經歷與個人資料並依欄位分類產出建檔資料
 3.提供HR進行HRM系統建檔、核薪與考核表填寫參考(直接導出模板)

## 客戶提供的資料來源
人事基本資料表、HRM導入模板

## EgentHub 知識庫上傳指引

### 建議的知識庫檔案
- HR 系統 MCP（員工資料、組織異動 API）／類型：MCP 工具
- 新進報到清單.pdf（需建立的系統與帳號清單）／類型：全文查找
- 員工主檔.sql（人事資料表）／類型：SQL 表格

### 上傳步驟
1. 登入 EgentHub 工作室
2. 進入「知識庫管理」
3. 依上述清單上傳對應檔案
4. 設定每個檔案的處理類型（向量／SQL／全文）
5. 在 Agent 設定中綁定本知識庫

### 注意事項
- L0 級機密圖檔（客戶 CAD、製程參數）請走地端 only 路由
- SQL 表格類請先整理為 CSV / XLSX 格式
- 全文文件建議 PDF / DOCX
- 向量檢索適合長文件、案例庫、SOP
