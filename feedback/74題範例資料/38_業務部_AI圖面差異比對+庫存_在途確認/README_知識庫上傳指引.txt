# AI圖面差異比對+庫存/在途確認（題號 #38）知識庫範例

## 基本資訊
- 部門：業務部
- 提案人：詹翊佳
- 架構：D ‧ 流程自動化
- 建議知識庫類型：SQL+全文 混合
- 出席時段：XT 11:00-12:00
- 共通專題：手工重複 ‧ 套用

## 輸入（客戶提供）
將客戶圖面上傳，自動疊合兩張圖檔比對差異

## 預期輸出（客戶提供）
自動產出差異報告，AI生成差異清單和標註圖；
 確認廠內是否有在途與庫存

## 處理邏輯（客戶提供）
將客戶圖面上傳，自動疊合兩張圖檔(如PDF、CAD或照片)，標示出新舊版本的差異。
 AI自動查詢庫存和在途，供業務判斷是否需要同時進版

## 客戶提供的資料來源
1. 客戶圖面
 2. 翔名品號資訊(品號、品名、規格)
 3. 即時各倉庫存量
 4. SFT

## EgentHub 知識庫上傳指引

### 建議的知識庫檔案
- AI圖面差異比對+庫存/在途確認_系統 MCP（對應系統 API 串接）／類型：MCP 工具
- 操作 SOP.pdf（標準作業流程）／類型：向量檢索
- 主檔資料.sql（相關資料表）／類型：SQL 表格

### 上傳步驟
1. 登入 EgentHub 工作室
2. 進入「知識庫管理」
3. 依上述清單上傳對應檔案
4. 設定每個檔案的處理類型（向量／SQL／全文）
5. 在 Agent 設定中綁定本知識庫

### 注意事項
- L0 級機密圖檔（客戶 CAD、製程參數）請走地端 only 路由
- SQL 表格類請先整理為 CSV / XLSX 格式
- 全文文件建議 PDF / DOCX
- 向量檢索適合長文件、案例庫、SOP
