# Agent 09 — Line@ 客戶耗材詢問助手（業務部 / 黃子濬）

## 一、可行性
**可做（半人工輔助）。** 「比對客戶採購紀錄＋推薦加購＋產出話術」LLM 都能做，是高頻、高 ROI 的場景（每月 100 筆 × 15 人 = 高效益）。卡關不是技術，是「Line@ 對話從哪裡進來」。

## 二、模型建議
- 預設：`claude-sonnet-4-6`（中文話術自然、能依語境調整親密度）

## 三、知識庫設定

| 知識庫名稱 | 類型 | 狀態 |
|-----------|------|------|
| 客戶機台與耗材使用記錄 | 表格查詢 | ✅ 與 Agent 08 共用（待 IT 從 EIP 匯出） |
| 客戶採購歷史 | 表格查詢 | ✅ 與 Agent 08 共用 |
| 印表機耗材規格表 | 表格查詢 | ✅ 與 Agent 03 共用 |
| 優惠活動表 | 表格查詢 | ✅ 與 Agent 03 共用 |
| Line@ 話術指南 | 全文查找 | **待黃子濬補**（附範本） |

## 四、結構性限制與替代方案

### 限制：Line@ 對話需「人工貼上」（短期）／串接 MCP（長期）
本 Prompt 設計為「業務貼上客戶訊息 → AI 給話術 → 業務貼回 Line@」的半自動模式。**完全自動化的兩個方案**：
- **方案 A（建議優先）**：保留人工貼話術。優點：業務有完全控制權、不會誤發、可加修飾。每筆處理時間從 5 分鐘 → 1 分鐘。
- **方案 B**：請 IT 開發 Line Notify／Line Messaging API MCP，讓 Agent 直接從 Line@ 拉訊息＋發訊息。要做但需開發資源。

短期先用方案 A 上線，業務的 ROI 已經很高。

## 五、待釐清事項
1. 業務客戶通常用「型號代碼」還是「我要黑色的那個」這種口語？影響 Prompt 對「模糊描述」的處理深度。
2. 加購推薦的策略：寬鬆（只要週期接近就推）還是保守（嚴格判斷才推）？目前 Prompt 預設保守（30 天內到期才推）。
3. 是否要區分「老客戶」與「新客戶」的話術風格？

## 六、驗測情境建議
1. 業務貼「TP0003-45 客戶說：黑色碳粉快沒了」→ 預期：自動辨識為 K.M C368 黑色碳粉匣，給報價＋優惠＋加購建議。
2. 客戶說：「上次那個黃色的還有貨嗎？」→ 預期：查歷史採購找出「黃色」對應品名，給報價。
3. 客戶說：「現在有什麼活動？」→ 預期：列出當下適用的優惠活動。
4. 同一客戶在不同訊息中反覆詢問 → 預期：話術不死板，能依累積對話脈絡調整。
