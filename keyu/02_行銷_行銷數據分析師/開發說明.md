# Agent 02 — 行銷數據分析師（行銷部 / 蘇郁婷）

## 一、可行性
**部分可做。** 「資料分析、找異常、給建議、輸出 Markdown 報表」可做。但原任務描述「自動生成簡報」**EgentHub 平台不支援直接產出 .pptx**，需改為輸出 Markdown 報告由人工套用簡報模板（或之後接 MCP 工具）。

## 二、模型建議
- 預設：`claude-sonnet-4-6`（資料分析、結構化輸出穩）
- 高要求情境（含複雜歸因分析）：`claude-opus-4-6`

## 三、知識庫設定
需要建立 3 個「表格查詢知識庫」，目前**都待補**：

| 知識庫名稱 | 來源 | 狀態 |
|-----------|------|------|
| 銷售數據表 | 蝦皮後台／官網後台匯出 | **待客戶確認實際欄位** |
| 流量與轉換表 | GA4、FB Ads 後台匯出 | **待客戶確認實際欄位** |
| 檔期活動表 | 行銷部活動規劃表 | **待客戶確認實際欄位** |

Drive 連結內提供了 3-4 月各項數據，但**實際欄位結構未知**，需要蘇郁婷確認：
- 蝦皮後台匯出的銷售報表「實際欄位名稱」是什麼？
- 流量資料來自 GA4 還是蝦皮自己的後台？
- 廣告花費資料由誰彙整？頻率？

## 四、限制與替代方案

### 限制 1：無法直接產出 .pptx
**替代**：Agent 輸出 Markdown 報表，每週／每月由人工套用簡報模板。或之後請 IT 接「Markdown → pptx」的 MCP 工具。

### 限制 2：歷史回朔需要長期累積資料
建議在 EgentHub 上每週／每月固定上傳當期資料，建立累積基礎；初期 2-3 個月內，可能因比對基期不足而給出建議有限。

### 限制 3：Token 限制
若單次資料量超過 20K tokens，需要先做摘要或拆批分析。建議：原始數據在 Excel 端先做「日 → 週」或「商品 → 類別」彙總，再上傳到表格知識庫。

## 五、待釐清事項
1. 「警戒線」的標準由誰定義？目前 Prompt 設預設值（銷量降幅 ≥20%、轉換率 <1%、ROI <1），需確認可否接受。
2. 「自動生成簡報」是必要還是 nice-to-have？若是必要，要排 MCP 工具開發。
3. 分析頻率：每週還是每月？

## 六、驗測情境建議
1. 「分析 4 月銷售，找出表現最差的 5 樣商品」
2. 「618 預熱活動和 6 月畢業季哪個檔期表現好？」
3. 「FB Ads vs Google Ads，這個月哪個 ROI 高？」
4. 「我沒上傳數據，幫我看一下」→ 預期：禮貌請補資料
