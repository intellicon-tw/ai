## 二、建議上線順序

### 階段 1（本週可上線）— 可立即進場的綠燈題
- **Agent 03 — 印表機耗材詢答**：資料齊全，作為首發示範。
- **Agent 10 — Line@ 四色人格分析**：知識庫範本完整，LLM 強項，可立刻測。

### 階段 2（補完知識庫後 1–2 週可上線）
- **Agent 04 — 電商文案 SEO**：王鈺婷補品牌語氣 + 3-5 篇成功文案範例。
- **Agent 09 — Line@ 客戶耗材詢問**：業務團隊試用半自動模式。
- **Agent 02 — 行銷數據分析師**：蘇郁婷確認數據欄位、上傳當月銷售流量資料。

### 階段 3（需 IT 或客戶協作）
- **Agent 01 — 設備排查客服助手**：IT 匯出保固清單、客服彙整故障代碼表、上傳維修手冊。
- **Agent 05 — 差旅費審核**：採「歷史平均里程」可上線；要精確 Google Map 需開發 MCP。
- **Agent 06 — 出貨異常查核**：建議先用 Python 做大批量比對，Agent 做摘要。
- **Agent 07 — 招募流程自動化**：拆解後做履歷篩選＋信件草稿＋測驗分析。
- **Agent 08 — 業務本週行程預排**：IT 每週日匯出 EIP 三張表 CSV，業務週一早上即可用。
- **Agent 11 — 業務動能分析**：先與業務團隊溝通定位（不是抓人），再上線。

## 三、共通限制（EgentHub 平台層級）

以下限制對所有 Agent 都適用：

1. **無法直接產出 .pptx／.xlsx 檔案**：可輸出 Markdown 表格，使用者在 EgentHub 介面下載 .xlsx。
2. **對話框不支援上傳 .xlsx／.csv**：替代為「表格查詢知識庫」、貼上文字、或上傳 PDF/TXT。
3. **單次上下文約 20K tokens**：資料過大時需先摘要或分批處理。
4. **無原生 Google Map／日曆／104／Line@ 整合**：需透過 MCP 或外部工具串接。
5. **EIP 系統無 API**：影響 Agent 01、08、09、11 四個 Agent — 短期一律靠 IT 定期匯出 CSV 上傳到表格知識庫。

## 五、檔案結構說明

每個 Agent 子資料夾包含：
- `prompt.md` — 可直接複製到 EgentHub 後台的系統提示詞
- `開發說明.md` — 該 Agent 的可行性、知識庫設定、待補項、驗測情境
- `知識庫/` — 該 Agent 需要的知識庫檔案（已備齊為 CSV／TXT；待補的為 `_範本_` 開頭的範本檔）
