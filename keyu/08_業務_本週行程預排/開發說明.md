# Agent 08 — 業務人員本週行程預排（業務部 / 黃子濬）

## 一、可行性
**部分可做。** 客戶優先序評分、地理區塊分組、報價單生成都可做。卡關在「EIP 無 API」與「無原生 Google Map 路徑最佳化」。本 Prompt 採用「地理區塊聚類」替代精確路徑計算。

## 二、模型建議
- 預設：`claude-sonnet-4-6`（多表整合、評分排序、長文輸出穩）

## 三、知識庫設定

| 知識庫名稱 | 類型 | 狀態 |
|-----------|------|------|
| 客戶機台與耗材使用記錄 | 表格查詢 | **待 IT 從 EIP 匯出** |
| 客戶採購歷史 | 表格查詢 | **待 IT 從 EIP 匯出** |
| 報價單格式範本 | 全文查找 | **待黃子濬補**（附範本） |
| 耗材價格表 | 表格查詢 | ✅ 可重用 Agent 03 的印表機耗材規格表 |

## 四、結構性限制與替代方案

### 限制 1：EIP 系統無 API
與 Agent 01、11 相同。**短期方案（建議）**：請 IT 每週日晚間／週一早上匯出三張表 CSV 並上傳至 EgentHub 知識庫，業務早上 8 點即可使用。**長期方案**：開發 EIP MCP。

### 限制 2：無 Google Map 路徑最佳化
原始任務希望「安排路線」。EgentHub 沒有原生地圖工具，無法做 TSP（旅行推銷員問題）最佳解。**本 Prompt 採用**：
- 用「地理區塊」欄位分組（業務區、台北市東區、台北市西區、新北板橋、桃園等預先標註）
- 每天集中拜訪同一區塊
- 區塊內按地址順序排（不做精確距離計算）

若要更精確，未來可：(A) 串 Google Maps API（IT 開發 MCP）；(B) 業務同仁拿到名單後自己用 Google Maps 規劃。

## 五、客戶優先序評分權重
本 Prompt 預設權重如下，黃子濬可調整：
- 預估下次採購日落在本週內：+40 分
- 已過預估採購日（逾期）：+30 分
- 上次拜訪 > 60 天：+15 分
- 高採購頻次客戶（過去 3 個月 ≥ 3 次採購）：+15 分

## 六、待釐清事項
1. 「預估下次採購日」如何計算？目前 Prompt 假設用「上次採購日 + 平均耗材週期」。實際公司怎麼算？
2. 業務的「地理區塊」歸類由誰維護？需建主檔。
3. 是否要把「庫存狀況」納入優先序？目前未納入。
4. 報價單是否需要客戶 logo 或業務簽名？這部分 Agent 無法做，需要再人工套版。

## 七、驗測情境建議
1. 「我是台北東區業務黃子濬，幫我排這週」 → 預期：列出該區塊優先序前 15 名 + 5 天行程 + 重點客戶報價單。
2. 「下週還有 3 天，幫我選 9 個客戶」 → 預期：按業務指定天數排，每天 3 個客戶。
3. 「沒有上傳客戶資料」 → 預期：禮貌請補 / 提示要先準備好 CSV。
