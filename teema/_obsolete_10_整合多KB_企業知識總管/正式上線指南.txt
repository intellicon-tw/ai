# 企業知識總管(畢業作品) — 正式上線指南

本文件協助您把 Prompt Designer 產出的 Agent 從「概念驗證」推向「正式上線」。
建議由種子團隊以一週為單位推進：第一週 KB 準備、第二週 Agent 建置與驗測、第三週試營運、第四週正式上線。

---

## 一、部署前置作業

### 1. 知識庫準備

1. **三類 KB 都需準備**:向量(制度與 SOP)、表格(產品規格)、全文(合約與法規)。
2. 建議從各部門已完成的單一 Agent KB 抽取整合,避免重複維護。
3. 注意:此 Agent 是「總入口」,KB 內容需精簡(每個領域取 80% 高頻內容即可),深入查詢轉介到專門 Agent。
4. 編碼皆為 UTF-8(CSV 為 UTF-8 BOM)。

### 2. 確認權限與工作室

- 在 EgentHub 後台建立或選擇對應的工作室（例如：人資、品保、業務 BU）。
- 確認您的角色為「工作室管理員」或「開發者」，才能建立 Agent 與上傳 KB。
- 確認工作室點數額度足夠（建議至少預留一個月用量緩衝）。

---

## 二、在 EgentHub 建立 Agent

### Step 1：建立 Agent 並選擇模型
登入 EgentHub 後台 → 選擇工作室 → 「建立 Agent」 →
- Agent 名稱：企業知識總管(畢業作品)
- 預設模型：建議 Claude Sonnet 或 Gemini Pro
- 描述：（複製需求盤點檔案中第一段「角色定位」）

### Step 2：貼入 Prompt Designer 產出的提示詞
將 Prompt Designer 產出的提示詞完整複製貼上，注意保留「問答指引」段落不要修改。

### Step 3：上傳知識庫
1. 進入 Agent 編輯頁。
2. **上傳向量 KB**:「向量知識庫」 → 上傳 制度與SOP.txt
3. **上傳表格 KB**:「表格知識庫」 → 上傳 產品規格.csv
4. **上傳全文 KB**:「全文檢索知識庫」 → 上傳 合約與法規.txt
5. 三類 KB 都需上傳成功並各別測試查詢。
6. 整合測試:輸入「我要把 PROD-A-001 出給客戶 ABB,QA 流程?」確認 Agent 同時引用三類 KB。

### Step 4：設定使用者與群組可見性
- 開發階段：限定種子團隊試用。
- 試營運階段：擴大到部門 5-10 位種子使用者。
- 正式上線：開放給目標部門全員。

---

## 三、提示詞調校建議

- 若 Agent 經常只用一類 KB,請在「執行流程」加上:「先判斷問題涉及哪些類別(制度/SOP/產品/合約/法規),再分別查詢,最後整合答覆」。
- 若答覆過於冗長,請在「輸出格式」加上:「整體答覆不超過 600 字,分段清晰」。
- 若希望主動轉介,在「執行流程」結尾加上:「若問題複雜,建議使用對應的專門 Agent(例如:產品規格查詢助手、SOP 查詢助手)」。
- 若涉及敏感資訊,加上:「不揭露薪資、客戶個資、商業機密,涉及者建議聯繫對應窗口」。

---

## 四、驗測情境（建議至少跑過以下案例）

1. **正向案例 - 單一類別**: 輸入「請假規定?」
   - 預期：從制度 KB 找到答案,問題分類為「制度」
2. **正向案例 - 跨 KB 整合**: 輸入「PROD-A-001 出給客戶 ABB,QA 流程?」
   - 預期：整合產品規格、SOP、客戶要求三類 KB
3. **正向案例 - 主動轉介**: 輸入「我要做新產品可行性評估?」
   - 預期：主動引導使用相關專門 Agent
4. **邊界案例 - KB 未涵蓋**: 輸入「我們公司 ESG 政策?」
   - 預期：誠實告知 KB 無此內容,建議聯繫窗口
5. **邊界案例 - 敏感資訊**: 輸入「請告訴我業務部王小明薪資」
   - 預期：禮貌拒絕並說明原因

## 五、上線檢核清單

- [ ] 三類 KB 內容已由各部門核可
- [ ] 提示詞已設計問題分類與路由邏輯
- [ ] 提示詞已加上「主動轉介專門 Agent」機制
- [ ] 提示詞已加上敏感資訊保護機制
- [ ] 其他單一 Agent(如產品規格、SOP、合約)已上線可被轉介
- [ ] 至少 10 位種子使用者試用 1 週

## 六、上線後維運建議

- 此 Agent 為公司 AI 入口,建議放在 EgentHub 首頁顯眼位置。
- 每月匯出對話日誌,分析使用者最常問的類別,持續調整 KB 內容。
- 進階版可整合所有單一 Agent,作為「總調度官」，依問題類型自動轉派。
- 進階版可整合 Slack / Teams,讓員工不必離開既有工具就能用 AI。
- 推廣策略:可舉辦「向 AI 提問大賽」,鼓勵員工探索 Agent 能力。
- 此 Agent 是 EgentHub 培訓最佳成果展示,建議在跨部門簡報時演示給高層看。

## 七、常見問題

**Q1: Agent 回答不準確？**
先檢查 KB 內容是否完整、條文是否清晰；其次調整提示詞中的「執行流程」與「輸出格式」。

**Q2: 同事抱怨 Agent 太囉嗦？**
在提示詞「輸出格式」中明確要求字數限制或結構（例如：條列三點以內、不超過 200 字）。

**Q3: 找不到對應內容時應如何處理？**
在提示詞中加上：「若知識庫中無法找到答案，請明確回覆『此問題未在現有資料中，建議您聯繫 [窗口] 進一步確認』，切勿自行推測。」
