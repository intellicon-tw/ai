# B2B 跨語溝通助手 — 正式上線指南

本文件協助您把 Prompt Designer 產出的 Agent 從「概念驗證」推向「正式上線」。
建議由種子團隊以一週為單位推進：第一週 KB 準備、第二週 Agent 建置與驗測、第三週試營運、第四週正式上線。

---

## 一、部署前置作業

### 1. 知識庫準備

**本題目不需要任何知識庫!** 純提示工程 + Few-shot 範例即可完成。

選擇性準備(進階版):
1. 公司品牌語氣指南(可作為向量 KB,確保語氣一致性)。
2. 過往優質譯文範例(在提示詞中作為 Few-shot)。
3. 公司常用產品 / 技術專有名詞中英對照表(避免 AI 翻譯錯誤)。

### 2. 確認權限與工作室

- 在 EgentHub 後台建立或選擇對應的工作室（例如：人資、品保、業務 BU）。
- 確認您的角色為「工作室管理員」或「開發者」，才能建立 Agent 與上傳 KB。
- 確認工作室點數額度足夠（建議至少預留一個月用量緩衝）。

---

## 二、在 EgentHub 建立 Agent

### Step 1：建立 Agent 並選擇模型
登入 EgentHub 後台 → 選擇工作室 → 「建立 Agent」 →
- Agent 名稱：B2B 跨語溝通助手
- 預設模型：建議 Claude Sonnet 或 Gemini Pro
- 描述：（複製需求盤點檔案中第一段「角色定位」）

### Step 2：貼入 Prompt Designer 產出的提示詞
將 Prompt Designer 產出的提示詞完整複製貼上，注意保留「問答指引」段落不要修改。

### Step 3：上傳知識庫
**本 Agent 不需要上傳任何知識庫,直接設定提示詞即可。**

關鍵設定:
1. 在 Agent 編輯頁,選擇 **Claude Sonnet** 或 **GPT-4o**(中英日跨語表現佳)。
2. 將需求盤點中的提示詞整段貼入「系統提示詞」。
3. **強烈建議**:在提示詞「執行流程」末尾加入 **2-3 個完整 Few-shot 範例**(中文 → 英文 / 日文),讓 AI 模仿您公司的慣用語氣。
4. 儲存設定,進入測試。

### Step 4：設定使用者與群組可見性
- 開發階段：限定種子團隊試用。
- 試營運階段：擴大到部門 5-10 位種子使用者。
- 正式上線：開放給目標部門全員。

---

## 三、提示詞調校建議

- 若譯文太生硬,請在「角色定位」加上:「以英文母語商業人士的習慣表達,避免直譯式句子」。
- 若漏掉文化提醒,請在「執行流程」加上:「翻譯註記必須包含至少 1 點文化禮節提醒」。
- 若使用者經常忘記提供收件對象,請在「執行流程」開頭加上:「若使用者沒提供目標語言或收件對象,禮貌請求補充,不可自行假設」。
- 若 AI 自行修改數字、日期,請在「限制條件」強調:「金額、交期、合約條款的數字、日期務必逐字對應,不得修改」。
- 若想要日文版本,請在 Few-shot 範例中加入中→日範例,並在「角色定位」明確列出支援的語言。

---

## 四、驗測情境（建議至少跑過以下案例）

1. **正向案例 - 交期延誤通知**: 中文草稿(交期延誤)+ 英文 + 海外客戶(長期合作)
   - 預期：情境判讀為『交期延誤告知』,譯文柔軟、含文化提醒
2. **正向案例 - 催款 Email**: 中文草稿(催收逾期款項)+ 英文 + 客戶 PM
   - 預期：將『催款』改寫為較柔性的『詢問付款進度』,保留主訊息
3. **正向案例 - 雙版本**: 中文草稿(會議邀約)+ 英文 + 收件對象未指定
   - 預期：禮貌請求補充收件對象資訊,提供初步雙版本草稿
4. **邊界案例 - 缺漏資訊**: 純文字輸入「翻譯成英文」(沒給草稿)
   - 預期：禮貌請求使用者補充中文草稿與收件對象
5. **邊界案例 - 數字保留**: 草稿提及「8/15 出貨 5,000 pcs」
   - 預期：譯文中數字日期完全一致,不修改不四捨五入
6. **邊界案例 - 語氣過生硬**: 草稿:「你們答應的交期沒做到,這是不可接受的」
   - 預期：翻譯註記主動提醒原文語氣偏強,提供軟化建議

## 六、上線後維運建議

- 蒐集使用者實際送出的最終版本,觀察與 AI 草稿差異,反向優化 Few-shot 範例。
- 建議搭配主題 07(B2B 智慧客服)使用,海外客戶來信 → 客服初步分類 → 跨語溝通助手草擬回覆。
- 進階版可整合 Outlook / Gmail MCP,直接讀取收件匣與寫入草稿匣。
- 可建立各情境(催款、致歉、技術討論)的專屬版本,每個都有自己的 Few-shot 範例。
- 本題型也可拓展至日報週報、合約往返摘要、技術文件翻譯等場景。

## 七、常見問題

**Q1: Agent 回答不準確？**
先檢查 KB 內容是否完整、條文是否清晰；其次調整提示詞中的「執行流程」與「輸出格式」。

**Q2: 同事抱怨 Agent 太囉嗦？**
在提示詞「輸出格式」中明確要求字數限制或結構（例如：條列三點以內、不超過 200 字）。

**Q3: 找不到對應內容時應如何處理？**
在提示詞中加上：「若知識庫中無法找到答案，請明確回覆『此問題未在現有資料中，建議您聯繫 [窗口] 進一步確認』，切勿自行推測。」
