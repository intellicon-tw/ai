# RFQ 應答草稿助手 — 正式上線指南

本文件協助您把 Prompt Designer 產出的 Agent 從「概念驗證」推向「正式上線」。
建議由種子團隊以一週為單位推進：第一週 KB 準備、第二週 Agent 建置與驗測、第三週試營運、第四週正式上線。

---

## 一、部署前置作業

### 1. 知識庫準備

1. **表格 KB**:由產品 PM 整理產品報價表,每月更新。
2. **向量 KB**:由業務主管彙整公司簡介、認證能力、QA 流程、付款條件、常見 FAQ。
3. 注意:報價表中的價格區間僅供業務內部參考,不得對外直接送出。
4. 兩種 KB 編碼皆為 UTF-8。

### 2. 確認權限與工作室

- 在 EgentHub 後台建立或選擇對應的工作室（例如：人資、品保、業務 BU）。
- 確認您的角色為「工作室管理員」或「開發者」，才能建立 Agent 與上傳 KB。
- 確認工作室點數額度足夠（建議至少預留一個月用量緩衝）。

---

## 二、在 EgentHub 建立 Agent

### Step 1：建立 Agent 並選擇模型
登入 EgentHub 後台 → 選擇工作室 → 「建立 Agent」 →
- Agent 名稱：RFQ 應答草稿助手
- 預設模型：建議 Claude Sonnet 或 Gemini Pro
- 描述：（複製需求盤點檔案中第一段「角色定位」）

### Step 2：貼入 Prompt Designer 產出的提示詞
將 Prompt Designer 產出的提示詞完整複製貼上，注意保留「問答指引」段落不要修改。

### Step 3：上傳知識庫
1. 進入 Agent 編輯頁。
2. **上傳表格 KB**:「表格知識庫」 → 上傳 產品報價.csv。
3. **上傳向量 KB**:「向量知識庫」 → 上傳 公司FAQ.txt。
4. 兩種 KB 都需上傳成功並測試查詢。
5. 測試:輸入「客戶詢問 12V 變壓器,要 UL 認證」,確認 Agent 能同時引用兩種 KB。

### Step 4：設定使用者與群組可見性
- 開發階段：限定種子團隊試用。
- 試營運階段：擴大到部門 5-10 位種子使用者。
- 正式上線：開放給目標部門全員。

---

## 三、提示詞調校建議

- 若 Agent 偏向只用其中一種 KB,請在「執行流程」加上:「先從表格 KB 查產品,再從向量 KB 查公司能力與 FAQ,最後整合為應答草稿」。
- 若應答草稿過於冗長,請在「輸出格式」加上:「整體草稿不超過 600 字,清晰分段」。
- 若希望更貼近業務口吻,請在「角色定位」加上:「語氣專業而友善,符合台灣 B2B 業務溝通風格」。
- 若使用者經常忘了確認重要事項,在執行流程加上:「每份草稿結尾必須列出 3-5 項『PM 確認事項』」。

---

## 四、驗測情境（建議至少跑過以下案例）

1. **正向案例 - 標準 RFQ**: 輸入「客戶要 12V 變壓器 5000 pcs UL 認證」
   - 預期：正確推薦 PROD-A-001,引用報價表 + FAQ 中的付款、產能、認證資訊
2. **正向案例 - 公司能力詢問**: 輸入「新客戶詢問我司 QA 與認證能力」
   - 預期：從 FAQ 整理 ISO 9001、IATF 16949、三道品檢流程
3. **邊界案例 - 不存在規格**: 輸入「客戶要 100V/30A 變壓器」
   - 預期：誠實告知產品線無此規格,建議洽 PM 評估客製
4. **邊界案例 - 大訂單與特殊條款**: 輸入「客戶要 50 萬 pcs,要求 90 天月結」
   - 預期：提示需業務主管核可,並列出 PM 確認事項

---

## 五、上線檢核清單

- [ ] 產品報價表已由 PM 確認,每月更新機制建立
- [ ] 公司 FAQ 已由業務主管核可
- [ ] 提示詞已強調「正式報價需業務主管核可」
- [ ] 提示詞已要求結尾列出 PM 確認事項
- [ ] 至少 3 位業務 PM 種子使用者試用 1 週

---

## 六、上線後維運建議

- 每月由 PM 助理更新報價表 CSV。
- 每季由業務主管檢視 FAQ 是否需更新。
- 蒐集業務送出客戶的最終版本與草稿差異,持續優化提示詞。
- 進階版:整合 CRM 與客戶歷史訂單,做到一鍵個人化。

---

## 七、常見問題

**Q1: Agent 回答不準確？**
先檢查 KB 內容是否完整、條文是否清晰；其次調整提示詞中的「執行流程」與「輸出格式」。

**Q2: 同事抱怨 Agent 太囉嗦？**
在提示詞「輸出格式」中明確要求字數限制或結構（例如：條列三點以內、不超過 200 字）。

**Q3: 找不到對應內容時應如何處理？**
在提示詞中加上：「若知識庫中無法找到答案，請明確回覆『此問題未在現有資料中，建議您聯繫 [窗口] 進一步確認』，切勿自行推測。」
