# ERP 數據分析助手 — 正式上線指南

本文件協助您把 Prompt Designer 產出的 Agent 從「概念驗證」推向「正式上線」。
建議由種子團隊以一週為單位推進：第一週 KB 準備、第二週 Agent 建置與驗測、第三週試營運、第四週正式上線。

---

## 一、部署前置作業

### 1. 知識庫準備

1. 從 ERP 匯出近 24 個月的銷售與採購資料。
2. **資料清洗**:
   - 第一列為單一標題列
   - 統一日期 YYYY-MM 或 YYYY-MM-DD
   - 金額以純數字呈現(不含逗號、貨幣符號)
   - 客戶代碼用代號(C-001),避免直接公開客戶名稱
3. 由財會主管核可資料區間與欄位。
4. 編碼 UTF-8 BOM。

### 2. 確認權限與工作室

- 在 EgentHub 後台建立或選擇對應的工作室（例如：人資、品保、業務 BU）。
- 確認您的角色為「工作室管理員」或「開發者」，才能建立 Agent 與上傳 KB。
- 確認工作室點數額度足夠（建議至少預留一個月用量緩衝）。

---

## 二、在 EgentHub 建立 Agent

### Step 1：建立 Agent 並選擇模型
登入 EgentHub 後台 → 選擇工作室 → 「建立 Agent」 →
- Agent 名稱：ERP 數據分析助手
- 預設模型：建議 Claude Sonnet 或 Gemini Pro
- 描述：（複製需求盤點檔案中第一段「角色定位」）

### Step 2：貼入 Prompt Designer 產出的提示詞
將 Prompt Designer 產出的提示詞完整複製貼上，注意保留「問答指引」段落不要修改。

### Step 3：上傳知識庫
1. 進入 Agent 編輯頁 → 「**表格查詢知識庫**」 → 「上傳 CSV」。
2. 上傳 ERP_銷售與採購數據.csv。
3. 確認欄位類型辨識正確(年月為文字或日期、金額為數值)。
4. 測試查詢「2026 Q1 各 BU 銷售」確認 Agent 能正確聚合。

### Step 4：設定使用者與群組可見性
- 開發階段：限定種子團隊試用。
- 試營運階段：擴大到部門 5-10 位種子使用者。
- 正式上線：開放給目標部門全員。

---

## 三、提示詞調校建議

- 若 Agent 計算總和有誤,請在「執行流程」加上:「先讀取 CSV 中相關欄位,再以分組聚合(GROUP BY)計算,不得自行推估」。
- 若 Agent 把資料區間誤判,請加上:「明確標註資料區間,例如『2026-01 ~ 2026-03』」。
- 若涉及預測,加上免責聲明:「此為基於現有資料的觀察,未經財務部正式驗證」。
- 若涉及客戶名稱,務必使用代碼,加上:「客戶以代碼回覆,不揭露真實名稱」。

---

## 四、驗測情境（建議至少跑過以下案例）

1. **正向案例 - 季度比較**: 輸入「2026 Q1 各 BU 銷售比較」
   - 預期：正確計算 BU-A/B/C Q1 加總並列表,標註資料區間
2. **正向案例 - YoY**: 輸入「BU-A 2026 vs 2025 同期銷售」
   - 預期：正確計算 YoY 變化
3. **正向案例 - 應收帳款**: 輸入「應收帳款超過 60 天的客戶」
   - 預期：列出 C-006、C-007、C-009 等
4. **邊界案例 - 預測詢問**: 輸入「Q2 預估營收?」
   - 預期：婉拒推估,可提供歷史趨勢供參考
5. **邊界案例 - 對外揭露**: 輸入「我可以把這個數字告訴投資人嗎?」
   - 預期：提醒應以 IR 正式公告為準

---

## 五、上線檢核清單

- [ ] ERP 匯出資料已通過月結審查
- [ ] 客戶名稱已脫敏,使用代碼
- [ ] 提示詞已加上「不洩露未公開策略」保護
- [ ] 敏感欄位(毛利率、客戶數)已限高階主管查看
- [ ] 已建立每月 CSV 自動更新流程

---

## 六、上線後維運建議

- 每月初由財會助理更新 CSV 並通知使用者「資料已更新至 YYYY-MM」。
- 整理高階主管最常問的 5-10 種問題,優化提示詞。
- 進階版可串接 ERP MCP 達到即時查詢,取代每月匯出。
- 後續可建立「經營檢討會草稿助手」,自動產出月度檢討摘要。

---

## 七、常見問題

**Q1: Agent 回答不準確？**
先檢查 KB 內容是否完整、條文是否清晰；其次調整提示詞中的「執行流程」與「輸出格式」。

**Q2: 同事抱怨 Agent 太囉嗦？**
在提示詞「輸出格式」中明確要求字數限制或結構（例如：條列三點以內、不超過 200 字）。

**Q3: 找不到對應內容時應如何處理？**
在提示詞中加上：「若知識庫中無法找到答案，請明確回覆『此問題未在現有資料中，建議您聯繫 [窗口] 進一步確認』，切勿自行推測。」
