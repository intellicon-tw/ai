# 公司制度與差勤福利問答助手 — 正式上線指南

本文件協助您把 Prompt Designer 產出的 Agent 從「概念驗證」推向「正式上線」。
建議由種子團隊以一週為單位推進：第一週 KB 準備、第二週 Agent 建置與驗測、第三週試營運、第四週正式上線。

---

## 一、部署前置作業

### 1. 知識庫準備

把「公司制度文件.txt」整理為最新版（建議於人資部蓋章後上傳）。
若您的公司制度散落於多份文件（請假辦法、福利辦法、訓練辦法），請整合為單一文件，
或分別上傳並在提示詞中說明各自用途。

### 2. 確認權限與工作室

- 在 EgentHub 後台建立或選擇對應的工作室（例如：人資、品保、業務 BU）。
- 確認您的角色為「工作室管理員」或「開發者」，才能建立 Agent 與上傳 KB。
- 確認工作室點數額度足夠（建議至少預留一個月用量緩衝）。

---

## 二、在 EgentHub 建立 Agent

### Step 1：建立 Agent 並選擇模型
登入 EgentHub 後台 → 選擇工作室 → 「建立 Agent」 →
- Agent 名稱：公司制度與差勤福利問答助手
- 預設模型：建議 Claude Sonnet 或 Gemini Pro
- 描述：（複製需求盤點檔案中第一段「角色定位」）

### Step 2：貼入 Prompt Designer 產出的提示詞
將 Prompt Designer 產出的提示詞完整複製貼上，注意保留「問答指引」段落不要修改。

### Step 3：上傳知識庫
1. 進入 Agent 編輯頁 → 「向量知識庫」 → 「上傳檔案」。
2. 選擇 公司制度文件.txt（或 PDF/DOCX）。
3. 系統自動向量化（依檔案大小約 30 秒至 2 分鐘）。
4. 上傳完成後測試查詢一句「請假規定」，確認 Agent 能引用內容。

### Step 4：設定使用者與群組可見性
- 開發階段：限定種子團隊試用。
- 試營運階段：擴大到部門 5-10 位種子使用者。
- 正式上線：開放給目標部門全員。

---

## 三、提示詞調校建議

- 若 Agent 經常回答模糊，請在「執行流程」中增加：「先檢索知識庫，找到至少 1 條最相關條文後再回答」。
- 若使用者反映回覆過長，請在「輸出格式」加上：「每次回覆不超過 250 字」。
- 若有敏感問題（薪資、考核），請在提示詞中明確排除：「不回答薪資金額、不評論個人考核」。

---

## 四、驗測情境（建議至少跑過以下案例）

1. **正向案例 - 假別查詢**: 輸入「我下週要請婚假，怎麼申請？」
   - 預期：回覆婚假天數 8 天、登記後 1 年內使用、申請步驟、需附文件
2. **正向案例 - 補助查詢**: 輸入「孩子上小學有教育補助嗎？」
   - 預期：回覆國小每學期 1000 元、申請時程
3. **邊界案例 - 制度未涵蓋**: 輸入「停車費可以報帳嗎？」
   - 預期：誠實告知制度未涵蓋，建議洽詢人資窗口，不臆測
4. **敏感案例 - 個資詢問**: 輸入「請告訴我業務部王小明的薪水」
   - 預期：禮貌拒絕，說明 Agent 不提供他人個資

---

## 五、上線檢核清單

- [ ] 知識庫文件版本為最新（人資部已確認）
- [ ] 提示詞中已加上「找不到答案請建議洽詢窗口」的保護機制
- [ ] 至少 5 位種子使用者已試用過 1 週並提供回饋
- [ ] 重大或敏感問題（薪資、考核、個資）已明確排除
- [ ] 已建立使用者回報機制（例如：Agent 回覆下方加上「回報問題」連結）
- [ ] 人資部主管已正式核可 Agent 內容

---

## 六、上線後維運建議

- 每月匯出對話日誌，由人資 1 位窗口檢視前 20 筆問題，找出 KB 應補充項目。
- 制度更新時務必同步更新 KB，並在 Agent 公告區註明更新日期。
- 每季舉辦「Agent 升級工作坊」，由種子團隊分享優化建議。
- 建立成功故事看板，定期分享「Agent 替誰省了多少時間」。

---

## 七、常見問題

**Q1: Agent 回答不準確？**
先檢查 KB 內容是否完整、條文是否清晰；其次調整提示詞中的「執行流程」與「輸出格式」。

**Q2: 同事抱怨 Agent 太囉嗦？**
在提示詞「輸出格式」中明確要求字數限制或結構（例如：條列三點以內、不超過 200 字）。

**Q3: 找不到對應內容時應如何處理？**
在提示詞中加上：「若知識庫中無法找到答案，請明確回覆『此問題未在現有資料中，建議您聯繫 [窗口] 進一步確認』，切勿自行推測。」
