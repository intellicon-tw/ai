# 培訓教材:10 個 POC Agent 教學包

## 適用對象
精誠資訊 EgentHub 第一波客戶種子成員聯合培訓使用。

## 教學包結構
本教材依「知識庫類型」分為四個區段，由淺入深漸進式設計：

### 第一段：純向量知識庫(RAG)入門
| # | 題目 | KB 類型 | 教學重點 |
|---|---|---|---|
| 01 | 公司制度與差勤福利問答助手 | 向量 KB | RAG 入門、單一 KB 應用 |
| 02 | SOP 與工法查詢助手 | 向量 KB | RAG 進階、跨多份文件比對 |

### 第二段：表格知識庫(SQL/CSV)
| # | 題目 | KB 類型 | 教學重點 |
|---|---|---|---|
| 03 | 產品規格查詢助手 | 表格 KB | 結構化資料查詢、欄位篩選 |
| 04 | 經營儀表板問答助手 | 表格 KB | 表格 KB 進階、聚合運算與趨勢 |

### 第三段：全文檢索(法規與合約)
| # | 題目 | KB 類型 | 教學重點 |
|---|---|---|---|
| 05 | 認證法規查詢助手 | 全文檢索 | 長文逐條檢索與引用 |
| 06 | 合約條款查詢助手 | 全文檢索 | 法務情境、條款比對 |

### 第四段:整合應用(混合 KB 與純提示工程)
| # | 題目 | KB 類型 | 教學重點 |
|---|---|---|---|
| 07 | RFQ 應答草稿助手 | 向量 + 表格 | 兩類 KB 整合、結構化草稿生成 |
| 08 | 客訴 8D 報告草稿助手 | 向量 + 全文 | 兩類 KB 整合、案例式生成 |
| 09 | 會議紀錄與行動項助手 | 無 KB(純提示工程) | 提示設計、結構化輸出 |
| 10 | 企業知識總管(畢業作品) | 三類 KB 整合 | 路由與分流、跨 KB 整合 |

## 每一題目的內容
每個題目資料夾中包含三類檔案:

1. **需求盤點.txt**
   學員可直接複製內容貼入 EgentHub Prompt Designer 的「需求描述」欄位。
   Prompt Designer 會引導學員產出對應的 Agent Prompt。

2. **知識庫_*.txt 或 .csv**
   開發測試所需的範例知識庫內容。
   學員可直接上傳到 EgentHub 對應 Agent 的知識庫設定中。

3. **正式上線指南.txt**
   完整的部署指南,包含:
   - 部署前置作業
   - 在 EgentHub 建立 Agent 步驟
   - 知識庫上傳步驟
   - 提示詞調校建議
   - 驗測情境
   - 上線檢核清單
   - 上線後維運建議

## 建議授課流程

### 第 1 場:導論與第一段(2 小時)
- AI Agent 概念介紹
- EgentHub 平台導覽
- 動手做題目 01-02(向量 KB 入門)

### 第 2 場:表格 KB 與全文檢索(2 小時)
- 動手做題目 03-04(表格 KB)
- 動手做題目 05-06(全文檢索)

### 第 3 場:整合應用與畢業作品(2 小時)
- 動手做題目 07-08(混合 KB)
- 動手做題目 09(純提示工程)
- 介紹題目 10「企業知識總管」概念

### 第 4 場:畢業作品實作與分享(2 小時)
- 學員實作題目 10
- 各組分享心得與成果展示

## 學員成果驗收標準
完成本教材後,學員應能:

1. 獨立判斷一個業務需求應採用哪一類知識庫。
2. 撰寫清晰的「需求盤點」描述,讓 Prompt Designer 產出可用的初版 Prompt。
3. 整理對應的知識庫資料(向量 / 表格 / 全文)。
4. 在 EgentHub 上獨立建立 Agent、上傳 KB、進行驗測。
5. 跟客戶說明各 Agent 適用情境與限制,評估上線可行性。

## 給講師的提醒
- 第 1-9 題建議讓學員「自己動手做」,題目 10 可以由講師示範或讓進階學員嘗試。
- 強調「正式上線指南」的重要性 — POC 上線後的維運機制比 PoC 本身重要。
- 若有時間限制,優先推進題目 1、3、5、7、10(每個區段選 1 題)。
- 每完成一題,鼓勵學員試用後分享一個讓自己驚訝的 Agent 表現,建立內部成功故事。
