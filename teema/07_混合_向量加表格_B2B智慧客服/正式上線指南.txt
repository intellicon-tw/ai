# B2B 智慧客服 — 正式上線指南

本文件協助您把 Prompt Designer 產出的 Agent 從「概念驗證」推向「正式上線」。
建議由種子團隊以一週為單位推進：第一週 KB 準備、第二週 Agent 建置與驗測、第三週試營運、第四週正式上線。

---

## 一、部署前置作業

### 1. 知識庫準備

1. **向量 KB**:由業務與客服共同整理 FAQ 文件,涵蓋產品、保固、出貨、付款、認證、退換貨等常見問題。
2. **表格 KB**:從 ERP/CRM 匯出訂單資料,**客戶名稱用代碼或匿名化**(例如 C-001 / 客戶 A),避免洩露真實客戶資料。
3. 編碼 UTF-8 BOM。

### 2. 確認權限與工作室

- 在 EgentHub 後台建立或選擇對應的工作室（例如：人資、品保、業務 BU）。
- 確認您的角色為「工作室管理員」或「開發者」，才能建立 Agent 與上傳 KB。
- 確認工作室點數額度足夠（建議至少預留一個月用量緩衝）。

---

## 二、在 EgentHub 建立 Agent

### Step 1：建立 Agent 並選擇模型
登入 EgentHub 後台 → 選擇工作室 → 「建立 Agent」 →
- Agent 名稱：B2B 智慧客服
- 預設模型：建議 Claude Sonnet 或 Gemini Pro
- 描述：（複製需求盤點檔案中第一段「角色定位」）

### Step 2：貼入 Prompt Designer 產出的提示詞
將 Prompt Designer 產出的提示詞完整複製貼上，注意保留「問答指引」段落不要修改。

### Step 3：上傳知識庫
1. 進入 Agent 編輯頁。
2. 上傳向量 KB:「向量知識庫」 → 上傳 公司產品與服務FAQ.txt。
3. 上傳表格 KB:「表格查詢知識庫」 → 上傳 訂單表.csv。
4. 兩種 KB 都需上傳成功並各別測試。
5. 整合測試:輸入「ORD-2026-00125 到哪了?」確認 Agent 從訂單表查詢。
6. 整合測試:輸入「保固期多久?」確認 Agent 從 FAQ 查詢。

### Step 4：設定使用者與群組可見性
- 開發階段：限定種子團隊試用。
- 試營運階段：擴大到部門 5-10 位種子使用者。
- 正式上線：開放給目標部門全員。

---

## 三、提示詞調校建議

- 若 Agent 偏向只用一種 KB,請在「執行流程」加上:「先判斷問題類型(產品/訂單/客訴),再使用對應 KB」。
- 若 Agent 過於冰冷,請在「溝通風格指引」加上:「親切但專業,使用『您』而非『你』」。
- 若 Agent 在客訴情境直接給方案缺乏同理心,加上:「客訴情境必須先表達同理心(『非常抱歉造成您的不便』),再提供方案」。
- 若 Agent 嘗試處理改訂單、退款等請求,加上:「超出權限範圍,禮貌說明無法處理並提供正確窗口」。
- 若 Agent 洩露其他客戶資料,加上:「僅查詢當前對話客戶的訂單,不洩露其他客戶資訊」。

---

## 四、驗測情境（建議至少跑過以下案例）

1. **正向案例 - 產品諮詢**: 輸入「保固期多久?」
   - 預期：從 FAQ 引用 Q1 答覆,標準保固 1 年
2. **正向案例 - 訂單查詢**: 輸入「ORD-2026-00125 到哪了?」
   - 預期：從訂單表查到客戶 B、200 pcs、已出貨、物流號 SF-2026-005
3. **正向案例 - 組合問題**: 輸入「ORD-2026-00120 是什麼產品?保固多久?」
   - 預期：從訂單表 + FAQ 整合答覆
4. **邊界案例 - 客訴處理**: 輸入「上週收到 5 個不良,你們是怎麼搞的!」
   - 預期：先表達同理心,再說明客訴流程,提供品保窗口
5. **邊界案例 - 超出權限**: 輸入「幫我把訂單改成 1000 個」
   - 預期：禮貌說明無法處理,提供業務窗口
6. **邊界案例 - 跨客戶查詢**: 輸入「ORD-2026-00121 是不是客戶 X 訂的?」
   - 預期：禮貌拒絕,不洩露其他客戶資訊

- [ ] 訂單表已脫敏(客戶名稱代碼化)

## 六、上線後維運建議

- 蒐集 Agent 升級給人工的案例,分析升級原因,擴充 FAQ。
- 進階版:整合 LINE 官方帳號 / 網頁 Widget,提供 24 小時客服。
- 進階版:整合 CRM,自動建立客訴單與追蹤。
- 與「庫存管理助手」整合,客戶詢問交期時可直接查到庫存與在途狀態。

## 七、常見問題

**Q1: Agent 回答不準確？**
先檢查 KB 內容是否完整、條文是否清晰；其次調整提示詞中的「執行流程」與「輸出格式」。

**Q2: 同事抱怨 Agent 太囉嗦？**
在提示詞「輸出格式」中明確要求字數限制或結構（例如：條列三點以內、不超過 200 字）。

**Q3: 找不到對應內容時應如何處理？**
在提示詞中加上：「若知識庫中無法找到答案，請明確回覆『此問題未在現有資料中，建議您聯繫 [窗口] 進一步確認』，切勿自行推測。」
