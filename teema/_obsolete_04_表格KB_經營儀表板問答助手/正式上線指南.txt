# 經營儀表板問答助手 — 正式上線指南

本文件協助您把 Prompt Designer 產出的 Agent 從「概念驗證」推向「正式上線」。
建議由種子團隊以一週為單位推進：第一週 KB 準備、第二週 Agent 建置與驗測、第三週試營運、第四週正式上線。

---

## 一、部署前置作業

### 1. 知識庫準備

1. 與財會部門共同定義 KPI 表欄位（建議：年月、BU、產品線、營收、毛利、毛利率、出貨量、新增訂單、客戶數、客訴件數）。
2. 由財會每月初匯出最近 24 個月的數據（保留滾動 24 個月）。
3. 確認資料已通過月結審查、與正式財報一致。
4. 編碼 UTF-8 BOM。

### 2. 確認權限與工作室

- 在 EgentHub 後台建立或選擇對應的工作室（例如：人資、品保、業務 BU）。
- 確認您的角色為「工作室管理員」或「開發者」，才能建立 Agent 與上傳 KB。
- 確認工作室點數額度足夠（建議至少預留一個月用量緩衝）。

---

## 二、在 EgentHub 建立 Agent

### Step 1：建立 Agent 並選擇模型
登入 EgentHub 後台 → 選擇工作室 → 「建立 Agent」 →
- Agent 名稱：經營儀表板問答助手
- 預設模型：建議 Claude Sonnet 或 Gemini Pro
- 描述：（複製需求盤點檔案中第一段「角色定位」）

### Step 2：貼入 Prompt Designer 產出的提示詞
將 Prompt Designer 產出的提示詞完整複製貼上，注意保留「問答指引」段落不要修改。

### Step 3：上傳知識庫
1. 進入 Agent 編輯頁 → 「表格知識庫」 → 「上傳 CSV」。
2. 上傳月度 KPI 表.csv。
3. 確認 EgentHub 自動辨識的欄位類型（年月為文字或日期，數字為數值型）。
4. 測試查詢「2026 Q1 BU-A 營收」確認 Agent 能正確聚合。

### Step 4：設定使用者與群組可見性
- 開發階段：限定種子團隊試用。
- 試營運階段：擴大到部門 5-10 位種子使用者。
- 正式上線：開放給目標部門全員。

---

## 三、提示詞調校建議

- 若 Agent 數據常有錯誤，請在「執行流程」加上：「先讀取 CSV 中相關欄位，再進行加總與計算，不得自行推估」。
- 若使用者反映回覆過於冷冰，可在「角色定位」加上：「在數據後補充 1-2 句趨勢觀察」。
- 若涉及預測或預估，請加上免責聲明：「此為基於現有資料的觀察，未經財務部正式驗證」。

---

## 四、驗測情境（建議至少跑過以下案例）

1. **正向案例 - 季度比較**: 輸入「2026 Q1 各 BU 毛利率比較」
   - 預期：正確列出 BU-A/B/C 三季毛利率與觀察
2. **正向案例 - 趨勢分析**: 輸入「BU-A 過去 12 個月毛利率變化」
   - 預期：顯示月度毛利率序列並指出趨勢
3. **正向案例 - YoY**: 輸入「2026/01 vs 2025/01 整體營收 YoY」
   - 預期：正確計算並列出年增率
4. **邊界案例 - 預測詢問**: 輸入「Q2 預估營收？」
   - 預期：婉拒推估，建議聯繫財務部，可提供歷史趨勢供參考

---

## 五、上線檢核清單

- [ ] KPI 表已通過月結審查
- [ ] 敏感欄位（毛利率、客戶數）權限已設定為高階主管
- [ ] 提示詞已加上「不洩露未公開策略」的保護機制
- [ ] 已建立每月 CSV 自動更新流程
- [ ] 經營層、財會主管已試用並核可

---

## 六、上線後維運建議

- 每月初由財會助理更新 CSV，並通知使用者「資料已更新至 YYYY-MM」。
- 蒐集主管最常問的 5-10 種問題，回頭優化提示詞中的「執行流程」與「輸出格式」。
- 進階版可串接 ERP 與 BI 工具（透過 MCP），達到即時查詢。
- 後續可建立「經營檢討會草稿助手」自動產出月度檢討會摘要。

---

## 七、常見問題

**Q1: Agent 回答不準確？**
先檢查 KB 內容是否完整、條文是否清晰；其次調整提示詞中的「執行流程」與「輸出格式」。

**Q2: 同事抱怨 Agent 太囉嗦？**
在提示詞「輸出格式」中明確要求字數限制或結構（例如：條列三點以內、不超過 200 字）。

**Q3: 找不到對應內容時應如何處理？**
在提示詞中加上：「若知識庫中無法找到答案，請明確回覆『此問題未在現有資料中，建議您聯繫 [窗口] 進一步確認』，切勿自行推測。」
