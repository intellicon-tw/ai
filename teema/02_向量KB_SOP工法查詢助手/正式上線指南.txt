# SOP 與工法查詢助手 — 正式上線指南

本文件協助您把 Prompt Designer 產出的 Agent 從「概念驗證」推向「正式上線」。
建議由種子團隊以一週為單位推進：第一週 KB 準備、第二週 Agent 建置與驗測、第三週試營運、第四週正式上線。

---

## 一、部署前置作業

### 1. 知識庫準備

1. 整合各部門 SOP 文件（建議 5-10 份），統一命名與編號。
2. 確認文件版本為最新，並由品保部蓋章核可。
3. 若 SOP 散落多個系統（紙本、雲端、ERP），先抽出最高頻使用的 5-10 份建立 KB。

### 2. 確認權限與工作室

- 在 EgentHub 後台建立或選擇對應的工作室（例如：人資、品保、業務 BU）。
- 確認您的角色為「工作室管理員」或「開發者」，才能建立 Agent 與上傳 KB。
- 確認工作室點數額度足夠（建議至少預留一個月用量緩衝）。

---

## 二、在 EgentHub 建立 Agent

### Step 1：建立 Agent 並選擇模型
登入 EgentHub 後台 → 選擇工作室 → 「建立 Agent」 →
- Agent 名稱：SOP 與工法查詢助手
- 預設模型：建議 Claude Sonnet 或 Gemini Pro
- 描述：（複製需求盤點檔案中第一段「角色定位」）

### Step 2：貼入 Prompt Designer 產出的提示詞
將 Prompt Designer 產出的提示詞完整複製貼上，注意保留「問答指引」段落不要修改。

### Step 3：上傳知識庫
1. 進入 Agent 編輯頁 → 「向量知識庫」 → 「上傳檔案」。
2. 上傳整合好的 SOP 文件（建議單一檔案，或同類型分開上傳）。
3. 系統自動向量化。
4. 測試查詢「來料檢驗 AQL」確認可正確引用 SOP 編號。

### Step 4：設定使用者與群組可見性
- 開發階段：限定種子團隊試用。
- 試營運階段：擴大到部門 5-10 位種子使用者。
- 正式上線：開放給目標部門全員。

---

## 三、提示詞調校建議

- 若 Agent 引用 SOP 編號常常錯誤，請在「執行流程」中增加：「回覆前必須確認 SOP 編號與版次與知識庫一致」。
- 若 Agent 對安全議題回覆過於委婉，請在提示詞加上：「涉及安全的內容必須使用粗體標示『安全提醒』，並列出具體危害與應變」。
- 若涉及多份 SOP 衝突，請在提示詞加上：「以最新版本為準，並提示使用者新舊差異」。

---

## 四、驗測情境（建議至少跑過以下案例）

1. **正向案例 - 製程參數**: 輸入「SMT 回焊爐溫度設定？」
   - 預期：回覆預熱、均熱、回焊、冷卻四區參數，引用 SOP-MFG-003 第 4 條
2. **正向案例 - 安全規範**: 輸入「化學品儲存規定？」
   - 預期：回覆儲存規範並標註安全提醒，引用 SOP-EHS-001
3. **邊界案例 - SOP 未收錄**: 輸入「機台 OOO 的故障排除 SOP？」
   - 預期：誠實告知 KB 未收錄，建議聯繫品保部
4. **邊界案例 - 跨 SOP 整合**: 輸入「我要做 ESD 區的化學品檢驗，要注意什麼？」
   - 預期：整合 SOP-EHS-001 與 SOP-EHS-005 的相關條文

---

## 五、上線檢核清單

- [ ] SOP 文件版本為最新（品保部已確認）
- [ ] 提示詞已要求引用 SOP 編號與版次
- [ ] 安全相關內容已要求標註「安全提醒」
- [ ] 至少 10 位種子使用者試用過 1 週
- [ ] 已建立 SOP 變更時的同步更新機制
- [ ] 品保部主管已正式核可 Agent 內容

---

## 六、上線後維運建議

- 建立 SOP 變更通知機制：每次 SOP 更新後 3 日內同步更新 KB。
- 每月匯出對話日誌，找出最常被問但 KB 未涵蓋的 SOP，列入下一波 KB 補充。
- 鼓勵第一線同仁透過 Agent 對話下方的「回報」機制主動提交 SOP 改善建議。
- 把 Agent 整合到新人到職訓練中，作為新人查找 SOP 的第一站。

---

## 七、常見問題

**Q1: Agent 回答不準確？**
先檢查 KB 內容是否完整、條文是否清晰；其次調整提示詞中的「執行流程」與「輸出格式」。

**Q2: 同事抱怨 Agent 太囉嗦？**
在提示詞「輸出格式」中明確要求字數限制或結構（例如：條列三點以內、不超過 200 字）。

**Q3: 找不到對應內容時應如何處理？**
在提示詞中加上：「若知識庫中無法找到答案，請明確回覆『此問題未在現有資料中，建議您聯繫 [窗口] 進一步確認』，切勿自行推測。」
