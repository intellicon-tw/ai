# 交通費報帳助手(入門練習) — 正式上線指南

本文件協助您把 Prompt Designer 產出的 Agent 從「概念驗證」推向「正式上線」。
建議由種子團隊以一週為單位推進：第一週 KB 準備、第二週 Agent 建置與驗測、第三週試營運、第四週正式上線。

---

## 一、部署前置作業

### 1. 知識庫準備

**本題目不需要知識庫!** 純提示工程 + Vision 視覺辨識即可完成。

選擇性準備(進階版):
1. 公司差旅政策文件(可作為向量 KB,讓 Agent 標示違反政策的單據)。
2. 公司常用的報帳系統匯出格式(可在提示詞中加入 few-shot 範例)。
3. 員工差旅申請紀錄(進階版可串接驗證行程合理性)。

### 2. 確認權限與工作室

- 在 EgentHub 後台建立或選擇對應的工作室（例如：人資、品保、業務 BU）。
- 確認您的角色為「工作室管理員」或「開發者」，才能建立 Agent 與上傳 KB。
- 確認工作室點數額度足夠（建議至少預留一個月用量緩衝）。

---

## 二、在 EgentHub 建立 Agent

### Step 1：建立 Agent 並選擇模型
登入 EgentHub 後台 → 選擇工作室 → 「建立 Agent」 →
- Agent 名稱：交通費報帳助手(入門練習)
- 預設模型：建議 Claude Sonnet 或 Gemini Pro
- 描述：（複製需求盤點檔案中第一段「角色定位」）

### Step 2：貼入 Prompt Designer 產出的提示詞
將 Prompt Designer 產出的提示詞完整複製貼上，注意保留「問答指引」段落不要修改。

### Step 3：上傳知識庫
**本 Agent 不需要上傳任何知識庫,直接設定提示詞即可。**

但需要做一件重要的設定:

1. 在 Agent 編輯頁中,確認**選擇支援 Vision 的模型**。
   - Claude Sonnet(建議,中文 OCR 表現佳)
   - Gemini Pro
   - **不要使用** GPT-3.5 或舊版模型(無視覺能力)

2. 啟用「附件閱讀」功能,讓使用者可以上傳圖片或 PDF。

3. 測試上傳一張交通單據照片,確認 Agent 能讀取。

### Step 4：設定使用者與群組可見性
- 開發階段：限定種子團隊試用。
- 試營運階段：擴大到部門 5-10 位種子使用者。
- 正式上線：開放給目標部門全員。

---

## 三、提示詞調校建議

- 若 Agent 經常虛構資料(例如填了實際看不到的車號),請在「執行流程」明確加上:「若任一欄位在圖片上看不清楚,該欄位填『待確認』,絕不臆測」。
- 若 Agent 漏掉某些單據(例如多張並排的合照只認出 3 張),請在「執行流程」加上:「逐張掃描,確認每張獨立單據都被處理,若有疑似遺漏明確列出」。
- 若民國年份沒轉成西元,請在「輸出格式」加上:「日期統一轉為西元 YYYY-MM-DD 格式,民國 115 年 = 西元 2026 年」。
- 若 Agent 的小計計算有誤,請在「輸出格式」加上:「小計與總金額需重新加總一次以確保正確」。
- 若使用者反映輸出表格在 Excel 中貼上後變形,可在「輸出格式」加上:「Markdown 表格欄位之間用 | 分隔,確保格式正確」。

## 四、驗測情境（建議至少跑過以下案例）

1. **正向案例 - 單張計程車收據**: 上傳一張清晰的 LINE GO 計程車收據(195 元 / 4.2 km)
   - 預期：正確辨識日期、起迄、金額、車號;備註欄含距離資訊
2. **正向案例 - 整月單據 PDF**: 上傳本題目附的範例 PDF(2026 年 1 月份交通費)
   - 預期：正確逐張辨識所有單據,產出完整報帳表格與小計、總金額
3. **邊界案例 - 模糊或缺漏資訊**: 上傳一張只有手寫的計程車運價證明(無起迄)
   - 預期：車資金額正確辨識,起迄欄標示「待確認」,不臆測
4. **邊界案例 - 補印單據**: 上傳一張標示「補印 (COPY)」的計程車證明
   - 預期：備註欄標註「補印單據,請避免重複報帳」
5. **邊界案例 - 加油發票**: 上傳一張加油發票
   - 預期：標註車種「加油」並提醒「若公司不允許加油報帳,請排除」
6. **邊界案例 - 沒上傳圖片**: 純文字輸入「我要報交通費」
   - 預期：禮貌請使用者上傳交通單據照片或 PDF

## 六、上線後維運建議

- 建議在公司內以「省時 1 小時/月」作為推廣口號,讓員工快速感受價值。
- 蒐集員工回饋,看哪幾類單據(例如:特殊外幣收據、停車證上的車牌)辨識率較低,作為提示詞優化依據。
- 進階版可整合公司差旅政策 KB,自動標示違反政策的單據。
- 更進階可串接報帳系統 API,讓 Agent 直接建立報帳單草稿。
- 本題目是『AI 真的有用』的最佳推廣案例,建議拍成 Demo 影片,在公司內部成功故事看板分享。

## 七、常見問題

**Q1: Agent 回答不準確？**
先檢查 KB 內容是否完整、條文是否清晰；其次調整提示詞中的「執行流程」與「輸出格式」。

**Q2: 同事抱怨 Agent 太囉嗦？**
在提示詞「輸出格式」中明確要求字數限制或結構（例如：條列三點以內、不超過 200 字）。

**Q3: 找不到對應內容時應如何處理？**
在提示詞中加上：「若知識庫中無法找到答案，請明確回覆『此問題未在現有資料中，建議您聯繫 [窗口] 進一步確認』，切勿自行推測。」
