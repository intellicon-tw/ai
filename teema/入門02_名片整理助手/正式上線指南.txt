# 名片整理助手(入門練習) — 正式上線指南

本文件協助您把 Prompt Designer 產出的 Agent 從「概念驗證」推向「正式上線」。
建議由種子團隊以一週為單位推進：第一週 KB 準備、第二週 Agent 建置與驗測、第三週試營運、第四週正式上線。

---

## 一、部署前置作業

### 1. 知識庫準備

**本題目不需要知識庫!** 純提示工程 + Vision 即可完成。

選擇性準備:
1. 公司常用聯絡人欄位範本(讓 Agent 輸出對齊 CRM 欄位)。
2. CRM 系統匯入格式範例(可在提示詞加入 few-shot)。

### 2. 確認權限與工作室

- 在 EgentHub 後台建立或選擇對應的工作室（例如：人資、品保、業務 BU）。
- 確認您的角色為「工作室管理員」或「開發者」，才能建立 Agent 與上傳 KB。
- 確認工作室點數額度足夠（建議至少預留一個月用量緩衝）。

---

## 二、在 EgentHub 建立 Agent

### Step 1：建立 Agent 並選擇模型
登入 EgentHub 後台 → 選擇工作室 → 「建立 Agent」 →
- Agent 名稱：名片整理助手(入門練習)
- 預設模型：建議 Claude Sonnet 或 Gemini Pro
- 描述：（複製需求盤點檔案中第一段「角色定位」）

### Step 2：貼入 Prompt Designer 產出的提示詞
將 Prompt Designer 產出的提示詞完整複製貼上，注意保留「問答指引」段落不要修改。

### Step 3：上傳知識庫
**本 Agent 不需要上傳任何知識庫,直接設定提示詞即可。**

但需要做這些設定:

1. 在 Agent 編輯頁,**選擇支援 Vision 的模型**(Claude Sonnet 或 Gemini Pro)。
2. 啟用「附件閱讀」功能,讓使用者可以上傳圖片或 PDF。
3. 測試上傳一張名片照片,確認 Agent 能讀取。

### Step 4：設定使用者與群組可見性
- 開發階段：限定種子團隊試用。
- 試營運階段：擴大到部門 5-10 位種子使用者。
- 正式上線：開放給目標部門全員。

---

## 三、提示詞調校建議

- 若 Agent 經常虛構資料(例如自行補上 Email),請在「執行流程」加上:「若任一欄位看不清楚,該欄位填『待確認』,絕不臆測」。
- 若辨識多張並排名片時漏失,請在「執行流程」加上:「先掃描整張圖找出所有獨立名片邊界,再逐張辨識」。
- 若電話格式不一致,請在「輸出格式」加上:「電話統一為 02-XXXX-XXXX、手機統一為 09XX-XXX-XXX」。
- 若公司名稱誤合併到地址,請加上:「逐項對齊欄位,避免跨欄位混淆」。

---

## 四、驗測情境（建議至少跑過以下案例）

1. **正向案例 - 單張中文名片**: 上傳一張清晰的中文名片
   - 預期：正確辨識所有欄位,輸出表格與 CSV
2. **正向案例 - 中英對照名片**: 上傳一張中英對照的名片
   - 預期：中文與英文姓名、公司皆正確辨識
3. **邊界案例 - 多張合照**: 上傳一張含 3-4 張名片的合照
   - 預期：逐張辨識,結果列為多列
4. **邊界案例 - 模糊照片**: 上傳一張部分模糊的名片照
   - 預期：可辨識欄位正常,模糊欄位標『待確認』
5. **邊界案例 - 沒上傳圖片**: 純文字輸入「我要整理名片」
   - 預期：禮貌請使用者上傳名片照片

## 五、上線檢核清單

- [ ] 已選擇支援 Vision 的模型
- [ ] 已啟用附件閱讀功能
- [ ] 提示詞已加上「不臆測」保護機制
- [ ] 提示詞已統一電話格式
- [ ] 至少 3 位種子使用者試用

## 六、上線後維運建議

- 推廣時可結合展會、商業活動主題,讓員工帶回名片立即試用。
- 蒐集無法辨識的名片類型(特殊字型、外文等),作為提示詞優化依據。
- 進階版可整合 CRM API,直接建立聯絡人。

---

## 七、常見問題

**Q1: Agent 回答不準確？**
先檢查 KB 內容是否完整、條文是否清晰；其次調整提示詞中的「執行流程」與「輸出格式」。

**Q2: 同事抱怨 Agent 太囉嗦？**
在提示詞「輸出格式」中明確要求字數限制或結構（例如：條列三點以內、不超過 200 字）。

**Q3: 找不到對應內容時應如何處理？**
在提示詞中加上：「若知識庫中無法找到答案，請明確回覆『此問題未在現有資料中，建議您聯繫 [窗口] 進一步確認』，切勿自行推測。」
