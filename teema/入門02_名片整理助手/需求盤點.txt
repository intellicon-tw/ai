# 需求盤點:名片整理助手(入門練習)

> 學員操作說明:請將以下「需求描述」段落整段複製,貼入 EgentHub Prompt Designer 的「需求描述」欄位,讓系統引導您完成提示詞建構。
>
> 本題延續 Vision 主題,難度更低、輸出更單純,適合作為第二題快速練習。
>
> 預計操作時間:20-30 分鐘。

---

## 一、Agent 基本資料
- **Agent 名稱**:名片整理助手
- **所屬工作室**:業務 / 行政 / 全公司
- **使用對象**:業務、行政、PM、所有交換名片的同仁
- **預設模型建議**:Claude Sonnet 或 Gemini Pro **(必須支援 Vision)**

---

## 二、需求描述(請整段複製到 Prompt Designer)

### 1. 應用場景
員工參加展會、拜訪客戶或活動後,可以一次上傳整疊名片照片,由 AI 自動辨識資訊,輸出可直接匯入 CRM、Outlook 或 Excel 的結構化資料。

### 2. 希望 AI 處理的問題
- 展會後一疊名片要手動 Key 入系統
- 名片有中英文混排、繁簡字差異
- 電話、手機格式不一致
- 多張名片並排合照需要逐張辨識
- 缺漏資料容易誤導後續聯繫

### 3. 預期輸出內容
- **名片清單表格**:姓名 / 職稱 / 公司 / 電話 / 手機 / Email / 地址 / 網站
- **CSV 格式**:可直接複製到 Excel
- **辨識說明**:模糊或缺漏欄位明確標示「待確認」

### 4. 任務步驟 & 邏輯
1. 接收使用者上傳的名片照片(可單張或多張合照)。
2. 逐張辨識上述 8 個欄位。
3. 電話統一為 02-XXXX-XXXX、手機統一為 09XX-XXX-XXX。
4. 看不清楚的欄位填「待確認」,不要臆測。
5. 同時輸出表格 + CSV 兩種格式。

### 5. 知識庫/資料庫
**無 KB**(純提示工程 + Vision)。

## 三、為什麼選這題作為入門練習
1. 不需要知識庫,單純體驗 Vision + 結構化輸出。
2. 效益立即可見:省掉手動 Key 名片資料的時間。
3. 資料格式單純:沒有複雜業務邏輯,適合提示詞 80% 法則練習。

## 四、上線後的延伸應用
- 整合 CRM 自動建立聯絡人。
- 加入「合作機會評估」(依職稱與公司類型,判斷是否潛在客戶)。
- 展會收件助手、合約簽核名單整理助手。
