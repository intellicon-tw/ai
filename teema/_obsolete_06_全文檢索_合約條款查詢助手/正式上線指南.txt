# 合約條款查詢助手 — 正式上線指南

本文件協助您把 Prompt Designer 產出的 Agent 從「概念驗證」推向「正式上線」。
建議由種子團隊以一週為單位推進：第一週 KB 準備、第二週 Agent 建置與驗測、第三週試營運、第四週正式上線。

---

## 一、部署前置作業

### 1. 知識庫準備

1. 由法務部整理常用合約樣本(建議:採購、銷售、NDA、技術授權、員工聘僱、租賃)。
2. 確認樣本為最新核可版本,並去除具名客戶或敏感資訊。
3. 將每份合約以條號清楚編排為 TXT 或 PDF。
4. 由法務主管核可後上傳。

### 2. 確認權限與工作室

- 在 EgentHub 後台建立或選擇對應的工作室（例如：人資、品保、業務 BU）。
- 確認您的角色為「工作室管理員」或「開發者」，才能建立 Agent 與上傳 KB。
- 確認工作室點數額度足夠（建議至少預留一個月用量緩衝）。

---

## 二、在 EgentHub 建立 Agent

### Step 1：建立 Agent 並選擇模型
登入 EgentHub 後台 → 選擇工作室 → 「建立 Agent」 →
- Agent 名稱：合約條款查詢助手
- 預設模型：建議 Claude Sonnet 或 Gemini Pro
- 描述：（複製需求盤點檔案中第一段「角色定位」）

### Step 2：貼入 Prompt Designer 產出的提示詞
將 Prompt Designer 產出的提示詞完整複製貼上，注意保留「問答指引」段落不要修改。

### Step 3：上傳知識庫
1. 進入 Agent 編輯頁 → 「全文檢索知識庫」 → 「上傳檔案」。
2. 上傳合約樣本(可多份)。
3. 系統自動建立全文索引。
4. 測試查詢「NDA 期限」確認 Agent 能引用條款。

### Step 4：設定使用者與群組可見性
- 開發階段：限定種子團隊試用。
- 試營運階段：擴大到部門 5-10 位種子使用者。
- 正式上線：開放給目標部門全員。

---

## 三、提示詞調校建議

- 若 Agent 引用條款混淆,請在「執行流程」加上:「先確認使用者詢問哪一份合約,再檢索條款」。
- 若涉及法律建議,請強化免責聲明:「本助手僅供條款查詢,正式法律意見請洽法務」。
- 若涉及具體個案,請加上:「實際合約以最終簽署版本為準」。

---

## 四、驗測情境（建議至少跑過以下案例）

1. **正向案例 - 條款查詢**: 輸入「採購合約付款條件?」
   - 預期：正確引用第 5 條,列出標準付款條件、大型專案、新供應商三類
2. **正向案例 - 多合約交叉**: 輸入「保密期限多久?」
   - 預期：回覆採購合約第 9 條 3 年、NDA 第 7 條 3 年並延續 5 年
3. **邊界案例 - 個案合約**: 輸入「我們與 ABC 客戶的合約有沒有獨家條款?」
   - 預期：誠實告知 KB 為樣本,個案以實際合約為準,建議洽法務
4. **邊界案例 - 法律建議**: 輸入「對方違約我可不可以告他?」
   - 預期：說明合約終止條款,但提醒正式法律意見請洽法務或律師

---

## 五、上線檢核清單

- [ ] 合約樣本已由法務主管核可
- [ ] 樣本中已去除具名客戶與敏感資訊
- [ ] 提示詞已強化「不提供法律建議」的免責聲明
- [ ] 至少 3 位業務 / 採購 / 法務種子使用者試用
- [ ] 已建立合約樣本更新流程

---

## 六、上線後維運建議

- 每季由法務檢視合約樣本是否需更新。
- 蒐集業務最常問的條款,優先補強樣本內容。
- 與「認證法規查詢助手」整合,作為合規查詢的雙引擎。
- 進階版可串接合約管理系統(CLM),達到實際合約版本即時查詢。

---

## 七、常見問題

**Q1: Agent 回答不準確？**
先檢查 KB 內容是否完整、條文是否清晰；其次調整提示詞中的「執行流程」與「輸出格式」。

**Q2: 同事抱怨 Agent 太囉嗦？**
在提示詞「輸出格式」中明確要求字數限制或結構（例如：條列三點以內、不超過 200 字）。

**Q3: 找不到對應內容時應如何處理？**
在提示詞中加上：「若知識庫中無法找到答案，請明確回覆『此問題未在現有資料中，建議您聯繫 [窗口] 進一步確認』，切勿自行推測。」
