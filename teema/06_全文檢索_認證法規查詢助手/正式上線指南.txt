# 認證法規查詢助手 — 正式上線指南

本文件協助您把 Prompt Designer 產出的 Agent 從「概念驗證」推向「正式上線」。
建議由種子團隊以一週為單位推進：第一週 KB 準備、第二週 Agent 建置與驗測、第三週試營運、第四週正式上線。

---

## 一、部署前置作業

### 1. 知識庫準備

1. 蒐集貴公司常用的法規條文 (建議從 RoHS、REACH、UL、CE、IEC、AEC-Q 等開始)。
2. 法規原文以 PDF 或 TXT 整理。建議由法務或品保整理為易讀的版本(可保留條號)。
3. 確認版本為最新公告,並標註發布日期。
4. 法規來源建議標註原始連結,方便使用者深入查閱。

### 2. 確認權限與工作室

- 在 EgentHub 後台建立或選擇對應的工作室（例如：人資、品保、業務 BU）。
- 確認您的角色為「工作室管理員」或「開發者」，才能建立 Agent 與上傳 KB。
- 確認工作室點數額度足夠（建議至少預留一個月用量緩衝）。

---

## 二、在 EgentHub 建立 Agent

### Step 1：建立 Agent 並選擇模型
登入 EgentHub 後台 → 選擇工作室 → 「建立 Agent」 →
- Agent 名稱：認證法規查詢助手
- 預設模型：建議 Claude Sonnet 或 Gemini Pro
- 描述：（複製需求盤點檔案中第一段「角色定位」）

### Step 2：貼入 Prompt Designer 產出的提示詞
將 Prompt Designer 產出的提示詞完整複製貼上，注意保留「問答指引」段落不要修改。

### Step 3：上傳知識庫
1. 進入 Agent 編輯頁 → 「全文檢索知識庫」 → 「上傳檔案」。
2. 上傳法規條文(可多份分別上傳)。
3. 系統自動建立全文索引(依檔案大小約 1-3 分鐘)。
4. 測試查詢「RoHS 限用物質」確認 Agent 能正確引用條號。

### Step 4：設定使用者與群組可見性
- 開發階段：限定種子團隊試用。
- 試營運階段：擴大到部門 5-10 位種子使用者。
- 正式上線：開放給目標部門全員。

---

## 三、提示詞調校建議

- 若 Agent 引用條號常出錯,請在「執行流程」加上:「回覆前必須核對知識庫條文,引用條號需與原文一致」。
- 若回覆過於模糊,請在「輸出格式」加上:「核心要求需用粗體標示」。
- 若涉及合規判斷,請加上免責聲明:「正式合規判斷請洽法務或外部顧問」。
- 若使用者用簡稱(例:RoHS, REACH),請在執行流程加上:「先解析使用者問題的法規縮寫,再檢索」。

---

## 四、驗測情境（建議至少跑過以下案例）

1. **正向案例 - 條文查詢**: 輸入「RoHS 限制了哪些重金屬?」
   - 預期：正確列出 10 項限用物質與容許值,引用 RoHS 2.0 第 4 條
2. **正向案例 - 義務查詢**: 輸入「REACH 第 33 條告知義務?」
   - 預期：說明 SVHC 0.1% 告知義務,提及 SCIP
3. **邊界案例 - 法規未涵蓋**: 輸入「美國加州 Prop 65 限制?」
   - 預期：誠實告知 KB 未涵蓋,建議洽詢法務或補充 KB
4. **邊界案例 - 合規判斷**: 輸入「我這款 LED 模組符合 RoHS 嗎?」
   - 預期：說明條文要求並建議洽法務正式評估

---

## 五、上線檢核清單

- [ ] 法規條文版本為最新並標註版本日期
- [ ] 提示詞中已強調「以官方公告為準」的免責聲明
- [ ] 提示詞中已加上「不提供法律建議」的保護機制
- [ ] 至少 3 位法務 / 品保種子使用者試用
- [ ] 已建立法規更新時的同步更新流程

---

## 六、上線後維運建議

- 建立法規更新訂閱機制(歐盟、ECHA、UL 等),並由法務每月更新 KB。
- 蒐集使用者最常問的法規條文,作為 KB 補充優先項。
- 進階版可整合「合約條款查詢助手」與「產品認證資料庫」,達到端到端合規查詢。
- 與外部合規顧問建立 SLA,作為 Agent 無法解答時的後援機制。

---

## 七、常見問題

**Q1: Agent 回答不準確？**
先檢查 KB 內容是否完整、條文是否清晰；其次調整提示詞中的「執行流程」與「輸出格式」。

**Q2: 同事抱怨 Agent 太囉嗦？**
在提示詞「輸出格式」中明確要求字數限制或結構（例如：條列三點以內、不超過 200 字）。

**Q3: 找不到對應內容時應如何處理？**
在提示詞中加上：「若知識庫中無法找到答案，請明確回覆『此問題未在現有資料中，建議您聯繫 [窗口] 進一步確認』，切勿自行推測。」
