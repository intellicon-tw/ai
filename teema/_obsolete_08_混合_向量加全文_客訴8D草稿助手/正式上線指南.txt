# 客訴 8D 報告草稿助手 — 正式上線指南

本文件協助您把 Prompt Designer 產出的 Agent 從「概念驗證」推向「正式上線」。
建議由種子團隊以一週為單位推進：第一週 KB 準備、第二週 Agent 建置與驗測、第三週試營運、第四週正式上線。

---

## 一、部署前置作業

### 1. 知識庫準備

1. **向量 KB**:由品保整理 5-10 件代表性歷史 8D 案例,涵蓋不同產品線、不同根因類型。
2. **全文 KB**:由業務與品保整理 5-10 個主要客戶的品質要求文件,匿名化處理。
3. 注意:客戶名稱、聯絡資訊建議使用代號或匿名化。
4. 兩種 KB 編碼皆為 UTF-8。

### 2. 確認權限與工作室

- 在 EgentHub 後台建立或選擇對應的工作室（例如：人資、品保、業務 BU）。
- 確認您的角色為「工作室管理員」或「開發者」，才能建立 Agent 與上傳 KB。
- 確認工作室點數額度足夠（建議至少預留一個月用量緩衝）。

---

## 二、在 EgentHub 建立 Agent

### Step 1：建立 Agent 並選擇模型
登入 EgentHub 後台 → 選擇工作室 → 「建立 Agent」 →
- Agent 名稱：客訴 8D 報告草稿助手
- 預設模型：建議 Claude Sonnet 或 Gemini Pro
- 描述：（複製需求盤點檔案中第一段「角色定位」）

### Step 2：貼入 Prompt Designer 產出的提示詞
將 Prompt Designer 產出的提示詞完整複製貼上，注意保留「問答指引」段落不要修改。

### Step 3：上傳知識庫
1. 進入 Agent 編輯頁。
2. **上傳向量 KB**:「向量知識庫」 → 上傳 8D歷史案例.txt。
3. **上傳全文 KB**:「全文檢索知識庫」 → 上傳 客戶品質要求.txt。
4. 兩種 KB 都需上傳成功。
5. 測試:輸入「客戶 ABB 反映短路問題」,確認 Agent 同時引用兩種 KB。

### Step 4：設定使用者與群組可見性
- 開發階段：限定種子團隊試用。
- 試營運階段：擴大到部門 5-10 位種子使用者。
- 正式上線：開放給目標部門全員。

---

## 三、提示詞調校建議

- 若 Agent 過度依賴歷史案例,請加上:「歷史案例僅供參考,實際根因仍須現場確認」。
- 若客戶 SLA 提醒不夠醒目,請在「輸出格式」加上:「客戶 SLA 必須以粗體 + 結構化方式呈現」。
- 若涉及敏感客訴(人員安全、召回),請加上:「立即提示『升級至最高等級客訴流程』」。
- 若有多個客戶詢問,請在執行流程加上:「先確認是哪一個客戶,再檢索對應的品質要求」。

---

## 四、驗測情境（建議至少跑過以下案例）

1. **正向案例 - 標準客訴**: 輸入「客戶 ABB 變壓器短路 200 pcs 客訴」
   - 預期：正確產出 D1-D4,引用案例 #2024-0456 與 ABB SLA
2. **正向案例 - 多 KB 整合**: 輸入「客戶 XYZ LED 亮度不均」
   - 預期：正確引用 XYZ 客戶品質要求 + 案例 #2025-0102
3. **邊界案例 - 新客戶**: 輸入「客戶 [新客戶] 反映 ...」
   - 預期：誠實告知 KB 中無此客戶,建議業務補充客戶資料
4. **邊界案例 - 安全客訴**: 輸入「車用感測器短路導致客戶測試車 ECU 損壞」
   - 預期：立即標示『升級至最高等級客訴流程』並引用 ACME SLA(若適用)

---

## 五、上線檢核清單

- [ ] 歷史 8D 案例已由品保主管核可
- [ ] 客戶品質要求已由業務主管核可,客戶名稱已匿名化
- [ ] 提示詞已要求草稿須經品保主管簽核
- [ ] 提示詞已加上敏感客訴升級機制
- [ ] 至少 3 位品保 / 品工種子使用者試用

---

## 六、上線後維運建議

- 每月由品保新增 1-2 件代表性 8D 案例,豐富 KB。
- 每季由業務檢視客戶品質要求是否更新。
- 蒐集草稿與最終 8D 報告差異,持續優化提示詞。
- 進階版:整合品保系統(QMS),自動拉入批次號、生產資料。
- 建立『敏感客訴自動升級』流程,確保人員安全相關案件能即時通報高層。

---

## 七、常見問題

**Q1: Agent 回答不準確？**
先檢查 KB 內容是否完整、條文是否清晰；其次調整提示詞中的「執行流程」與「輸出格式」。

**Q2: 同事抱怨 Agent 太囉嗦？**
在提示詞「輸出格式」中明確要求字數限制或結構（例如：條列三點以內、不超過 200 字）。

**Q3: 找不到對應內容時應如何處理？**
在提示詞中加上：「若知識庫中無法找到答案，請明確回覆『此問題未在現有資料中，建議您聯繫 [窗口] 進一步確認』，切勿自行推測。」
