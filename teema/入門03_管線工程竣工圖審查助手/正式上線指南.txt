# 管線工程竣工圖審查助手(入門練習) — 正式上線指南

本文件協助您把 Prompt Designer 產出的 Agent 從「概念驗證」推向「正式上線」。
建議由種子團隊以一週為單位推進：第一週 KB 準備、第二週 Agent 建置與驗測、第三週試營運、第四週正式上線。

---

## 一、部署前置作業

### 1. 知識庫準備

1. 取得「臺北自來水事業處管線工程竣工圖繪製作業須知」最新版本。
   - 本教學包提供「示範摘要」版本作為練習用。
   - 正式上線時請取得官方最新版本。
2. 整理為易讀的條文結構(章節編號清楚、欄位列表化)。
3. 由工程審查單位主管核可後上傳。
4. 編碼採 UTF-8。

### 2. 確認權限與工作室

- 在 EgentHub 後台建立或選擇對應的工作室（例如：人資、品保、業務 BU）。
- 確認您的角色為「工作室管理員」或「開發者」，才能建立 Agent 與上傳 KB。
- 確認工作室點數額度足夠（建議至少預留一個月用量緩衝）。

---

## 二、在 EgentHub 建立 Agent

### Step 1：建立 Agent 並選擇模型
登入 EgentHub 後台 → 選擇工作室 → 「建立 Agent」 →
- Agent 名稱：管線工程竣工圖審查助手(入門練習)
- 預設模型：建議 Claude Sonnet 或 Gemini Pro
- 描述：（複製需求盤點檔案中第一段「角色定位」）

### Step 2：貼入 Prompt Designer 產出的提示詞
將 Prompt Designer 產出的提示詞完整複製貼上，注意保留「問答指引」段落不要修改。

### Step 3：上傳知識庫
1. 進入 Agent 編輯頁。
2. **選擇支援 Vision 的模型**(Claude Sonnet 或 Gemini Pro)。
3. 啟用「附件閱讀」功能。
4. 上傳向量 KB:「向量知識庫」 → 上傳《作業須知摘要.txt》。
5. 系統自動向量化(約 1-2 分鐘)。
6. 測試查詢「標題欄必要欄位」確認 Agent 能引用條文。
7. 上傳本題目附的範例竣工圖 PDF,測試完整流程。

### Step 4：設定使用者與群組可見性
- 開發階段：限定種子團隊試用。
- 試營運階段：擴大到部門 5-10 位種子使用者。
- 正式上線：開放給目標部門全員。

---

## 三、提示詞調校建議

- 若 Agent 經常自行制定規範(非 KB 中內容),請在「執行流程」加上:「審查項目必須引用 KB 中的條文編號,若 KB 無對應條文,標示『規範未明確要求』」。
- 若 Agent 對圖面細節判斷不準,請在「執行流程」加上:「圖面看不清楚的項目標示『無法判斷,建議高解析度版本複核』,不臆測」。
- 若 Agent 過度涉入實質工程判斷,請在「角色定位」加上:「不對工法合理性、設計方案做判斷,只做形式合規審查」。
- 若報告過於冗長,請在「輸出格式」加上:「審查項目以表格呈現,每項說明不超過 1 行」。

---

## 四、驗測情境（建議至少跑過以下案例）

1. **正向案例 - 完整竣工圖**: 上傳本題目附的範例 PDF(管線工程竣工圖)
   - 預期：正確列出基本資料,逐項審查並標示符合 / 不符合,引用條文
2. **正向案例 - 規範條文查詢**: 純文字輸入「標題欄必要欄位有哪些?」
   - 預期：從 KB 引用第 3.2 條,列出 13 項必要欄位
3. **邊界案例 - 缺少必要項**: 上傳一份缺少北方標的圖面
   - 預期：明確標示『北方標缺失,違反第 4.1 條』,提出改善建議
4. **邊界案例 - 解析度不足**: 上傳一份解析度低的圖面
   - 預期：對細部項目標示『無法判斷』,不臆測
5. **邊界案例 - 規範未涵蓋**: 詢問「廠商是否可以用 PE 管做主幹管?」
   - 預期：從 KB 第 6.2 條判斷,說明 PE 不在公共管線材質列表中

## 五、上線檢核清單

- [ ] 已選擇支援 Vision 的模型
- [ ] 已啟用附件閱讀功能
- [ ] 規範 KB 已由工程審查主管核可
- [ ] 提示詞已要求引用條文,不自行制定
- [ ] 提示詞已限定為形式審查,不涉及實質工程判斷
- [ ] 至少 3 位審查人員試用 1 週
- [ ] 已建立規範更新時的 KB 同步流程

## 六、上線後維運建議

- 規範更新時務必同步更新 KB,並於 Agent 公告區註明更新日期。
- 建議僅作為「初步審查」,實質工程審查仍須資深工程師複核。
- 蒐集審查人員實際使用後的回饋,持續優化提示詞與審查項目。
- 進階版可串接工程管理系統,自動產生退補件通知。
- 本題型可複製到其他『規範審查』場景:電力工程、ISO 文件、合約審查等。
- 本題目是『AI 替代形式審查、人力做實質審查』的最佳示範案例。

## 七、常見問題

**Q1: Agent 回答不準確？**
先檢查 KB 內容是否完整、條文是否清晰；其次調整提示詞中的「執行流程」與「輸出格式」。

**Q2: 同事抱怨 Agent 太囉嗦？**
在提示詞「輸出格式」中明確要求字數限制或結構（例如：條列三點以內、不超過 200 字）。

**Q3: 找不到對應內容時應如何處理？**
在提示詞中加上：「若知識庫中無法找到答案，請明確回覆『此問題未在現有資料中，建議您聯繫 [窗口] 進一步確認』，切勿自行推測。」
