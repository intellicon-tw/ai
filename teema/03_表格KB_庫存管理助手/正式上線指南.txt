# 庫存管理助手 — 正式上線指南

本文件協助您把 Prompt Designer 產出的 Agent 從「概念驗證」推向「正式上線」。
建議由種子團隊以一週為單位推進：第一週 KB 準備、第二週 Agent 建置與驗測、第三週試營運、第四週正式上線。

---

## 一、部署前置作業

### 1. 知識庫準備

1. 從 ERP 匯出最新庫存表為 Excel 或 CSV。
2. **資料清洗 SQL 規範**(避免 AI 讀取錯誤):
   - 第一列為單一標題列(欄位名稱)
   - 拆解合併儲存格,每格填入對應值
   - 統一日期格式為 YYYY-MM-DD
   - 欄位命名避免 /、,、() 等特殊符號
   - 移除空白列與 Excel 公式引用
3. 確認包含核心欄位:料號、品名、規格、類別、現有庫存、單位、倉庫、安全庫存、在途量、預計進貨日、折扣率、折扣到期日。
4. 編碼採 UTF-8 BOM(utf-8-sig)。

### 2. 確認權限與工作室

- 在 EgentHub 後台建立或選擇對應的工作室（例如：人資、品保、業務 BU）。
- 確認您的角色為「工作室管理員」或「開發者」，才能建立 Agent 與上傳 KB。
- 確認工作室點數額度足夠（建議至少預留一個月用量緩衝）。

---

## 二、在 EgentHub 建立 Agent

### Step 1：建立 Agent 並選擇模型
登入 EgentHub 後台 → 選擇工作室 → 「建立 Agent」 →
- Agent 名稱：庫存管理助手
- 預設模型：建議 Claude Sonnet 或 Gemini Pro
- 描述：（複製需求盤點檔案中第一段「角色定位」）

### Step 2：貼入 Prompt Designer 產出的提示詞
將 Prompt Designer 產出的提示詞完整複製貼上，注意保留「問答指引」段落不要修改。

### Step 3：上傳知識庫
1. 進入 Agent 編輯頁 → 「**表格查詢知識庫**」 → 「上傳 CSV」。
2. 上傳 庫存表.csv。
3. 確認 EgentHub 自動辨識欄位類型(數字、文字、日期)。
4. 測試查詢「鋁線有庫存嗎?」確認 Agent 能正確讀取。

### Step 4：設定使用者與群組可見性
- 開發階段：限定種子團隊試用。
- 試營運階段：擴大到部門 5-10 位種子使用者。
- 正式上線：開放給目標部門全員。

---

## 三、提示詞調校建議

- 若 Agent 把欄位混淆(例如把單位填到金額),請在「執行流程」加上:「逐項對齊欄位,避免跨欄位混淆」。
- 若 Agent 漏掉折扣資訊,請在「執行流程」加上:「若有折扣欄位非空,一併回覆折扣率與有效期限」。
- 若 Agent 沒主動回報安全庫存比較,請加上:「若現有庫存低於安全庫存,以 ⚠️ 標示並提醒」。
- 若使用者用模糊名稱(例如「鋁線」),請在「執行流程」加上:「先從品名/類別模糊比對,若有多筆相符,列出全部選項供選擇」。
- 若 Agent 自行推測缺漏資料,請在「限制條件」明確加上:「不要自行推測未出現在資料中的資訊;查無資料請回覆『目前查無相關庫存資料,建議聯繫倉管確認』」。

---

## 四、驗測情境（建議至少跑過以下案例）

1. **正向案例 - 模糊查詢**: 輸入「鋁線有庫存嗎?」
   - 預期：列出 AL-1006、AL-1208、AL-1510 三筆,各自顯示庫存量
2. **正向案例 - 折扣查詢**: 輸入「目前哪些連接器有折扣?」
   - 預期：列出 CT-USB-A 15%、CT-USB-C 8%,並標註折扣到期日
3. **邊界案例 - 安全庫存提醒**: 輸入「A02-1 馬達還有嗎?」
   - 預期：回覆庫存 14 組,以 ⚠️ 標示已低於安全庫存 20 組
4. **邊界案例 - 查無資料**: 輸入「碳化鎢刀片庫存?」
   - 預期：回覆「查無此料號,建議聯繫倉管確認」
5. **邊界案例 - 多倉查詢**: 輸入「銅線在 A 倉與 B 倉分別多少?」
   - 預期：依倉庫分別列出 CU-1006(A 倉)、CU-1208/1510(B 倉)

## 五、上線檢核清單

- [ ] 庫存表已由 ERP 系統最新匯出,並完成資料清洗
- [ ] 提示詞已加上「不推測、查無回報窗口」保護機制
- [ ] 提示詞已要求安全庫存比較與折扣主動回報
- [ ] 已建立每日 ERP 自動匯出流程(建議 7:00 前完成)
- [ ] 至少 5 位採購 / 業務種子使用者試用 1 週
- [ ] 倉管主管已確認 Agent 不洩露其他客戶的保留資訊

## 六、上線後維運建議

- 進階版:整合 ERP MCP API,直接即時查詢取代每日匯出。
- 監控查詢日誌,蒐集最常問但 KB 缺漏的料件,作為主檔補充參考。
- 與「ERP 數據分析助手」、「B2B 智慧客服」整合,形成端到端的詢價回應體系。
- 建議與業務 SOP 整合:業務在客戶詢價時先查 Agent,再根據結果決定報價策略。

## 七、常見問題

**Q1: Agent 回答不準確？**
先檢查 KB 內容是否完整、條文是否清晰；其次調整提示詞中的「執行流程」與「輸出格式」。

**Q2: 同事抱怨 Agent 太囉嗦？**
在提示詞「輸出格式」中明確要求字數限制或結構（例如：條列三點以內、不超過 200 字）。

**Q3: 找不到對應內容時應如何處理？**
在提示詞中加上：「若知識庫中無法找到答案，請明確回覆『此問題未在現有資料中，建議您聯繫 [窗口] 進一步確認』，切勿自行推測。」
