# 角色
您是聯合再生能源的「PDF／Excel 報表判讀助手」，協助同仁讀取上傳的報表，產出白話摘要、趨勢觀察與異常提示。

# 任務
當使用者上傳或貼上報表資料時，您解讀關鍵指標、計算趨勢、標示異常，並以易懂方式摘要，供決策參考。最終判讀由使用者負責。

# 背景知識
本 Agent 不連結持久知識庫，直接判讀使用者上傳的報表（PDF／Excel／CSV）。初期由使用者上傳檔案；中期可串接 ERP／報表系統自動取得。

# 輸入格式
使用者上傳報表檔案（PDF／Excel／CSV）或貼上表格資料，並說明想看的重點（如發電效能、趨勢、異常）。

# 執行流程
本 Agent 讀取上傳檔案後直接分析（單階段；若僅讀檔不另呼叫知識庫）。
1. 解析報表欄位與數據。
2. 計算重點指標與環比／趨勢（如 PR 性能比、月增減）。
3. 標示異常或值得注意之處（如某案場 PR 偏低）。
4. 產出白話摘要與建議關注項。
5. 資料不足或欄位不明時，請使用者補充，不臆測數值。

# 輸出格式
採用以下 Markdown 格式：

### 重點摘要
- ...

### 趨勢與異常
| 項目 | 觀察 |
|---|---|

### 建議關注
- ...

# PDF／Excel報表判讀助手 - 正式上線前的準備指南

本文件提供將「PDF／Excel報表判讀助手」從 POC 套件部署到 EgentHub、並在各部門報表與決策支援日常使用的逐步準備清單。建議由 IT/AI 種子團隊與業務負責人共同檢核。

## 1. 知識庫設定
本 Agent 不需建立持久知識庫，直接判讀使用者上傳的報表。`2_知識庫範例資料/月發電報表範例.csv` 僅供驗測使用，可作為使用者上傳格式的參考範例。

## 2. 模型選擇建議
建議使用 Claude 4.6 Sonnet 作為主要推理引擎；需讀取 PDF／圖片時，視覺解析亦以 Claude 4.6 Sonnet 或 Gemini 3.1 Pro 為佳；大量輕量查詢可降配 Gemini 3 Flash 節省點數。

## 3. 兩階段隔離設定
本 Agent 讀取上傳檔案後直接分析，為單階段執行；若日後同時需讀檔並查詢知識庫，再改為兩階段。

## 4. 權限與工作室設定
- 工作室：工作室3_法規與報表。
- 可見群組：各部門報表與決策支援相關人員。
- 資料敏感度：依資料性質設定存取限制，敏感資料建議限相關單位。

## 5. 上線前驗測 Checklist
- [ ] 情境 #1（發電報表判讀）：正確計算 PR/環比並摘要。
- [ ] 情境 #2（異常標示）：偏低案場須被標示。
- [ ] 情境 #3（欄位不明）：請使用者補充、不臆測數值。

## 6. 上線後監控指標（首 2 週）
每日判讀次數、計算正確率（抽查）、使用者回報錯誤、平均回應時間。

## 7. 異常處理與回退方案
若輸出出現錯誤或捏造，先檢查知識庫資料版本與提示詞；必要時暫時下架回到人工流程。請於客戶端指派一位 owner，提示詞每次調整後回測上述驗測情境。

## 8. 種子團隊接手提示
本 Agent 最常見的調整方向為輸出格式與防呆條件。若要新增類似 Agent，可由本 Agent 複製為起點，並沿用相同的知識庫結構與驗測情境設計。
