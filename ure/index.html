共 6 款 Agent POC範圍 高可行、人機協作、零至低串接來源 各部門課前調查真實需求

以下每一頁對應各部門課前調查的真實需求，皆已開發為可在 EgentHub 落地的「完整可跑 POC」。點進任一題可看到：需求與痛點＋開發前 5 問 → 開發思路 → 完整系統提示詞 → 知識庫設計（附可下載範例檔）→ 建置流程 → 驗測情境。

#1.1

合約條款比對審查助手

審查比對型（合約初檢）

業務與法務審閱跨國合約時，要快速比對標準範本、抓出高風險與缺漏條款並產出摘要。

 #1.2

跨國翻譯潤稿助手

內容生成型（中／英／泰翻譯潤稿）

跨國案件（泰／英／中）書信與技術文件翻譯潤稿耗時，且須保住術語與數據正確。

 #2.1

太陽能技術知識問答助手

知識問答型（規格＋話術）

第一線業務與技術客服回答太陽能專業問題時，需精準且一致；本 Agent 輔助第一線人員，不直接面向終端客戶。

 #2.2

客訴與異常重點彙整助手

數據解讀型（彙整＋關聯分析）

品質與客服需把大量客訴與異常整理成結構化重點，並找出重複／批量型態。

 #3.1

法規查詢與彙整助手

知識問答型（法規／認證）

法規、品保、認證與稽核單位需查詢太陽能與再生能源法規認證並彙整重點。

 #3.2

PDF／Excel 報表判讀助手

數據解讀型（報表判讀）

各部門要快速判讀 PDF／Excel 報表，計算趨勢、標示異常並產出摘要。
