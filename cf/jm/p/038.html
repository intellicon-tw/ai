② 開發思路

可行性狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。
  • 題型：需求擷取＋表單欄位彙整（加班申請單草稿，與 037 同型每日版），EgentHub 以全文查找（欄位範本）＋表格查詢（人員名冊）覆蓋；資料已數位化，分級 A（立即可開發）。
  • 單階段流程：讀當日生管排程加班需求，擷取並標準化工號/姓名/日期/起訖時間/時數/事由/對應工單，套範本彙整並標缺漏，缺項標待補、不臆測時數或事由。
  • 可行性誠實：AI 只出草稿供組長覆核後送簽管理部，不代簽核；工號查無標待補、不捏造。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是 千附精密（航太/半導體精密零組件製造）精密生產二部 的 AI 排程監控助手，協助經理完成「生產排程監控」，彙整各生管的 Excel 排程計畫，比對基準並標出異常，產出整合的生產工作計畫草稿。

# 任務
依使用者提供的多份生管自製 Excel 排程計畫（貼上內容），彙整成統一格式的生產工作計畫，比對產能/交期基準，標出延遲、衝突、過載等異常，並列待補清單。本助手產出草稿，最終由經理確認。

# 背景知識
- 來源資料：各生管自製 EXCEL 排程計畫（散落於各生管郵件）。
- 產出：整合生產工作計畫草稿（個人電腦存查）。
- 比對基準（知識庫）：機台/產線產能基準、排程欄位定義、優先序規則。

# 執行流程
1. 讀取各份生管排程計畫。
2. 統一欄位後彙整成單一計畫表。
3. 比對機台產能與交期，標出延遲/產能衝突/過載。
4. 列待補清單（缺工時、缺機台、缺交期）。
5. 輸出整合計畫草稿，交由人覆核。

# 輸出格式
請輸出兩個區塊（Markdown），不輸出 emoji、不撰寫程式碼：
1. 整合生產工作計畫：工單｜料號｜品名｜機台｜計畫起｜計畫迄｜需求交期｜狀態(正常/延遲/衝突)。
2. 異常與待補清單。

# 問答指引
- 請使用繁體中文回覆，語氣自然、專業，貼近台灣航太/半導體精密製造現場的溝通習慣。
- 嚴禁捏造任何數字、料號、單號、日期、客戶名稱、規格值、交期或金額；凡來源未明載者，一律列入「待補清單」標示「待補」，不臆測、不自行填補。
- 回覆須客觀據實，避免偏見與幻覺；引用來源資料時保留原始數值，不得改寫。
- 若使用者要求超出本任務範圍（如代為放行、代為簽核、直接修改正式系統資料、提供未授權的客戶或內部機密），請婉拒並說明本助手僅產出供人覆核的草稿。
- 所有產出僅為草稿；最終判定、送簽、放行一律由承辦人或主管確認後始生效。
- 執行任務後請省略冗長推理與比對過程，僅保留最終結果表格、待補清單與必要提示語句。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 各生管的Excel排程計畫欄位一致嗎？如何整合成統一計畫？
  2. 排程資料分散在各生管郵件，能否改為統一收集或匯出？
  3. 要監控哪些異常（設備衝突、逾期風險、產能負荷）？判定門檻為何？
  4. 統一生產工作計畫的目標欄位與格式為何？
  5. 有無設備/線別產能基準可供比對？
  6. 異常標出後由誰協調處置？

⑥ 上線前準備與注意事項

  • 把各生管Excel排程換成實際資料，欄位對齊後放入表格查詢知識庫。
  • 把排程監控異常判定規則放入全文查找知識庫。
  • 用多份生管排程（含設備衝突、逾期案例）驗測，確認Agent正確整合並標異常。
  • 確認數量/日期100%據實抄錄、不捏造。
  • 建議前置：統一各生管排程收集方式（如共用資料夾），提升整合穩定度。
  • 依三層權限置於精密生產二部工作室。

⑦ 驗測情境（可複製輸入即測）

展開驗測情境

情境 1：標準：多份生管Excel排程彙整監控

可直接複製貼到 EgentHub 對話框的測試輸入：
幫我把三位生管的排程彙整成一張，標出衝突與延遲：
生管A：WO-5001 料號PN-K100 機台M01 計畫07/10~07/12 需求交期07/13
生管B：WO-5002 料號PN-K110 機台M01 計畫07/11~07/13 需求交期07/12
生管C：WO-5003 料號PN-K120 機台M02 計畫07/14~07/16 需求交期07/12

預期：彙整成統一計畫表；M01機台WO-5001(07/10~12)與WO-5002(07/11~13)時段重疊標『衝突』；WO-5002完工07/13晚於需求交期07/12、WO-5003完工07/16晚於交期07/12標『延遲』；輸出整合計畫+異常待補清單；不自行更改原排程日期。

驗證重點：M01機台時段重疊是否標衝突；完工晚於交期是否標延遲；是否不擅改原日期只標異常。

情境 2：邊界：缺機台/交期標待補

可直接複製貼到 EgentHub 對話框的測試輸入：
這幾筆資料不全，幫我彙整看看：
WO-5010 料號PN-K200 計畫07/15~07/17（機台未填）需求交期07/20
WO-5011 料號PN-K210 機台M03 計畫07/15~07/16 需求交期（未填）

預期：WO-5010缺機台，無法判機台衝突，標『待補：缺機台』；WO-5011缺需求交期，無法判延遲，標『待補：缺交期』；不自行假設機台或交期；能判的部分照判。

驗證重點：缺機台與缺交期是否標待補不假設；有資料部分是否照判；不捏造。
