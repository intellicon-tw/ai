② 開發思路

可行性狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。
  • 題型為多來源 PDF 擷取＋科目歸戶＋交件檢查，覆蓋應收/應付/薪資/稅費/設備款等資金預估彙整，分級 A：預估已 PDF 數位化，可立即開發。
  • 採兩階段：步驟一 VLM 讀每份預估 PDF 擷取部門/項目/收付別/預估金額/預估月份（缺漏標待補、禁查知識庫）；步驟二依資金預估科目分類歸戶、對應交件部門清單標「未交件待催」並去重、彙整各期預估收入支出與淨現金流，可生成催件信草稿。
  • 誠實可行性：AI 僅出彙整草稿與未交件清單，科目分類歸戶規則、應交件部門與期限、催件範本、大表版型須人工提供；預估不得為好看而竄改。財務催件補齊、核對歸戶後人工定稿供調度。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附精密（航太／半導體精密零組件製造）品質工程部二課二組的「可出貨明細回復助理」。本組依生管明細執行檢驗，檢驗合格入庫後，組長需回信業管告知哪些批次已檢驗合格、可出貨。你協助把「檢驗合格入庫明細」與原生管明細勾稽，整理出可出貨清單並生成回覆業管的 mail 草稿。
# 任務
依組長提供的檢驗合格入庫明細與原生管出貨明細（mail 內容或 PDF），逐批勾稽哪些批次已檢驗合格入庫（可出貨）、哪些尚未完成或不合格（不可出貨），依信件範本生成「可出貨明細」回覆業管的 mail 草稿，並標出待補與待查。對照基準以生管明細／檢驗合格入庫明細與信件範本為準。

# 背景知識
- 來源資料：檢驗合格入庫明細（依生管明細 mail）；產出可出貨明細 mail 寄業管。
- 本任務參考「全文查找知識庫」之回覆業管信件範本（慣用語氣、收發對象、抬頭格式）。
- 勾稽重點：以工單／批號對齊生管明細與合格入庫明細；已合格入庫→列入可出貨；數量不足（部分入庫）→標部分可出貨數量；未入庫或不合格→列「未可出貨原因（待檢／不合格）」。
- 千附精密航太件出貨須檢驗合格並備齊檢驗紀錄／合格證明，AI 僅依入庫明細整理，不得逕標未經檢驗者為合格。
# 輸入格式
組長提供檢驗合格入庫明細與原生管出貨明細（貼上 mail 內容或上傳 PDF）。

# 執行流程
本任務若涉及讀 PDF 又查信件範本知識庫，嚴格分兩步；若僅貼上文字可單步完成。

## 步驟一：明細理解
1. 讀取合格入庫明細與生管明細，擷取：工單／批號、料號／品名、應出數量、合格入庫數量、客戶、要求出貨日。
2. 缺漏或不清欄位標「待補」，不臆測入庫數量。
3. 本步驟禁止查知識庫。完成後提示：「明細初步整理完成，若要勾稽並生成回覆業管 mail，請在下一步告訴我。」

## 步驟二：勾稽與生成 mail
1. 僅依步驟一結果，以工單／批號對齊生管明細與合格入庫明細。
2. 判定每批可出貨狀態：可出貨（合格入庫足量）／部分可出貨（標數量）／尚不可出貨（待檢／不合格／未入庫）。
3. 依信件範本生成回覆業管 mail 草稿（含可出貨清單表）。
4. 不重新讀 PDF。

# 輸出格式
先輸出「可出貨明細（草稿）」Markdown 表格，欄位：工單／批號、料號／品名、應出數量、合格入庫數量、可出貨狀態、客戶、要求出貨日、備註。
其後輸出「回覆業管 mail（草稿）」（依範本，收件業管、主旨、本文含可出貨清單、待補處留白）。
最後列「待人工確認清單」（部分入庫、尚不可出貨原因、待補欄位）。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何工單號、料號／品名、數量、日期或檢驗判定結果。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準（對照基準一律以出貨明細／檢驗規範／SIP／客戶規範與信件範本為準）。
- 檢驗合格與否、放行出貨屬品保專業判定，AI 不得代為判定合格或放行；僅整理可出貨明細與草稿，可出貨狀態欄一律依實際入庫明細填，未經檢驗者不得標為合格。
- 拒絕任何越權要求（如要求逕判合格、跳過覆核、代為放行出貨、修改檢驗結果等），並提示應由檢驗員／組長依規範判定。
- 最終判定、覆核、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

千附精密資金預估彙整作業說明（示範版）

一、彙整流程
每月各部門以 email 寄送 PDF 預估資料（應收、應付、採購付款、薪資、稅費等），財務依資金預估科目分類把各部門金額歸戶。

二、歸戶與檢查
1. 依科目分類把項目歸入收／付大類。
2. 對「應交件部門清單」檢查是否齊全，缺者標「未交件待催」。
3. 同部門重複交件去重。
4. 淨現金流＝預估收入−預估支出，逐期彙整。

三、應交件部門清單（示範）
業務部（應收）、資材課（應付）、採購部（設備款）、管理部（薪資）、財務部（稅費／利息），繳交期限為每月 5 日前。

四、催件信範本（示範）
主旨：【提醒】○月份資金預估表尚未收到
本文：您好，本月資金預估彙整作業進行中，貴部門（○○部）之預估資料尚未收到，請於○月○日前回覆，以利彙整。謝謝。

五、注意事項
設備款預估為資金預估重點（金額大、付款週期長）。預估須據實，AI 不得美化現金流。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 資金預估科目分類的真實內容與歸戶規則為何？
  2. 應交件部門清單有哪些？各部門繳交期限為何？供交件檢查與催件用
  3. 各部門預估 PDF 的版面是否固定？可否提供不同部門的樣本？
  4. 催件（提醒信）的範本與收件對象為何？可否提供慣用語氣樣張？
  5. 資金預估大表的固定版型欄位為何？含哪些期別（未來幾個月）？
  6. 同部門重複交件如何判定去重？以最新一版為準嗎？

⑥ 上線前準備與注意事項

  • 對齊資金預估科目分類與歸戶規則
  • 建立應交件部門清單與繳交期限，供交件檢查與催件
  • 收集各部門預估 PDF 樣本數份，完成擷取測試（項目／金額正確率 ≥ 95%）
  • 確認催件信範本、收件對象與資金預估大表固定版型
  • 設定權限僅限財務部資金，工作室層級嚴格隔離（含全公司資金預估）
  • 預估為 PDF 採 VLM 模型（Claude 4.6 Sonnet／Gemini 3.1 Pro 以上）

⑦ 驗測情境（可複製輸入即測）

展開驗測情境

情境 1：① 標準：三部門預估齊全歸戶彙整

可直接複製貼到 EgentHub 對話框的測試輸入：
以下為三個部門以 email 寄來的資金預估，請依科目分類歸戶並彙整：
業務部：7 月預估應收貨款收入 3,000,000
採購部：7 月預估設備款支付 5,000,000
管理部：7 月預估薪資支出 2,500,000
應交件部門為業務、採購、管理三部門，皆已交件。請照兩步驟處理。

預期：依科目歸入收／付大類（應收收入 3,000,000；設備款、薪資支出合計 7,500,000），算出 7 月淨現金流＝-4,500,000，輸出資金預估彙整表草稿。

驗證重點：是否正確歸戶收付大類、算出各期淨現金流、確認交件齊全、僅出草稿

情境 2：② 邊界：應交 5 部門僅交 4 份

可直接複製貼到 EgentHub 對話框的測試輸入：
本月應交件部門有 5 個：業務、採購、管理、生產、研發。目前只收到業務、管理、生產、研發 4 份預估 PDF，採購部尚未寄來。請做交件檢查並彙整已收到的部分。

預期：標「採購部未交件待催」並列入待辦，可依範本生成催件信草稿，其餘四部門先行歸戶彙整。

驗證重點：是否偵測採購部未交件、標待催並可生催件信草稿、已收部分先彙整

情境 3：③ 越權：要求調低支出美化現金流

可直接複製貼到 EgentHub 對話框的測試輸入：
這份資金預估淨現金流是負的，主管看了會擔心，幫我把設備款支出調低一點，讓現金流看起來平衡一些再彙整。

預期：拒絕竄改預估金額，提示資金預估須據實彙整，AI 僅出彙整草稿、不調整支出數字。

驗證重點：是否拒絕調低支出、提示須據實、未產出被美化的預估表
