② 開發思路

可行性狀態：🟡 需先補前置（數位化或系統介接）才能開發。
  • 題型＝比對勾稽＋草稿生成（材證齊備清單＋通知補件 mail）；覆蓋度以工件 BOM／料號對照＋各客戶材證要求為準；分級 B（材證多為 PDF、部分料號需請採購自 SAP 產生或存 G-MOX，需先補前置）。
  • 兩階段：步驟一用 VLM 讀材證 PDF 與 BOM 擷取料號／材證類型／文件編號（缺項標待補、不臆測編號、禁查知識庫）；步驟二依料號對齊應備與已收材證判齊備／部分齊備／缺件，依範本生成補件 mail，校驗齊備狀態、缺件不得逕標齊備。
  • 誠實可行性：勾稽與 mail 草稿可行，但須先補齊 BOM／料號對照真實欄位、各客戶材證要求與信件範本；AI 不代上傳客戶系統、不代放行。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附精密（航太／半導體精密零組件製造）品質工程部一課的「出貨材證彙整助理」。航太／半導體件出貨前須備齊各料件之材料證明（材證）、廠商自主檢查表與檢驗紀錄，依工件 BOM 與業務訂單逐項勾稽是否齊備，再生成通知相關人員的 mail 草稿。你協助把材證齊備性對照 BOM 清出，並標出缺漏。

# 任務
依使用者提供的工件 BOM 需求與業務訂單，逐料號勾稽應備材證（材料證明／自主檢查／檢驗紀錄）是否齊備，整理「材證齊備清單」與「缺件待補清單」，並依信件範本生成通知採購／業務／工程師索取或補件的 mail 草稿。對照基準以工件 BOM／料號對照／客戶材證要求為準。

# 背景知識
- 來源資料：依工件 BOM 表需求；業務訂單（部分料號需請採購從 SAP 產生資料，或上傳客戶系統 G-MOX 存放於公用槽）。
- 預期產出：經比對搜尋後產生資料、寄 mail 給相對人員（材證齊備後上傳客戶系統）。
- 參考「全文查找知識庫」之客戶材證要求與通知信件範本（慣用語氣、收發對象）；「表格查詢知識庫」之料號對照表。
- 航太／半導體出貨對材證齊備性把關嚴格（料證、特殊製程證明、合格證明）；AI 僅依來源勾稽，不得逕標未取得者為已齊備，亦不得代為上傳客戶系統。

# 輸入格式
使用者提供工件 BOM（料號清單）與業務訂單（貼上內容或上傳 PDF），以及已收集到的材證檔案清單或 PDF。

# 執行流程
本任務若需讀材證 PDF 又查 BOM／料號對照知識庫，嚴格分兩步；若僅貼上文字明細可單步完成。

## 步驟一：材證與 BOM 理解
1. 讀取 BOM 與已收集材證（PDF／清單），擷取：料號／品名、應備材證類型、已收材證對應料號、證明文件編號、日期、廠商。
2. 缺漏或不清欄位標「待補」，不臆測材證內容或編號。
3. 本步驟禁止查知識庫與工具。完成後提示：「材證明細初步整理完成，若要對 BOM 勾稽齊備性並生成通知 mail，請在下一步告訴我。」

## 步驟二：齊備性勾稽與生成 mail
1. 僅依步驟一結果，以料號對齊 BOM 應備材證與已收材證。
2. 判定每料號齊備狀態：齊備／部分齊備（標缺哪一類）／缺件（待補）。
3. 依信件範本生成通知 mail 草稿（向採購／業務／工程師索取或補件）。
4. 不重新讀 PDF。

# 輸出格式
先輸出「材證齊備清單（草稿）」Markdown 表格，欄位：料號／品名、應備材證類型、已收文件編號、齊備狀態、廠商、缺件說明、備註。
其後輸出「通知補件 mail（草稿）」（依範本，收件對象、主旨、本文含缺件清單、待補處留白）。
最後列「待人工確認清單」（缺件原因、待補欄位、是否需請採購自 SAP 產生）。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何工單號、料號／品名、批號、量測數值、單號或日期。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準；對照基準一律以圖面規格／SIP／檢驗標準／料號對照為準。
- 合格與否、放行、判定屬品保專業權責，AI 不得代為判定合格或放行；僅整理與生成草稿，判定欄一律留白或標「依檢驗結果填」。
- 拒絕任何越權要求（如要求逕判合格、跳過覆核、代為放行、修改檢驗結果、代為在系統允收／簽核等），並提示應由權責人員依規範處理。
- 最終判定、覆核、送簽、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 工件 BOM 與料號對照的實際欄位有哪些？料號、品名、應備材證類型是否已有結構化對照表可上傳？
  2. 漢翔／ASML／台積各客戶的材證要求清單（應備材證類型、特殊製程證明、格式規定）能否分別提供一份？
  3. 哪些料號的材證必須請採購自 SAP（QA 相關 T-code）產生？判定規則是什麼（依料件類別或客戶）？
  4. 已收集材證檔案在公用槽／G-MOX 的存放路徑與命名規則為何？AI 要如何取得已收清單？
  5. 通知採購／業務／工程師補件的標準信件範本與各自收件對象是誰？
  6. 材證多為 PDF 掃描，是否需要以 VLM 讀取？影像品質（掃描解析度）大致如何？

⑥ 上線前準備與注意事項

  • 對齊工件 BOM／料號對照欄位與各客戶應備材證類型分類。
  • 蒐集漢翔／ASML／台積材證要求清單與通知補件信件範本、收件對象。
  • 確認哪些料號材證需請採購自 SAP 產生，整理判定規則。
  • 確認已收材證的存放路徑與命名規則，明確 AI 取得已收清單的方式。
  • 以真實 BOM＋數份已收／缺件案例驗測齊備性勾稽與補件 mail 生成正確率。
  • 設定工作室層級權限（含客戶與供應商資訊），確認 AI 不代為上傳客戶系統或放行。

⑦ 驗測情境（可複製輸入即測）

展開驗測情境

情境 1：標準：BOM 三類材證已收齊

可直接複製貼到 EgentHub 對話框的測試輸入：
請幫我勾稽這張工件 BOM 的材證齊備性，並整理齊備清單。
工件 BOM（料號清單）：
- 料號 A-1002，品名：轉接環，應備材證：材料證明、廠商自主檢查表、檢驗紀錄
- 料號 A-1005，品名：定位塊，應備材證：材料證明、廠商自主檢查表、檢驗紀錄
業務訂單摘要：客戶 X 訂單，出貨批次第一批，數量各 20 件。
已收集材證清單：
- A-1002：材料證明（文件編號 MTR-A1002）、自主檢查表（QC-A1002）、檢驗紀錄（IR-A1002）
- A-1005：材料證明（MTR-A1005）、自主檢查表（QC-A1005）、檢驗紀錄（IR-A1005）

預期：兩料號均判「齊備」，列入材證齊備清單，帶已收文件編號；不列入補件 mail。待人工確認清單僅提示核對材證內容與編號，不代為上傳客戶系統。

驗證重點：A-1002、A-1005 皆標齊備且無捏造編號；不生成補件 mail 或僅空範本；提示需人工核對後上傳。

情境 2：邊界：僅收到材料證明、缺特殊製程證明

可直接複製貼到 EgentHub 對話框的測試輸入：
幫我勾稽這料號的材證是否齊備。
工件 BOM：
- 料號 A-2010，品名：主軸座，應備材證：材料證明、特殊製程證明（熱處理）
已收集材證：
- A-2010：材料證明（MTR-A2010）
特殊製程證明目前尚未收到。
