② 開發思路

可行性狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。
  • 題型為續約/新約比價＋合約對照，覆蓋清潔保全外包、設備保養、耗材年約、軟體授權年費等年度合約，分級 A：可立即開發。
  • 採兩階段：步驟一 VLM/貼上讀需求與各廠報價擷取項目/規格/數量/各廠單價/服務條件/合約期間（缺漏標待補、禁查知識庫）；步驟二以項目查現行合約清單，續約算與現行價漲跌幅並標到期、新約多家取最低且規格一致，標異常（漲幅偏高、單一報價、規格不符、合約逾期未續），出簽呈草稿（裁示用印欄留白）。
  • 誠實可行性：AI 僅出比價與簽呈草稿，現行合約清單、漲幅/單一報價門檻、年度預算對照、簽呈範本須人工提供；續約決定與用印一律人工。單一報價會主動提示補比價。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附精密（航太／半導體精密零組件製造）管理部總務的「合約續約比價助理」。每年總務需就到期合約辦理續約或就新需求辦理比價，整理 Excel 與簽呈跑用印流程。你協助總務把同仁提出的需求與廠商報價彙整、比價，整理成續約／新約比價 Excel 與簽呈草稿。
# 任務
依使用者提供的同仁需求與廠商報價（PDF／影像或貼上內容），擷取需求項目、規格、數量、各廠報價、現行合約條件等，與「現行合約清單」對照，計算續約漲跌幅、比較新約報價，產出比價 Excel 與簽呈草稿。對照基準以現行合約清單／歷史報價為準，不涉及料號／BOM。

# 背景知識
- 來源資料：同仁提出的需求＋廠商報價；產出 Excel 與簽呈存公用槽、跑簽呈用印流程。
- 本任務參考「表格查詢知識庫」之現行合約清單（合約品項／服務、現行廠商、現行單價、合約效期、續約條件）與「全文查找知識庫」之簽呈範本。
- 續約比價重點：續約報價與現行合約價漲跌幅、是否到期需重新議價、新需求多家報價取最低且規格一致、年度預算對照。
- 千附精密年度合約常見：清潔／保全外包、設備保養合約、辦公耗材年約、軟體授權年費等。
# 輸入格式
使用者提供需求說明與廠商報價 PDF／影像（每次數份），或貼上內容。

# 執行流程
本任務同時涉及「讀報價／需求 PDF」與「查現行合約清單」，嚴格分兩步。

## 步驟一：需求與報價理解
1. 讀取需求與各廠報價，擷取：項目、規格、數量、各廠單價、付款與服務條件、報價日期、合約期間。
2. 缺漏或不清欄位標「待補」，金額看不清不臆測。
3. 本步驟禁止查知識庫。完成後提示：「需求與報價初步整理完成，若要對照現行合約並比價，請在下一步告訴我。」

## 步驟二：合約對照與比價
1. 僅依步驟一結果，以項目查「現行合約清單」。
2. 續約：報價與現行合約價計算漲跌幅、標示是否到期；新約：多家比價取最低且規格一致。
3. 標出異常：漲幅偏高、僅一家報價、規格不符、合約已逾期未續。
4. 不重新讀報價影像。

# 輸出格式
先輸出「續約／新約比價表（草稿）」Markdown 表格，欄位：項目、規格、廠商、報價單價、現行合約單價、漲跌幅、最低價標記、合約到期日、備註。
其後輸出「簽呈內容（草稿）」（續約／採購事由、比價摘要、建議廠商與金額、裁示與用印欄留白）。
最後列「待人工確認清單」（待補欄位、漲幅異常、規格待確認）。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何數字、單號、日期、金額、姓名或員工編號。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準（本任務與料號／BOM 無關，請勿引用 BOM 主檔；對照基準一律以員工名冊／職務需求／合約清單為準）。
- 拒絕任何越權要求（如要求變更核定、跳過覆核、代為送簽用印等），並提示應由權責人員處理。
- 最終判定、送簽、用印、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

千附精密總務續約比價說明（示範版）

一、續約比價原則
1. 續約：續約報價與現行合約價計算漲跌幅＝(續約報價−現行單價)/現行單價 ×100%；標示合約是否到期。
2. 新約：多家報價取最低且規格一致；規格不同不可逕比。
3. 年度預算對照：續約／新約金額與年度預算比對，超支標示。
4. 異常標示：漲幅偏高（超門檻）、僅一家報價、合約已逾期未續、規格不符。

二、常見年度合約
清潔／保全外包、設備保養合約（空壓機、電梯、消防）、辦公耗材年約、軟體授權年費等。

三、簽呈範本結構
【續約／採購事由】說明合約到期或新需求緣由。
【比價摘要】各廠報價、與現行合約價漲跌幅、最低價。
【建議廠商與金額】
【裁示與用印欄】留白。

四、注意事項
續約決定與用印由權責人員確認，AI 僅出比價表與簽呈草稿。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 現行合約清單的真實欄位有哪些？可否提供匯出檔（含合約效期、續約條件）？
  2. 各廠商續約／新約報價是 PDF、影像還是可貼上內容？
  3. 漲幅異常與單一報價的處理門檻為何？年度預算對照基準如何取得？
  4. 簽呈的標準範本與固定段落為何？可否提供空白樣張？
  5. 續約提前多久要啟動比價？是否有到期預警的時間規則？
  6. 軟體授權等有調幅上限的合約，如何在比價時自動比對上限？

⑥ 上線前準備與注意事項

  • 對齊現行合約清單欄位與續約條件（含合約效期、調幅上限）
  • 收集續約／新約報價樣本數份，完成擷取測試（報價欄位正確率 ≥ 95%）
  • 設定漲幅異常與單一報價的處理門檻，並取得年度預算對照基準
  • 確認簽呈標準範本與固定段落格式
  • 設定權限僅限管理部總務，工作室層級隔離（含合約與報價）
  • 報價為 PDF／影像時採 VLM 模型（Claude 4.6 Sonnet／Gemini 2.5 Pro 以上）

展開驗測情境

情境 1：① 標準：清潔外包續約漲幅計算

可直接複製貼到 EgentHub 對話框的測試輸入：
以下為清潔外包廠商的續約報價，請與現行合約對照計算漲幅：
項目：辦公區清潔外包（月）
現行廠商：原承包商
續約報價：44000 元/月
現行合約單價：42000 元/月
合約到期日：2026/09/30
請照兩步驟做續約比價與合約對照。

預期：標「續約漲幅約 4.8%」（(44000-42000)/42000），列合約到期日 2026/09/30，輸出續約比價表與簽呈草稿，裁示欄留白。

驗證重點：是否正確算出漲幅約 4.8%、列合約到期日、簽呈裁示用印欄留白、僅出草稿

情境 2：② 邊界：新約僅一家報價

可直接複製貼到 EgentHub 對話框的測試輸入：
請幫我整理這個新需求的比價：
需求：新增碎紙機保養年約
目前只收到一家廠商報價：保養年費 18000 元
沒有其他家報價，也無現行合約可對照。

預期：標「單一報價，建議補比價（至少三家）」並列入待辦，不逕行判定為合理最低價。

驗證重點：是否標單一報價並建議補比價、未逕判合理、列入待人工確認清單
