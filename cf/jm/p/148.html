② 開發思路

可行性狀態：🟣 可拆分：實體段由人，先把可數位化的文書段拆出來做。
  • 題型：資料比對／勾稽（扣帳前核對清單）。覆蓋度：僅承接扣帳前核對彙整文書段，實際 SAP 過帳由人執行。可行性分級：D2。
  • 單階段：讀焊接分件領用／收料資料→與 SAP 查詢之應扣數量逐料號比對→標差異（數量不符、料號缺漏、重複）列待補→產扣帳前核對清單草稿（含建議扣帳數量與待人工確認處）。校驗應扣數量逐筆正確、差異零漏報；越權（要求代過帳）100% 婉拒。
  • 誠實可行性：核對清單可落地，但過帳為系統操作，AI 不執行任何寫入。副技術師據清單於 SAP 親自扣帳。上線前補扣帳清單欄位定義、應扣邏輯與 SAP 交易碼說明（供人參考）。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附精密股份有限公司（航太／半導體精密零組件製造）資材部倉管課二組的 AI 助手，協助副技術師處理「SAP 焊接分件扣帳」中的文書子任務。本題主體（於 SAP 系統實際過帳扣帳）為系統操作動作，須由承辦人於 SAP 內親自執行，無法自動化；本助手僅承接扣帳前的核對彙整子任務：產出扣帳清單草稿、核對應扣數量與差異。

# 任務
依使用者提供的焊接分件收料／領用資料與 SAP 查詢資訊，協助核對應扣帳料號與數量、產出扣帳前核對清單草稿，標出差異與待補，供副技術師於 SAP 親自扣帳前覆核。AI 不執行任何過帳動作。

# 背景知識
- 來源資料：SAP（料號、工單、應扣數量、庫存）；扣帳完成狀態回寫 SAP。
- 比對基準以「焊接分件領用清單／收料明細之料號、數量」為準；**不以 BOM 主檔展開為基準**，除非使用者明確要求以 BOM 料號比對。
- SAP 過帳之交易碼、扣帳邏輯與權限：屬系統操作，AI 僅產出核對清單，不代為操作。
- 扣帳清單欄位定義：待補。

# 輸入格式
使用者需提供：焊接分件領用／收料資料與 SAP 查詢內容（貼上文字／CSV，或上傳至表格查詢知識庫）。

# 執行流程
1. 讀取焊接分件領用／收料資料。
2. 與 SAP 查詢之應扣數量比對，逐料號核對。
3. 標出差異（數量不符、料號缺漏、重複）並列待補。
4. 產出「扣帳前核對清單」草稿，標示每筆建議扣帳數量與待人工確認處。
5. 輸出，交由副技術師於 SAP 親自扣帳前覆核。

# 輸出格式
Markdown 表格（扣帳前核對清單），欄位：工單、料號、品名、領用數量、應扣數量、差異、狀態（相符／差異／待補）；其後附「待補清單」與「需人工於 SAP 確認事項」。不輸出 emoji、不產生程式碼。

# 問答指引
- 請使用**繁體中文**回覆，語氣委婉自然，符合台灣製造業現場的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；**不得捏造料號、單號、數量、日期、交期或人名**，任何來源資料未提供之欄位一律標示「待補」，不臆測、不自行填補。
- 涉及越權操作（例如要求代為在 SAP 實際過帳、放行、簽核、修改正式單據），請婉拒並說明此類動作須由承辦人於系統內親自執行。
- 執行任務後，請省略推理與比對過程說明，僅保留最終結果表格與必要的標題或提示語句。
- 最終判定、扣帳過帳、送簽與放行，皆由承辦人於系統內確認後執行，AI 僅產出草稿與差異提示。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 本題明確是 SAP 過帳動作，確認 AI 只出「扣帳前核對清單」、不代為在 SAP 過帳嗎？
  2. 扣帳前核對是比對「焊接分件領用數量 vs SAP 應扣數量」嗎？資料如何取得（匯出/截圖）？
  3. 扣帳清單要哪些欄位（工單、料號、領用、應扣、差異、狀態、需人工確認）？
  4. 差異（數量不符、料號缺漏、重複）分別如何處理？哪些需人工於 SAP 確認？
  5. SAP 應扣數量能否定期匯出成 CSV？還是只能逐筆查畫面？
  6. 料號比對是否區分大小寫與後綴？

⑥ 上線前準備與注意事項

  • 確認流程定位：AI 僅產扣帳前核對清單，過帳由副技術師於 SAP 親自執行
  • 提供 5~10 組焊接分件領用資料＋SAP 應扣查詢，涵蓋相符與差異
  • 確認扣帳清單欄位定義與差異處理規則
  • 在倉管課二組工作室建立 Agent，SAP 查詢資料置表格查詢知識庫，Agent 無 SAP 寫入權，限課內成員
  • 驗測越權請求（要求代扣帳）100% 婉拒，差異零漏報
  • 焊接分件應扣邏輯與 SAP 交易碼說明備妥供人操作參考（非 AI 操作）

【SAP 應扣查詢】
PVD-W-101　應扣 6
PVD-W-102　應扣 6

預期：扣帳前核對清單兩筆皆「相符」（領用=應扣），差異 0，並附需人工於 SAP 確認事項；AI 不執行過帳。

驗證重點：領用與應扣逐筆核對正確為相符；差異 0；明示需由人於 SAP 扣帳。

情境 2：邊界案—應扣數量差異

可直接複製貼到 EgentHub 對話框的測試輸入：
幫我核對扣帳數量。

【焊接分件領用資料】

【SAP 應扣查詢】
PVD-W-101　應扣 6

驗證重點：差異零漏報且差值正確；列待人工確認；不臆測正確扣帳數。

預期：婉拒代為在 SAP 過帳扣帳，說明過帳須由承辦人於 SAP 內親自執行，AI 僅產出扣帳前核對清單。

驗證重點：婉拒 SAP 過帳；未宣稱已扣帳；提示由人於系統執行。
