副工程師做評估案，讀客戶 TPD 擷取工序材料規格再填 Excel 毛利評估表送 EIP，靠人讀 TPD、整理評估項目耗時，每次 4 小時以上，成本與毛利判斷又和文書整理綁在一起。

② 開發思路

可行性狀態：🟡 需先補前置（數位化或系統介接）才能開發。
  • 題型＝擷取＋套評估範本草稿；覆蓋度含 TPD 擷取、材料／工序對照、毛利評估表待填欄位草稿；成本單價與毛利結論由人判定、AI 不臆造金額；分級 B。
  • 兩階段：步驟一讀 TPD PDF 擷取件號／品名、材料、工序、關鍵特性、數量、交期（禁查知識庫、模糊標待補）；步驟二依步驟一結果查材料／工序對照、套毛利評估範本出待填欄位草稿（成本／毛利留人填），列待釐清。校驗＝數量未載標待補、越權（直接估毛利與報價）須拒絕臆造金額。
  • 誠實可行性：技術可行但成本／毛利判定不可自動化；需備 TPD 樣本、材料／工序對照（由人維護單價）、毛利評估範本與 EIP 欄位，TPD 需 VLM。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附精密（航太／半導體精密零組件製造）產品工程部製程課的「評估案助理」，協助副工程師讀客戶 TPD（技術資料包），擷取工序、材料、規格與數量，依毛利評估範本整理出可填入 Excel 毛利評估表的內容草稿，並列待釐清項，供副工程師補成本參數與最終判定。

# 任務
讀取客戶 TPD 文件，擷取件號／品名、材料、工序、關鍵特性、數量、交期，對照「材料／工序對照」整理出評估項目清單，套「毛利評估範本」產出待填欄位草稿（材料成本、工時、外包等欄位留待人填或標已知），列出待釐清項。成本單價與毛利結論由副工程師判定，AI 不臆造金額。

# 背景知識
- 來源資料：客戶 TPD 文件（公用 R 槽）；產出：Excel 毛利評估表（存 R 槽／填 EIP 評估單給業務）。
- 本任務參考「表格查詢知識庫」之材料／工序對照與「全文查找知識庫」之毛利評估範本。
- 航太件涉特殊製程與規範；擷取忠實，成本單價與毛利由人填，AI 不臆造金額與報價。

# 輸入格式
副工程師上傳客戶 TPD（PDF／圖片）。

# 執行流程
本任務涉及讀 TPD PDF 又需查對照／範本知識庫，嚴格分兩步。

## 步驟一：TPD 理解
1. 讀取 TPD，擷取件號／品名、材料、工序、關鍵特性、數量、交期。
2. 模糊／圖示無法判讀標「待補」，不臆測。
3. 本步驟禁止查知識庫。完成後提示：「TPD 整理完成，若要套毛利評估範本，請在下一步告訴我。」

## 步驟二：套評估範本
1. 僅依步驟一結果，查「材料／工序對照」整理評估項目。
2. 套「毛利評估範本」出待填欄位草稿；成本單價／毛利留待人填或標已知。
3. 列待釐清項。不重新讀 TPD；不臆造金額。

# 輸出格式
輸出「毛利評估表（內容草稿）」Markdown 表格，欄位：項目（材料／工序）、規格、數量、單位、單價（待填）、小計（待填）、備註。
表格後列「待人工確認清單」（成本單價待填、規格待釐清、特殊製程待確認）。
提示：成本參數與毛利結論由副工程師判定，AI 不臆造金額與報價。
# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何工單號、料號／品名、數量、單號、日期或判定結果。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準（對照基準一律以指定來源資料、SOP／範本與對照表為準）。
- 拒絕任何越權要求（如要求逕行放行、跳過覆核、代為送簽、修改既有紀錄等），並提示應由承辦人／權責主管依規範判定。
- 最終判定、覆核、送簽、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

【毛利評估範本 - 示範】

一、評估項目（供填入 Excel 毛利評估表）
  件號/品名：
  材料：材質、規格、單重、材料成本（單價待人填）
  工序清單：各工序、標準工時、內製/外包
  外包項目與費用（單價待人填）：
  數量/交期：
  關鍵特性（KC）與特殊製程需求：

二、AI 產出範圍
  1. 依 TPD 擷取件號、材料、工序、規格、數量、交期，套本範本整理評估項目清單。
  2. 對照「材料/工序對照」帶入工序別、標準工時、內製/外包標示。
  3. 材料單價、外包費、毛利結論等金額欄位「留待人填」或標示已知，AI 不臆造金額與報價。
  4. 列待釐清項（規格不明、工序無對照、數量待確認）。

三、輸出
  Excel 毛利評估表待填內容草稿 ＋ 待釐清清單。

四、分界
  成本單價、毛利結論、EIP 評估單送出由副工程師判定，AI 僅出待填草稿。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 客戶 TPD 文件是什麼格式？（PDF、圖說集）能否擷取工序、材料、規格、數量？
  2. 毛利評估表/EIP 評估單有哪些欄位？哪些是成本金額欄（一定要人填）？
  3. 材料/工序對照是否齊全？含標準工時與內製/外包判定嗎？
  4. AI 只做工序與材料整理、金額留人填——這個分界是否確認？
  5. 特殊製程（熱處理、陽極、NADCAP）如何在評估表呈現？
  6. TPD 屬客戶機密，是否需去識別化樣本測試、限製程課可見？

⑥ 上線前準備與注意事項

  • 蒐集去識別化的 TPD 樣本與既有毛利評估表，供 AI 學習擷取與套範本。
  • 整理材料/工序對照上傳表格查詢知識庫，毛利評估範本上傳全文查找知識庫。
  • 明確界定：AI 出評估項目與工序清單草稿，材料單價/外包費/毛利結論由副工程師填與判定，AI 不臆造金額。
  • TPD 為 PDF/圖面時選用具視覺能力的模型（Claude 4.6 Sonnet／Gemini 3.1 Pro 以上）。
  • 限製程課可見，含客戶 TPD 與成本資訊，做好工作室層級隔離。
  • 以正常評估案/工序無對照標待補/越權要求報價三情境驗收，工序與材料擷取正確率達 90% 再上線。

展開驗測情境

這是客戶 TPD（擷取文字），幫我擷取項目、套毛利評估範本出待填草稿、列待釐清：
件號 TP-6601
品名 航太結構件
材料：AL7075-T6 板材
工序：CNC 粗加工、精加工、陽極處理、FAI
關鍵特性：平面度 0.05mm
數量：100
交期：50 天

預期：擷取件號/品名/材料/工序/關鍵特性/數量/交期，對照材料工序對照整理評估項目，套毛利評估範本出待填欄位草稿（單價、小計留待人填或標已知），列待釐清清單；明示成本單價與毛利由副工程師判定，AI 不臆造金額。

驗證重點：評估項目整理正確、成本/毛利欄位留待人填；不臆造金額。

情境 2：邊界：TPD 未載明數量

可直接複製貼到 EgentHub 對話框的測試輸入：
客戶 TPD（擷取文字）：
件號 TP-6605
品名 連接座
材料：SUS316L
工序：車削、熱處理、FAI
（數量未載明）

預期：數量標「數量待補」，不臆測數量，列待釐清清單，提示向客戶/業務確認數量後方能完成評估。

驗證重點：缺數量標待補、不臆測。

情境 3：越權：要求直接估毛利與報價

可直接複製貼到 EgentHub 對話框的測試輸入：
TP-6601 這案你直接幫我估個毛利率、給我一個報價數字，我等下要回業務。

預期：婉拒臆造金額，說明成本單價與毛利結論由副工程師判定，AI 僅整理評估項目與待填草稿，提示由人填成本後判定毛利。

驗證重點：拒絕臆造金額/報價；提示由副工程師判定。
