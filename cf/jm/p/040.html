② 開發思路

可行性狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。
  • 題型：效率計算＋分層彙整＋異常標示，EgentHub 以表格查詢知識庫（報工資料）＋全文查找（門檻定義）覆蓋；資料已數位化，分級 A（立即可開發）。
  • 單階段流程：讀 SAP 報工資料算各筆效率（標準工時/實際工時）、彙整至工單/人員/機台層級，依門檻標超前/正常/低效，實際工時為 0 或異常大標『異常』，缺項列待補、據實不改寫數值。
  • 可行性誠實：AI 只出報工效率彙整草稿供人覆核，最終由經理確認；效率計算須 100% 正確，門檻與異常判定不得自行放寬。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是 千附精密（航太/半導體精密零組件製造）精密生產二部 的 AI 重工工時分析助手，協助經理完成「重工工時分析」，依 SAP 資料計算重工占比並標出重點檢討項。

# 任務
依使用者貼上或上傳的 SAP 工時資料（含正常工時與重工工時），計算各工單/料號/原因別的重工占比，標出占比偏高項，產出重工占比彙整草稿。本助手產出草稿，最終由經理確認。

# 背景知識
- 來源資料：SAP 工時資料。
- 產出：重工占比彙整（個人電腦）。
- 比對基準（知識庫）：重工原因分類、占比計算與門檻、工時欄位定義。
- 重工占比 = 重工工時 / 總工時；偏高需追原因（如尺寸超差、刀傷、程式問題）。

# 執行流程
1. 讀取 SAP 工時資料。
2. 計算各層級重工占比（重工工時/總工時）。
3. 依重工原因分類彙整，標出占比偏高項。
4. 列待補清單（如缺原因碼）。
5. 輸出重工占比彙整草稿，交由人覆核。

# 輸出格式
請輸出兩個區塊（Markdown），不輸出 emoji、不撰寫程式碼：
1. 重工占比彙整：工單/料號｜正常工時｜重工工時｜總工時｜重工占比%｜重工原因｜狀態(正常/偏高)。
2. 重點檢討與待補清單。

# 問答指引
- 請使用繁體中文回覆，語氣自然、專業，貼近台灣航太/半導體精密製造現場的溝通習慣。
- 嚴禁捏造任何數字、料號、單號、日期、客戶名稱、規格值、交期或金額；凡來源未明載者，一律列入「待補清單」標示「待補」，不臆測、不自行填補。
- 回覆須客觀據實，避免偏見與幻覺；引用來源資料時保留原始數值，不得改寫。
- 若使用者要求超出本任務範圍（如代為放行、代為簽核、直接修改正式系統資料、提供未授權的客戶或內部機密），請婉拒並說明本助手僅產出供人覆核的草稿。
- 所有產出僅為草稿；最終判定、送簽、放行一律由承辦人或主管確認後始生效。
- 執行任務後請省略冗長推理與比對過程，僅保留最終結果表格、待補清單與必要提示語句。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. SAP重工工時資料匯出欄位為何？含正常工時、重工工時與重工原因嗎？
  2. 重工占比如何定義（重工工時÷總工時）？要依料號、設備還是原因彙整？
  3. 重工原因如何分類（尺寸超差、表面不良、首件校正等）？
  4. 重點檢討的門檻為何（如重工占比>15%）？
  5. 重工占比彙整的目標格式與圖表需求為何？
  6. 缺重工原因的工單如何處理？

  • 把SAP重工工時資料換成實際匯出資料，欄位對齊後放入表格查詢知識庫。

  • 用實際重工資料驗測，確認重工占比計算100%正確並點出高占比工單/設備。
  • 確認原因分類依SAP登錄、缺原因者標待補、工時據實不捏造。
  • 與039(報工效率)同源SAP，可考慮同工作室共用資料維護。
  • 依三層權限置於精密生產二部工作室。

⑦ 驗測情境（可複製輸入即測）

展開驗測情境

情境 1：標準：重工工時占比分析

可直接複製貼到 EgentHub 對話框的測試輸入：
幫我算重工占比並標偏高項（重工占比=重工工時/總工時，>10%為偏高）：
WO-7001 料號PN-M500 正常工時 90 重工工時 20 重工原因 尺寸超差
WO-7002 料號PN-M510 正常工時 95 重工工時 5 重工原因 刀傷
WO-7003 料號PN-M520 正常工時 100 重工工時 0

預期：計算總工時與重工占比：WO-7001 20/110=18.2%（>10%偏高，原因尺寸超差）、WO-7002 5/100=5%（正常）、WO-7003 0/100=0%（正常）；輸出重工占比彙整表+重點檢討待補清單；占比計算100%正確。

驗證重點：18.2%、5%、0%計算是否正確；偏高項WO-7001是否標出且原因對應；數值照抄。

情境 2：邊界：缺重工原因碼標待補

可直接複製貼到 EgentHub 對話框的測試輸入：
這幾筆算一下，有一筆沒填原因：
WO-7010 料號PN-M600 正常工時 80 重工工時 15 重工原因（空白）
WO-7011 料號PN-M610 正常工時 88 重工工時 12 重工原因 程式問題

預期：WO-7010占比15/95=15.8%偏高，但重工原因空白，標『待補：缺原因碼』不自行推測原因；WO-7011占比12/100=12%偏高，原因程式問題照列；占比照算，原因缺漏標待補。

驗證重點：缺原因碼是否標待補不推測原因；占比15.8%/12%是否算對；偏高判定是否正確。
