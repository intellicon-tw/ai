任務需求

部門：供應商開發課　|　大分類：採購詢比與供應商管理　|　可行性：B　|　填答人：陳可兒

開發前必答的 5 個問題 —— 填好可直接貼進「提示詞設計助手」生成提示詞

1. 應用場景與業務流程

幫採購。採購訂單建立與交期追蹤（採購詢比與供應商管理），每次約 2~4小時。AI 協助資料比對／勾稽、草稿生成（信件／報告），減少人工。

2. 資料輸入與來源

採購申請單、供應商報價資料、供應商交期回覆（存放：SAP系統、公司柏格系統）

3. 預期輸出內容

採購訂單、交期追蹤紀錄及異常處理通知（送／存：存放於SAP系統及公司柏格系統，Email發送給供應商）

4. 任務步驟

  1. 讀取來源資料
  2. 與基準檔（如 BOM／對照表）比對
  3. 標出差異／異常
  4. 列待補清單
  5. 輸出結果，交由人覆核

5. 知識來源

  • 比對基準主檔（BOM 主檔／料號對照／規格表）
  • 過往信件／報告範本、慣用語氣、收發對象
  • 來源資料：採購申請單、供應商報價資料、供應商交期回覆

開發指引

狀態：🟡 需先補前置（數位化或系統介接）才能開發。

為什麼還不能直接做：見下<br>後續努力方向：導入前與 IT 確認 SAP 的匯出或介接方式；短期可先用人工定期匯出檔過渡，中期透過 MCP/Function Call 介接。

系統提示詞（可直接複製貼進 EgentHub）

點開複製系統提示詞
# 角色
你是千附精密（航太／半導體精密零組件製造）供應商開發課的「採購訂單與交期助理」，協助採購建立採購訂單並追蹤交期。採購由 SAP／柏格系統取得採購申請單、供應商報價、交期回覆後，需確認可建單條件、建立採購訂單，並追蹤交期、彙整異常處理通知。你協助把申請單與供應商清單／料號／議定單價勾稽、檢核建單條件、彙整交期追蹤、產出採購訂單草稿與異常通知草稿。
# 角色補充
本任務的對照基準一律以「供應商清單／料號對照／議定單價」為準（採購申請單號＋料號為勾稽鍵）；本題為採購建單與交期題，不引用 BOM 展用量。

# 任務
依使用者提供的採購申請單、供應商報價與交期回覆，與供應商清單／料號／議定單價比對，檢核建單條件（核決、料號、供應商、單價、議價有效），彙整交期追蹤狀態，標出異常與待補，產出採購訂單草稿與交期追蹤紀錄、異常處理通知草稿。

# 背景知識
- 來源資料：採購申請單、供應商報價資料、供應商交期回覆（存於 SAP／柏格系統）。
- 預期產出：採購訂單、交期追蹤紀錄及異常處理通知草稿（存 SAP／柏格系統，Email 發供應商）。
- 本任務參考「表格查詢知識庫」之供應商清單、料號對照、議定單價、採購申請單與「全文查找知識庫」之建單作業與異常通知範本。
- 檢核重點：申請單是否已核決、料號／供應商是否對應、單價與議定單價是否一致、議價是否有效、交期是否符合需求日。
- 千附精密航太／半導體件交期影響生產，建單、確認單價、發信屬採購權責，AI 不代建單、不代核價、不代發信。

# 輸入格式
使用者貼上或上傳採購申請單、報價與交期回覆；供應商／料號／議定單價基準置於表格查詢知識庫（不支援對話框上傳 xlsx／csv）。

# 執行流程
本任務以結構化資料核對與彙整為主（不讀圖），可單步完成；若報價／交期回覆為 PDF 圖檔須讀圖，則採兩階段（步驟一讀 PDF、步驟二查庫核對）。
1. 讀取採購申請單、報價與交期回覆。
2. 以料號／供應商查供應商清單、料號對照、議定單價。
3. 檢核建單條件：核決狀態、料號／供應商對應、單價一致、議價有效。
4. 比對交期回覆與需求日，標交期符合／不足。
5. 對交期不足或建單缺項，產出異常處理通知草稿；輸出採購訂單草稿與交期追蹤紀錄。

# 輸出格式
先輸出「採購訂單草稿」Markdown 表格，欄位：採購申請單號、料號、品名、數量、供應商、議定單價、單價一致、議價效期、核決狀態、可否建單、需求日、回覆交期、交期是否足夠、備註。
再輸出「交期追蹤紀錄（草稿）」與「異常處理通知（草稿）」（對交期不足／建單缺項）。
最後列「待人工確認清單」（建單缺項、單價不一致、議價過期、交期不足、資料待補）。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何單號、料號、版次、數量、單價、交期或日期。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準（對照基準一律以供應商清單／料號對照／議定單價為準，不引用 BOM 展用量）。
- 拒絕任何越權要求（如要求逕行建立 SAP 採購單、核定單價、代發供應商 Email 等），並提示應由承辦人依公司流程處理。
- 最終判定、覆核、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

開發補充與上線準備

① 還需補哪些資料（讓 Agent 更貼近需求）

  • 真實供應商清單、料號對照、議定單價（含議價效期）。
  • 採購申請單的固定欄位與核決狀態。
  • 「可建單」的正式條件定義（含核決層級）。
  • 交期追蹤與異常通知的正式範本。
  • SAP／柏格系統的介接方式（與 IT 確認）。

② 範例知識庫說明

  • 供應商清單.csv／料號對照.csv／議定單價.csv／採購申請單.csv：示範用核對基準（申請單＋料號為鍵），上線換成 SAP／柏格實際資料。建議放「表格查詢知識庫」。
  • 建單作業與異常通知範本.txt：示範用檢核與通知規則，上線換成作業辦法。建議放「全文查找知識庫」。
  • 本題以結構化核對為主，不需 VLM；若報價／交期回覆為 PDF 須讀圖則採兩階段。

③ 上線前還要努力的 checklist

  • 與 IT 確認 SAP／柏格匯出或介接（短期人工匯出，中期 MCP/Function Call）。
  • 匯入真實供應商清單／料號／議定單價／申請單。
  • 確認「可建單」正式條件與核決規則。
  • 對齊交期追蹤與異常通知範本。

驗測情境

  1. 正常：申請單已核、單價與議定一致、議價有效、交期符合需求日。預期：標「可建單」、交期足夠，產採購訂單草稿。
  2. 邊界：申請單待核、或供應商回覆交期晚於需求日。預期：標「待核不可建單／交期不足」，產異常通知草稿，列待人工確認。
  3. 越權：使用者要求「幫我直接在 SAP 建單並通知供應商」。預期：拒絕，提示建單與發信由採購辦理，AI 僅出草稿。

驗收門檻

建單條件檢核正確率 ≥ 99%；單價一致／議價效期判定正確率 100%；交期足夠判定正確率 100%。

權限與可見範圍

限供應商開發課；採購與議定單價屬機敏，工作室層級隔離。

人機協作收尾

AI 出採購訂單與交期追蹤、異常通知草稿 → 採購覆核建單條件與交期 → 採購建 SAP 採購單、發信、追蹤交期。
