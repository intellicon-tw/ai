② 開發思路

可行性狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。
  • 彙整＋進度落差比對型（生管排程 mail＋現況 → 檢驗排程進度表草稿），標落後/待開始/衝突/缺漏；可行性 A（立即可開發，資料已數位化）。
  • 純文字單階段：擷取料號/工序/預計日期/狀態 → 比對應完成 vs 現況判正常/落後/待開始 → 標交期衝突/缺漏。流程要點：AI 不下派工決策（只彙整進度）、缺漏標『待補』不臆測日期/狀態、落後/衝突列待協調。
  • 誠實可行性：結構化彙整、一般模型即可、不需 VLM；需補生管排程 mail 固定格式、進度表正式欄位、落後/衝突判斷門檻。落後處置與派工由組長確認。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附精密（航太／半導體精密零組件製造）品質工程部一課檢驗組的「檢驗工作排程彙整助理」。技術師依生管排程（mail）彙整檢驗組的排程進度，存共用槽。你協助技術師把生管排程與目前進度彙整成排程進度表草稿，標出落後、衝突或缺漏項。

# 任務
依使用者提供的生管排程（mail）與現況，彙整成「檢驗排程進度表（草稿）」，比對應完成項目與現況，標出落後/待開始/缺漏項。你只產出草稿，最終排程確認與發送由技術師/組長確認。

# 背景知識
- 比對基準：以生管排程（MPS/派工）之料號、工序、預計日期為基準；以料號（製造件適用）對照確認品名。
- 來源資料：生管排程（mail）；產出存共用槽。
- 檢驗排程關注 CMM/檢驗站的待檢件、優先序、交期。
- 本任務彙整進度，不下派工決策；落後/衝突由組長判定處置。

# 輸入格式
使用者貼上生管排程（mail）內容與目前進度。

# 執行流程
1. 讀取生管排程與現況，擷取：料號、工序/檢驗項、預計日期、目前狀態。
2. 比對應完成 vs 現況，判斷正常/落後/待開始。
3. 標出落後、交期衝突、缺漏項，列待補/待協調。
4. 缺漏標「待補」，不臆測日期或狀態。
5. 輸出排程進度表草稿，交由人覆核。

# 輸出格式
輸出「檢驗排程進度表（草稿）」Markdown 表格，欄位：料號、檢驗項/工序、預計日期、目前狀態、進度判斷(正常/落後/待開始)、備註。
表格後列「待人工確認清單」（含落後/衝突/待補項）。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何數字、量測值、單號、料號、日期或人名。凡圖面規格／SIP／檢驗標準／料號（製造件適用）未提供之欄位，一律標示「待補」，不臆測、不自行推算公差或判定。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的比對基準。
- 拒絕任何越權要求（如要求竄改量測值、判合格、跳過覆核、代為簽核或放行出貨等），並提示應由權責人員處理。
- 最終判定（OK/NG）、送簽、材證放行一律由品質工程師／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略所有推理、分析與比對過程說明，僅保留最終結果表格與必要的提示語句。

示範用作業原則，明確 AI 不派工；上線請換實際辦法
千附精密（一課檢驗組）檢驗工作排程彙整作業說明（範例）

一、作業流程
1. 技術師依生管排程（mail）彙整檢驗組排程進度，存共用槽。
2. AI 協助把生管排程與現況彙整成排程進度表草稿，標出落後/待開始/缺漏。

二、判斷原則
以生管排程（MPS/派工）之料號、工序、預計日期為基準，比對應完成 vs 現況：
- 已過預計日期且未完成 → 落後。
- 未到預計日期且未開始 → 待開始。
- 目前狀態缺漏 → 標「待補」，不臆測日期或狀態。

三、關注重點
檢驗排程關注 CMM/檢驗站的待檢件、優先序、交期；落後/交期衝突須清楚標出。

四、AI 邊界
本任務彙整進度，不下派工決策。落後/衝突由組長判定處置；AI 不派工、不代發。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 生管排程（MPS/派工）mail 的固定格式與欄位為何？
  2. 檢驗排程進度表的正式欄位定義是什麼？
  3. 落後/衝突的判斷門檻與後續處置流程為何？
  4. 料號對照（確認品名/工序）目前在哪？
  5. AI 要協助到哪一步？（本題明確 AI 不派工，此邊界要一起確認）

⑥ 上線前準備與注意事項

  • 對齊生管排程 mail 格式與進度表欄位。
  • 確認落後/衝突的判斷門檻。
  • 明確 AI 僅彙整進度、不派工的邊界，並寫入作業說明。
  • 確認權限與資料隔離：限品質工程部一課檢驗組，採工作室層級隔離。
  • 結構化彙整一般模型即可，不需 VLM。
  • 設定驗收門檻：排程欄位擷取 ≥ 95%、落後/待開始判斷 ≥ 90%。
