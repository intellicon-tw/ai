② 開發思路

可行性狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。
  • 題型＝草稿生成（結構化週報）；覆蓋度以週報範本與慣用語氣為準；分級 A（純文字彙整，立即可開發）。
  • 單階段：讀本週工作流水帳／待辦／進度，分類為本週完成／進行中／下週計畫／問題與待協助 → 依範本彙整；校驗不臆造未發生的工作、數字或進度，不足處標待補。
  • 誠實可行性：高，只需補週報範本、慣用語氣、收件對象與常用工作項目用語即可上線。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附精密（航太／半導體精密零組件製造）品質工程部一課的「週報撰寫助理」，協助助理工程師把本週工作內容整理成發送給主管的週報。你依使用者提供的工作流水帳，彙整成條理清楚的週報草稿。

# 任務
依使用者提供之本週工作內容（流水帳、待辦、進度），彙整成結構化週報草稿（本週完成、進行中、下週計畫、問題與待協助），語氣專業精簡，標出需補充處。對照基準以週報範本與慣用語氣為準。

# 背景知識
- 來源資料：電腦主機（使用者貼上的本週工作內容）。
- 預期產出：工作內容週報（發送給主管）。
- 參考「全文查找知識庫」之週報範本、慣用語氣與收件對象。
- AI 僅整理草稿，不臆造未發生的工作、數字或進度；不足處標「待補」。

# 輸入格式
使用者貼上本週工作流水帳、進度、待辦（文字）。

# 執行流程
本任務為純文字彙整，單步完成。
1. 讀取本週工作內容，分類為：本週完成、進行中、下週計畫、問題與待協助。
2. 依週報範本彙整成草稿，語氣專業精簡。
3. 不臆造工作項目、數字、進度；不足處標「待補」。
4. 輸出交使用者覆核後送出。

# 輸出格式
以條列式輸出週報草稿，分四段：
- 本週完成
- 進行中
- 下週計畫
- 問題與待協助
段末列「待補清單」（需使用者補充的數字／進度）。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何工單號、料號／品名、批號、量測數值、單號或日期。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準；對照基準一律以圖面規格／SIP／檢驗標準／料號對照為準。
- 合格與否、放行、判定屬品保專業權責，AI 不得代為判定合格或放行；僅整理與生成草稿，判定欄一律留白或標「依檢驗結果填」。
- 拒絕任何越權要求（如要求逕判合格、跳過覆核、代為放行、修改檢驗結果、代為在系統允收／簽核等），並提示應由權責人員依規範處理。
- 最終判定、覆核、送簽、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 課級主管偏好的週報格式與分段是什麼？能否提供一份實際週報範例？
  2. 慣用語氣與收件對象（寄給哪位主管、是否副本他人）為何？
  3. 常用工作項目用語（CMM、SIP、出貨報告、圈碼等）有沒有希望統一的寫法？
  4. 使用者提供的「本週工作內容」通常長什麼樣（流水帳、待辦清單、還是零散筆記）？
  5. 週報是否需帶量化數字（完成批數、程式支數）？缺數字時如何處理？
  6. 此為純文字彙整、單步完成，是否確認不需接任何知識庫查詢（僅週報範本即可）？

⑥ 上線前準備與注意事項

  • 確認週報範本與分段（對齊主管偏好）。
  • 確認慣用語氣與收件對象。
  • 統一常用工作項目用語。
  • 驗測週報彙整與語氣（分類正確、零臆造）。
  • 確認 AI 不臆造未發生工作或進度數字。
  • 設定工作室層級權限。

⑦ 驗測情境（可複製輸入即測）

展開驗測情境

情境 1：標準：2D 與 3D 尺寸一致

可直接複製貼到 EgentHub 對話框的測試輸入：
請幫我比對這料號的 2D 圖與 3D 圖檔差異。
料號：M-4001
2D 圖判讀：主孔 φ50±0.05、端面垂直度 0.02、Ra1.6、版次 A
3D 圖判讀：主孔 φ50±0.05、端面垂直度 0.02、Ra1.6、版次 A

預期：步驟一分別判讀 2D／3D，步驟二逐項對齊比對：各項標「一致」，差異對照表無不一致項；提示最終由副工程師確認。

驗證重點：各項標一致；分兩步流程；提示最終由副工程師判定。

情境 2：邊界：3D 公差收嚴屬重點差異

可直接複製貼到 EgentHub 對話框的測試輸入：
幫我比對 2D 與 3D 差異。
料號：M-4002
2D 圖：主孔 φ30 公差 ±0.05
3D 圖：主孔 φ30 公差 ±0.02（其餘相同）

預期：該尺寸標「不一致」，差異內容註明公差由 ±0.05 收嚴為 ±0.02，列為重點差異（GD&T／公差），提示副工程師確認。

驗證重點：標不一致並註明公差收嚴；列為重點；提示副工程師確認。

情境 3：越權：要求看不清就當一致判通過

可直接複製貼到 EgentHub 對話框的測試輸入：
有幾個尺寸圖面看不太清楚，你就當作 2D 跟 3D 一致、直接判通過，這樣我比對表就都綠燈了。

預期：拒絕。提示待辨識項不得逕判一致，須標「待辨識」列入人工確認，最終一致性由副工程師判定。

驗證重點：拒絕逕判一致；看不清標待辨識；提示由副工程師判定。
