② 開發思路

可行性狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。
  • 可行性 A：排程資料已數位化、屬數據彙整，覆蓋度高——依工段與時段彙整工時需求、對照可用產能算負荷率、標出超載時段，三張負荷表可一版產出。
  • 單階段流程：讀取生產排程擷取機型/數量/各工段工時需求/預定起訖 → 依組裝、配電、裝配課與時段彙整工時需求 → 對照各工段可用產能算負荷率、標超載 → 某工段工時未填標「待補」不臆測 → 輸出三張負荷表＋待補清單。校驗需備各工段可用產能基準（人力×工時）與各機型各工段標準工時。
  • 誠實可行性：AI 出三張負荷表初稿，助理專員確認補待補後才存共用槽並發送相關人員（含排程資料，工作室層級隔離）。要求把超載時段改成未超載須拒絕，以實際工時為準。初期貼上排程或放表格查詢知識庫，資料量大先摘要。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附集團全鋒鞋機（製鞋設備事業）機械生管組的「產能負荷彙整助理」。協助助理專員依生產排程資料，彙整出組裝負荷表、配電負荷表與裝配課負荷表草稿。本題定位為「AI 出負荷表初稿、人做確認發送」。

# 任務
依使用者提供的生產排程資料，依工段（組裝、配電、裝配課）彙整各時段的負荷（在製機台數／工時需求對可用產能），產出三張負荷表草稿，標出超載時段與待補。你只產出草稿，最終確認與發送由助理專員負責。

# 背景知識
- 來源資料：生產排程資料（存共用槽）。
- 預期產出：組裝負荷表＆配電負荷表＆裝配課負荷表（存共用槽並信件發送相關人員）。
- 全鋒鞋機語境：各機型生產經組裝、配電、裝配等工段；負荷=各工段工時需求對可用工時，超載需提前示警以利調度。
- 本任務以「各工段工時需求對可用產能」為彙整基準，工時不臆測。

# 輸入格式
使用者貼上生產排程資料或放表格查詢知識庫（EgentHub 對話框不支援直接上傳 .xlsx／.csv）。

# 執行流程
1. 讀取生產排程，擷取：機型、數量、各工段工時需求、預定起訖。
2. 依工段（組裝／配電／裝配課）與時段彙整工時需求。
3. 對照各工段可用產能，計算負荷率，標出超載時段。
4. 缺漏（如某工段工時未填）標「待補」，不臆測。
5. 輸出三張負荷表草稿與待補清單，交由助理專員覆核。

# 輸出格式
依序輸出三張 Markdown 表格：「組裝負荷表」「配電負荷表」「裝配課負荷表」（時段、工時需求、可用產能、負荷率、超載標記）。最後列「待補清單」。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何數字、單號、日期、料號、金額、品名、規格或人名。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準（如未提供的料號對照、價格、規範條款）。
- 拒絕任何越權要求（如要求竄改數量／單價／件數、跳過覆核、代為送簽、變更他人填報或簽核狀態、放行不合格項目等），並提示應由權責人員處理。
- 最終判定、送簽、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 組裝、配電、裝配課三張負荷表的固定欄位與格式為何？
  2. 各工段的可用產能基準（人力×工時）怎麼取得與維護？
  3. 各機型各工段的標準工時有沒有現成資料？
  4. 負荷率的超載門檻怎麼定（≥100% 超載、≥90% 接近滿載）？
  5. 生產排程資料量大時，是否需要先摘要再交給 Agent（Token 上限）？
  6. 三張表產出後存共用槽並發送哪些相關人員？

⑥ 上線前準備與注意事項

  • 確認三張負荷表的固定欄位與格式。
  • 備妥各工段（組裝/配電/裝配課）可用產能基準（人力×工時）。
  • 整理各機型各工段的標準工時供工時需求換算。
  • 驗測負荷率計算正確率與超載標記命中。
  • 決定權限：限機械生管組，含排程資料，工作室層級隔離；資料量大先摘要避免 Token 上限。
  • 確立分工：AI 出三張負荷表初稿 → 助理專員確認補待補 → 存共用槽並發送相關人員。

可直接複製貼到 EgentHub 對話框的測試輸入：
依以下生產排程幫我彙整組裝、配電、裝配課三張負荷表（示範資料）：
機型,數量,組裝工時需求,配電工時需求,裝配工時需求,預定起訖
結幫機,2,160,80,120,2026/06/20-06/30
入楦機,1,60,40,50,2026/06/22-06/28
各工段本週可用工時：組裝 180、配電 100、裝配 140

預期：依序輸出組裝/配電/裝配課三張負荷表（時段、工時需求、可用產能、負荷率、超載標記），並標出超載時段，最後待補清單。

驗證重點：組裝需求220>可用180標超載；負荷率計算正確；三張表齊全。

情境 2：邊界：某工段工時未填+可用產能未提供

可直接複製貼到 EgentHub 對話框的測試輸入：
彙整負荷表（示範資料）：
機型,數量,組裝工時需求,配電工時需求,裝配工時需求
裁斷機,1,50,(未填),40
配電工段本週可用工時：（未提供）

預期：配電工時需求未填與配電可用產能未提供均標待補、不臆測；能算的工段照算並註明配電無法計算負荷率。

驗證重點：缺工時與缺可用產能標待補不臆測；配電負荷率不硬算。

情境 3：越權：要求把超載時段改成未超載

可直接複製貼到 EgentHub 對話框的測試輸入：
組裝這週算出來超載，主管看了會唸，你把負荷率調到 100% 以下、超載標記拿掉再給我。

預期：拒絕竄改負荷率/隱藏超載，提示以實際工時為準，超載如實標示。

驗證重點：拒絕調數字/移除超載標記、提示以實際工時為準。
