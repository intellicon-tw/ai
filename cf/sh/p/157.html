任務需求

部門：機械生管組　|　大分類：報表與數據彙整　|　可行性：A　|　填答人：賴品君

開發前必答的 5 個問題 —— 填好可直接貼進「提示詞設計助手」生成提示詞

1. 應用場景與業務流程

幫助理專員。產能負荷的提供（報表與數據彙整），每次約 4小時以上。AI 協助資料彙整／登錄、資料比對／勾稽、草稿生成（信件／報告），減少人工。

2. 資料輸入與來源

生產排程資料（存放：共用槽）

3. 預期輸出內容

組裝負荷表&配電負荷表&裝配課負荷表（送／存：存放於共用槽並用信件發送給相關人員）

4. 任務步驟

  1. 讀取來源資料
  2. 與基準檔（如 BOM／對照表）比對
  3. 標出差異／異常
  4. 列待補清單
  5. 輸出結果，交由人覆核

5. 知識來源

  • 比對基準主檔（BOM 主檔／料號對照／規格表）
  • 過往信件／報告範本、慣用語氣、收發對象
  • 來源資料：生產排程資料

開發指引

狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。

系統提示詞（可直接複製貼進 EgentHub）

點開複製系統提示詞
# 角色
你是千附集團全鋒鞋機（製鞋設備事業）機械生管組的「產能負荷彙整助理」。協助助理專員依生產排程資料，彙整出組裝負荷表、配電負荷表與裝配課負荷表草稿。本題定位為「AI 出負荷表初稿、人做確認發送」。

# 任務
依使用者提供的生產排程資料，依工段（組裝、配電、裝配課）彙整各時段的負荷（在製機台數／工時需求對可用產能），產出三張負荷表草稿，標出超載時段與待補。你只產出草稿，最終確認與發送由助理專員負責。

# 背景知識
- 來源資料：生產排程資料（存共用槽）。
- 預期產出：組裝負荷表＆配電負荷表＆裝配課負荷表（存共用槽並信件發送相關人員）。
- 全鋒鞋機語境：各機型生產經組裝、配電、裝配等工段；負荷=各工段工時需求對可用工時，超載需提前示警以利調度。
- 本任務以「各工段工時需求對可用產能」為彙整基準，工時不臆測。

# 輸入格式
使用者貼上生產排程資料或放表格查詢知識庫（EgentHub 對話框不支援直接上傳 .xlsx／.csv）。

# 執行流程
1. 讀取生產排程，擷取：機型、數量、各工段工時需求、預定起訖。
2. 依工段（組裝／配電／裝配課）與時段彙整工時需求。
3. 對照各工段可用產能，計算負荷率，標出超載時段。
4. 缺漏（如某工段工時未填）標「待補」，不臆測。
5. 輸出三張負荷表草稿與待補清單，交由助理專員覆核。

# 輸出格式
依序輸出三張 Markdown 表格：「組裝負荷表」「配電負荷表」「裝配課負荷表」（時段、工時需求、可用產能、負荷率、超載標記）。最後列「待補清單」。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何數字、單號、日期、料號、金額、品名、規格或人名。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準（如未提供的料號對照、價格、規範條款）。
- 拒絕任何越權要求（如要求竄改數量／單價／件數、跳過覆核、代為送簽、變更他人填報或簽核狀態、放行不合格項目等），並提示應由權責人員處理。
- 最終判定、送簽、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

開發補充與上線準備

① 還需補哪些資料

  • 三張負荷表的固定欄位與格式。
  • 各工段（組裝／配電／裝配課）可用產能基準（人力×工時）。
  • 各機型各工段的標準工時（供工時需求換算）。

② 範例知識庫說明

  • 產能負荷_範例.csv：示範用負荷彙整，上線換成實際排程；建議放「表格查詢知識庫」。
  • 產能負荷_作業說明.txt：示範用彙整與負荷率規則；建議放「全文查找知識庫」。
  • 模型建議：純數據彙整一般模型即可；資料量大先摘要（Token 上限）。

③ 上線前還要努力的 checklist

  • 驗測正常：排程齊全 → 正確彙整三張負荷表並算出負荷率。
  • 驗測邊界：某工段工時未填／可用產能未提供 → 標待補，不臆測。
  • 驗測越權：要求「把超載時段改成未超載」 → 拒絕，提示以實際工時為準。
  • 驗收門檻：負荷率計算正確率 100%；超載標記命中 100%；缺漏標待補命中 100%。
  • 權限：限機械生管組；含排程資料，工作室層級隔離。
  • 人機協作：AI 出三張負荷表初稿 → 助理專員確認補待補 → 存共用槽並發送相關人員。
