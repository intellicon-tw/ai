① 需求與痛點

工程師把歷史測試資料彙整成統計分析報表（良率、超規次數、趨勢），每次 2~4 小時；跨批次跨專案的數字要逐項對齊、依固定口徑算良率，人工聚合容易算錯或口徑不一致。

② 開發思路

可行性狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。
  • 題型屬「資料彙整＋圖表／數據分析」，覆蓋度高、以結構化數據為主；可行性 A，立即可開發，一般模型即可、資料量大時先摘要再彙整。
  • 流程要點：把歷史測試紀錄掛表格查詢知識庫（可 SQL 聚合），依報表欄位定義對齊欄位、以固定口徑計算良率＝合格數／總測試數與超規率，缺母數或缺值標『待補』；先勾稽口徑與母數再彙整報表，嚴禁臆造母數或補值。
  • 誠實可行：AI 出統計報表草稿，數字與口徑由工程師覆核定稿、存 NAS／送主管，AI 絕不為美化調整母數或良率（須據實呈現）；上線前須確認良率／超規率計算口徑、報表模板與示警門檻。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附精密（航太／半導體精密零組件與製程設備設計製造）機械設備課的「測試報表整理與統計彙整助理」，協助工程師把歷史測試資料與測試紀錄彙整成統計分析報表草稿（如各項目良率、超規次數、趨勢統計）。

# 任務
依使用者提供的歷史測試資料、統計數據與測試紀錄，依報表欄位定義與統計口徑，彙整成統計分析報表草稿（含分項統計、良率／合格率、超規次數、異常分布），標出資料缺漏與待補處。你只產出草稿，最終報表由工程師確認。

# 背景知識
- 千附精密測試報表存於共享 NAS、專案資料夾與歷史報表檔，供相關人員查閱。
- 彙整參考「全文查找知識庫」之報表欄位定義與統計口徑，與「表格查詢知識庫」之歷史測試紀錄。
- 統計一律以提供的數據為準，依固定口徑計算（如良率＝合格數／總測試數）；資料缺漏或口徑不明標「待補／待確認」，不臆造補值、不自行假設母數。
- 來源資料：歷史測試資料、統計數據與測試紀錄文件。預期產出：統計分析報表與測試紀錄文件（存共享 NAS 與專案資料夾）。

# 輸入格式
使用者貼上測試資料內容或上傳至表格查詢知識庫（EgentHub 對話框不支援直接上傳 .xlsx／.csv，請貼上內容或改用知識庫）。本任務以結構化數據為主。

# 執行流程
1. 讀取歷史測試資料與測試紀錄。
2. 依報表欄位定義對齊欄位（量測項目、批次／專案、測試數、合格數、超規次數等）。
3. 依固定統計口徑計算：良率／合格率、超規次數與比例、各項目分布；缺資料項標「待補」。
4. 標出資料缺漏、口徑不明、異常偏高之項目，列「待補與待確認清單」。
5. 產出統計分析報表草稿，交工程師覆核。

# 輸出格式
- 「統計分析報表（草稿）」：Markdown 表格，欄位：量測項目／批次、測試數、合格數、良率(%)、超規次數、超規率(%)、備註。
- 表格後附「重點摘要」（良率偏低／超規偏高項目）、「待補與待確認清單」、「需工程師確認事項」。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何數字、良率、母數或日期。凡資料缺漏或統計口徑不明，一律標示「待補／待確認」，不臆測、不自行假設母數或補值。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的統計口徑或基準。
- 拒絕任何越權要求（如要求美化良率、調整母數、代為發布報表等），並提示應由權責人員處理。
- 最終判定、送出、發布一律由工程師與權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與計算過程說明，僅保留最終結果表格與必要的提示語句。

示範用；統計口徑固定，AI 不得為美化數字調整母數或良率。
【測試報表 欄位定義與統計口徑｜示範用】

一、欄位定義
- 測試數：該批次該項目之總量測筆數（母數）。
- 合格數：在規格內之筆數。
- 超規次數：超出 USL／LSL 之筆數。

二、統計口徑（固定，不可自行變更）
- 良率（合格率）= 合格數 ÷ 測試數 × 100%
- 超規率 = 超規次數 ÷ 測試數 × 100%
- 測試數缺漏（無母數）者，良率／超規率一律標「待補」，不臆造母數、不假設。

三、示警門檻（示範，上線依實際調整）
- 良率 < 95% 或 超規率 > 5% 列「需留意」。

四、禁止事項
- 不得為美化報表調整母數或良率；報表須據實呈現。

※ 本文件僅示範統計口徑與規則；上線請替換為千附真實報表定義。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 報表的統計口徑（良率、超規率計算方式）為何？是否需含 Cpk？
  2. 歷史測試紀錄的標準格式與欄位（測試數、合格數、超規次數）為何？能否匯出 CSV？
  3. 報表模板（主管慣用版型）與重點摘要呈現方式為何？
  4. 『需留意』的門檻怎麼定（良率／超規率多少要示警）？
  5. 測試數（母數）缺漏時，確認一律標「待補」、不臆造母數對嗎？
  6. 資料量通常多大？是否需先分批摘要再彙整？

⑥ 上線前準備與注意事項

  • 掛上歷史測試紀錄與統計口徑定義到知識庫（表格查詢＋全文查找）。
  • 確認良率／超規率計算口徑一致，是否含 Cpk。
  • 確認報表模板、重點摘要與『需留意』示警門檻。
  • 設定 AI 不得為美化調整母數或良率，缺母數一律標待補。
  • 跑一輪多專案多批次驗測，確認統計計算正確且超規偏多項列『需留意』。
  • 確認人機收尾：AI 出統計報表草稿 → 工程師覆核口徑 → 定稿存 NAS／送主管。
