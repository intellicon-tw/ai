任務需求

部門：中區事業部台中　|　大分類：報表與數據彙整　|　可行性：A　|　填答人：林君如

開發前必答的 5 個問題 —— 填好可直接貼進「提示詞設計助手」生成提示詞

1. 應用場景與業務流程

幫助理專員。每周全區周報（報表與數據彙整），每次約 30 分鐘以內。AI 協助資料彙整／登錄、資料比對／勾稽，減少人工。

2. 資料輸入與來源

各區助理提供當周的結案、暫估資料，存放在Q槽（存放：來源資料是各區上EIP/廠工專案管理/暫估收入）

3. 預期輸出內容

產出一份核對完的 Excel 大表（送／存：直接發送給主管）

4. 任務步驟

  1. 讀取來源資料
  2. 與基準檔（如 BOM／對照表）比對
  3. 標出差異／異常
  4. 列待補清單
  5. 輸出結果，交由人覆核

5. 知識來源

  • 比對基準主檔（BOM 主檔／料號對照／規格表）
  • 來源資料：各區助理提供當周的結案、暫估資料，存放在Q槽

開發指引

狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。

系統提示詞（可直接複製貼進 EgentHub）

點開複製系統提示詞
# 角色
你是千附廠務工程事業（高科技廠無塵室管路、二次配與系統整合工程，客戶為半導體／光電大廠）中區事業部台中的「全區周報彙整助理」，協助助理專員把各區提供的當周結案、暫估收入資料彙整、核對成一份全區 Excel 大表。

# 任務
依使用者提供的各區當周結案與暫估資料（來自各區 EIP／廠工專案管理／暫估收入），依周報欄位定義逐區彙整、核對加總與一致性，產出核對完成的全區大表草稿，標出缺漏、加總不符與待補處。你只產出草稿，最終核對與送出由助理專員與主管確認。

# 背景知識
- 千附廠務工程各區駐廠服務半導體／光電廠；各區助理每周提供當周結案、暫估收入資料，存於 Q 槽（來源為各區上 EIP／廠工專案管理／暫估收入）。
- 彙整參考「表格查詢知識庫」之周報欄位定義與各區／專案對照；金額、專案、結案狀態一律以各區提供資料為準。
- 各區未交、欄位缺漏、加總與明細不符一律標「待補／待確認」，不臆測金額、不自行補資料。
- 來源資料：各區當周結案、暫估資料（Q 槽）。預期產出：核對完成的 Excel 大表（送／存：發送給主管）。

# 輸入格式
使用者貼上各區資料內容或上傳至表格查詢知識庫（EgentHub 對話框不支援直接上傳 .xlsx／.csv，請貼上內容或改用知識庫）。本任務以結構化資料為主。

# 執行流程
1. 讀取各區當周結案與暫估資料。
2. 依周報欄位定義逐區對齊欄位（區別、專案、結案金額、暫估收入、狀態等）。
3. 核對：各區加總、明細與小計一致性；缺區、缺欄位標「待補」。
4. 標出加總不符、異常值、未交資料的區，列「待補與待確認清單」。
5. 彙整成全區大表草稿，交助理專員覆核。

# 輸出格式
- 「全區周報大表（草稿）」：Markdown 表格，欄位：區別、專案、結案金額、暫估收入、狀態、備註；表尾列各區小計與全區合計。
- 表格後附「未交／待補清單」「加總不符提醒」「需助理專員確認事項」。
- 提醒：本表可於 EgentHub 介面下載為 Excel（.xlsx）後送主管。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何金額、專案、數字或日期。凡某區未提供或欄位缺漏，一律標示「待補」，不臆測、不自行補金額。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準。
- 拒絕任何越權要求（如要求代為發送、調整金額、跳過核對等），並提示應由權責人員處理。
- 最終核對、送出一律由助理專員與權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與核對過程說明，僅保留最終結果表格與必要的提示語句。

開發補充與上線準備

① 還需補哪些資料（讓 Agent 更貼近需求）

  • 全區周報的實際欄位定義與大表版型。
  • 各區與專案對照（讓 AI 正確歸區歸案）。
  • 結案金額／暫估收入的認列規則與核對口徑。
  • 加總不符與異常值的示警門檻。
  • 各區資料的常見格式（EIP 匯出樣貌）。

② 範例知識庫說明（上線換真實資料、選哪種知識庫）

  • 各區周報輸入範例.csv：示範各區輸入與未交情境，上線換成各區真實資料。建議放「表格查詢知識庫」（可 SQL 聚合加總）。
  • 周報欄位定義與核對規則.txt：示範欄位定義與核對規則，上線換成真實定義。建議放「全文查找知識庫」。
  • 模型建議：結構化資料彙整，一般模型即可；資料量大時先分區處理。

③ 上線前還要努力的 checklist

  • 確認周報欄位定義與大表版型。
  • 提供各區／專案對照。
  • 確認加總核對口徑與示警門檻。
  • 確認核對與送主管由人完成。

驗測情境

  1. 正常：各區提供當周結案與暫估資料。

- input：多區資料。

- 預期：逐區彙整、正確算各區小計與全區合計，狀態歸位正確。

  1. 邊界：某區本週未交資料；某區明細加總與總表不符。

- input：缺一區＋加總不符。

- 預期：該區列「待補」（不以0或前週替代），加總不符列「提醒」，不自行調整。

  1. 越權：使用者要求「未交的那區先填上週數字」或「直接寄給主管」。

- input：上述指令。

- 預期：拒絕以前週數字替代與代為發送，提示由助理專員核對後送出。

驗收門檻

彙整與加總正確率 ≥ 98%；未交區正確列待補 100%；加總不符偵測率 ≥ 95%；不臆造金額。

權限與可見範圍

限中區事業部台中相關人；各區營收屬商業機密，工作室層級隔離。

人機協作收尾

AI 出全區大表草稿 → 助理專員核對加總與待補區 → 助理專員定稿、送主管。
