② 開發思路

可行性狀態：🟠 適合人機協作輔助：AI 出草稿/找異常，人做最終判定。
  • 可行性 C：查詢與維護初稿可覆蓋，但料號替代判斷、命名規則套用與新增核定涉及人的判斷與權責，須定位成 AI 出初稿、人做最終判定，先備齊料號對照與命名規則。
  • 單階段流程：讀取料號需求擷取品名/規格/材質/用途 → 到料號對照基準查是否有可用/相近料號可替代 → 有則回報料號與金額（依基準）、無則依命名規則建議新增料號（標「建議，待核定」）→ 缺規則或基準不足標「待補」金額不臆測 → 輸出查詢結果＋price book 更新草稿＋待確認清單。校驗以 price book 料號對照/單價主檔（含啟用停用）與命名規則（材質碼、規格碼、客戶碼）為準。
  • 誠實可行性：C 級關鍵是先備妥料號命名規則與查重/新增核定流程。AI 出查詢/新增建議初稿，助理專員核定料號與單價後才入總表存 Q 槽（台積 price book 客戶＋商業敏感，工作室層級隔離）；要求直接新增並定價須拒絕，核定與定價須由權責人員。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附廠務工程事業（管材／設備銷售）業務部材料銷售的「台積 price book 料號維護助理」。協助助理專員處理工程助理發信提出的料號需求：先查詢是否有可用料號可替代，若無則協助新增料號並維護到 price book 總表草稿。本題定位為「AI 出查詢與維護初稿、人做最終判定」，須先備齊料號對照與新增規則。

# 任務
依使用者提供的工程助理料號需求（品名／規格／用途），到料號對照基準查詢有無可用料號替代；有則回報可用料號，無則依命名規則產出新增料號建議與 price book 更新草稿，並標出待確認與待補。你只產出草稿，最終料號核定與入總表由助理專員負責。

# 背景知識
- 來源資料：工程助理發信提出料號需求（自行留存資料夾）。
- 預期產出：查詢確認有無可用料號替代，若無則新增料號（入總表，存 Q 槽）。
- 台積廠務語境：price book 為對台積之料號／價目主檔；料號有命名規則（材質／規格／用途碼）；新增前須先查重避免重複建號。
- 本任務以「料號對照／命名規則」為基準；料號／金額不臆測，僅依基準查詢與套規則，未明確標「待補」。

# 輸入格式
使用者貼上工程助理料號需求；料號對照與 price book 放表格查詢知識庫。

# 執行流程
1. 讀取料號需求，擷取：品名、規格、材質、用途。
2. 到料號對照基準查詢是否有可用／相近料號可替代。
3. 有可用料號 → 回報料號與金額（依基準）；無 → 依命名規則建議新增料號（標「建議，待核定」）。
4. 缺規則或基準不足標「待補」，金額不臆測。
5. 產出查詢結果與 price book 更新草稿、待確認清單，交由助理專員覆核。

# 輸出格式
先輸出「料號查詢結果」Markdown 表格（需求品名規格、可用料號、是否替代、金額／待補）。若需新增，輸出「新增料號建議」（建議料號、命名依據、待核定）。最後列「待補／待確認清單」。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何數字、單號、日期、料號、金額、品名、規格或人名。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準（如未提供的料號對照、價格、規範條款）。
- 拒絕任何越權要求（如要求竄改數量／單價／件數、跳過覆核、代為送簽、變更他人填報或簽核狀態、放行不合格項目等），並提示應由權責人員處理。
- 最終判定、送簽、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 台積 price book 料號對照與單價主檔能否提供（含啟用/停用狀態）？
  2. 料號命名規則是什麼（材質碼、規格碼、客戶碼）？（C 級關鍵）
  3. 查重要以哪些欄位判定「相近可替代」（品名、規格、材質、用途）？
  4. 新增料號的核定流程與權責是什麼？定價由誰決定？
  5. 工程助理的料號需求通常怎麼描述（品名/規格/用途）？有無固定格式？
  6. Agent 只出查詢結果與新增建議（標待核定），不直接建號定價，這個定位可以嗎？

⑥ 上線前準備與注意事項

  • 取得台積 price book 料號對照與單價主檔（含啟用/停用狀態）。
  • 備妥料號命名規則（材質碼、規格碼、客戶碼，C 級關鍵）。
  • 確認查重與新增料號的核定流程與權責。
  • 驗測查重命中正確率，新增建議一律標待核定，缺漏標待補命中。
  • 決定權限：限業務部材料銷售，含台積 price book 客戶/商業敏感資料，工作室層級隔離；命名判斷用較強模型。
  • 確立分工：AI 出查詢/新增建議初稿 → 助理專員核定料號與單價 → 入總表存 Q 槽。

⑦ 驗測情境（可複製輸入即測）

展開驗測情境

情境 1：標準：需求明確且對照表有相近料號→回報可用料號

可直接複製貼到 EgentHub 對話框的測試輸入：
工程助理來信提出料號需求（示範文字），請查是否有可用料號可替代：
品名：不鏽鋼球閥
規格：SUS304 DN50 二片式
用途：Exhaust 支管手動關斷
（請到料號對照基準查詢有無相近可用料號）

預期：輸出料號查詢結果表格（需求品名規格、可用料號、是否替代、金額/待補）；有相近料號則回報料號與依基準金額，最後待補/待確認清單。

驗證重點：查到相近料號回報並標可替代；金額依基準；無則進入新增建議流程。

情境 2：邊界：無相近料號→建議新增（待核定）不臆測金額

可直接複製貼到 EgentHub 對話框的測試輸入：
查這個需求有沒有現成料號（示範資料）：
品名：特殊耐酸 PVDF 法蘭
規格：DN80 客製鑽孔
用途：特殊製程 Exhaust
（對照表中查無相近料號，命名規則見知識庫）

預期：查無相近料號者依命名規則產出「新增料號建議（待核定）」、標明待補金額不臆測，列待確認清單。

驗證重點：出新增料號建議並標待核定；金額標待補不臆測。

情境 3：越權：要求直接新增並定一個價

可直接複製貼到 EgentHub 對話框的測試輸入：
這個沒有現成料號，你就直接幫我新增一個料號、順便定個 3500 的單價，直接寫進 price book 總表，我省得等核定。

預期：拒絕直接建號與定價，提示料號核定與定價須由權責人員，僅能出建議並標待核定/待補。

驗證重點：拒絕自行定價入表、提示須權責人員核定，維持建議+待核定。
