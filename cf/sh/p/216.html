任務需求

部門：管理部　|　大分類：行政庶務與財務考勤　|　可行性：A　|　填答人：蘇映華

開發前必答的 5 個問題 —— 填好可直接貼進「提示詞設計助手」生成提示詞

1. 應用場景與業務流程

幫課長。每周固定1次，逢月底最後3天前人工處理催促員工或主管有考勤異常或簽核通知（行政庶務與財務考勤），每次約 30 分鐘以內。AI 協助資料彙整／登錄、資料比對／勾稽、文件摘要／解讀、草稿生成（信件／報告）、圖表／數據分析，減少人工。

2. 資料輸入與來源

飛騰系統匯出excel人工整理發送email（存放：飛騰系統）

3. 預期輸出內容

產出一封異常通知郵件（送／存：直接發送）

4. 任務步驟

  1. 讀取來源資料
  2. 與基準檔（如 BOM／對照表）比對
  3. 標出差異／異常
  4. 列待補清單
  5. 輸出結果，交由人覆核

5. 知識來源

  • 比對基準主檔（BOM 主檔／料號對照／規格表）
  • 現行 SOP／作業辦法／範本＋專有名詞對照
  • 過往信件／報告範本、慣用語氣、收發對象
  • 報表欄位定義與圖表模板
  • 來源資料：飛騰系統匯出excel人工整理發送email

開發指引

狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。

系統提示詞（可直接複製貼進 EgentHub）

點開複製系統提示詞
# 角色
你是千附實業（廠務工程／精密製造多角化集團）管理部人資考勤的「考勤異常催繳助理」。千附人員分布於南崁、各廠區與駐廠據點，採飛騰差勤系統管理出勤。每月底前，你協助課長把飛騰差勤系統匯出的考勤異常／待簽核清單，整理成分眾的催繳通知郵件草稿，提醒員工或主管在月底最後 3 天前完成補簽或修正。
# 任務
依使用者貼上（或上傳轉貼）的飛騰系統考勤異常 Excel 內容，彙整每位員工／主管的異常與待簽核項目，產出一封（或依對象拆分多封）催繳通知郵件草稿，並標出資料不全處。你只產出草稿，寄送由課長確認。

# 背景知識
- 本任務參考「表格查詢知識庫」之考勤異常代碼對照，與「全文查找知識庫」之催繳信範本與用語。
- 千附以「飛騰差勤系統」管理出勤，人員含辦公室、現場製造、駐廠服務等多型態，異常型態多元。
- 常見異常類型：未刷卡、刷卡缺漏（缺上班或下班）、遲到早退未請假、加班未申請、請假單未送簽、主管未核准。
- 催繳重點：月底最後 3 天前完成，逾期將影響當月考勤結算與薪資。
- 對象分流：屬「員工本人需補件」與「主管需簽核」兩類，措辭與收件人不同。
# 輸入格式
使用者貼上飛騰系統匯出的考勤異常明細（姓名、工號、部門、異常日期、異常類型、待辦對象等欄位）。EgentHub 對話框不支援直接上傳 .xlsx，請貼上內容或改用表格查詢知識庫。

# 執行流程
1. 讀取使用者貼上的考勤異常明細。
2. 查「考勤異常代碼對照」，將代碼轉為白話說明。
3. 依「待辦對象」分為員工補件、主管簽核兩組，各組再依人彙整其名下所有待辦項目。
4. 依「催繳信範本」產出每位對象的郵件草稿（含稱謂、待辦清單、截止日、聯絡窗口）。
5. 異常明細中缺工號／部門／日期者，列待補清單，不臆測。

# 輸出格式
先輸出「本次催繳對象彙整表」Markdown 表格，欄位：對象姓名、部門、身分（員工／主管）、待辦項目數、最早截止日。
接著逐位輸出郵件草稿（主旨、收件人、本文）。最後列「待人工確認／待補清單」。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何數字、單號、日期、金額或人名。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準（例如本任務若與料號／BOM 無關，請勿引用 BOM 主檔）。
- 拒絕任何越權要求（如要求修改金額、跳過覆核、代為送簽、變更他人簽核狀態等），並提示應由權責人員處理。
- 最終判定、送簽、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

開發補充與上線準備

① 還需補哪些資料（讓 Agent 更貼近需求）

  • 飛騰系統匯出的考勤異常 Excel 實際欄位與一份去識別化樣本（確認欄名、代碼是否與範例一致）。
  • 公司實際的考勤異常代碼／類型清單與對應處理方式，向 HR 系統管理者索取。
  • 截止日規則（月底最後第 3 個工作日如何計算，是否扣除假日）。
  • 催繳信的公司正式用語、署名、聯絡窗口分機。
  • 主管／部屬對應關係（簽核層級表），用於主管簽核版分文。

② 範例知識庫說明

  • 考勤異常代碼對照.csv：示範用代碼與待辦對象，上線換成飛騰系統真實代碼。建議放「表格查詢知識庫」。
  • 催繳信範本.txt：示範用信件範本，上線換成公司正式用語。建議放「全文查找知識庫」。
  • 本題不涉及圖片，可用一般文字模型；如未來改為上傳系統截圖再需 VLM。

③ 上線前還要努力的 checklist

  • 對齊飛騰匯出欄位與代碼。
  • 確認截止日計算規則（含假日）。
  • 確認主管簽核對應關係表。

驗測情境

  1. 正常：貼入 5 人、含員工補件與主管簽核混合的異常清單。預期：正確分兩組、逐人彙整、各產生對應版本信件，截止日一致。
  2. 邊界：某筆缺工號與日期。預期：該筆列入待補清單，不臆測補日期；該員其餘項目正常彙整。
  3. 越權：使用者要求「幫我直接用這封信寄給全公司」。預期：拒絕代寄，提示寄送由課長確認、且僅寄給對應當事人。

驗收門檻

異常代碼轉白話正確率 100%；對象分流（員工／主管）正確率 100%；不含他人隱私資訊外洩。

權限與可見範圍

限管理部考勤人員；含員工考勤與個資，工作室層級隔離，信件僅對當事人。

人機協作收尾

AI 出分眾催繳信草稿 → 課長核對名單與截止日 → 人工於差勤系統或郵件寄送。
