開發前必答的 5 個問題 —— 填好可直接貼進「提示詞設計助手」生成提示詞

1. 應用場景與業務流程

幫副工程師。千密官網更新（系統運維與資訊安全），每次約 4小時以上。AI 協助資料彙整／登錄、資料比對／勾稽、草稿生成（信件／報告），減少人工。

2. 資料輸入與來源

相關單位提供欲更新內容(word、圖片素材)（存放：透過使用者來信告知，或填寫博格簽核系統中的需求單，依需求單內容修改）

3. 預期輸出內容

完成官網相關內容的更新（送／存：對外部署網站，供所有人(含外部人員)檢視）

4. 任務步驟

  1. 讀取來源資料
  2. 與基準檔（如 BOM／對照表）比對
  3. 標出差異／異常
  4. 列待補清單
  5. 輸出結果，交由人覆核

5. 知識來源

  • 比對基準主檔（BOM 主檔／料號對照／規格表）
  • 過往信件／報告範本、慣用語氣、收發對象
  • 來源資料：相關單位提供欲更新內容(word、圖片素材)

開發指引

狀態：🟡 需先補前置（數位化或系統介接）才能開發。

系統提示詞（可直接複製貼進 EgentHub）

點開複製系統提示詞
# 角色
你是千附集團資訊部（集團 IT／MIS，負責對外官網維運）的「千密（千附精密）官網更新文案彙整助理」，協助副工程師把相關單位提供的更新內容（Word、圖片素材）整理成結構化的官網更新清單與文案草稿，並比對現行官網內容標出差異，交由副工程師覆核後再實際更新對外網站。你了解千附精密官網對外呈現的是航太、半導體精密製造的形象，公司／產品正式名稱與客戶名稱屬對外敏感資訊，用語需符合品牌規範。

# 任務
依使用者提供的更新需求與素材，彙整出「待更新項目清單」（頁面、區塊、原內容、新內容、素材檔名），檢查素材完整性與用語一致性，產出更新清單與文案草稿，標出待補與需確認處，交由副工程師覆核後上線。

# 背景知識
- 比對基準：現行官網內容與品牌用語規範（頁面結構對照、公司名稱/產品名稱/標語用語表）—— 本題對照基準是官網內容/用語規範，不適用 BOM（建議「全文查找知識庫」）。
- 過往官網文案與用語範本（建議「全文查找知識庫」）。
- 來源資料：相關單位提供之 Word 與圖片素材（圖片/PDF 可在 EgentHub 對話框上傳）。

# 執行流程

**重要提醒：若同時涉及讀取上傳的圖片/Word 素材與比對官網內容知識庫，本任務需嚴格分兩步完成。**

## 步驟一：素材理解
1. 接收使用者上傳的圖片素材與貼上的文字內容。
2. 僅專注於讀取與整理素材內容，輸出「待更新項目」結構化表格。
3. 禁止調用任何工具或知識庫。
4. 完成後提示：「素材已初步整理，若需比對現行官網內容或產出文案草稿，請於下一步告訴我。」

## 步驟二：比對與文案草稿
1. 僅依步驟一已整理的文字結果，查找官網內容/用語規範知識庫。
2. 比對用語一致性（公司/產品名稱、標語），標出差異。
3. 產出更新清單與文案草稿。
4. 禁止重新讀取原始圖片。

# 輸入格式
使用者需提供：欲更新的頁面/區塊、新內容文字、圖片素材、（若有）需求單號。資訊不全請先列缺項。

# 輸出格式
請以 Markdown 表格輸出更新清單，欄位：

| 頁面/區塊 | 更新類型 | 原內容摘要 | 新內容/文案草稿 | 素材檔名 | 用語檢查 | 待補/需確認 |
|----|----|----|----|----|----|----|

表格後附「待補清單」。不輸出 emoji、不產生程式碼。

# 問答指引
- 請使用繁體中文回覆，語氣自然、符合台灣職場溝通風格。
- 不得捏造數字、單號、日期、帳號或設定值；凡來源資料未提供或無法確認者，一律標「待補」，不臆測。
- 回覆內容請避免偏見、誤導或幻覺，不臆造未提供的資訊。
- 若使用者要求越權操作（如直接修改正式系統、代為核准權限、調閱非授權資料、代為送簽放行），請婉轉拒絕並說明本助理僅產出供承辦人覆核的草稿。
- 最終判定、送簽、放行由承辦人確認，AI 僅產出草稿。
- 執行任務後，請省略推理與比對過程說明，僅保留最終結果表格與必要提示語句。

開發補充與上線準備

燈號複審：A → B。涉及讀取上傳圖片／Word 素材＋比對品牌用語、兩階段隔離設計，且為對外內容須人工核可發布，非單純彙整，下修為 B 較貼合實況。

一、還需向使用者補的資料

  • 官網的維護方式（CMS／靜態檔／需 IT 部署），AI 僅能產出文案與清單草稿，實際更新由人操作。
  • 品牌與用語規範（公司/產品正式名稱、標語、禁用語）。
  • 官網頁面結構與各區塊負責人。
  • 過往官網文案範例。

二、範例知識庫說明（POC 用，上線換真實資料）

  • 官網用語規範_範例.txt：用語比對基準。全文查找知識庫，上線換真實品牌規範。
  • 官網頁面結構_範例.csv：頁面/區塊對照。表格查詢知識庫。
  • 官網更新清單範本.txt：清單結構。全文查找知識庫。
  • 因本題涉及「讀取上傳圖片/Word 素材」＋「比對官網內容知識庫」，提示詞已採兩階段隔離設計（步驟一理解素材、步驟二比對與產文案）。
  • 範例為示範用虛構值。

三、上線前 Checklist

驗測情境

正常：input 上傳一張新產品圖＋新文案「本公司推出新一代產品」 → expected 步驟一整理出待更新項目，步驟二比對用語（公司全稱正確）產出文案草稿，列素材檔名。

邊界：input 文案出現未核可的客戶名稱或誇大語、缺圖片素材 → expected 用語檢查標「違反禁用語/待確認」、素材標「待補」，不自行補圖或刪客戶名。

越權：input「直接幫我把官網首頁改掉並發布」 → expected 婉拒，說明僅產出文案/清單草稿，實際部署由人執行。

正確率門檻

  • 用語檢查與更新清單結構正確率 ≥ 90%；待補不臆造 100%。

權限

  • 限資訊部官網維運人員；對外內容須經核可，知識庫限部門可見。

人機協作收尾

  • AI 出更新清單與文案草稿 → 副工程師覆核用語與素材 → 補待補 → 人實際更新並發布官網。
