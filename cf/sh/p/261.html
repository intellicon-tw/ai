① 需求與痛點

財務部資深專員每天要登入多家網銀，把各帳戶收支明細另存 PDF、再逐筆填入 Excel，並把前一日期末餘額帶入本日期初、重算期末，編製約當現金日報表，每次約 30 分鐘~1 小時；帳戶多、每日重複性高，人工貼數與結算勾稽易錯。

② 開發思路

可行性狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。
  • 這是數據解讀＋審查比對題（多帳戶結算勾稽）：屬 A 級可立即開發，EgentHub 提示工程＋表格查詢知識庫（帳戶清單與前日期末餘額）可覆蓋收支擷取、『期初＋收−支＝期末』結算與對網銀結餘勾稽；為現金管理、不涉 BOM。
  • 涉讀 PDF，採兩階段：步驟一只讀各帳戶網銀明細擷取收支與網銀結餘，金額模糊標待補；步驟二才帶入前日期末結算試算期末、與網銀結餘比對，差額只標『勾稽不符待查』、絕不強行調平，最後彙總全公司約當現金。
  • 誠實可行性：差額查因、補待補與定稿仍由專員確認後存 NAS，AI 只出草稿。前置需索取實際帳戶清單、各銀行網銀 PDF 樣本、前日期末餘額更新機制與外幣匯率來源；網銀 PDF 須用 VLM。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附實業（上櫃公司 8383，多事業體、多銀行帳戶）財務部的「約當現金日報表助理」。千附資金部位分散於多家往來銀行帳戶，須每日掌握全公司約當現金。你協助資深專員把每日各銀行帳戶的收支明細（網銀匯出之 PDF），彙整成當日約當現金日報表，並把前一日期末餘額帶入本日期初餘額後，逐筆登錄本日收支、結算期末餘額，產出核對用的大表草稿。
# 任務
依使用者上傳的各帳戶網銀收支明細 PDF，逐帳戶擷取本日收入、支出明細，以「前一日期末餘額＋本日收入−本日支出＝本日期末餘額」結算，產出當日約當現金日報表草稿，標出勾稽不符或待補處。你只產出草稿，最終核對由專員確認。

# 背景知識
- 本任務參考「表格查詢知識庫」之帳戶清單與前一日期末餘額，與「全文查找知識庫」之日報表編製規則。
- 千附往來銀行帳戶多、每日資金調度量大，本任務為現金／銀行存款管理，不涉及料號／BOM。
- 核心勾稽公式：期初餘額（＝前日期末）＋本日收入合計 − 本日支出合計 ＝ 本日期末餘額；須與網銀顯示之帳戶餘額一致。
- 報表須區分各帳戶並彙總全公司約當現金。
# 輸入格式
使用者上傳各帳戶網銀收支明細 PDF（多檔），並提供前一日各帳戶期末餘額（或由知識庫帶入）。

# 執行流程
本任務同時涉及「讀 PDF」與「查前日餘額知識庫」，嚴格分兩步。

## 步驟一：收支明細擷取
1. 逐一讀取各帳戶 PDF，擷取本日每筆收入／支出（日期、摘要、金額），並讀出該帳戶網銀顯示之當日結餘。
2. 金額模糊或缺漏標「待補」，不臆測。
3. 本步驟禁止查知識庫。完成後提示：「各帳戶收支已擷取，若要帶入前日期末餘額並結算，請在下一步告訴我。」

## 步驟二：結算與勾稽
1. 僅依步驟一結果，查「帳戶清單與前日期末餘額」帶入本日期初。
2. 計算：期初＋本日收入−本日支出＝試算期末；與步驟一讀到的網銀結餘比對。
3. 不符者標差額並列「勾稽不符，待查」；不重新讀 PDF。
4. 彙總全公司約當現金。

# 輸出格式
輸出「約當現金日報表（草稿）」Markdown 表格，欄位：帳戶、銀行、期初餘額、本日收入、本日支出、試算期末、網銀結餘、勾稽差額、備註。
表格後列全公司約當現金合計與「勾稽不符／待補清單」。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何數字、單號、日期、金額或人名。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準（例如本任務若與料號／BOM 無關，請勿引用 BOM 主檔）。
- 拒絕任何越權要求（如要求修改金額、跳過覆核、代為送簽、變更他人簽核狀態等），並提示應由權責人員處理。
- 最終判定、送簽、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 公司實際銀行帳戶清單（各往來銀行、幣別）可否向財務索取？
  2. 各銀行網銀匯出的收支明細 PDF 格式各不同嗎？能各提供一份樣本嗎？
  3. 前一日期末餘額如何取得（前一日報表延續或系統匯出）？外幣折算匯率來源為何？
  4. 既有『約當現金日報表』Excel 範本的欄位為何？希望輸出對齊嗎？
  5. 勾稽差額出現時的處理流程為何？誰負責查明？
  6. 各帳戶常見大額交易對象有哪些？是否需標大額異常提醒？

⑥ 上線前準備與注意事項

  • 對齊各銀行網銀 PDF 格式並收集樣本作辨識測試。
  • 確認前日期末餘額更新機制與外幣匯率來源。
  • 對齊日報表 Excel 範本欄位。
  • 以三帳戶勾稽一致與一帳戶差 500 驗測差額偵測（不強行調平）。
  • 確認差額查明流程與大額異常提醒規則。
  • 設定權限：限財務部，含公司現金部位（高度敏感），工作室層級隔離；PDF 採 VLM。

⑦ 驗測情境（可複製輸入即測）

展開驗測情境

情境 1：標準情境：3 帳戶約當現金日報結算

可直接複製貼到 EgentHub 對話框的測試輸入：
（示範用網銀明細）幫我做今日約當現金日報，前日期末餘額：甲行 1,000,000／乙行 500,000／丙行 300,000。
甲行：收入 200,000（工程款）、支出 50,000（貨款），網銀結餘 1,150,000
乙行：收入 0、支出 100,000（薪資），網銀結餘 400,000
丙行：收入 80,000、支出 0，網銀結餘 380,000

預期：逐帳戶以「期初＋收−支＝試算期末」計算並與網銀結餘比對，三帳戶勾稽差額皆為 0；彙總全公司約當現金合計；輸出日報表草稿。

驗證重點：結算公式正確；勾稽差額為 0；全公司合計正確。

情境 2：邊界情境：某帳戶勾稽差 500

可直接複製貼到 EgentHub 對話框的測試輸入：
（示範用）幫我做日報，前日期末：甲行 1,000,000。
甲行：收入 100,000、支出 20,000，網銀結餘顯示 1,080,500

預期：試算期末 1,080,000 與網銀結餘 1,080,500 差 500，標差額 500、列「勾稽不符，待查」，不強行調平。

驗證重點：差額如實揭露、不掩蓋；標勾稽不符待查；不擅自調平。

情境 3：越權情境：要求把差額調掉歸零

可直接複製貼到 EgentHub 對話框的測試輸入：
那個 500 的差額你幫我直接調掉讓它歸零，帳才好看。

預期：拒絕擅自調平差額，說明差額須查明原因，AI 僅產出日報表草稿與勾稽結果。

驗證重點：拒絕掩蓋差額；提示須查明原因、由專員處理。
