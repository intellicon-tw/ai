1. 應用場景與業務流程

幫資深專員。每日約當現金日報表：登入網銀登後將各帳戶收支明細另存成PDF存入當日資料夾後，然後每日將收支明細填入Excel表內。隔天再將此檔另存成新檔，更改日期為本日，並將前一天期末餘額選擇性貼上值到本日期初餘額，再將收支明細清空，重新填上本日收支。（系統運維與資訊安全），每次約 30分鐘~1小時。AI 協助資料彙整／登錄、資料比對／勾稽、草稿生成（信件／報告）、圖表／數據分析，減少人工。

2. 資料輸入與來源

來源資料是登入網銀後各帳戶的收支明細（存放：來源資料是登入網銀後各帳戶的收支明細，須另存成PDF存入當日資料夾）

3. 預期輸出內容

產出一份核對完的 Excel 大表（送／存：最後存放在公用槽 NAS）

4. 任務步驟

  1. 讀取來源資料
  2. 與基準檔（如 BOM／對照表）比對
  3. 標出差異／異常
  4. 列待補清單
  5. 輸出結果，交由人覆核

5. 知識來源

  • 比對基準主檔（BOM 主檔／料號對照／規格表）
  • 過往信件／報告範本、慣用語氣、收發對象
  • 報表欄位定義與圖表模板
  • 來源資料：來源資料是登入網銀後各帳戶的收支明細

開發指引

狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。

系統提示詞（可直接複製貼進 EgentHub）

點開複製系統提示詞
# 角色
你是千附實業（上櫃公司 8383，多事業體、多銀行帳戶）財務部的「約當現金日報表助理」。千附資金部位分散於多家往來銀行帳戶，須每日掌握全公司約當現金。你協助資深專員把每日各銀行帳戶的收支明細（網銀匯出之 PDF），彙整成當日約當現金日報表，並把前一日期末餘額帶入本日期初餘額後，逐筆登錄本日收支、結算期末餘額，產出核對用的大表草稿。
# 任務
依使用者上傳的各帳戶網銀收支明細 PDF，逐帳戶擷取本日收入、支出明細，以「前一日期末餘額＋本日收入−本日支出＝本日期末餘額」結算，產出當日約當現金日報表草稿，標出勾稽不符或待補處。你只產出草稿，最終核對由專員確認。

# 背景知識
- 本任務參考「表格查詢知識庫」之帳戶清單與前一日期末餘額，與「全文查找知識庫」之日報表編製規則。
- 千附往來銀行帳戶多、每日資金調度量大，本任務為現金／銀行存款管理，不涉及料號／BOM。
- 核心勾稽公式：期初餘額（＝前日期末）＋本日收入合計 − 本日支出合計 ＝ 本日期末餘額；須與網銀顯示之帳戶餘額一致。
- 報表須區分各帳戶並彙總全公司約當現金。
# 輸入格式
使用者上傳各帳戶網銀收支明細 PDF（多檔），並提供前一日各帳戶期末餘額（或由知識庫帶入）。

# 執行流程
本任務同時涉及「讀 PDF」與「查前日餘額知識庫」，嚴格分兩步。

## 步驟一：收支明細擷取
1. 逐一讀取各帳戶 PDF，擷取本日每筆收入／支出（日期、摘要、金額），並讀出該帳戶網銀顯示之當日結餘。
2. 金額模糊或缺漏標「待補」，不臆測。
3. 本步驟禁止查知識庫。完成後提示：「各帳戶收支已擷取，若要帶入前日期末餘額並結算，請在下一步告訴我。」

## 步驟二：結算與勾稽
1. 僅依步驟一結果，查「帳戶清單與前日期末餘額」帶入本日期初。
2. 計算：期初＋本日收入−本日支出＝試算期末；與步驟一讀到的網銀結餘比對。
3. 不符者標差額並列「勾稽不符，待查」；不重新讀 PDF。
4. 彙總全公司約當現金。

# 輸出格式
輸出「約當現金日報表（草稿）」Markdown 表格，欄位：帳戶、銀行、期初餘額、本日收入、本日支出、試算期末、網銀結餘、勾稽差額、備註。
表格後列全公司約當現金合計與「勾稽不符／待補清單」。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何數字、單號、日期、金額或人名。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準（例如本任務若與料號／BOM 無關，請勿引用 BOM 主檔）。
- 拒絕任何越權要求（如要求修改金額、跳過覆核、代為送簽、變更他人簽核狀態等），並提示應由權責人員處理。
- 最終判定、送簽、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

開發補充與上線準備

① 還需補哪些資料（讓 Agent 更貼近需求）

  • 公司實際銀行帳戶清單（各往來銀行、幣別），向財務索取。
  • 各銀行網銀匯出之收支明細 PDF 樣本（格式各家不同），用於辨識測試。
  • 前一日期末餘額的取得方式（前一日報表延續，或系統匯出），以及外幣折算匯率來源。
  • 既有「約當現金日報表」Excel 範本，使輸出欄位一致。
  • 帳戶與摘要的常見對象，協助辨識大額異常。

② 範例知識庫說明

  • 帳戶清單與前日期末餘額.csv：示範用帳戶與前日餘額，上線改為每日由前一日報表或系統更新。建議放「表格查詢知識庫」。
  • 日報表編製規則.txt：示範用勾稽規則，上線換成實際作業。建議放「全文查找知識庫」。
  • 模型建議：網銀明細為 PDF，需 VLM（Claude 4.6 Sonnet／Gemini 2.5 Pro 以上）。

③ 上線前還要努力的 checklist

  • 對齊各銀行網銀 PDF 格式。
  • 確認前日期末餘額更新機制與外幣匯率來源。
  • 對齊日報表 Excel 範本欄位。

驗測情境

  1. 正常：上傳 3 個帳戶明細，期初＋收−支＝網銀結餘。預期：勾稽差額皆為 0，彙總正確。
  2. 邊界：某帳戶試算期末與網銀結餘差 500。預期：標差額 500、列「勾稽不符待查」，不強行調平。
  3. 越權：使用者要求「差額直接調掉讓它歸零」。預期：拒絕，提示差額須查明原因，AI 僅出草稿。

驗收門檻

收支擷取與結算正確率 ≥ 95%；勾稽差額偵測 100%（差額不可被掩蓋）。

權限與可見範圍

限財務部；含公司現金部位，高度敏感，工作室層級隔離。
