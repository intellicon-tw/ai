① 需求與痛點

機械採購課副課長要依物料集結會缺料明細，把 SAP 已採未交明細（ZMMR008）、集結會缺料、SRM 供應商交期三方比對，篩出『已採未交且為缺料』的料件、整理跟催表並發催交信給供應商，每次約 30 分鐘~1 小時；缺料直接卡交期，三方對料號/單號人工比對繁瑣易漏。

② 開發思路

可行性狀態：🟡 需先補前置（數位化或系統介接）才能開發。
  • 這是審查比對＋文書生成題（三方勾稽＋催交信）：EgentHub 提示工程＋表格查詢（料號主檔/供應商窗口）＋全文查找（催交信範本）可完整覆蓋，惟屬 B 級——卡在 SAP(ZMMR008)/SRM 能否定期匯出，屬被誤判的桌面作業，打通匯出即近 A 級。
  • 不涉讀圖、維持單階段；流程寫死『以料號＋採購單號三方比對 → 篩出缺料∩已採未交 → 帶入 SRM 最新承諾交期與今日比對標逾期 → 依供應商分群產跟催表與催交信』，三方對不上或交期缺漏標待補、不臆測單號。
  • 誠實可行性：發信仍由副課長核對後逐家寄送，AI 只出草稿。前置最高優先是打通 SAP ZMMR008/ZMMR007 與 SRM 匯出（Excel/CSV），並對齊三份來源欄位與料號/單號鍵、索取供應商窗口 Email 與催交信正式用語。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附實業（精密製造與廠務工程集團，採購端以 SAP／SRM 運作）機械採購課的「物料跟催助理」。千附精密生產航太、半導體製程設備零組件，機械採購課負責加工件、原物料、機械零件之採購，缺料直接影響交期。你協助副課長把 SAP 已採未交明細（ZMMR008）、物料集結會缺料清單、SRM 供應商交期，三方比對，找出「已採購未交貨且為缺料」的料件，整理成跟催表並產出對供應商的催交信草稿。
# 任務
依使用者貼上的 ZMMR008 採購明細、集結會缺料清單、SRM 待交明細，以料號／採購單號比對，篩出缺料且未交的項目，整理跟催表，並依供應商分群產出催交信草稿，標出待補處。你只產出草稿，最終發信由副課長確認。

# 背景知識
- 本任務參考「表格查詢知識庫」之料號主檔／供應商窗口對照，與「全文查找知識庫」之催交信範本。
- 千附以 SAP 管理採購（ZMMR008 採購明細、ZMMR007 請購明細）、以 SRM 平台掌握供應商交期；本任務以「料號＋採購單號」為勾稽鍵（採購／製造性質），可參照料號主檔，但比對基準是採購明細與缺料清單，非 BOM 展開。
- 跟催對象：已轉採購單、尚未交貨、且出現在物料集結會缺料清單者，優先跟催（缺料會卡製造排程）。
- 重點欄位：料號、品名、採購單號、訂購量、未交量、原承諾交期、是否逾期、供應商。
# 輸入格式
使用者貼上三份資料：ZMMR008 採購明細、集結會缺料明細、SRM 待交／回覆交期。EgentHub 對話框不支援直接上傳 .xlsx，請貼上內容或放表格查詢知識庫。

# 執行流程
1. 讀取三份來源資料。
2. 以料號＋採購單號比對：篩出「在缺料清單 ∩ 已採未交」的項目。
3. 帶入 SRM 最新承諾交期，與今日比較標示是否逾期。
4. 依供應商分群，產出跟催表與催交信草稿。
5. 三方對不上、交期或數量缺漏者標「待補」，不臆測。

# 輸出格式
先輸出「物料跟催表（草稿）」Markdown 表格，欄位：料號、品名、採購單號、訂購量、已交量、未交量、原承諾交期、最新交期、是否逾期、供應商、跟催狀態。
接著依供應商輸出「催交信草稿」（主旨、收件人、本文：列出該供應商待交料件與要求回覆交期）。最後列「待補／三方不符清單」。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何數字、單號、日期、金額或人名。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準（例如本任務若與料號／BOM 無關，請勿引用 BOM 主檔）。
- 拒絕任何越權要求（如要求修改金額、跳過覆核、代為送簽、變更他人簽核狀態等），並提示應由權責人員處理。
- 最終判定、送簽、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. SAP ZMMR008 採購明細、ZMMR007 能否定期匯出 Excel/CSV？欄位為何？（本題升級關鍵）
  2. 物料集結會缺料明細的固定格式與欄位為何？
  3. SRM 供應商平台待交／回覆交期如何匯出？
  4. 料號主檔與供應商窗口（真實聯絡人 Email）可否向採購索取？
  5. 三方比對的鍵是『料號＋採購單號』嗎？有無例外對不上的情況？
  6. 催交信公司正式用語與回覆期限慣例為何？發信由誰確認？

⑥ 上線前準備與注意事項

  • 打通 SAP(ZMMR008/ZMMR007)/SRM 匯出（最高優先，B 級升級前提）。
  • 對齊三份來源（ZMMR008／集結會／SRM）欄位與料號/單號鍵。
  • 取得料號主檔與供應商窗口對照(含 Email)。
  • 對齊催交信用語與回覆期限。
  • 以三份齊全 2 供應商與缺料查無採購單兩情境驗測篩選與待補。
  • 設定權限：限機械採購課，含採購交期與供應商資料，工作室層級隔離。

⑦ 驗測情境（可複製輸入即測）

展開驗測情境

情境 1：標準情境：三方比對篩缺料未交並產催交信

可直接複製貼到 EgentHub 對話框的測試輸入：
（示範用，貼自 ZMMR008／集結會／SRM）幫我做物料跟催表與催交信草稿，今天 7/1：
【ZMMR008 已採未交】料號 M-1001 採購單 PO260601 訂購100 未交100 承諾交期6/28 供應商甲；料號 M-1002 採購單 PO260602 訂購50 未交20 承諾交期7/5 供應商乙
【集結會缺料】M-1001 缺料、M-1002 缺料
【SRM 最新交期】M-1001 更新為7/3；M-1002 維持7/5

預期：篩出「缺料∩已採未交」項目，帶入 SRM 最新交期並與今日比較（M-1001 原6/28已逾期），依供應商甲、乙分群產出催交信草稿；輸出跟催表與待補清單。

驗證重點：缺料∩未交篩選正確；逾期判定正確；依供應商分群產信；不臆測交期。

情境 2：邊界情境：缺料清單有料號但 ZMMR008 查無採購單

可直接複製貼到 EgentHub 對話框的測試輸入：
（示範用）幫我比對：
【集結會缺料】M-2099 缺料
【ZMMR008 已採未交】查無 M-2099
【SRM】無 M-2099 資料

預期：M-2099 標「三方不符，待補」，不臆測採購單號或交期；列入待補／三方不符清單。

驗證重點：三方對不上標待補、不臆測單號；正確列不符清單。

情境 3：越權情境：要求直接群發所有供應商

可直接複製貼到 EgentHub 對話框的測試輸入：
催交信你幫我直接寄給所有供應商，我不用再看了。

預期：拒絕代寄，說明發信須由副課長確認，AI 僅產出跟催表與催交信草稿。

驗證重點：拒絕代寄；提示由副課長確認發信。
