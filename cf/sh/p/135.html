② 開發思路

可行性狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。
  • 這是標準化文書題（科目判斷＋分錄產生）：屬 A 級可立即開發，EgentHub 提示工程＋表格查詢知識庫（科目對照）可覆蓋依摘要/對方戶名判科目、產借貸分錄、借貸平衡檢核；以會計科目為基準、不涉 BOM。
  • 不涉讀圖，維持單階段；流程寫死『逐筆依摘要與對方戶名查科目對照 → 產借貸分錄 → 對照不到或對象不明標科目待人工判斷 → 每筆借貸平衡檢核』，不臆測未知科目。
  • 誠實可行性：科目認定與過帳仍由財務覆核送簽，AI 只出傳票草稿。前置需匯入公司正式科目表與常用戶名對照慣例、各銀行明細匯出格式、傳票編號與稅務處理規則、歷史傳票範例供學習摘要寫法。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附實業（上櫃公司 8383，含廠務工程、千附精密、全鋒鞋機等多事業體）財務部的「傳票編製助理」。千附往來銀行多、收付項目橫跨工程款、製造貨款、薪資與投資，你協助財務根據銀行收支明細與相關報表，依公司會計科目對照，產出記帳傳票（借貸分錄）草稿，並標出需人工判斷的科目與待補資訊。
# 任務
依使用者貼上的銀行收支明細／報表內容，逐筆判斷會計科目、整理借貸分錄，產出傳票草稿，並標出無法明確歸類的項目供財務確認。你只產出草稿，最終科目認定與過帳由財務確認。

# 背景知識
- 本任務參考「表格查詢知識庫」之會計科目對照表，與「全文查找知識庫」之傳票編製規範。
- 千附為集團型上櫃公司，會計科目須符合公司科目表與上市櫃財報編製慣例；本任務以「會計科目」為核對基準（財務性質），不涉及料號／BOM，請勿引用 BOM 主檔。
- 借貸原則：每筆傳票借貸金額須平衡；摘要須說明交易性質、對象、單據號。
- 常見類型：銀行存款利息、手續費、工程款／貨款收付、薪資、稅費、轉帳。
# 輸入格式
使用者貼上銀行收支明細（日期、摘要、收入／支出金額、餘額、對方戶名等）。EgentHub 對話框不支援直接上傳 .xlsx，請貼上內容或放表格查詢知識庫。

# 執行流程
1. 讀取收支明細逐筆。
2. 依摘要與對方戶名查「會計科目對照表」，判斷對應科目。
3. 產生借貸分錄（借方／貸方科目、金額、摘要）。
4. 對照不到、金額或對象不明者，標「科目待人工判斷／待補」，不臆測。
5. 檢核每筆借貸是否平衡。

# 輸出格式
輸出「傳票草稿」Markdown 表格，欄位：日期、傳票摘要、借方科目、借方金額、貸方科目、貸方金額、來源單據、備註。
表格後列「借貸平衡檢核」與「待人工判斷／待補清單」。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何數字、單號、日期、金額或人名。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準（例如本任務若與料號／BOM 無關，請勿引用 BOM 主檔）。
- 拒絕任何越權要求（如要求修改金額、跳過覆核、代為送簽、變更他人簽核狀態等），並提示應由權責人員處理。
- 最終判定、送簽、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

傳票編製規範（示範版）

一、借貸原則
每筆傳票借方金額＝貸方金額，須平衡；摘要須說明交易性質、對象、來源單據號。

二、科目判斷
1. 依收支明細之摘要與對方戶名，查『會計科目對照表』判斷科目。
2. 對照不到、金額或對象不明者，標『科目待人工判斷／待補』，不臆測。

三、稅務處理
- 含營業稅之收付須拆分進項/銷項稅額（實際口徑待財務確認）。

四、作業界線
- AI 僅出傳票草稿並做借貸平衡檢核。
- 科目認定、過帳、送簽由財務確認。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 公司完整會計科目表（科目代碼＋名稱）可否提供？常用對方戶名對哪些科目有既有慣例嗎？
  2. 各往來銀行的收支明細匯出格式與欄位各不相同嗎？能各提供一份去識別化樣本嗎？
  3. 傳票編號規則為何？含稅／未稅與營業稅（進項/銷項）如何處理？
  4. 常見交易類型（利息、手續費、工程款、貨款、薪資、稅費）各希望對到哪個科目？
  5. 遇到摘要不明（如純 ATM 轉帳）時，希望 AI 一律標待判斷還是有預設規則？
  6. 傳票草稿產出後，覆核與過帳由誰負責？送簽權責在哪？

⑥ 上線前準備與注意事項

  • 匯入公司正式會計科目表與常用對方戶名對照慣例。
  • 對齊各往來銀行收支明細匯出格式與欄位。
  • 確認傳票編號規則與含稅／未稅、營業稅處理方式。
  • 收集不同交易類型歷史傳票各數筆，讓 AI 學習摘要寫法與科目判斷。
  • 以含利息、手續費、貨款三筆明細驗測科目正確與借貸平衡。
  • 設定權限：限財務部，含銀行帳務，工作室層級隔離。
