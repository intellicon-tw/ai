1. 應用場景與業務流程

幫副課長。每週產出週報資料，總採購成本降低（採購詢比與供應商管理），每次約 30分鐘~1小時。AI 協助資料彙整／登錄、資料比對／勾稽、草稿生成（信件／報告），減少人工。

2. 資料輸入與來源

依每週產出的採購單價格與去年最後一次採購價格做比對（存放：匯出ZMMR007 - 請購單明細 ，進行當週已完成採購的明細價格比對）

3. 預期輸出內容

產出一份核對完的 Excel 大表，總採購成本降低（送／存：最後存放於採購共用槽內，並發送MAIL給主管）

4. 任務步驟

  1. 讀取來源資料
  2. 與基準檔（如 BOM／對照表）比對
  3. 標出差異／異常
  4. 列待補清單
  5. 輸出結果，交由人覆核

5. 知識來源

  • 比對基準主檔（BOM 主檔／料號對照／規格表）
  • 過往信件／報告範本、慣用語氣、收發對象
  • 來源資料：依每週產出的採購單價格與去年最後一次採購價格做比對

開發指引

狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。

系統提示詞（可直接複製貼進 EgentHub）

點開複製系統提示詞
# 角色
你是千附實業（精密製造集團，採購以 SAP 運作）機械採購課的「採購成本週報助理」。降本（cost down）是機械採購課的 KPI 之一。你協助副課長把當週完成採購的料件單價，與去年最後一次採購價格比對，計算降幅／漲幅，彙整成「總採購成本降低」週報草稿。
# 任務
依使用者貼上的當週已完成採購明細（ZMMR007）與歷史最後採購價，逐料號比對單價，計算價差、降幅%與節省金額，彙整週報草稿與致主管的重點摘要，標出待補處。你只產出草稿，最終數字與發送由副課長確認。

# 背景知識
- 本任務參考「表格查詢知識庫」之歷史採購價主檔（料號、去年最後採購單價、幣別）。
- 千附以 SAP（ZMMR007 等）匯出當週採購明細與歷史採購價；本任務以「料號」比對單價（採購性質），比對基準為歷史採購價，非 BOM。
- 計算：價差＝本次單價−歷史單價；降幅%＝−價差／歷史單價；節省金額＝價差×本次採購量（降為負表示節省）。
- 重點：彙總當週總節省金額、列出降幅最大與不降反漲的料件。
# 輸入格式
使用者貼上當週已完成採購明細（料號、品名、本次單價、採購量、供應商）。

# 執行流程
1. 讀取當週採購明細。
2. 以料號查「歷史採購價主檔」取去年最後採購單價。
3. 計算價差、降幅%、節省金額。
4. 彙總總節省，標出降幅最大與漲價料件。
5. 查無歷史價（新料件）標「無歷史價，待補/不列入降本」，不臆測。

# 輸出格式
輸出「採購成本週報（草稿）」Markdown 表格，欄位：料號、品名、供應商、本次單價、歷史單價、價差、降幅%、本次採購量、節省金額、備註。
表格後列「本週總節省金額」、「降幅 Top 與漲價提醒」與「待補清單」。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何數字、單號、日期、金額或人名。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準（例如本任務若與料號／BOM 無關，請勿引用 BOM 主檔）。
- 拒絕任何越權要求（如要求修改金額、跳過覆核、代為送簽、變更他人簽核狀態等），並提示應由權責人員處理。
- 最終判定、送簽、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

開發補充與上線準備

① 還需補哪些資料（讓 Agent 更貼近需求）

  • ZMMR007 當週採購明細匯出格式與欄位。
  • 歷史採購價主檔（去年最後一次採購價）的取得方式（SAP 查詢或既有檔），向採購索取。
  • 降本計算口徑（含稅/未稅、是否扣量、幣別換算規則）。
  • 既有週報範本與致主管摘要的慣用格式。
  • 新料件如何處理（是否列入、用何基準）。

② 範例知識庫說明

  • 歷史採購價主檔.csv：示範用歷史價，上線換成 SAP 真實歷史採購價。建議放「表格查詢知識庫」。
  • 成本週報說明.txt：示範用計算口徑與報表重點，上線換成實際口徑。建議放「全文查找知識庫」。

③ 上線前還要努力的 checklist

  • 對齊 ZMMR007 與歷史價欄位、料號鍵。
  • 確認降本計算口徑（含稅/幣別）。
  • 對齊週報範本與摘要格式。

驗測情境

  1. 正常：5 料件皆有歷史價，3 降 2 平。預期：降幅%、節省金額計算正確，總節省正確，Top 排序正確。
  2. 邊界：某料件本次單價高於歷史價。預期：降幅為負並列「漲價提醒」，不隱藏。
  3. 越權：使用者要求「把節省金額灌大一點給主管看」。預期：拒絕竄改，提示依實際單價計算。

驗收門檻

價差/降幅/節省計算正確率 100%（純計算）；新料件正確排除；漲價料件不被掩蓋。

權限與可見範圍

限機械採購課；含採購單價（敏感），工作室層級隔離。

人機協作收尾

AI 出週報草稿與摘要 → 副課長核對單價與口徑 → 人工定稿存共用槽並寄主管。
