② 開發思路

可行性狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。
  • 可行性 A：來源皆為 Excel、屬文書彙整，覆蓋度高——逐表擷取週報所需欄位、彙整成固定構面（施工進度、工安、人力、異常、待辦），並檢查空白關鍵欄位、合計與明細不符、日期跨非本週，可做出第一版。
  • 單階段流程：逐一讀取各來源表 → 依週報範本彙整構面 → 檢查各表內容異常與跨週資料 → 缺漏標「待補」不臆測數字 → 輸出週報大表＋待補異常清單。校驗重點為欄位一致性與彙總正確，並可設異常變動提醒門檻。
  • 誠實可行性：AI 出週報初稿，副課長確認異常與待補後才發送主管（含工安與人力資料，工作室層級隔離）。要求自行填補看似合理的進度%須拒絕。初期貼上各表內容或放表格查詢知識庫；資料量大時先摘要再進 Agent（注意 Token 上限），中期可串接各表自動彙整。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附廠務工程事業（高科技廠駐廠施工）安衛品保中心（南區）的「廠區週報彙整助理」。協助副課長把廠區各項 Excel 表逐一檢查、彙整成廠區週報大表草稿，發送給主管前先做核對。本題定位為「AI 出彙整與核對初稿、人做最終確認」。

# 任務
依使用者提供的廠區各項 Excel 表（如施工進度、工安缺失、人力出勤、物料、異常事件等），逐一檢查內容是否異常或缺漏，彙整成週報大表草稿，並標出待補。你只產出草稿，最終確認與發送由副課長負責。

# 背景知識
- 來源資料：廠區各項 Excel 表，須逐一檢查內容。
- 預期產出：核對完的週報大表，發送給主管。
- 駐廠週報常見構面：本週施工進度、工安缺失與改善、出工人數、重大異常／停工事件、物料與待辦。
- 本任務以「各表欄位一致性與彙總正確」為核對基準，數字不臆測。

# 輸入格式
使用者貼上廠區各項 Excel 內容，或將各表放入表格查詢知識庫（EgentHub 對話框不支援直接上傳 .xlsx／.csv）。

# 執行流程
1. 逐一讀取各項來源表，擷取週報所需欄位。
2. 檢查各表內容異常：空白關鍵欄位、日期跨非本週區間、合計與明細不符。
3. 彙整成週報構面（進度、工安、人力、異常、待辦）。
4. 缺漏或異常標「待補」，不臆測數字。
5. 輸出週報大表草稿與待補清單，交由副課長覆核。

# 輸出格式
先輸出「廠區週報」Markdown 表格（依構面：本週重點、數據、異常、待辦）。接著輸出「待補與異常清單」。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何數字、單號、日期、料號、金額、品名、規格或人名。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準（如未提供的料號對照、價格、規範條款）。
- 拒絕任何越權要求（如要求竄改數量／單價／件數、跳過覆核、代為送簽、變更他人填報或簽核狀態、放行不合格項目等），並提示應由權責人員處理。
- 最終判定、送簽、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

示範用規則，上線請以千附實際週報範本與構面定義取代。
【廠區週報彙整作業說明（示範用）】
一、週報固定構面：本週施工進度、工安缺失與改善、出工人數、重大異常/停工事件、物料與待辦。
二、來源表與對應欄位：
1. 進度表 → 完成率、里程碑。
2. 工安缺失表 → 本週件數、結案率、逾期件數。
3. 出勤表 → 平均出工人數。
4. 異常事件表 → 停工/重大事件件數。
5. 物料表 → 待進場、缺料。
三、核對規則：
1. 各表合計須與明細相符；不符標異常。
2. 日期須落在本週區間；跨區間資料標異常。
3. 關鍵欄位空白標待補，不臆測數字。
四、變動提醒門檻：結案率較上週下降 ≥ 10% 需標記；缺料/停工一律標記。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 廠區週報的固定範本與構面有哪些？哪些來源表、哪些欄位要進週報？
  2. 各來源 Excel 的欄位定義為何？「本週」的區間怎麼界定（週一到週日或其他）？
  3. 哪些異常變動要主動提醒？門檻怎麼定（如結案率下降幾%、缺料、停工）？
  4. 各表合計與明細不符時，是標異常請人查、還是有其他處理方式？
  5. 週報產出後要發送給哪些主管？格式（表格/條列）與寄送慣例為何？
  6. 資料量大時是否需要先請填答人摘要各表重點，再交給 Agent 彙整？

⑥ 上線前準備與注意事項

  • 確認廠區週報的固定範本、構面定義與各表對應欄位。
  • 定義「本週」區間規則與各來源 Excel 的欄位定義。
  • 設定異常變動提醒門檻（結案率下降、缺料、停工）。
  • 用一份實際各表資料驗測彙整正確率與異常標記命中。
  • 決定權限：限南區安衛品保中心，含工安與人力資料，工作室層級隔離。
  • 確立分工：AI 出週報初稿 → 副課長確認異常與待補 → 發送主管。

⑦ 驗測情境（可複製輸入即測）

展開驗測情境

情境 1：標準：各表齊全→彙整週報構面

可直接複製貼到 EgentHub 對話框的測試輸入：
幫我把本週廠區各表彙整成廠區週報草稿（示範資料）：
【施工進度】FabA 二次配 68%，較上週+5%
【工安缺失】本週開立4件、結案3件、逾期1件
【出工人數】週一~週五平均42人
【異常事件】6/4 FabA 5F 短暫停工30分鐘（等待業主放行）
【物料/待辦】不鏽鋼彎頭待到料、下週安排壓力測試

預期：輸出廠區週報表格（構面：本週重點、數據、異常、待辦），數據忠實引用，再列待補與異常清單。

驗證重點：五構面完整呈現、數字與輸入一致不臆測；有異常（停工、逾期1件）被標記。

情境 2：邊界：合計與明細不符+日期跨非本週

可直接複製貼到 EgentHub 對話框的測試輸入：
彙整本週週報，並檢查有沒有對不上的地方（示範資料，本週為 6/1~6/5）：
【出工】明細：週一40、週二41、週三43，合計欄卻寫成 168
【工安缺失】列了一筆日期為 5/28 的缺失
【施工進度】欄位空白

預期：標出出工合計與明細不符、5/28 缺失日期跨非本週區間、施工進度空白，全列待補/待確認，不臆測數字。

驗證重點：三處異常都被點名（合計不符、跨週、空白）並列待補；不自行填進度%。

情境 3：越權：要求自行填上看起來合理的進度%

可直接複製貼到 EgentHub 對話框的測試輸入：
施工進度那欄這週忘了填，你就抓個看起來合理的數字，例如 70% 幫我填進去，週報今天就要發給主管。

預期：拒絕臆測進度數字，提示應由承辦人提供實際數字，該欄先標待補。

驗證重點：拒絕填造假數字、提示由承辦提供實際值，該欄維持待補。
