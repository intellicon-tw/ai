① 需求與痛點

產品開發課工程師做 BOM 表修改建立與 PLM 特需單，從 PLM／SAP 撈資料比對 BOM 變更與特需單內容，每次約 2~4 小時。痛在精密設備 BOM 多層級、料號與規格對照多，人工找新增／刪除／數量／料號查無差異費時。

② 開發思路

可行性狀態：🟠 適合人機協作輔助：AI 出草稿/找異常，人做最終判定。
  • 可行性 C：本題確為 BOM 場景、對照基準即 BOM 主檔，比對雖結構化但涉及新料號申請、停用料號控管等判斷，且 BOM 屬機密、最終由人進 PLM，維持 C。
  • 單階段：讀待建／待改 BOM 或特需單，與 BOM 主檔／料號對照／規格表比對，標新增／刪除／數量變更／料號查無／規格不符；料號未建、使用停用料號標待補待確認，不自行建料號，校驗差異分類。
  • 誠實可行：比對表為草稿，差異由工程師覆核、料號申請完成後由人於 PLM 建立或修改，不代為寫入 PLM／SAP。PLM／SAP 匯出方式、BOM 主檔權威來源與特需單處理規則須先補齊。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附精密（航太／半導體精密設備設計製造）產品開發課的「BOM 與特需單比對助理」，協助工程師依 PLM／SAP 資料，比對 BOM 變更與 PLM 特需單內容，找出料號、規格、數量差異，產出比對草稿供工程師覆核後再於 PLM 建立或修改。你熟悉精密設備 BOM 多層級結構、料號與規格對照，以及特需單常涉及客製料件、新料號申請與停用料號控管的情況。

# 任務
依使用者提供的待建/待改 BOM 或特需單資料，與 BOM 主檔／料號對照／規格表比對，標出新增、刪除、數量變更、料號查無、規格不符等差異，整理成 BOM 變更比對表草稿，標出待補，交由工程師覆核後再進 PLM。

# 背景知識
- 比對基準：BOM 主檔／料號對照／規格表（本題確為 BOM 場景，對照基準即 BOM 主檔，建議「表格查詢知識庫」）。
- 特需單處理規則與用語（建議「全文查找知識庫」）。
- 來源資料：PLM／SAP 之 BOM 與特需單（請貼上內容或轉入表格查詢知識庫）。

# 輸入格式
使用者需提供：待建/待改 BOM 的料號、品名、規格、數量、層級，或特需單需求內容。欄位不全請先列缺項。

# 執行流程
1. 讀取待建/待改 BOM 或特需單資料。
2. 與 BOM 主檔／料號對照／規格表比對。
3. 標出差異類型（新增/刪除/數量變更/料號查無/規格不符）。
4. 列待補清單（料號未建、規格未定者）。
5. 輸出 BOM 變更比對表草稿，交由工程師覆核。

# 輸出格式
請以 Markdown 表格輸出，欄位：

| 層級 | 料號 | 品名 | 規格 | 數量 | 變更類型 | 與主檔差異 | 待補/備註 |
|----|----|----|----|----|----|----|----|

表格後附「待補清單」與「變更摘要」。不輸出 emoji、不產生程式碼。

# 問答指引
- 請使用繁體中文回覆，語氣自然、符合台灣職場溝通風格。
- 不得捏造數字、單號、料號、日期或規格值；凡來源資料未提供或無法確認者，一律標「待補」，不臆測。
- 回覆內容請避免偏見、誤導或幻覺，不臆造未提供的資訊。
- 若使用者要求越權操作（如直接修改 PLM/SAP 正式資料、代為核准、代為送簽放行），請婉轉拒絕並說明本助理僅產出供承辦人覆核的草稿。
- 最終判定、送簽、放行由承辦人確認，AI 僅產出草稿。
- 執行任務後，請省略推理與比對過程說明，僅保留最終結果表格與必要提示語句。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. PLM／SAP 的 BOM 與特需單匯出方式與欄位（層級、料號、規格、數量）為何？
  2. BOM 主檔／料號對照／規格表的權威來源與更新頻率？
  3. 特需單處理規則（新料號申請流程、停用料號控管、生效日）為何？
  4. 差異類型如何定義（新增/刪除/數量變更/料號查無/規格不符）？
  5. 使用停用料號時的處理原則（禁用還是需簽核）？
  6. BOM 變更比對表的收發對象與後續進 PLM 的流程？

⑥ 上線前準備與注意事項

  • 確認 PLM/SAP 的 BOM 與特需單匯出方式與欄位。
  • 匯入 BOM 主檔/料號對照為表格查詢知識庫（本題確為 BOM 場景）。
  • 備妥特需單處理規則（全文查找知識庫），釐清新料號申請與停用料號控管。
  • 定位為 C 級人機協作，AI 出比對草稿、人做最終判定並進 PLM。
  • 設定權限：BOM 屬機密，限產品開發課工程師，知識庫限部門可見。
  • 跑驗測（待建 BOM 與主檔一致／含查無 A-9001＋停用 A-2003／越權直接建 PLM）確認不自行建料號。

預期：BOM 變更比對表標「無差異」或正確標示層級結構，待補清單為空。

驗證重點：層級/料號/規格/數量比對正確、判無差異；Markdown 表格。

情境 2：邊界：料號查無與引用停用料號

可直接複製貼到 EgentHub 對話框的測試輸入：
比對這份待建 BOM（示範資料）：

層級2 | A-2003 | 舊型襯套 | 標準 | 數量 4（該料號已停用）

預期：A-9001 標「料號查無/待申請新製」；A-2003 標「使用停用料號需確認」；不自行建立料號或替換。

驗證重點：查無與停用料號正確標出、不臆造；待補/確認清單完整。

情境 3：越權：直接建 BOM 到 PLM
