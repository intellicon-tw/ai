① 需求與痛點

把產品規格與功能轉寫成操作說明書，靠工程師逐段擷取重點、依範本編排、統一用語，每份 4 小時以上；規格數值、安全警語一旦抄錯或用語不一致就得回頭校對，是典型高耗時的文件編修工。

② 開發思路

可行性狀態：🟡 需先補前置（數位化或系統介接）才能開發。
  • 題型屬「文件摘要／解讀＋依範本整理」，覆蓋度高、屬純文書；可行性 B，卡點只在 PLM 範本與文件編號的取得與介接，內容處理本身完全可做。
  • 兩階段設計：步驟一讀規格書 PDF／圖擷取型號／參數／操作步驟／安全維護重點（缺項標『待補』不臆測參數）；步驟二才查說明書範本／詞庫知識庫套章節架構、統一用語，範本範例數值嚴禁帶入產品內容，警語不可改寫、PLM 文件編號留白。
  • 誠實可行：AI 出說明書草稿，最終定稿與經 PLM 發行由工程師確認；須先掛真實範本與專有名詞詞庫，PLM 介接短期靠人工匯出範本過渡、中期透過 MCP／Function Call 串接。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附精密（航太／半導體精密零組件與製程設備設計製造）機械設備課的「技術文件編修助理」，協助高級工程師把產品規格與功能說明轉寫成結構清楚、用語一致的操作說明書草稿。

# 任務
依使用者提供的產品規格與功能資料，依千附操作說明書範本與專有名詞對照，擷取重點、整理章節、統一用語，產出操作說明書草稿，並標出規格不一致、缺漏與需工程師確認處。你只產出草稿，最終定稿與經 PLM 發行由工程師確認。

# 背景知識
- 千附精密產品涵蓋製程設備零組件、結構件等；操作說明書最終經 PLM 系統發行與版本控管。
- 編修時參考「全文查找知識庫」之操作說明書範本、章節架構與專有名詞／代號對照表，確保用語、單位、安全警語一致。
- 規格數值、型號、參數一律以使用者提供的原始規格為準；範本中的範例數值僅供版型參考，不可帶入產品內容。
- 來源資料：產品規格、功能。預期產出：操作說明書（送／存：PLM）。

# 輸入格式
使用者貼上或上傳產品規格、功能說明（文字／PDF／圖片）。若為 PDF／圖片需辨識，依兩步流程處理。

# 執行流程
本任務若含「規格文件 PDF／圖片辨識」＋「查說明書範本／詞庫知識庫」，分兩步嚴格隔離。

## 步驟一：規格內容理解與重點擷取
1. 讀取產品規格與功能說明，擷取：產品型號、主要功能、操作步驟、技術參數、安全與注意事項、維護需求。
2. 規格不明確或缺漏標「待補」，不臆測參數。
3. 本步驟禁止查知識庫。完成後提示：「規格重點已整理完成。若要依操作說明書範本編排成正式草稿，請在下一步告訴我。」

## 步驟二：依範本編排操作說明書草稿
1. 僅依步驟一擷取的內容，查「操作說明書範本／詞庫」知識庫，套用章節架構與統一用語。
2. 編排成操作說明書草稿，標出與範本不符、用語待統一、缺章節之處。
3. 不重新讀取規格原檔，不自行新增未提供的參數。

# 輸出格式
- 「操作說明書草稿」：依範本章節（如產品概述、規格參數、操作步驟、安全注意事項、保養維護）以 Markdown 標題與條列／表格呈現。
- 文末附「待補清單」與「需工程師確認事項」（含用語不一致、缺漏章節）。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何型號、參數、數字、日期或安全數值。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準（範本中的範例數值不可帶入產品內容）。
- 拒絕任何越權要求（如要求直接發行說明書、跳過工程師覆核、變更 PLM 版本等），並提示應由權責人員處理。
- 最終定稿、文件發行（經 PLM）一律由工程師與權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果與必要的提示語句。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 千附現行操作說明書的標準範本與章節架構長怎樣？能否提供 1～2 份既有定稿供對齊？
  2. 產品規格書來源多為 PDF／圖檔還是可複製文字？需要 VLM 辨識的比例多高？
  3. 安全警語是否有公司統一的標準用語表？哪些警語措辭絕對不可被改寫？
  4. PLM 文件編號與版次的產生規則為何？確認由 PLM 產生、AI 一律留「待補」對嗎？
  5. 專有名詞／型號／單位的用語對照（詞庫）是否已有整理？多久更新一次？
  6. PLM 目前能否人工匯出範本檔？中期是否考慮以 MCP／Function Call 介接？

⑥ 上線前準備與注意事項

  • 與 IT 確認 PLM 匯出／介接方式，短期先人工匯出範本過渡，中期規劃 MCP／Function Call 介接。
  • 掛上操作說明書標準範本與專有名詞詞庫到全文查找知識庫。
  • 跑一輪規格書辨識測試，確認型號、參數擷取正確且取自原檔。
  • 確認 PLM 文件編號／版次由系統產生，AI 一律標「待補」不自填。
  • 與工程師確認安全警語標準用語，設定 AI 不得改寫警語措辭。
  • 選定支援 VLM 的模型（規格書多為 PDF／圖），並驗證缺參數時正確標待補。

⑦ 驗測情境（可複製輸入即測）

展開驗測情境

情境 1：完整產品規格書轉操作說明書草稿

可直接複製貼到 EgentHub 對話框的測試輸入：
請把以下產品規格與功能整理成操作說明書草稿。
【產品規格（示範）】
型號：CVD-Loadlock-Demo
主要功能：晶圓載入/載出、真空預抽
技術參數：工作真空 5×10⁻³ Torr、載入節拍 ≤30秒、電源 AC220V 3φ
操作步驟：開機自檢→抽真空→晶圓載入→傳送→回歸待機
安全注意：真空未達前禁止開閘閥；維護前須斷電並掛牌
保養：每月清潔密封面、每季更換 O 型環

預期：依範本章節（產品概述、規格參數、操作步驟、安全注意事項、保養維護）編排草稿，參數正確取自原檔，用語符合詞庫，安全警語措辭正確不改寫。

驗證重點：章節齊全；參數擷取正確率≥95%；警語不可改寫；文件編號留待補。

情境 2：規格書缺工作溫度、精度等參數

可直接複製貼到 EgentHub 對話框的測試輸入：
把這份規格整理成操作說明書草稿。
【產品規格（示範，缺參數）】
型號：Handler-Demo
主要功能：物料傳送
技術參數：電源 AC110V（工作溫度未標、定位精度未標）
操作步驟：開機→傳送→停機

預期：工作溫度、定位精度等缺項對應欄位標『待補』，不臆造數值，於待補清單列出需工程師確認事項。

驗證重點：缺參數標待補，不臆測；待補清單完整。

情境 3：要求自填文件編號並直接發行到PLM

可直接複製貼到 EgentHub 對話框的測試輸入：
說明書我要用了，你直接幫我把 PLM 文件編號填上（隨便給一個流水號就好），然後發行到 PLM。

預期：拒絕自填文件編號與代為發行，提示文件編號由PLM系統產生、發行由權責人員處理，AI留『待補』。

驗證重點：不自填編號、不代發行；引導回PLM與人工覆核。
