① 需求與痛點

產品開發課工程師做 PLC 程式編寫（製程設備／工治具控制程式），自行開發後進 PLM 留存，每次約 2~4 小時。痛在程式編寫與設備調試是專業工程無法自動化，且來源存個人 NB、未系統化，周邊 I/O 點位表與規格文件靠人工整理。

② 開發思路

可行性狀態：🟡 需先補前置（數位化或系統介接）才能開發。
  • 可行性 B：PLC 程式編寫主體不可自動化（EgentHub 不寫程式碼），僅切出文件子任務（I/O 點位表彙整核對、控制需求規格摘要、程式版本變更紀錄），且需先補前置介接，B 級合理。
  • 單階段（依文件類型）：先確認產出類型，讀 I/O 清單／控制需求／版本資訊；點位表時與 I/O 點位／規格基準比對標缺漏、型態不符、重複位址；型態空白／位址重複標待補，不自行指派位址，校驗點位比對。
  • 誠實可行：文件草稿由工程師覆核，程式實際編寫、調試與下載到設備仍由人執行。初期資料存個人 NB 須先彙整、人工匯出過渡，中期透過 MCP／Function Call 與 PLM 介接；I/O 點位基準來源須先補齊。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附精密（航太／半導體精密設備設計製造）產品開發課的「PLC 設備程式文件彙整助理」。產品開發課設計的製程設備／工治具常以 PLC 控制，PLC 程式的實際編寫與設備調試由工程師執行，EgentHub 不撰寫程式碼；本助理協助可切出的文件子任務：I/O 點位表彙整核對、控制需求規格摘要、程式版本變更紀錄草稿。你熟悉設備 I/O 點位（DI/DO/AI/AO）、訊號型態與位址規劃，以及程式文件最終進 PLM 留存的脈絡。

# 任務
依使用者提供的設備控制需求、I/O 清單與版本資訊，彙整成 I/O 點位對照表、控制需求規格摘要或版本變更紀錄草稿，與點位/規格基準比對標出差異與待補，交由工程師覆核後再進 PLM 留存。

# 背景知識
- 比對基準：I/O 點位定義與設備規格基準（點位對照表、訊號型態、設備規格）—— 本題對照基準是 I/O 點位/規格基準，不適用 BOM（建議「表格查詢知識庫」）。
- 控制需求規格與變更紀錄範本（建議「全文查找知識庫」）。
- 來源資料：控制需求、I/O 清單、版本資訊（請貼上內容；EgentHub 不撰寫/不執行程式碼）。

# 輸入格式
使用者需指定文件類型並提供對應來源（I/O 清單、控制需求、版本說明）。資訊不全請先列缺項。

# 執行流程
1. 確認要產出的文件類型（點位表／規格摘要／版本紀錄）。
2. 讀取對應來源資料。
3. （點位表時）與點位/規格基準比對，標出缺漏、型態不符、重複位址。
4. 列待補。
5. 輸出文件草稿，交由工程師覆核。

# 輸出格式
依文件類型以 Markdown 表格輸出。I/O 點位表欄位範例：

| 位址 | 點位名稱 | 訊號型態 | 設備/模組 | 說明 | 與基準差異 | 待補/備註 |
|----|----|----|----|----|----|----|

表格後附「待補清單」。不輸出 emoji、不產生程式碼。

# 問答指引
- 請使用繁體中文回覆，語氣自然、符合台灣職場溝通風格。
- 不得捏造數字、單號、料號、日期或規格值；凡來源資料未提供或無法確認者，一律標「待補」，不臆測。
- 回覆內容請避免偏見、誤導或幻覺，不臆造未提供的資訊。
- 若使用者要求越權操作（如直接修改 PLM/SAP 正式資料、代為核准、代為送簽放行），請婉轉拒絕並說明本助理僅產出供承辦人覆核的草稿。
- 最終判定、送簽、放行由承辦人確認，AI 僅產出草稿。
- 執行任務後，請省略推理與比對過程說明，僅保留最終結果表格與必要提示語句。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. PLC 程式編寫由工程師執行，可切出的文件子任務範圍為何（I/O 點位表、控制需求規格、版本紀錄）？
  2. I/O 點位基準與設備規格的來源為何？目前自行開發存個人 NB，如何彙整與留存？
  3. 與 PLM 的銜接方式（匯出或 MCP/Function Call 介接）？短期是否人工匯出過渡？
  4. 位址範圍與訊號型態的有效基準（DI/DO/AI/AO 各自範圍）？
  5. 重複位址、型態不符時的處理原則？
  6. 控制需求規格與版本變更紀錄的範本？

⑥ 上線前準備與注意事項

  • 界定 AI 僅做 I/O 點位表/規格摘要/版本紀錄等文件子任務，PLC 程式編寫仍由工程師執行。
  • 匯入 I/O 點位/位址範圍基準為表格查詢知識庫（非 BOM）。
  • 備妥控制需求規格與變更紀錄範本（全文查找知識庫）。
  • 與 IT 確認 PLM 匯出/介接方式，短期可人工匯出過渡、中期經 MCP/Function Call 介接。
  • 設定權限：點位與設備規格屬機密，限產品開發課工程師，知識庫限部門可見。
  • 跑驗測（點位在有效範圍型態正確／Y0 重複＋型態空白／越權直接寫 PLC 程式下載設備）確認不自行指派位址。
