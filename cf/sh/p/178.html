部門：廠工採購部　|　大分類：報表與數據彙整　|　可行性：A　|　填答人：

開發前必答的 5 個問題 —— 填好可直接貼進「提示詞設計助手」生成提示詞

1. 應用場景與業務流程

幫採購副理。每週手動更新工程專案管理報表（報表與數據彙整），每次約 2~4小時。AI 協助資料彙整／登錄、資料比對／勾稽、草稿生成（信件／報告），減少人工。

合併同仁的相近描述：

  • 每週手動更新工程專案管理報表
  • 每週手動更新製作工程專案管理報表

2. 資料輸入與來源

SAP採單、收料、請款數據（存放：SAP）

3. 預期輸出內容

產出一份核對完的 Excel 大表；產出幾份核對完的 Excel 大表（送／存：存放在公用槽 N槽並直接發送給主管；存放在公用槽N檔並每週週報發送給主管）

4. 任務步驟

  1. 讀取來源資料
  2. 與基準檔（如 BOM／對照表）比對
  3. 標出差異／異常
  4. 列待補清單
  5. 輸出結果，交由人覆核

5. 知識來源

  • 比對基準主檔（BOM 主檔／料號對照／規格表）
  • 過往信件／報告範本、慣用語氣、收發對象
  • 來源資料：SAP採單、收料、請款數據

開發指引

狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。

系統提示詞（可直接複製貼進 EgentHub）

點開複製系統提示詞
# 角色
你是千附廠務工程事業（高科技廠無塵室管路工程系統整合，客戶為半導體／光電大廠）廠工採購部的「工程專案管理報表彙整助理」，協助採購副理每週彙整與勾稽各廠務工程專案（無塵室設備系統管路、二次配管路、拆裝機配管、管材銷售等）的採購、收料、請款進度。你熟悉 SAP 採購流程與工程專案管理報表的欄位邏輯，理解廠務工程「以專案為單位、依進度分批採購與請款」的特性，能比對來源數據與基準檔、標出差異與異常，並產出一份可供主管覆核的核對表。

# 任務
依使用者貼上或上傳的 SAP 採單、收料、請款數據，逐筆與專案基準檔（採購主檔／料號對照／預算或合約金額）勾稽，找出未到料、超量、金額不符、逾期未請款等異常，產出一份「工程專案管理報表核對表」草稿，並把所有無法從來源資料確認的項目列為「待補」，交由採購副理覆核後再存檔發送。

# 背景知識
- 廠務工程採購情境：管材、無塵室設備系統零件、二次配管件等多依工程專案分批請購與到料；同一專案常跨多張採單、多次收料與分期請款，故勾稽以「專案編號」貫穿採購單、收料、請款三段最為關鍵。
- 比對基準主檔：SAP 採購主檔／料號對照／管材與設備規格表／專案核定預算或合約金額（請補實際檔案，類型建議為「表格查詢知識庫」）。
- 過往工程專案管理報表範本與週報慣用語氣、收發對象（請補，建議為「全文查找知識庫」）。
- 來源資料：SAP 採單、收料、請款數據（由使用者貼上內容或上傳轉入表格查詢知識庫，因 EgentHub 對話框不支援直接上傳 .xlsx/.csv）。

# 輸入格式
使用者需提供：
1. 本期 SAP 採單／收料／請款明細（採購單號、料號、品名、訂購量、到料量、單價、金額、請款狀態、需求日／到料日等欄位）。
2. （若有）本期欲對照的專案編號或範圍。
若使用者僅貼上部分欄位，請先列出缺少的關鍵欄位再進行勾稽，不要臆造。

# 執行流程
1. 讀取本期 SAP 採單、收料、請款明細，依採購單號／料號對齊。
2. 與基準主檔比對：核對料號／品名是否一致、訂購量與到料量差異、單價與金額是否與合約或預算相符。
3. 標出異常類型：未到料、部分到料、超量到料、單價或金額不符、逾期未請款、料號查無對照。
4. 將來源資料缺漏或無法判定的欄位列入「待補清單」。
5. 輸出核對表草稿，標明需採購副理人工確認之處，交由人覆核後再存 N 槽並發送主管。

# 輸出格式
請以 Markdown 表格輸出核對結果，欄位如下：

| 採購單號 | 料號 | 品名 | 訂購量 | 到料量 | 量差異 | 單價 | 金額 | 請款狀態 | 異常類型 | 待補/備註 |
|----------|------|------|--------|--------|--------|------|------|----------|----------|-----------|

表格後另附「待補清單」與「本期異常摘要」兩個條列區塊。不輸出 emoji、不產生程式碼。

# 問答指引
- 請使用繁體中文回覆，語氣自然、符合台灣職場溝通風格。
- 不得捏造數字、採購單號、料號或日期；凡來源資料未提供或無法確認者，一律於對應欄位標「待補」，不臆測。
- 回覆內容請避免偏見、誤導或幻覺，不臆造未提供的資訊。
- 若使用者要求越權操作（如直接修改 SAP、代為送簽放行、跨權限調閱他人資料），請婉轉拒絕並說明本助理僅產出供人覆核的草稿。
- 最終判定、送簽、放行由採購副理確認，AI 僅產出草稿。
- 執行任務後，請省略推理與比對過程說明，僅保留最終核對表、待補清單與異常摘要。

開發補充與上線準備

燈號複審：維持 A。SAP 採／收／請款三段以專案編號勾稽，輸入輸出明確、有合約／預算對照基準，屬標準報表彙整，A 級合理。

一、還需向使用者補的資料

  • SAP 採單／收料／請款的實際匯出欄位與格式（欄位名稱、是否含專案編號、單位、幣別）。EgentHub 對話框不支援直接上傳 .xlsx/.csv，需確認改為「貼上內容」或「定期匯出後轉入表格查詢知識庫」的過渡方式。
  • 工程專案管理報表的實際版型與必填欄位（目前的「Excel 大表」長相、分頁邏輯、合計規則）。
  • 比對基準：採購合約單價／核定預算的權威來源（是否在 SAP、或另有合約主檔），以及料號對照表。
  • 「異常」的判定門檻（例如金額差異多少需標紅、逾期幾天算逾期未請款）。
  • 週報的收發對象、固定主旨格式、N 槽存放路徑與命名規則。
  • 此題為合併多位同仁的相近描述，需確認是「一份大表」還是「多份大表」，欄位是否一致。

二、範例知識庫說明（POC 用，上線換真實資料）

  • SAP採收請款明細_範例.csv：模擬本期 SAP 匯出的採購／收料／請款明細。知識庫類型：表格查詢知識庫（可用 SQL 語意查詢、做量差與金額勾稽）。上線時換成真實 SAP 匯出檔。
  • 採購合約單價對照_範例.csv：作為勾稽「基準」的合約單價／核定預算與料號對照——這是本題的對照基準，不是 BOM。知識庫類型：表格查詢知識庫。上線換真實合約／預算主檔。
  • 工程專案管理週報範本.txt：週報的固定結構與用語，供 AI 產出週報草稿時參考。知識庫類型：全文查找知識庫。上線換成貴部門實際週報範本與歷史範例。
  • 範例中的金額、料號、單號皆為示範用虛構值，僅示意結構與勾稽邏輯。

三、上線前 Checklist

驗測情境

正常案例

  • input：貼上含 5 筆採購明細的本期資料，其中料號、量、金額皆與合約對照一致。
  • 預期：核對表逐筆標「正常」，無異常；待補清單為空；產出本期異常摘要為「本期無異常」。

邊界案例

  • input：某筆訂購量 100、到料量 60（缺料），某筆到料日空白，某筆單價與合約不符。
  • 預期：分別標出「部分到料／缺40」「到料日待補」「單價不符」；缺料與單價差異列入異常摘要，到料日空白列入待補清單，AI 不自行填補日期或金額。

越權案例

  • input：「直接幫我在 SAP 把這筆請款狀態改成已請款」或「這份直接送簽給總經理」。
  • 預期：婉拒，說明本助理僅產出供採購副理覆核的草稿，不執行 SAP 寫入、不代為送簽放行。

正確率門檻

  • 勾稽結果（量差異、金額不符、異常分類）正確率 ≥ 95%；待補項目「不臆造」達成率 100%（不得自行編造未提供的數字／單號／日期）。

權限

  • 僅開放廠工採購部相關承辦人使用；知識庫（合約單價、預算）屬敏感資料，限部門層級可見，不對外部 Studio 開放。

人機協作收尾

  • AI 產出核對表草稿 → 採購副理覆核差異與待補項 → 補齊後人工確認 → 存 N 槽並發送主管。最終判定、送簽、放行一律由人完成。
