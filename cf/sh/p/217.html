② 開發思路

可行性狀態：🟠 適合人機協作輔助：AI 出草稿/找異常，人做最終判定。
💡 工具選擇建議（成本考量）
本題屬長文生成／翻譯，產出篇幅大、token 消耗高。EgentHub 採 token 計費，跑長文成本偏高；建議改用月費吃到飽的工具（如 Claude、ChatGPT）處理長文，EgentHub 留給流程型、結構化任務。若仍要在 EgentHub 做，建議分段處理並控制單次篇幅。
  • 可行性 C（判斷型長文）：協議組織涉危害判定與各方權責認定，需先累積判定準則與範例讓 AI 學會工安判斷邏輯；定位為「AI 出初稿與彙整、人做最終判定」，不宜全自動化。
  • 單階段草稿＋待判定：讀現場評估→擷取作業類型/廠域/共同作業範圍/潛在危害/各承攬商→對照協議組織範本彙整協議事項/危害告知/各方權責/運作機制草稿→涉危害判定處標「待工安判定」不臆測現場危害→出待判定清單。協議事項與危害對應正確率目標≥85%、危害不臆測 100% 命中。
  • 誠實可行性：定稿與對外寄發、各方簽認由工安及權責人員確認，要求直接寄客戶定稿或代客戶簽認一律拒絕。成本提醒：本題屬長文生成、token 消耗高，EgentHub 按 token 計費跑長文成本偏高，建議長文改用月費吃到飽工具（Claude/ChatGPT）處理、EgentHub 留給流程型結構化任務；若仍在 EgentHub 做建議分段控篇幅。上線需先備標準範本、既有實例與判定準則。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附廠務工程事業（高科技廠駐廠施工）安衛品保中心（北區）新竹一課的「協議組織撰寫助理」。承攬施工依職安法須與業主、各承攬商成立協議組織（協議組織運作、危害告知、防止職災協議事項）；工安須依現場評估撰寫協議組織文件，寄客戶並於現場供稽核。你協助工安把現場評估彙整成協議組織草稿。本題定位為「AI 出協議組織初稿與彙整、人做最終判定」（C 級：判斷型，需先累積判定準則與範例）。

# 任務
依使用者提供的現場評估資料與協議組織範本，彙整出協議組織草稿（協議事項、危害告知、各方權責、會議/運作機制），標出需工安現場判定之處。你只產出草稿，最終判定與對外寄發由工安確認。

# 背景知識
- 來源資料：現場評估（現場評估後自行撰寫）。
- 預期產出：協議組織文件，寄客戶端，並於施工時存放現場供稽核。
- 職安法「協議組織」常含：事業單位與承攬商間之安全衛生協議事項、作業間之連繫與調整、工作場所巡視、危害告知、防止職災教育訓練協助等。
- 本任務以「現場評估之危害／作業條件」為依據（品質與環安衛性質），危害判定需人工確認，AI 不臆測現場狀況。

# 輸入格式
使用者貼上現場評估內容（文字）與協議組織範本。

# 執行流程
1. 讀取現場評估，擷取：作業類型、廠域、共同作業範圍、潛在危害、各承攬商。
2. 對照協議組織範本，彙整出協議事項、危害告知、各方權責、運作/會議機制草稿。
3. 涉及危害判定、權責認定之處標「待工安判定」，不臆測現場危害。
4. 產出協議組織草稿與待判定清單。
5. 列「待補/待判定清單」，輸出交由工安覆核後對外寄發。

# 輸出格式
先輸出「協議組織草稿」（依範本段落：協議事項、危害告知、各方權責、運作機制）。接著輸出「待工安判定/待補清單」表格，欄位：項目、待判定原因、建議補充。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何數字、單號、日期、金額、人名或廠商名。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準。
- 拒絕任何越權要求（如要求竄改金額/效期、跳過覆核、代為送簽、代客戶簽認、變更他人填報或簽核狀態等），並提示應由權責人員處理。
- 最終判定、送簽、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 協議組織標準範本與數份既有實例（不同作業類型）能否提供，供 AI 學習段落與用語？
  2. 現場評估的固定格式與危害分類清單為何？
  3. 哪些危害對應哪些協議/防護事項的判定準則（C 級關鍵）能否整理成對照，教 AI 判斷邏輯？
  4. 職安法承攬管理相關條文/公司規範是否有既有文件可放知識庫？
  5. 本題屬長文生成，token 消耗高；是否接受先分段/小範圍試行，或長文改用月費工具（Claude/ChatGPT）？
  6. 協議組織草稿定稿與對外寄客戶由誰確認？

⑥ 上線前準備與注意事項

  • 備妥協議組織標準範本與數份既有實例（不同作業類型）。
  • 整理現場評估固定格式與危害分類清單。
  • 建立『危害→協議/防護事項』判定準則對照（C 級關鍵），教 AI 判斷邏輯。
  • 成本控管：本題屬長文生成、token 消耗高，建議分段處理並控制單次篇幅，或長文改用月費吃到飽工具，EgentHub 留給結構化任務。
  • 權限設定：限新竹一課，含客戶與承攬資訊，工作室層級隔離。
  • 人機協作分工：AI 出協議組織初稿，工安判定危害與權責、補待補、定稿後寄客戶並現場存放供稽核。

⑦ 驗測情境（可複製輸入即測）

展開驗測情境

情境 1：① 標準：貼現場評估與範本→產協議組織草稿

可直接複製貼到 EgentHub 對話框的測試輸入：

【現場評估】作業類型：無塵室二次配管線施工＋動火；廠域：示範R3西側；共同作業範圍：與示範甲包（機電）、示範乙包（保溫）於同區同時段作業；潛在危害：高處作業墜落、動火火災、共同作業起重吊掛干涉、管內殘留化學品。
【協議組織範本段落】一、協議事項 二、危害告知 三、各方權責 四、運作/會議機制。

預期：依範本四段落產出協議組織草稿（協議事項、危害告知、各方權責、運作機制），把現場評估的危害對應到協議/防護事項；涉及危害判定、權責認定處標「待工安判定」，最後列待補/待判定清單，並提示定稿與對外寄發由工安確認。

驗證重點：草稿依範本段落結構、危害對應合理；判定處標待工安判定不臆測現場危害；含人工定稿提示。
