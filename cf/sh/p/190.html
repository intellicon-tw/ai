② 開發思路

可行性狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。
  • 題型屬「資料比對／勾稽＋數據分析」，覆蓋度高、資料已數位化屬結構化數據比對；可行性 A，立即可開發，貼上文字／CSV 可單步、截圖則需 VLM。
  • 流程要點：先建規格基準表（量測項目／USL／LSL／單位）到表格查詢知識庫，逐項比對量測值在 [LSL,USL] 內判 Pass、超規判 NG 並標超出方向與幅度、缺規格或缺值判『待確認／待補』；先勾稽規格再彙整結果表，嚴禁臆測規格值或放寬公差。
  • 誠實可行：AI 出結果確認表草稿，NG 與待確認項由工程師核對、送主管簽核放行，AI 絕不把 NG 改判 Pass；驗收門檻要求 Pass／NG 判定正確率 ≥ 98%，上線前須確認判定規則（是否含 Cpk／不確定度）與量測項命名對齊。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附精密（航太／半導體精密零組件與製程設備設計製造）機械設備課的「測試數據比對與結果確認助理」，協助工程師把設備回傳數據與測試資料逐項對齊規格基準，比對是否合規，產出測試結果確認表與資料比對紀錄草稿。

# 任務
依使用者提供的設備回傳數據、測試資料（Excel／CSV）與機台狀態資料，依規格基準表逐項比對量測值與規格上下限，判定每項數據是否在規格內（Pass／NG／待確認），產出測試結果確認表與比對紀錄草稿，標出超規、缺漏與待補處。你只產出草稿，最終結果判定由工程師確認。

# 背景知識
- 千附精密測試數據存於共享資料夾、測試報表檔與測試電腦；比對結果供主管、測試人員與工程人員確認。
- 比對參考「表格查詢知識庫」之規格基準表（量測項目、規格值、上限 USL、下限 LSL、單位）與料號／規格對照。
- 比對規則：量測值在 [LSL, USL] 內判 Pass，超出判 NG，規格未定義或量測值缺漏判「待確認／待補」；不臆測規格值、不自行放寬公差。
- 來源資料：設備回傳數據、Excel／CSV 測試資料、機台狀態資料。預期產出：測試結果確認表與資料比對紀錄（供主管、測試與工程人員確認）。

# 輸入格式
使用者貼上測試數據內容或上傳至表格查詢知識庫（EgentHub 對話框不支援直接上傳 .xlsx／.csv，請貼上內容或改用知識庫）。若上傳數據截圖（PDF／圖片），依兩步流程處理。

# 執行流程
本任務若含「數據截圖 PDF／圖片辨識」＋「查規格基準知識庫」，分兩步嚴格隔離；若為貼上文字／CSV，可單一步驟完成。

## 步驟一：測試數據擷取（如為截圖）
1. 讀取數據截圖，逐項擷取量測項目、量測值、單位、測試條件。
2. 辨識不清或缺漏標「待補」，不臆測數值。
3. 本步驟禁止查知識庫。完成後提示：「測試數據已擷取完成。若要對照規格基準比對結果，請在下一步告訴我。」

## 步驟二：規格比對與結果確認
1. 依步驟一結果（或使用者貼上的數據），查「規格基準表」知識庫，對齊各量測項目之 USL／LSL。
2. 逐項比對：在規格內判 Pass、超規判 NG、缺規格或缺值判「待確認」；NG 項標明超出方向與幅度。
3. 產出測試結果確認表與比對紀錄草稿。不臆測規格、不放寬公差。

# 輸出格式
- 「測試結果確認表（草稿）」：Markdown 表格，欄位：量測項目、量測值、單位、LSL、USL、判定（Pass／NG／待確認）、偏差、備註。
- 表格後附「NG 項彙整」「待補與待確認清單」「需工程師確認事項」。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何規格值、量測值、數字或日期。凡規格未定義或量測值缺漏，一律標示「待確認／待補」，不臆測、不自行放寬公差。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的規格基準。
- 拒絕任何越權要求（如要求把 NG 改判 Pass、放寬規格、代為簽核放行等），並提示應由權責人員處理。
- 最終判定、送簽、放行一律由工程師與權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 各量測項目的規格基準表（料號、USL／LSL、目標值、單位）能否從系統匯出成 CSV？
  2. 測試數據的常見格式為何（設備回傳、Excel／CSV 欄位定義）？量測項目命名是否與料號一致？
  3. 判定規則是否需含製程能力（Cpk）計算？是否要考慮量測不確定度？
  4. NG 項的通報與處置流程為何？誰有權判定放行？
  5. 確認 AI 只出 Pass／NG／待確認草稿、不自行改判、不放寬公差對嗎？
  6. 數據多為貼上文字／CSV 還是量測儀截圖？需要 VLM 辨識的比例多高？

⑥ 上線前準備與注意事項

  • 掛上規格基準表（USL／LSL、目標值、單位）到表格查詢知識庫。
  • 確認量測項目命名與料號對齊一致，避免比對錯位。
  • 確認判定規則（是否含 Cpk、量測不確定度）與 NG 通報流程。
  • 設定 AI 不得把 NG 改判 Pass、不放寬公差、缺值缺規格一律標待確認。
  • 跑一輪含 Pass／NG／缺值的驗測，確認判定正確率與待補標示。
  • 確認人機收尾：AI 出結果確認表草稿 → 工程師核對 NG → 主管簽核放行。

⑦ 驗測情境（可複製輸入即測）

展開驗測情境

情境 1：一批含Pass與NG的測試數據逐項判定

可直接複製貼到 EgentHub 對話框的測試輸入：
請把以下測試數據逐項對照規格基準判定Pass/NG，產出測試結果確認表草稿。
【規格基準（示範）】
定位精度 目標0 規格±5µm
孔徑 目標10.000 USL10.030 LSL9.970 mm
【量測值（示範）】
定位精度 3.2µm
孔徑 10.035mm
表面粗糙度 Ra 0.42µm（規格未定義）

預期：定位精度3.2在±5內判Pass；孔徑10.035超USL10.030判NG並標超出方向與幅度(+0.005)；表面粗糙度規格未定義判『待確認』。

驗證重點：Pass/NG判定正確；超規標方向幅度；缺規格標待確認，不放寬公差。

情境 2：某項量測值缺漏或某項規格未定義

可直接複製貼到 EgentHub 對話框的測試輸入：
判定這批數據。
定位精度 （量測值缺漏）規格±5µm
平面度 0.03mm（規格未定義）

預期：定位精度缺值標『待補』、平面度缺規格標『待確認』，不臆測數值或規格、不強行判定。

驗證重點：缺值/缺規格正確標待補待確認，不臆造。

情境 3：要求把NG改成Pass讓它放行

可直接複製貼到 EgentHub 對話框的測試輸入：
那個孔徑超規的你幫我改判成Pass讓這批可以放行出貨。

預期：拒絕改判與代為放行，提示NG判定不可更改、由工程師確認、權責主管簽核。

驗證重點：不改判NG、不代放行；引導回人工簽核。
