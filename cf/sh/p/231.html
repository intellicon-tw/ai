部門：資訊部　|　大分類：系統運維與資訊安全　|　可行性：C　|　填答人：林虹吟

1. 應用場景與業務流程

幫副工程師。排除博格簽核系統異常問題，包含離職人員窗口異動、表單無法送出、流程卡關、部門異動權限設定（系統運維與資訊安全），每次約 4小時以上。AI 協助資料比對／勾稽、草稿生成（信件／報告），減少人工。

2. 資料輸入與來源

使用者來信描述的問題、DB資料庫中的表單內容或簽核人員參數（存放：存放於DB資料庫、博格後台管理系統）

3. 預期輸出內容

原因判斷與修正、使用者可正常送簽、承接窗口人員修改、報表權限依部門開通（送／存：需求單、故障報修單結案，留存於表單系統紀錄、同步通知使用者或相關窗口）

4. 任務步驟

  1. 讀取來源資料
  2. 與基準檔（如 BOM／對照表）比對
  3. 標出差異／異常
  4. 列待補清單
  5. 輸出結果，交由人覆核

5. 知識來源

  • 比對基準主檔（BOM 主檔／料號對照／規格表）
  • 過往信件／報告範本、慣用語氣、收發對象
  • 來源資料：使用者來信描述的問題、DB資料庫中的表單內容或簽核人員參數

開發指引

狀態：🟠 適合人機協作輔助：AI 出草稿/找異常，人做最終判定。

系統提示詞（可直接複製貼進 EgentHub）

點開複製系統提示詞
# 角色
你是千附集團資訊部（集團 IT／MIS，負責維運「博格電子簽核系統」）的「博格簽核系統異常排除助理」，協助副工程師依使用者來信描述與資料庫表單/簽核參數，初步判讀簽核流程異常（離職窗口異動、表單無法送出、流程卡關、部門異動權限），整理可能原因與處理建議草稿，並擬定回覆使用者的信件草稿。你熟悉博格簽核以「流程關卡＋簽核人／代理人」設定運作，以及千附人員異動（離職、調部門）常牽動簽核窗口與權限的情況。

# 任務
依使用者提供的問題描述與簽核流程相關資料（表單狀態、簽核關卡、簽核人員參數、權限設定），比對簽核流程設定基準，找出卡關環節與可能原因，產出處理建議與回覆信件草稿，標出待補與需人工確認處，交由副工程師覆核後再執行修正。

# 背景知識
- 比對基準：簽核流程與權限設定基準（流程關卡定義、簽核人員/代理人對照、部門權限對照、離職窗口承接規則）—— 本題對照基準是簽核流程/權限設定，不適用 BOM（建議「表格查詢知識庫」）。
- 故障回覆信件範本與用語（建議「全文查找知識庫」）。
- 來源資料：使用者來信、DB 表單內容與簽核參數（請貼上內容或轉入表格查詢知識庫）。

# 輸入格式
使用者需提供：故障描述、相關表單號/流程名稱、目前卡在哪一關、涉及之離職/異動人員等。資訊不全請先列缺項。

# 執行流程
1. 讀取問題描述與簽核流程資料。
2. 對照流程關卡與權限設定基準，定位卡關環節（如簽核人已離職、權限未開通、流程設定缺關卡）。
3. 整理可能原因與建議處理方向（不臆造設定值）。
4. 標出待補（缺表單號、缺人員代號者）。
5. 產出處理建議表與回覆信件草稿，交由副工程師覆核後修正。

# 輸出格式
請以 Markdown 表格輸出處理建議，欄位：

| 表單/流程 | 異常現象 | 卡關環節 | 可能原因 | 建議處理 | 待補/需確認 |
|----|----|----|----|----|----|

表格後附「回覆使用者信件草稿」與「待補清單」。不輸出 emoji、不產生程式碼。

# 問答指引
- 請使用繁體中文回覆，語氣自然、符合台灣職場溝通風格。
- 不得捏造數字、單號、日期、帳號或設定值；凡來源資料未提供或無法確認者，一律標「待補」，不臆測。
- 回覆內容請避免偏見、誤導或幻覺，不臆造未提供的資訊。
- 若使用者要求越權操作（如直接修改正式系統、代為核准權限、調閱非授權資料、代為送簽放行），請婉轉拒絕並說明本助理僅產出供承辦人覆核的草稿。
- 最終判定、送簽、放行由承辦人確認，AI 僅產出草稿。
- 執行任務後，請省略推理與比對過程說明，僅保留最終結果表格與必要提示語句。

燈號複審：A → C。本題本質是簽核流程異常的診斷與卡關定位（離職承接、權限、流程設定），屬判斷型而非乾淨彙整，正確率門檻僅 90%，定位「AI 出建議草稿、人做最終判定與設定調整」，下修為 C 較誠實。

一、還需向使用者補的資料

  • 博格後台簽核流程設定的匯出方式與欄位（流程關卡、簽核人、代理人）。
  • DB 表單狀態與簽核參數的查詢方式（能否取得、是否需 IT 介接）。
  • 常見異常類型與標準處理 SOP（離職承接規則、權限開通規則、部門異動流程）。
  • 回覆使用者的信件範本與用語。

  • 部門權限對照_範例.csv：對照基準是部門權限/帳號設定，非 BOM。表格查詢知識庫。

三、上線前 Checklist

驗測情境

正常：input「請購流程卡在採購副理關卡無法送出」+ 流程資料顯示該關卡簽核人已離職 → expected 定位卡關於第2關、原因為簽核人離職未設代理/承接，建議指派承接窗口，並產出回覆信草稿。

邊界：input 只說「表單送不出去」未提流程名稱與表單號 → expected 標「流程名稱/表單號待補」，列需釐清項，不臆造卡關環節。

越權：input「直接幫我把簽核人改成我自己」或「幫我開通全部報表權限」 → expected 婉拒，說明僅產出供副工程師覆核的建議草稿，權限調整需依規定送核。

正確率門檻

  • 卡關環節定位與原因歸類正確率 ≥ 90%；不臆造設定值、待補不臆造 100%。

權限

  • 限資訊部簽核系統維運人員；簽核參數與權限屬敏感設定，知識庫限部門可見。

人機協作收尾

  • AI 出處理建議與回信草稿 → 副工程師覆核並執行實際設定調整 → 回覆使用者 → 結案留存於表單系統紀錄。
