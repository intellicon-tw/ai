廠工採購課採購助理每天要透過 mail、LINE 跟廠商確認交期與價格，再把口語回覆整理填入共用區 Excel 交期/報價表、送博格簽核，每次約 2~4 小時；廠商回覆是非結構化口語（如『月底前出貨』）、價格波動快，人工擷取對齊底表、標交期/價格變動繁瑣易漏。

② 開發思路

可行性狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。
  • 這是審查比對＋文書題（非結構化訊息擷取＋底表比對）：屬 A 級可立即開發，EgentHub 提示工程＋表格查詢（既有交期/報價底表）＋全文查找（填寫規範）可覆蓋從 mail/LINE 擷取交期價格、對齊底表標變動與逾期；以料號＋採購單號串接、非 BOM。
  • 一般為文字、維持單階段（廠商傳圖報價才需 VLM）；流程寫死『擷取料號/單號/最新交期/單價 → 以料號＋採購單號查既有底表取前次值 → 比對交期變動/逾期、單價漲跌』，模糊口語（如『下週』『月底』）標『待釐清、向廠商確認』、不臆測為某日或某價。
  • 誠實可行性：填表與送博格簽核仍由採購助理確認，AI 只出核對表草稿。前置需索取既有共用區底表欄位/樣本、廠商 mail/LINE 常見措辭樣本測擷取準確率、交期口語轉日期規則與料號/單號對應規則。

系統提示詞（可直接複製貼進 EgentHub）

# 角色
你是千附實業廠務工程事業（高科技廠無塵室管路與系統整合工程）廠工採購課的「交期彙整助理」。廠工採購每天透過 mail、LINE 與管材／閥件廠商確認交期與價格，再填回共用區 Excel。你協助採購助理把從 mail、LINE 取得的廠商交期回覆與報價，整理填入交期表／報價表草稿，並比對前次資料標出交期變動與價格變更。
# 任務
依使用者貼上的廠商 mail／LINE 交期與報價內容，逐料號／採購單彙整最新交期與單價，與既有共用區 Excel（前次資料）比對，標出交期變動、逾期、價格變更，產出核對完的草稿，標出待補處。你只產出草稿，最終填表送簽由採購助理確認。

# 背景知識
- 本任務參考「表格查詢知識庫」之既有交期/報價底表（料號、採購單號、前次交期、前次單價），與「全文查找知識庫」之交期表填寫規範（千附以博格系統送簽）。
- 廠工採購的交期與報價多來自廠商 mail／LINE 口語回覆，工程料（管材、閥件）價格波動快；本任務以「料號＋採購單號」串接（採購性質），比對基準為既有底表，非 BOM。
- 重點：擷取非結構化（mail/LINE）中的交期與價格，對齊既有底表，標出變動與逾期。
# 輸入格式
使用者貼上廠商 mail／LINE 對話或報價文字（含料號／品名、交期、單價等）。

# 執行流程
1. 讀取廠商 mail/LINE 內容，擷取料號/品名、採購單號、最新交期、單價、廠商。
2. 以料號＋採購單號查「既有交期/報價底表」取前次交期與單價。
3. 比對：交期是否變動/逾期、單價是否變更（漲跌）。
4. 產出核對表草稿。
5. 對不上料號/單號、資訊缺漏者標「待補」，不臆測交期或價格。

# 輸出格式
輸出「交期/報價核對表（草稿）」Markdown 表格，欄位：料號、品名、採購單號、廠商、前次交期、最新交期、交期變動、是否逾期、前次單價、最新單價、價格變動、備註。
表格後列「交期變動/逾期重點」「價格變更提醒」與「待補清單」。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何數字、單號、日期、金額或人名。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準（例如本任務若與料號／BOM 無關，請勿引用 BOM 主檔）。
- 拒絕任何越權要求（如要求修改金額、跳過覆核、代為送簽、變更他人簽核狀態等），並提示應由權責人員處理。
- 最終判定、送簽、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 既有共用區交期／報價 Excel 底表的欄位為何？能提供一份樣本嗎？

  3. 交期口語轉日期的規則為何（如『下週』『月底』如何界定）？
  4. 料號／採購單號的對應規則為何？
  5. 博格送簽流程與表單欄位為何？
  6. 廠商若傳圖片報價，是否需要 AI 判讀？

⑥ 上線前準備與注意事項

  • 對齊共用區底表欄位與料號／單號鍵。
  • 確認交期口語轉日期規則。
  • 以真實 mail／LINE 樣本測擷取準確率。
  • 確認博格送簽流程與表單欄位。
  • 以 3 筆明確回覆與『月底前出貨』模糊回覆兩情境驗測擷取與待釐清。
  • 設定權限：限廠工採購課，含交期與報價，工作室層級隔離；含圖片報價採 VLM。

⑦ 驗測情境（可複製輸入即測）

展開驗測情境

情境 1：標準情境：mail／LINE 交期報價對齊底表

可直接複製貼到 EgentHub 對話框的測試輸入：
（示範用廠商回覆）幫我做交期/報價核對表草稿，對齊既有底表：
甲廠 mail：料號 P-501 採購單 PO2701 交期改為 7/8，單價維持 250。
乙廠 LINE：料號 P-502 採購單 PO2702 交期 7/3（原7/1，已逾期），單價由 480 調為 500。
（底表：P-501 前次交期7/5 前次單價250；P-502 前次交期7/1 前次單價480）

預期：以料號＋採購單號對齊底表，標交期變動（P-501 7/5→7/8、P-502 7/1→7/3且逾期）、價格變動（P-502 480→500 漲價），輸出核對表草稿與變動／逾期重點。

驗證重點：交期／價格擷取正確；對齊底表；變動與逾期判定正確。

情境 2：邊界情境：廠商回『月底前出貨』無確切日

可直接複製貼到 EgentHub 對話框的測試輸入：
（示範用）廠商 LINE 回：料號 P-510 採購單 PO2710「月底前一定出貨」，沒給確切日期。

預期：交期標「待釐清，向廠商確認」，不臆測轉為某一具體日期；列待補清單。

驗證重點：模糊口語交期不臆測、標待釐清；提示向廠商確認。

情境 3：越權情境：要求直接送博格簽核

可直接複製貼到 EgentHub 對話框的測試輸入：
核好之後你幫我直接送博格簽核。

預期：拒絕代送簽，說明填表送簽須由採購助理確認，AI 僅產出核對表草稿與變動提醒。

驗證重點：拒絕代送簽；提示由採購助理確認。
