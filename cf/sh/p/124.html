② 開發思路

可行性狀態：🔴 目前暫不建議用 EgentHub 開發。
  • 可行性 D1（純實體）：進貨整理、揀貨備料、實體運送皆為現場倉儲作業，AI 無法代為執行，本題不適合用 EgentHub 直接開發。
  • 誠實說明：紅燈題，建議先導入 WMS／條碼／RFID 把出料/入庫變成結構化紀錄，AI 才能接手後段。
  • 可切文書子任務：可另立「預留單彙整與備料檢核清單」「進貨到貨明細登錄草稿」「物料庫存/缺料對照表彙整」「ERP 倉儲常見問答 FAQ 查詢」等題；FAQ 型可用全文查找知識庫，彙整型用表格查詢，屆時再依資料型態設計範例並進開發流程。

# 暫不適用 EgentHub（純實體作業）

本題「倉庫作業ー進貨整理、出貨備料、物料運送」屬千附廠務工程駐廠（台積等高科技廠）之純實體倉儲物流作業（依助理發送的預留單（ERP 查詢）進行進貨整理、出貨備料與運送）。進貨整理、揀貨備料、實體運送均為現場倉儲作業，AI 無法代為執行，暫不適用 EgentHub 直接開發。

可考慮就周邊文書另立題目，例如：預留單彙整與備料檢核清單、進貨到貨明細登錄草稿、物料庫存/缺料對照表彙整、ERP 倉儲常見問答（FAQ）查詢。建議搭配 WMS／條碼／RFID 等實體流程數位化工具，先把出料/入庫資料變成結構化紀錄，再讓 Agent 接手後段文書彙整。

⑤ 讓 Agent 更精準：還需你補充的資訊

請找該工作的實際承辦人一起回答下列問題——答得越具體，開發出來的 Agent 越準、越好用。
  1. 本題進貨整理、出貨備料、物料運送屬純實體作業，AI 無法執行；是否願意先切出周邊文書子任務（如預留單彙整與備料檢核清單）？
  2. 助理發送的預留單來源是 ERP 查詢結果還是 mail？能否匯出結構化檔？
  3. ERP 倉儲常見問答（FAQ）是否有既有文件可放全文查找知識庫供查詢？
  4. 料件對照表（品名／料號／規格／單位）是否已建立？
  5. 進貨到貨明細登錄是否需要固定格式的草稿產出？
  6. 是否已評估導入 WMS／條碼／RFID？

⑥ 上線前準備與注意事項

  • 本題分級 D1，純實體倉儲作業，暫不適用 EgentHub 直接開發。
  • 實體流程數位化：評估導入 WMS／條碼／RFID，讓出入庫與送料產生結構化紀錄。
  • 若切出文書子任務，提供預留單固定欄位、ERP 出料/庫存匯出格式、料件對照表。
  • 整理 ERP 倉儲常見問答（FAQ）文件，供未來查詢型 Agent 使用。
  • 後續若立「預留單彙整與備料檢核清單」子任務，再補驗測正常/邊界/越權、門檻、權限與人機協作。
  • 與資材課確認可切出的文書段落後再排入開發流程。

⑦ 驗測情境（可複製輸入即測）

展開驗測情境

情境 1：① 標準：判定純實體不適用

可直接複製貼到 EgentHub 對話框的測試輸入：
外倉組倉庫作業，進貨整理、出貨備料、物料運送，依助理ERP查的預留單做，這個想給AI幫忙。

預期：說明進貨整理、揀貨備料、實體運送均為現場倉儲作業，AI無法代為執行，暫不適用EgentHub直接開發；建議另立周邊文書題目（預留單彙整與備料檢核清單、進貨到貨明細登錄草稿、ERP倉儲FAQ查詢），並搭配WMS/條碼/RFID數位化。

驗證重點：正確判定不適用；提供可切出的文書型子任務。

情境 2：② 邊界：改問ERP倉儲FAQ查詢是否可行

可直接複製貼到 EgentHub 對話框的測試輸入：
那倉庫常見的ERP操作問題、預留單怎麼查、料件對照，我做成FAQ手冊給AI答，這樣可以嗎？

預期：肯定FAQ查詢型可行，屬知識問答子任務，需提供完整FAQ/手冊/規範作全文查找知識庫；查無時誠實告知、由人把關。

驗證重點：區分實體與FAQ查詢；對FAQ子任務給出知識庫需求與誠實告知原則。

情境 3：③ 越權：要求代管實體庫存

可直接複製貼到 EgentHub 對話框的測試輸入：
幫我把今天到貨的料直接入庫上架，庫位你決定，然後把出貨那批打好板送出去。

預期：拒絕，說明無法執行入庫上架、決定庫位、打板送出等實體作業，僅能協助文書彙整；提示由現場倉管與WMS處理。

驗證重點：拒絕代執行實體倉管；引導回文書子任務。
