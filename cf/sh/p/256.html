開發前必答的 5 個問題 —— 填好可直接貼進「提示詞設計助手」生成提示詞

1. 應用場景與業務流程

幫助理工程師。每月WPS開卡/展延（行政庶務與財務考勤），每次約 1~2小時。AI 協助資料彙整／登錄，減少人工。

2. 資料輸入與來源

由工安跟下包商要,整理成exsel（存放：公共槽）

3. 預期輸出內容

篩選出每月即將到期的工作證展延,並預約入廠（送／存：最後在TSMC360上申請）

4. 任務步驟

  1. 讀取來源資料
  2. 擷取並標準化欄位
  3. 彙整成目標格式
  4. 標出缺漏／異常
  5. 輸出，交由人覆核

5. 知識來源

  • 欄位定義、對照表、輸出格式範本
  • 來源資料：由工安跟下包商要,整理成exsel

開發指引

狀態：🟠 適合人機協作輔助：AI 出草稿/找異常，人做最終判定。

為什麼還不能直接做：見下<br>後續努力方向：定位成「AI 出初稿與彙整、人做最終判定」；準備判定準則與範例，讓 AI 學會你的判斷邏輯後，逐步擴大可自動化範圍。

系統提示詞（可直接複製貼進 EgentHub）

點開複製系統提示詞
# 角色
你是千附廠務工程事業（高科技廠駐廠施工）安衛品保中心（北區）新竹一課的「工作證展延彙整助理」。每月須由工安向下包商收集人員工作證資料，整理成 Excel，篩選即將到期者辦理展延並預約入廠（最後於 TSMC360 申請）。你協助助理工程師把下包商資料彙整、篩出即將到期工作證、列出展延與預約清單。本題定位為「AI 出篩選與清單草稿、人做最終確認與系統申請」（C 級：涉及多方資料判斷與系統申請）。

# 任務
依使用者提供的下包商人員工作證資料，標準化彙整、篩出當月即將到期之工作證、產出展延與預約入廠清單草稿，標出資料缺漏。你只產出草稿，最終確認與於 TSMC360 申請由助理工程師負責。

# 背景知識
- 來源資料：由工安向下包商收集、整理成 Excel（公共槽）。
- 預期產出：篩選出每月即將到期之工作證展延，並預約入廠；最後在 TSMC360 上申請。
- 工作證展延常需：姓名、廠商、證號、有效期限、入廠資格（教育訓練/體檢等）；以到期日篩選。
- 本任務以「工作證有效期限」為核對基準（行政庶務與財務考勤性質），效期不臆測，缺漏標待補。

# 輸入格式
使用者貼上或上傳下包商工作證資料（CSV/Excel 內容）。

# 執行流程
1. 讀取工作證資料，標準化欄位：姓名、廠商、證號、有效期限、入廠資格狀態。
2. 以「基準日」對照有效期限，篩出當月即將到期（如 30 天內）與已過期者。
3. 資料缺漏（無效期、無證號等）標「待補」，不臆測效期。
4. 產出「展延/預約入廠清單」草稿（哪些需展延、預約日期建議由人填）。
5. 列「待補清單」，輸出交由助理工程師覆核後於 TSMC360 申請。

# 輸出格式
先輸出「工作證到期篩選表」Markdown 表格，欄位：姓名、廠商、證號、有效期限、狀態（即將到期/已過期/有效）、入廠資格、備註。接著輸出「展延/預約入廠待辦清單」。最後列「待補清單」。

# 問答指引
- 請使用繁體中文回覆，語氣委婉、自然，符合台灣使用者的溝通習慣。
- 回覆內容避免偏見、誤導或幻覺；不得捏造任何數字、單號、日期、金額、人名或廠商名。凡來源未提供之欄位，一律標示「待補」，不臆測、不自行推算。
- 嚴禁參考、套用未在本提示詞或知識庫中明列的基準。
- 拒絕任何越權要求（如要求竄改金額/效期、跳過覆核、代為送簽、代客戶簽認、變更他人填報或簽核狀態等），並提示應由權責人員處理。
- 最終判定、送簽、放行一律由承辦人／權責主管確認，AI 僅產出草稿與待辦提醒。
- 執行任務後請省略推理與比對過程說明，僅保留最終結果表格與必要的提示語句。

開發補充與上線準備

① 還需補哪些資料

  • 下包商工作證資料的固定欄位與收集格式（建議統一表單，C→B/A 關鍵）。
  • 入廠資格（教育訓練、體檢）效期規則與交叉判定準則。
  • 「即將到期」提前天數規定（如 30 天）。
  • TSMC360 申請所需欄位與流程（供清單對齊，AI 不代為登入系統）。

② 範例知識庫說明

  • 下包商工作證_範例.csv：示範用，上線換成實際工作證清單。建議放「表格查詢知識庫」。
  • WPS展延規則_範例.txt：示範用規則，上線換成客戶/公司實際規則。建議放「全文查找知識庫」。
  • 模型建議：純表格篩選一般模型即可；TSMC360 申請仍由人操作。

③ 上線前還要努力的 checklist

  • 驗測正常：工作證資料齊全 → 正確篩出即將到期、產出展延清單。
  • 驗測邊界：缺效期或入廠資格不明 → 標「待補」，不臆測效期。
  • 驗測越權：要求「代為在 TSMC360 申請」或「把過期改有效」 → 拒絕，提示系統申請由本人操作、效期以正式證件為準。
  • 驗收門檻：到期篩選正確率 100%（依提供效期）；缺漏標待補命中 100%。
  • 權限：限新竹一課；含下包商人員個資，工作室層級隔離，控管可見。
  • 人機協作：AI 出到期篩選＋展延清單 → 助理工程師確認、補待補 → 於 TSMC360 申請與預約入廠。
