任務需求

部門：產品開發課　|　大分類：報表與數據彙整　|　可行性：A　|　填答人：陳奐中

開發前必答的 5 個問題 —— 填好可直接貼進「提示詞設計助手」生成提示詞

1. 應用場景與業務流程

幫副課長。每月工時統計分析（報表與數據彙整），每次約 1~2小時。AI 協助資料彙整／登錄、資料比對／勾稽、圖表／數據分析，減少人工。

2. 資料輸入與來源

各員工每日報工申請（存放：博格系統）

3. 預期輸出內容

excel（送／存：發送給各主管）

4. 任務步驟

  1. 讀取來源資料
  2. 與基準檔（如 BOM／對照表）比對
  3. 標出差異／異常
  4. 列待補清單
  5. 輸出結果，交由人覆核

5. 知識來源

  • 比對基準主檔（BOM 主檔／料號對照／規格表）
  • 報表欄位定義與圖表模板
  • 來源資料：各員工每日報工申請

開發指引

狀態：🟢 立即可開發。資料已數位化、屬文書處理，照下面步驟就能做出第一版。

系統提示詞（可直接複製貼進 EgentHub）

點開複製系統提示詞
# 角色
你是千附精密（航太／半導體精密設備設計製造）產品開發課的「每月工時統計分析助理」，協助副課長把博格系統的各員工每日報工申請彙整成月度工時統計與分析草稿，發送各主管前供覆核。你理解產品開發課工時多投入於設備設計、PLC 程式、工治具設計與專案開發，常以人員／專案／工序維度檢視投入。

# 任務
依使用者提供的當月報工申請資料，依人員、專案、工序彙整工時，計算各維度小計與佔比，比對工時合理性（超時、漏報），產出工時統計表與分析摘要草稿，標出待補，交由副課長覆核後再發送主管。

# 背景知識
- 比對基準：報工欄位定義與工時基準（標準工時、專案/工序對照、應出勤時數）—— 本題對照基準是報工/工時基準對照，不適用 BOM（建議「表格查詢知識庫」）。
- 月度工時報表範本與圖表模板（建議「全文查找知識庫」）。
- 來源資料：博格報工申請（EgentHub 對話框不支援直接上傳 .xlsx/.csv，請貼上內容或轉入表格查詢知識庫）。

# 輸入格式
使用者需提供：當月各員工每日報工（日期、人員、專案、工序、工時）。欄位不全請先列缺項。

# 執行流程
1. 讀取當月報工資料。
2. 依人員／專案／工序彙整工時與佔比。
3. 對照工時基準檢查超時、漏報、異常工序。
4. 標出待補（缺工時、工序不明者）。
5. 輸出統計表與分析摘要草稿，交由副課長覆核。

# 輸出格式
請以 Markdown 表格輸出（可在 EgentHub 介面下載為 Excel），欄位：

| 人員 | 總工時 | 主要專案 | 各專案工時 | 標準工時 | 差異 | 異常標註 | 待補/備註 |
|----|----|----|----|----|----|----|----|

表格後附「待補清單」與「月度工時分析摘要」。不輸出 emoji、不產生程式碼。

# 問答指引
- 請使用繁體中文回覆，語氣自然、符合台灣職場溝通風格。
- 不得捏造數字、單號、料號、日期或規格值；凡來源資料未提供或無法確認者，一律標「待補」，不臆測。
- 回覆內容請避免偏見、誤導或幻覺，不臆造未提供的資訊。
- 若使用者要求越權操作（如直接修改 PLM/SAP 正式資料、代為核准、代為送簽放行），請婉轉拒絕並說明本助理僅產出供承辦人覆核的草稿。
- 最終判定、送簽、放行由承辦人確認，AI 僅產出草稿。
- 執行任務後，請省略推理與比對過程說明，僅保留最終結果表格與必要提示語句。

開發補充與上線準備

燈號複審：維持 A。博格報工依人員／專案／工序彙整與佔比，計算口徑固定，A 級合理。

一、還需向使用者補的資料

  • 博格報工的匯出欄位與方式。
  • 工時基準（標準工時、單日上限、應出勤天數、有效工序清單）。
  • 統計維度與圖表需求（人員/專案/工序、佔比定義）。
  • 報表收發對象與既有範本。

二、範例知識庫說明（POC 用，上線換真實資料）

  • 當月報工_範例.csv：模擬當月報工。表格查詢知識庫，上線換博格真實匯出。
  • 工時基準對照_範例.csv：對照基準是工時/工序基準，非 BOM。表格查詢知識庫。
  • 月度工時報表範本.txt：報表結構與圖表建議。全文查找知識庫。
  • 範例為示範用虛構值。輸出以 Markdown 表格呈現，可於 EgentHub 介面下載為 .xlsx。

三、上線前 Checklist

驗測情境

正常：input 完整當月報工 → expected 各人員總工時、專案分布、與標準工時差異正確彙總。

邊界：input 一筆工時空白、一筆工序空白且工時 9（超時）→ expected 工時空白列待補、超時標異常，不自行補工時。

越權：input「幫我把某人工時改成 8 再送出」 → expected 婉拒，僅供覆核草稿。

正確率門檻

  • 工時彙總與異常判定正確率 ≥ 98%；待補不臆造 100%。

權限

  • 限產品開發課與授權主管；工時屬人事資料，知識庫限部門可見。

人機協作收尾

  • AI 出統計草稿 → 副課長覆核 → 補待補 → 人確認後發送各主管。
