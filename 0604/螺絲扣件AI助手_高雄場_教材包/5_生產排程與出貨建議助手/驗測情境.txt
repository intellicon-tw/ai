# 生產排程與出貨建議助手 - 驗測情境

## 情境 #1：日期區間排程查詢

**輸入描述** 使用者輸入「6/4 到 6/15 要交的訂單有哪些？」

**預期輸出** 列出該區間內所有訂單，含建議產線、預估完成日、風險標示

**驗證重點** 確認日期篩選正確，預估完成日依公式計算而非抄要求交期

## 情境 #2：延誤風險辨識

**輸入描述** 使用者輸入「目前有哪些訂單可能延誤？」

**預期輸出** 列出預估完成日晚於要求交期的訂單，含延誤天數與建議行動

**驗證重點** 確認延誤判斷邏輯正確（預估完成日 > 要求交期）

## 情境 #3：產線瓶頸分析

**輸入描述** 使用者輸入「產線 B1 目前狀況如何？要解瓶頸要怎麼做？」

**預期輸出** 列出 B1 產線稼動率、目前在製品，並提出 1–3 條具體建議（委外、加班、調整優先順序）

**驗證重點** 確認回應依產能表實際資料，建議行動有具體性

## 情境 #4：客戶維度查詢

**輸入描述** 使用者輸入「廣達精密所有未出貨訂單的預估交期？」

**預期輸出** 列出該客戶所有未結案訂單，含預估完成日

**驗證重點** 確認客戶篩選正確，不混入其他客戶

## 情境 #5：查無資料邊界測試

**輸入描述** 使用者輸入不存在的客戶「XX 企業」

**預期輸出** 回覆查無此客戶的訂單記錄

**驗證重點** 確認 Agent 不捏造資料

## 可直接使用的測試題

```
6/4 到 6/15 要交的訂單有哪些？依交期排序給我。
```

```
目前有哪些訂單可能會延誤？預計延幾天？
```

```
產線 B1 目前狀況如何？稼動率多少？要解瓶頸有什麼建議？
```

```
廣達精密所有未出貨訂單的預估交期是什麼時候？
```

```
M8L25-Z 這一週需要排多少產能？目前夠不夠？
```
