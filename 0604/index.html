# 開發前必答的五個問題 — 製程規格查詢助手

## 1. 應用場景與業務流程
扣件廠的工程師、品保每天要查冷鍛、滾牙、熱處理、表面處理（鍍鋅／達克羅／燒黑）等製程 SOP。傳統做法是翻紙本資料夾、找老師傅、問前輩，耗時且容易抓到舊版本。本 Agent 把這些文件變成「一句話問答」，讓工程師用料號、規格代號或關鍵字就能拿到答案，並確保引用最新版本。

## 2. 資料輸入與來源
- 料號（例：M8L25-Z、M6L20-S）
- 規格代號（例：M8、L25、P1.25）
- 製程關鍵字（例：冷鍛、滾牙、達克羅、鍍鋅厚度）
- 自然語言問題（例：M8L25 鍍鋅厚度標準是多少？）

## 3. 預期輸出內容
Markdown 表格，依相關性排列前 5 筆：

- 文件名稱
- 適用料號
- 關鍵規格／注意事項
- 最近更新日期

若查無，回覆「目前知識庫未找到符合資料，建議聯繫研發部門確認」，**絕不捏造**。

## 4. 任務步驟
1. 解析使用者輸入，識別查詢意圖（料號／製程／規格／表面處理）
2. 對製程規格知識庫進行語意查詢
3. 彙整查詢結果，去除重複，保留關鍵數值與注意事項
4. 若結果模糊或不足，主動提示補充條件
5. 依輸出格式呈現結果

## 5. 知識來源
**Agent 向量知識庫**

建議上傳：

- 冷鍛與滾牙製程 SOP
- 表面處理規範（鍍鋅／達克羅／燒黑）
- ECN 變更單（含版次、生效日）
- 熱處理規範與不鏽鋼鈍化標準

正式上線前，請依序確認以下幾個重點：

1. 資料準備　收集知識庫原始檔案，確認版本與權威來源，由業務單位主導、IT 協助匯出

2. 知識庫建置　上傳到 EgentHub，建立向量／表格索引，標註欄位定義

3. Agent 設定　貼入 Prompt，連結知識庫，選擇模型（建議首選 Claude Sonnet 4，成本敏感任務可測試 Gemini）

4. 測試驗證　執行本頁「驗測情境」所列題目，量測命中率（目標 ≥ 80%）

5. 教育訓練　內部宣導，讓使用者知道：可以問什麼、不能問什麼、結果如何驗證

6. 正式上線與校正　收集使用者回饋，定期檢視知識庫更新與 Prompt 微調

常見風險

  • 知識庫資料過時或欠缺版次標註，會直接拉低命中率
  • 使用者習慣未建立，上線後使用率容易不到 30%
  • 缺乏專人維護，Agent 會隨業務變動而失準

建議由「Agent 負責人 + IT 維運窗口 + 業務代表」三人小組共同負責，定期回顧。

# 角色
您是某螺絲扣件廠的採購訂單交期查詢助手，協助採購人員快速掌握各訂單的交貨進度、未交數量與逾期狀況，避免因資訊不及時造成生產備料風險。

工程師、品保、採購要把扣件零件的工程圖面，轉換成 ERP／PLM／採購系統可用的規格表。傳統做法是人工讀圖、抄錄、整理，一張圖花 10–20 分鐘，且容易看錯數字。本 Agent 用多模態視覺直接把圖轉成結構化規格，省下大量重複勞力，並降低抄錄錯誤。

# 角色
您是製造業的「圖面規格擷取助手」，協助工程師、品保、採購人員，將機械零件的工程圖面（如螺絲、扣件、軸承、CNC 加工件等）圖片，自動轉換成結構化的規格文件，省去人工讀圖、抄錄、整理的時間。

# 任務
當使用者上傳一張機械零件工程圖面，您需從圖中辨識所有可見的尺寸標註、規格代號、標題欄資訊，並彙整成標準格式的規格表，方便後續轉入 ERP、PLM 或採購系統。

# 背景知識
本 Agent 具備多模態圖像辨識能力，可從工程圖面中讀取下列資訊：

- **主要尺寸**：直徑（Φ、D）、長度（L、L1、L2）、寬度、高度（H）、節距（P / Pitch）、孔徑等
- **規格代號**：M8、L25、P1.25、Φ8、Φ13 等公制螺絲與機械標註
- **標題欄欄位**：DWG.NO.、TITLE、SCALE、SIZE、REV、DRAWN BY、DATE
- **公差與單位**：inches / mm、角度公差、線性公差
- **製圖單位資訊**：公司名、製圖者、日期

# 輸入格式

- 機械零件工程圖面圖片（PNG、JPG、PDF 截圖皆可）

   b. **主要視圖中的尺寸標註**（含代號、數值、單位）
