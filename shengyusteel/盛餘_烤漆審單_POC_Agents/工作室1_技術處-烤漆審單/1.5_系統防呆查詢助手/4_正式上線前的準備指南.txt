# 系統防呆查詢助手 - 正式上線前的準備指南

本文件提供「系統防呆查詢助手」部署準備清單。本 Agent 從主審單助手獨立出來，因為系統防呆是「逐張訂單」的紀錄，混進主助手會造成跨單污染；獨立成「以訂單編號查詢」用途則安全且實用。

## 1. 知識庫設定
### 1.1 系統防呆與備註.csv（表格查詢知識庫）
- 類型：表格查詢知識庫；名稱「系統防呆與備註」；utf-8-sig。
- 內含 2,139 筆系統已產出之各產線防呆/備註（MKT/RAW/SOP 防呆＋PSR＋訂單修改記錄）。
- 正式替換：上線時可改接系統即時匯出，或定期更新。

## 2. 模型選擇建議
- 單純依編號查表，Gemini 3 Flash 或同級輕量模型即可省點數；需較佳中文整理可用 Claude 4.6 Sonnet。

## 3. 兩階段隔離設定
本 Agent 為單階段執行，無須額外設定。

## 4. 權限與工作室設定
- 工作室：技術處-烤漆審單；可見群組：審單人員。
- 敏感度：含訂單防呆內部資訊，限技術處內部。

## 5. 上線前驗測 Checklist
- [ ] 情境 #1（查存在的訂單）：正確回出該單 MKT/RAW/SOP 防呆。
- [ ] 情境 #2（查不存在的訂單）：明確回「查無」，不得拿別單頂替。

## 6. 上線後監控指標（首 2 週）
- 查詢命中率、回應正確率、使用量。

## 7. 異常處理與回退方案
- 查無資料先確認編號格式與知識庫版本。
- 維運窗口：建議由防呆/系統維護人員擔任 owner。

## 8. 種子團隊接手提示
- 核心維護＝定期更新系統防呆紀錄。
- 本 Agent 刻意與主審單助手 1.1 分離，請勿把本知識庫掛回 1.1，以免跨單污染。
