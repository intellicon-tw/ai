# 烤漆訂單審查助手 - 正式上線前的準備指南

本文件提供將「烤漆訂單審查助手」從 POC 套件正式部署到 EgentHub 平台、並在技術處烤漆技術課日常審單作業中穩定使用的逐步準備清單。建議由 IT/AI 種子團隊與審單負責人（大維哥）共同檢核。

## 1. 知識庫設定
本 Agent 的 `2_知識庫範例資料/` 內含四個檔案，請分別建立：
### 1.1 業務備註解讀指南.txt（Agent 向量知識庫 RAG）
- 知識庫類型：Agent 向量知識庫（RAG）
- 建議名稱：「業務備註解讀指南」
- 層級：Agent 層級
- 正式資料替換：由大維哥持續補充新案例，建議每季更新一次。
### 1.2 業務備註解讀對照表_範例.csv（表格查詢知識庫 SQL）
- 知識庫類型：表格查詢知識庫
- 建議名稱：「業務備註解讀對照表」
- 編碼：utf-8-sig
- 正式資料替換：POC 僅 15 類範例，上線前建議由審單團隊擴充至 100+ 真實案例。
### 1.3 防呆規則表_範例.csv（表格查詢知識庫 SQL）
- 知識庫類型：表格查詢知識庫
- 建議名稱：「OFP 防呆規則表」
- ⚠ 正式資料替換：POC 僅 8 條，上線前必須由附件27匯出完整 761 條（含改訂理由）。
### 1.4 相似訂單歷史檔_範例.csv（表格查詢知識庫 SQL）
- 知識庫類型：表格查詢知識庫
- 建議名稱：「相似訂單歷史檔」
- 正式資料替換：建議匯入 5–10 年烤漆歷史審單訂單。

## 2. 模型選擇建議
- 預設：Claude 4.6 Sonnet（一般推理＋多知識庫查詢，審單邏輯較複雜）。
- 若日後加入「上傳受訂生產通知單圖片」功能，視覺解析用 Claude 4.6 Sonnet 或 Gemini 3.1 Pro。

## 3. 兩階段隔離設定
POC 階段為純文字輸入＋知識庫查詢，本 Agent 為單階段執行，無須額外設定。
未來若開放上傳訂單截圖，再改為條件式兩階段（上傳圖片走步驟一圖像理解→步驟二查詢）。

## 4. 權限與工作室設定
- 工作室：技術處-烤漆審單。
- 可見群組：烤漆技術課審單人員、AI 種子團隊。
- 資料敏感度：訂單含客戶名稱與規格，屬營業敏感資料，限技術處內部存取，勿對全公司開放。

## 5. 上線前驗測 Checklist
執行 `3_驗測情境.txt` 全部情境，以下為 pass/fail gate：
- [ ] 情境 #1（標準烤漆單）：必須正確解讀限重/目標寬度並產出注意事項。
- [ ] 情境 #2（勿生產轉單）：必須標記【需人工確認】，不得判定可放行。
- [ ] 情境 #3（規範引用錯誤）：必須提示查核規範編號。

## 6. 上線後監控指標（首 2 週）
- 每日審單使用次數、使用人數。
- 高風險誤判率（該標未標、不該標亂標）。
- 審單人員主動回報的解讀錯誤次數。
- 平均回應時間與點數消耗。

## 7. 異常處理與回退方案
- 若解讀明顯失準：先查知識庫是否為最新版本（防呆規則、解讀對照表）。
- 重大誤判時暫時改回純人工審單，並回報 AI 種子團隊調整提示詞。
- 維運窗口：建議由大維哥擔任本 Agent owner，負責案例補充與規則更新。
- 提示詞調整後須回測情境 #1~#3。

## 8. 種子團隊接手提示
- 最常見調整方向：擴充業務備註解讀對照表的案例數、更新防呆規則。
- 新增類似 Agent 可由本 Agent 複製為起點。
- 資料依賴：本 Agent 與「防呆規則查詢助手」「相似訂單比對助手」共用知識庫，更新時請同步。
