## 2. 模型選擇建議
- 預設：Claude 4.6 Sonnet（可讀圖＋多知識庫查詢，審單邏輯較複雜）。本 Agent 支援上傳受訂生產通知單 PDF/圖片，**務必指派可視覺解析的模型**（Claude 4.6 Sonnet 或 Gemini 3.1 Pro），勿用純文字模型，否則上傳單據會無回應。
- 註：受訂生產通知單 PDF 內文常無法純文字抽取（字型編碼問題），必須靠視覺模型讀圖。

## 3. 兩階段隔離 ＋ 人類確認關卡（條件式，重要）
本 Agent 為「條件式兩階段＋人類確認」：上傳 PDF/圖片時走步驟一（讀圖判讀，禁查知識庫）→ **停下請使用者確認** → 步驟二（依已確認內容查知識庫審查）。純文字輸入則直接走步驟二。
- 請在 EgentHub Agent 設定頁啟用「兩階段隔離」。
- **人類確認關卡**：步驟一判讀完會停下並問「以上判讀是否正確？請回覆『確認』或指正」。使用者回覆「確認」後，agent 才在下一則回應做審查。此設計強制切出回合邊界，是目前防止「審查時抓錯訂單」的關鍵手段。
- 測試做法：
  1. 純文字輸入 → 直接給審查結果。
  2. 上傳 PDF → 先只輸出「步驟一判讀＋確認提問」，**不應同時出現審查結果**；回覆「確認」後才出審查。
  3. 若同一則回應裡同時出現兩次步驟一、或判讀完直接接審查並抓錯訂單 → 代表確認關卡或兩階段沒生效，請檢查兩階段設定，並確認使用的是最新版提示詞。
- 若上傳 PDF 後無回應：多半是未指派視覺模型，或未啟用兩階段。

## 4. 權限與工作室設定
- 工作室：技術處-烤漆審單。
- 可見群組：烤漆技術課審單人員、AI 種子團隊。
- 資料敏感度：訂單含客戶名稱與規格，屬營業敏感資料，限技術處內部存取，勿對全公司開放。

## 5. 上線前驗測 Checklist
執行 `3_驗測情境.txt` 全部情境，以下為 pass/fail gate：
- [ ] 情境 #1（標準烤漆單）：必須正確解讀限重/目標寬度並產出注意事項。
- [ ] 情境 #2（勿生產轉單）：必須標記【需人工確認】，不得判定可放行。
- [ ] 情境 #3（規範引用錯誤）：必須提示查核規範編號。

## 6. 上線後監控指標（首 2 週）
- 每日審單使用次數、使用人數。
- 高風險誤判率（該標未標、不該標亂標）。
- 審單人員主動回報的解讀錯誤次數。
- 平均回應時間與點數消耗。

## 7. 異常處理與回退方案
- 若解讀明顯失準：先查知識庫是否為最新版本（防呆規則、解讀對照表）。
- 重大誤判時暫時改回純人工審單，並回報 AI 種子團隊調整提示詞。
- 維運窗口：建議由大維哥擔任本 Agent owner，負責案例補充與規則更新。
- 提示詞調整後須回測情境 #1~#3。
