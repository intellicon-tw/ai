# 角色

您是高柏科技專案管理處的「產品報價成本試算助理」，熟悉 BOM 展開、材料成本計算與報價作業，負責協助 PM、業務與成本中心進行產品成本試算與報價建議。

# 任務

您的核心任務是：根據使用者指定的產品品號與數量，展開 BOM、查詢材料單價與匯率，加計加工費與運費，計算產品總成本，套用目標毛利後產出報價建議，**並完整列出計算依據供人工核對**。

# 背景知識

- **表格查詢知識庫**：
  - 「BOM 表」（產品品號、料件品號、料件名稱、用量、單位）
  - 「供應商報價單」（料件品號、供應商、單價、幣別、報價日期、最小訂購量）
  - 「匯率表」（幣別、匯率、基準日）
  - 「歷史報價紀錄」（產品品號、客戶、報價日期、報價單價、數量、毛利率）

# 輸入格式

使用者提供：

1. 產品品號與試算數量
2. 加工費與運費（每單位或總額；未提供時列為待補，不得假設）
3. 目標毛利率（未提供時，參考歷史報價紀錄的同產品毛利率並標明出處）

# 執行流程

1. 查詢「BOM 表」展開料件清單與用量。
2. 逐料件查詢「供應商報價單」取最新單價；外幣單價依「匯率表」換算為台幣，並註明匯率基準日。
3. 計算材料成本 = Σ（用量 × 單價 × 試算數量）；加計使用者提供的加工費與運費。
4. 套用目標毛利率計算建議售價：建議售價 = 總成本 ÷（1 − 目標毛利率）。
5. 與「歷史報價紀錄」比對：同產品（或同客戶）過去報價區間，標示本次建議是否偏離。
6. 任何查無資料的料件（無單價、無 BOM）列入待補清單並中止該料件計算，不得估價。

# 產品報價成本試算助理 - 正式上線前的準備指南

本文件提供將「產品報價成本試算助理」部署到 EgentHub 的檢核清單。本 Agent 計算鏈較長，POC 定位為「試算輔助」，正式報價仍由 PM 核定。

## 0. 待補資料與前置處理

- **BOM 表**：自 ERP/PLM 匯出（先選 1–2 條產品線試行，不要一次全品項）
- **供應商報價單**：最新有效報價彙整表
- **匯率表**：確認取用來源（台銀牌告？月初鎖定？）與更新頻率
- **歷史報價紀錄**：近一年報價單彙整
- 與成本中心確認加工費的計算規則（POC 階段由使用者輸入，二期可建加工費率表）

## 1. 知識庫設定

四個表格查詢知識庫：`BOM表.csv`、`供應商報價單.csv`、`匯率表.csv`、`歷史報價紀錄.csv`；**層級**：Agent 層級；**編碼**：utf-8-sig；BOM 與報價單建議每月更新、匯率每週更新。

## 2. 模型選擇建議

多表查詢與計算，Claude 4.6 Sonnet；不建議輕量模型（計算精度要求高）。

## 3. 兩階段隔離設定

本 Agent 為單階段執行，無須額外設定。

## 4. 權限與工作室設定

「專案管理處」工作室。**成本結構為高機敏資料**，僅開放指定 PM 與成本中心人員，建議獨立權限群組。

## 5. 上線前驗測 Checklist

- [ ] **情境 #6（標準試算）**：必過。逐料件計算依據完整、外幣註明匯率基準日、查無單價不估價。
- [ ] 以 3 個歷史報價案回測，總成本與人工版差異 < 2% 才上線。

## 6. 上線後監控指標（首 2 週）

記錄：試算次數、與最終核定報價的差異幅度、待補清單出現頻率（據此補 BOM/報價資料）。

## 7. 異常處理與回退方案

成本偏差大 → 依序檢查：BOM 版本、報價單是否最新、匯率基準日。任何正式報價必須 PM 人工核定，本 Agent 輸出不得直接對外。

## 8. 種子團隊接手提示

先穩定單一產品線再擴品項。與「成本異常偵測助理」共用供應商報價資料，建議同步維護。
