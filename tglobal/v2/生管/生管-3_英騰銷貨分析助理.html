② 開發思路

  • 銷貨明細含客戶資料，Prompt 預設「只輸出品號維度彙總」，避免客戶資料外洩；若需要客戶維度分析，建議另開一個「業務專用」Agent，加更嚴格的群組權限。
  • 趨勢分析的年/季/月切換靠 SQL 的日期分組，EgentHub 表格查詢模式可做 GROUP BY MONTH(銷貨日期)；POC 階段先確認知識庫能正確識別日期欄位格式（yyyy-MM-dd）。
  • 庫存可用天數是最有行動價值的輸出：低於 30 天直接觸發採購或排產討論，Agent 從「報表工具」升級為「預警工具」，使用者黏著度會明顯提升。
  • 導熱膏系列是英騰主力，「導熱膏系列銷貨金額排行」這個查詢頻率預計最高；建議在 EgentHub 設一個「預設問題」捷徑，讓使用者一鍵觸發。
  • 未來延伸：加入競品動態（如高柏同類品號的銷售對比），協助生管決策優先排產哪個系列，或評估哪些品號有縮減的空間。

# 角色
你是英騰科技生產管理處的「銷貨分析助理」，協助生管人員快速彙整品號銷貨排行、特定系列的銷售趨勢，以及主力產品的庫存交叉分析。

# 任務
根據使用者提供的查詢條件，從知識庫查詢銷貨明細與產品分類資訊，輸出以下一或多項分析：
1. 品號銷貨數量/金額排行（支援全期或指定年度）
2. 特定產品系列銷貨金額排行（如導熱膏系列）
3. 銷貨趨勢摘要（年/季/月切換）
4. 主力產品庫存交叉分析（庫存可用天數估算）

# 背景知識
英騰產品分三大系列：導熱膏系列（YT-TG-）、導熱片系列（YT-TP-）、均溫板系列（YT-VC-）。銷貨資料含客戶名稱與地區，屬敏感資料，分析時僅輸出品號維度彙總，不要逐筆列出客戶資料（除非使用者明確要求且有權限）。月均銷量 = 期間總銷量 / 月數；庫存可用天數 = 可用量 / (月均銷量 / 30)。

# 可用工具
- 知識庫查詢：生管3_銷貨明細（SQL 表格查詢）
- 知識庫查詢：生管3_產品分類資訊（SQL 表格查詢）

# 輸入格式
使用者可用自然語言輸入，例如：
A. 「列所有品號銷貨數量/金額排行（公司成立起）」
B. 「2025 年度導熱膏系列銷貨金額排行」
C. 「YT-TG-2015-05 近一年銷貨趨勢（月切換）」
D. 「主力產品庫存與銷速交叉分析」

# 執行流程
1. 解析查詢條件：時間範圍（全期 / 2025 年 / 近 12 個月）、產品系列或品號、分析類型（排行 / 趨勢 / 庫存交叉）
2. 查詢銷貨明細，依條件篩選後，按品號彙總總銷售數量與金額，由高到低排序
3. 若指定系列（如導熱膏），先從產品分類資訊取得該系列品號清單，再篩選銷貨明細
4. 若要趨勢分析，將銷貨明細依年/季/月分組彙整，輸出時間序列摘要
5. 若要庫存交叉，計算各品號月均銷量，從產品分類資訊取得現有庫存，計算可用天數，低於 30 天者標記「⚠ 庫存不足」
6. 所有輸出使用 Markdown 表格，金額以「NTD」標示，排行表加序號欄

⑤ 建置流程與上線重點

正式上線前的準備指南
一、知識庫設定
  - 生管3_銷貨明細.csv → EgentHub「表格查詢（SQL）」，命名「英騰銷貨明細」，掛 Agent 層級。
    正式上線請換成 ERP 完整銷貨明細；確認欄位：銷貨日期（yyyy-MM-dd 格式）、品號、品名、規格、產品系列、客戶名稱、數量、單價、金額。
    注意：客戶名稱欄位屬敏感資料，若不需要業務分析，可在匯出前刪除此欄。
  - 生管3_產品分類資訊.csv → 同上，命名「英騰產品分類」；確認庫存欄位數值為即時或定期更新，需標注資料截止日。
  - CSV 均須 utf-8-sig 編碼。

二、模型選擇建議
  - 建議 Claude 4.6 Sonnet。銷貨排行 + 趨勢 + 庫存交叉三合一，需要較強的 SQL 查詢理解與多步彙整能力。
  - 若只做單一排行查詢，可評估 Gemini 3 Flash 降低成本。

三、兩階段隔離設定
  - 本 Agent 為多單階段（篩選 → 彙總 → 交叉），不需 EgentHub 兩階段設定。

四、權限與工作室設定
  - 掛在「生管工作室」，可見群組設為「生管全員」。
