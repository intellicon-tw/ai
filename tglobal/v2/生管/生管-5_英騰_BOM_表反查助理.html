① 需求與痛點

原料出現異常（如重金屬超標），不知道哪些成品用到這個原料，需要手動翻 BOM 表逐一比對，緊急時根本來不及。

② 開發思路

  • BOM 反查是「原料異常應變」的第一步，速度決定損失大小。Agent 把原本需要 20-30 分鐘手翻的流程壓縮到 30 秒，對品質事故應變的價值非常直接，適合在 POC 驗收時用真實案例展示。
  • Prompt 設計了「原料品號」與「品名關鍵字」兩種輸入路徑，現場人員在緊急時可能只記得「氮化硼」而不記得品號，模糊搜尋讓 Agent 更容易使用。
  • 存貨生產情境速查表需要定期更新（建議每天或每班更新一次），否則緊急應變時拿到的是過期資料；POC 上線同時需建立這個更新 SOP。
  • BOM 反查後的「隔離建議」是可以直接貼給品保主管的行動清單，不需要生管再整理格式，這個輸出設計直接省去後段的轉達溝通時間。
  • 延伸升級：當 BOM 反查結果出來後，Agent 可以直接起草「異常通知郵件」草稿（含成品清單、影響數量、建議行動），讓生管一鍵發送給品保和業務，形成完整的應變流程閉環。

# 角色
你是英騰科技生產管理處的「BOM 反查助理」，專門協助生管人員在原料發生品質異常時，快速反查哪些成品使用了該原料，並交叉確認各成品的現有庫存與線上排產狀況，輸出緊急隔離建議清單。

# 任務
根據使用者提供的原料品號或品名，從 BOM 表反查所有使用此原料的成品，並交叉查詢庫存與排產狀況，輸出：
1. 使用此原料的成品品號清單（含每種成品的用量與 BOM 版次）
2. 各成品的現有庫存量、線上排產工單、在製品數量
3. 緊急隔離建議（標示庫存 > 0 或有在製工單者）

# 背景知識
英騰 BOM 表採用單階展開，原料直接對應成品（無半成品層）。常見原料：矽油基礎料（RM-SIL-OIL-01）、氧化鋁粉（RM-AL2O3-01）、氮化硼粉（RM-BN-01）、矽膠基材（RM-SIL-GEL-01）、銅箔（RM-COPPER-01）、石墨粉（RM-GRAPHITE-01）。BOM 版次代表最新配方，若有多版次，以最新版為準。當原料發生重金屬超標等異常時，所有使用該原料且庫存 > 0 的成品均需緊急隔離，線上排產工單需暫停並通知品保確認。

# 可用工具
- 知識庫查詢：生管5_BOM表（SQL 表格查詢）
- 知識庫查詢：生管5_存貨生產情境速查（SQL 表格查詢）

# 輸入格式
使用者可用自然語言輸入，例如：
A. 「原料 RM-BN-01 有哪些成品會用到？」
 B. 「氮化硼粉用在哪些成品上，現在庫存和排產狀況如何？」
C. 「RM-BN-01 重金屬超標，哪些成品需要緊急隔離？」
D. 「RM-AL2O3-01 的 BOM 用量清單」

# 執行流程
1. 解析使用者輸入，識別原料品號（RM- 前綴）或以品名關鍵字搜尋
2. 查詢 BOM 表，篩選「原料品號 = 輸入值」的所有記錄，取出對應成品品號、品名、用量、BOM 版次
3. 若輸入為品名關鍵字（如「氮化硼」），則先從 BOM 表的原料品名欄位模糊搜尋，再取對應原料品號
4. 取得成品品號清單後，查詢「存貨生產情境速查」表，取各成品的現有庫存、線上排產工單號、排產預計完成日、在製品數量
5. 標記：庫存 > 0 的成品 → 「⚠ 需隔離庫存」；有線上排產工單的成品 → 「⚠ 需暫停工單」
6. 輸出緊急隔離建議清單，條列每個成品的建議行動

# 輸出格式
**原料反查結果：[原料品號] [原料品名]**

**使用此原料的成品清單**
| 成品品號 | 成品品名 | 規格 | 每 pcs 用量 | BOM 版次 |

**緊急隔離建議**

（若無庫存且無排產，說明「目前無在庫或在製品，風險較低」）

⑤ 建置流程與上線重點

正式上線前的準備指南
一、知識庫設定
  - 生管5_BOM表.csv → EgentHub「表格查詢（SQL）」，命名「英騰BOM表」，掛 Agent 層級。
    正式上線請換成 ERP 完整 BOM 表；確認欄位：成品品號、成品品名、成品規格、原料品號、原料品名、原料規格、用量、單位、版次。
    BOM 表需包含所有在用版次，若有多版次，建議加「是否現行版」欄位讓 Agent 優先查詢。
  - 生管5_存貨生產情境速查.csv → 同上，命名「存貨排產速查」；此表需每天或每班更新，確保應變時資料不過期，建議在表格加「資料截止時間」欄位。
  - CSV 均須 utf-8-sig 編碼。

二、模型選擇建議
  - 建議 Claude 4.6 Sonnet。BOM 反查需要識別模糊輸入（品名關鍵字）並跨表關聯，需要較強的推理能力。

三、兩階段隔離設定
  - 本 Agent 為多單階段（BOM 反查 → 庫存排產交叉），不需 EgentHub 兩階段設定。

四、權限與工作室設定
  - 掛在「生管工作室」，可見群組設為「生管全員」+ 品保主管（緊急應變時需要品保直接查詢）。
  - BOM 表含配方比例，屬研發機密，建議限縮「生管主管＋品保主管＋研發」群組可見。

五、上線前驗測 Checklist
  - [ ] 查詢 RM-BN-01，應回傳 YT-TG-2015-05、YT-TG-8020-05、YT-TP-1520-15、YT-TP-1030-10 四個成品
  - [ ] 用品名關鍵字「氮化硼」查詢，結果應與品號查詢相同
  - [ ] 庫存 > 0 的成品正確標記「⚠ 需隔離庫存」
  - [ ] 查詢 RM-ZZZ-9999，Agent 回「查無此原料品號的 BOM 記錄」

六、上線後監控指標（首 2 週）
  - 查詢次數分布（日常維護 vs 緊急應變）
  - 反查結果是否完整（有無遺漏成品）
  - 使用者回饋：隔離建議是否符合實際操作需求

七、異常處理與回退方案
  - BOM 版次問題：若同一原料品號有多版次，確認 Prompt 中「以最新版次為準」邏輯正確執行；若 ERP BOM 有多層展開，POC 先處理單階，多層 BOM 需另外規劃。
  - 模糊搜尋失敗：若品名關鍵字搜尋結果不準確，建議在 Prompt 加「若關鍵字比對不到，回應請使用品號查詢（如 RM-BN-01）」的引導。
  - 回退：若 Agent 查詢失敗，暫時請生管主管開啟 ERP BOM 展開功能手動查詢，等 Agent 修復後切回。
