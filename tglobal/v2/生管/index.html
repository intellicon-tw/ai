這是 生產管理及資材處 的專屬資料夾，請把補充資料放這裡，方便我們對應每一題。

生管的盤點表在「想問什麼、想要什麼圖表」這件事上寫得相當具體，看得出來是真實痛點直接上來的需求——成本趨勢線圖、料工費圓餅、銷貨排行、庫齡分類、BOM 反查這些都很清楚。值得肯定。但「任務步驟/處理邏輯」欄幾乎空白，這對 Agent 開發是關鍵缺口：Agent 需要知道「先做什麼、再做什麼、遇到例外怎麼判斷」才能寫出真正可用的執行流程。另外，六個場景全部高度依賴 ERP 的結構化資料（成本表、工單明細、現場工時回饋、庫存明細帳、BOM、銷貨明細），要讓 Agent 正確分析，生管需要提供：實際欄位名稱清單、一份去識別化的資料樣本（各表 20–30 筆即可），並先確認哪些可以透過 API 接資料庫、哪些只能定期匯出 CSV——這個邊界決定了 POC 是「即時查詢」還是「定期報表」，兩者的架構差很多，建議優先釐清。以匯出檔模式先跑 POC，確認流程後再評估接 ERP 即時資料。以下每一頁對應一個需求，點進去可看到：需求與痛點 → 開發思路 → 完整提示詞 → 知識庫設計 → 建置流程 → 驗測情境。

資料分析型（庫存管理）

查庫存可用量、庫齡分類、水位狀態要翻好幾個 ERP 畫面，7 個人各自查、各自算，數字常常對不起來。

英騰 BOM 表反查助理

知識庫問答型（結構化查詢）

原料出現異常（如重金屬超標），不知道哪些成品用到這個原料，需要手動翻 BOM 表逐一比對，緊急時根本來不及。

英騰產線生產狀況分析助理

資料分析型（即時戰情）

要知道某產品線上哪些工單未完工、挑邊站有多少工單等待、或查歷史生產的模溫/環境溫度記錄，要同時開好幾個 ERP 畫面，8 個人各查各的。

 A · 生管-7

工時異常檢核與預警助理

資料分析型

生管需逐筆依標準工時人工核對實際工時，耗時且易漏抓異常，每週 30~60 分鐘。

缺料預警與樣品單管控助理

資料分析／勾稽型

ERP 無缺料預警，且樣品單優先領料常導致正貨缺料，每日 5 次以上、每次 30~60 分鐘。

出貨達交預測看板助理

資料分析型

無法快速判斷當日訂單能否準時出貨，每日 3~4 次、每次 30~60 分鐘。

 A · 生管-10

前製程配料換算與排程優化助理

資料分析／排程型

生管要依訂單厚度把數量換算成公斤、依產品分群(0.5/0.8~3.5/4.0以上)拆批，計算複雜、每日 3~4 次、每次 2~3 小時。

 A · 生管-11

後製程排程優化與派工建議助理

資料分析／排程型

後製程工位多、每日工單量大(100張以上)，生管難排程只能靠現場主管人工控單。
