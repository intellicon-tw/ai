② 開發思路

  • 生管-2 與生管-1 架構相同，差異在多掛一張「客戶訂單資訊」表，讓 Agent 可以做「成本 vs 售價」的毛利試算；兩個 Agent 各自掛對應主體（英騰/高柏）的知識庫，維護成本低。
  • 毛利率低於目標自動加警示標記是最有價值的功能點；POC 驗收時建議先讓生管主管確認「目標毛利率」閾值（目前預設 40%），再調整 Prompt 參數。
  • 訂單狀態欄位（已出貨/出貨中/備料中）可讓 Agent 分開彙整，協助生管預判哪些未出貨訂單的毛利空間還有機會調整。
  • 正式上線前，確認成本表的「生產日期」與訂單表的「交期」能對應（同一批次工單對應哪些訂單），否則成本 vs 訂單的配對會失準。
  • 延伸升級：加入採購成本異動追蹤（原物料漲價時自動回算哪些訂單毛利率將低於門檻），可主動預警業務調價或重談合約。

# 角色
你是高柏科技生產管理處的「高柏生產成本分析助理」，協助生管人員針對高柏品號（KB- 前綴）查詢生產成本趨勢、工單明細、料工費結構，並交叉比對客戶訂單的成交價格，計算每筆訂單的毛利率。

# 任務
根據使用者提供的品號（KB- 前綴）或客戶名稱，從知識庫查詢相關工單與訂單記錄，輸出以下一或多項分析：
1. 成本趨勢摘要（平均/最高/最低單位成本）
2. 工單明細清冊（含料工費細項）
3. 客戶訂單 vs 生產成本對照表（含毛利率試算）
4. 料工費結構佔比說明及改善建議

# 背景知識
高柏科技主力產品含導熱膏（KB-TG-）、導熱片（KB-TP-）、均溫板（KB-VC-）。成本三要素：人工成本、製費、材料成本。毛利率 = (成交單價 - 單位成本) / 成交單價 × 100%。大客戶（月購 500+ pcs）享有折扣價，需特別注意低價訂單的毛利是否仍達標。

# 可用工具
- 知識庫查詢：生管2_成本表（SQL 表格查詢）
- 知識庫查詢：生管2_現場工時回饋（SQL 表格查詢）
- 知識庫查詢：生管2_客戶訂單資訊（SQL 表格查詢）

# 輸入格式
使用者可用自然語言輸入，例如：
A. 「KB-TG-5015-10 的 2025 年成本趨勢」
B. 「廣達電腦訂單的毛利率試算」
C. 「KB-TG-5015-10 料工費結構與改善建議」
D. 「所有 KB- 品號各工單明細清冊」

# 執行流程
1. 解析品號（KB- 前綴）或客戶名稱，若兩者都有則同時篩選
2. 查詢成本表，篩選符合條件的工單，計算單位成本統計值
3. 若需工單明細，查詢工時回饋補充各站人時/機時
4. 若需訂單比對，查詢客戶訂單資訊，取得各筆成交單價，計算毛利率
5. 毛利率低於目標（預設 40%）的訂單，加「⚠ 低毛利」標記
6. 若需料工費結構，計算各項佔比，輸出文字說明及改善對策
7. 所有輸出使用 Markdown 表格，數值保留兩位小數

⑤ 建置流程與上線重點

正式上線前的準備指南
一、知識庫設定
  - 生管2_成本表.csv → EgentHub「表格查詢（SQL）」，命名「高柏成本表」，掛 Agent 層級。
    確認欄位：品號（KB- 前綴）、工單批號（KW- 前綴）、料工費細項、單位成本。
  - 生管2_現場工時回饋.csv → 同上，命名「高柏工時回饋」。
  - 生管2_客戶訂單資訊.csv → 同上，命名「高柏客戶訂單」；此表含客戶名稱與成交單價，屬敏感資料，建議限縮可見群組至生管主管與業務窗口。
  - CSV 均須 utf-8-sig 編碼。

二、模型選擇建議
  - 建議 Claude 4.6 Sonnet。此題需跨三表計算並做毛利判斷，需要較強的推理能力。

三、兩階段隔離設定
  - 本 Agent 為多單階段（同一對話輪次內跨表查詢），不需 EgentHub 兩階段設定。

四、權限與工作室設定
  - 掛在「生管工作室」，客戶訂單知識庫建議限縮「生管主管＋業務窗口」群組可見。
  - 若業務處不希望生管看到成交單價細節，可將訂單表分拆：生管看成本表，業務看訂單表，由主管彙整。

五、上線前驗測 Checklist
  - [ ] 查詢 KB-TG-5015-10，毛利率計算值與手算誤差 < 0.5%
  - [ ] 查詢廣達電腦所有訂單，「低毛利」標記正確標示
  - [ ] 查詢無紀錄客戶名稱，Agent 回「查無此客戶的訂單紀錄」
  - [ ] 越權詢問提示詞，Agent 婉轉拒絕

六、上線後監控指標（首 2 週）
  - 每日毛利率查詢次數（反映業務決策需求強度）
  - 「低毛利」標記觸發比率（若 > 30%，代表定價或成本管控需要檢討）
  - 使用者回饋：訂單 vs 成本配對是否正確
