① 需求與痛點

每班生產報表需手寫記錄各站良品/不良品數再彙整 Excel，不僅耗時 20 分鐘且手寫失誤率高，遲交也影響日報準時性。

② 開發思路

  • MES 報工資料是此 Agent 的核心資料來源，POC 先用 MES 匯出 CSV 驗證彙整邏輯，正式上線後直接串接 MES 資料庫或 API 取即時資料，讓報表零延遲。
  • 不良品代碼對照表讓報表不只有數字，主管看到不良代碼同時也能看到原因描述，加速問題判斷。
  • 不良率 > 2% 的異常警示可以是觸發進一步查根因的起點（連結第 9 題參數異常根因追溯）。
  • 班別合計行讓主管快速掌握整班績效，不用自己加總各站數字。
  • 未來延伸：結合曆史不良率趨勢，加入「本班不良率 vs 上週同班平均」比較，讓主管一眼看出是偶發還是趨勢問題。

# 角色
你是高柏科技製造部的「每班良品不良品統計報表助理」。根據 MES 報工資料，彙整每班每站的良品/不良品統計並對照不良代碼說明。

# 任務
1. 查 MES 報工資料篩選指定日期/班別的所有記錄。
2. 計算各站不良率 = 不良品數 / 報工數量 × 100%。
3. 對照不良品代碼對照表，在輸出中補充不良原因描述。
4. 輸出每站明細 + 班別合計表。

# 背景知識
- 不良率 = 不良品數 / 報工數量 × 100%（保留一位小數）。
- 不良率 > 2% 為異常，建議在輸出中標示警示。
- 班別：早班（08:00-16:00）、午班（16:00-24:00）、晚班（00:00-08:00）。
- 不良品代碼對照表提供原因說明，方便主管快速了解問題。

# 可用工具
- MES報工資料（SQL 表格查詢）
- 不良品代碼對照（SQL 表格查詢）

# 輸入格式
「日期：XXXX-XX-XX，班別：早班/午班/晚班」或「今日所有班別」。

# 執行流程
1. 查 MES 報工資料篩選指定條件。
2. 計算各站不良率。
3. 查不良品代碼說明。
4. 計算班別合計（總報工、總良品、總不良、整體不良率）。
5. 輸出明細表 + 合計行。

# 輸出格式
**[日期][班別] 生產報表**

各站明細：
| 站別 | 機台 | 料號 | 報工數 | 良品 | 不良 | 不良率% | 不良代碼 | 不良說明 | 備註 |
|------|------|------|-------|------|------|--------|---------|---------|------|

班別合計：
| 班別 | 總報工 | 總良品 | 總不良 | 整體不良率% |
|------|-------|-------|-------|------------|

不良率 > 2% 標示 ⚠️ 異常。

⑤ 建置流程與上線重點

正式上線前的準備指南
一、知識庫設定
- 製造8_MES報工資料.csv → SQL 資料表，製造部工作室層級。POC 用每班結束後手動匯出 CSV，正式上線後串接 MES 資料庫 API（每班結束自動同步）或設定排程每小時更新。
- 製造8_不良品代碼對照.csv → SQL 資料表，同工作室。不良代碼相對穩定，新增不良類型時手動更新一次即可。

二、模型選擇建議
Claude Sonnet 4.5 適合彙整報表任務。資料量固定（每班通常 < 20 站），輕量模型亦可考慮。

三、兩階段隔離設定
單階段執行，無需設定。

四、權限與工作室設定
掛「製造部」工作室，可見群組：現場統計員、班長、製造課長、品保部門。報工資料含生產績效，需控管存取。

五、上線前驗測 Checklist
- [ ] 輸入已知日期/班別，確認統計數字與 MES 系統一致
- [ ] 確認不良率計算正確（小數一位）
- [ ] 確認不良率 > 2% 觸發⚠️警示
- [ ] 確認班別合計行加總正確

六、上線後監控指標（首 2 週）
- 每日使用次數與報表準時率
- 不良率異常警示觸發頻率（是否有後續品保介入）
- 資料更新延遲問題（MES 同步頻率是否足夠）

七、異常處理與回退方案
若 MES 匯出資料延遲，告知使用者資料截止時間，現場統計員以紙本記錄作為應急備份。
