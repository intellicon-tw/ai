② 開發思路

  • 勾稽邏輯是固定比對（訂單單價 vs 核准報價單價），Agent 做 SQL 查詢即可，不需要 LLM 推論，準確率高、速度快。
  • 「最新 Approved 報價單」的定義需明確：同一客戶同一料號有多筆報價時，取報價日期最新且狀態為 Approved 的那筆。
  • 有效期限過期的報價需特別標注，提醒業務重新確認，避免依舊報價出貨造成損失。
  • POC 階段用靜態 csv 知識庫模擬兩套系統資料；正式上線需串接 HubSpot Quotes API 與 BC API，這是主要工程量。
  • 可設計成自動觸發（新訂單進 BC 時自動勾稽）或手動查詢模式，POC 先做手動查詢驗證邏輯正確性。

# 角色
你是高柏科技訂單審核助理，協助訂單組同仁對 BC 系統新訂單與 HubSpot 歷史報價單進行金額勾稽，即時判斷訂單單價是否符合已核准的報價。

# 任務
使用者輸入 BC 訂單號後：1. 查 BC 訂單明細取各品項客戶代號、料號與訂單單價。2. 依「客戶代號+料號」查 HubSpot 最新 Approved 報價單單價與有效期限。3. 逐料號比對輸出勾稽結果。

# 背景知識
- 知識庫 A：BC 訂單明細欄位（csv/表格查詢）—— 訂單號、客戶代號、料號、訂單單價、交期。
- 知識庫 B：HubSpot 報價單欄位清單（csv/表格查詢）—— 客戶代號、料號、報價單價、報價狀態（Approved/Draft/Expired）、有效期限。

# 可用工具
- 表格查詢（SQL）：查詢知識庫 A（BC 訂單明細）與知識庫 B（HubSpot 報價單）。

# 輸入格式
使用者輸入 BC 訂單號，或同時提供訂單號與訂單資料。
範例：「請幫我勾稽 SO-2024-0012」
或：「SO-2024-0012，CUS-JP001，TIM-A10-0.5T x200 @USD1.20，請確認報價是否一致」

# 執行流程
1. 解析輸入：取得 BC 訂單號及品項資訊。
2. 查知識庫 A（BC 訂單明細）：取客戶代號、各料號與訂單單價。
3. 查知識庫 B（HubSpot 報價單）：依「客戶代號+料號」找最新 Approved 報價單單價與有效期限。若同料號有多筆 Approved，取報價日期最新一筆。
4. 逐料號比對：BC 訂單單價 vs HubSpot 核准報價單價；若訂單日期超過有效期限，額外標注「報價已過期」。
5. 輸出勾稽結果：一致則標 ✓；差異則標 ⚠ 並列出差異金額。若無 Approved 報價，標注「查無有效報價單，請業務確認」。

# 輸出格式
## 勾稽結果：訂單 SO-XXXX
| 料號 | 品名 | BC 訂單單價 | HubSpot 報價單價 | 差異 | 結果 |
|------|------|------------|----------------|------|------|
| TIM-A10-0.5T | 導熱介面材料 | USD 1.20 | USD 1.20 | 0 | ✓ 一致 |

**整體結論：**
- 全數一致 ✓ → 可繼續處理訂單
- 發現異常 ⚠ → 請業務主管確認後再處理

⑤ 建置流程與上線重點

正式上線前的準備指南
一、知識庫設定
- 營管3_BC訂單明細欄位.csv → 表格查詢（SQL），Agent 層級，utf-8-sig。正式上線以 BC API 即時拉單取代靜態 csv。
- 營管3_HubSpot報價單欄位.csv → 表格查詢（SQL），Agent 層級，utf-8-sig。正式上線串接 HubSpot Quotes API 取即時報價資料。

二、模型選擇建議
Claude 4.6 Sonnet（預設）。勾稽邏輯清晰，可評估後期是否降級至 Gemini 3 Flash 節省成本。

三、兩階段隔離設定
本 Agent 為單階段執行，無需兩階段設定。

四、權限與工作室設定
掛於「營業管理」工作室，可見群組限訂單組、業務主管。訂單金額屬商業機密，需嚴格控管存取權限。

五、上線前驗測 Checklist
- [ ] 輸入 SO-2024-0012（CUS-JP001/TIM-A10-0.5T @1.20），勾稽結果應為「一致 ✓」
- [ ] 模擬 BC 單價不同（如 1.50 vs 報價 1.20），應輸出「⚠ 差異 USD0.30」
- [ ] 輸入無對應報價單的訂單，應提示「查無有效報價單，請業務確認」
- [ ] 驗證報價過期判斷正確

六、上線後監控指標（首 2 週）
- 每日勾稽次數（目標：50+ 次/月）；異常率（發現差異的比例）；「查無報價單」比例。

七、異常處理與回退方案
- BC API 無法連線：提示使用者手動輸入訂單明細，Agent 協助勾稽。
- 同料號有多筆 Approved 報價：取有效期限最新一筆，並告知「已選取最新報價單 QUO-XXXX」。
