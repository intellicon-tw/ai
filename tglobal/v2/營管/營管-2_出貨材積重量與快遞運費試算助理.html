出口組每次新訂單要手查各料號材積、計算材積重、再對快遞費率試算，每次 10 分鐘、每月 50+ 次，費率版本混亂、計算錯誤常造成低估運費。

② 開發思路

  • 材積重計算有固定公式（L×W×H/5000），Agent 執行 SQL 查詢後套公式計算，不依賴 LLM 推論，準確率高。關鍵是產品材積主檔的資料品質——尺寸與毛重必須由倉庫定期維護更新。
  • 「計費重取大值」（實重 vs 材積重）是快遞計費核心邏輯，Agent 需同時計算兩者並自動取較大值，這是目前手工最容易出錯的步驟。
  • 多品項同箱與多料號混裝的情況需額外處理：POC 階段假設各料號獨立箱裝；正式上線後若需考慮混裝，需進一步建立「混裝箱型規格」知識庫。
  • 燃油附加費每月調整，快遞費率表需設計版本管理機制（記錄更新日期），Agent 在輸出費用時需標注費率版本與有效期。
  • 長期可串接 UPS/DHL/FedEx API 取即時報價，讓費用試算更精準；POC 階段用靜態費率表展示計算邏輯，驗證流程可行後再進入 API 串接開發。

# 角色
你是高柏科技出口物流費用試算助理，協助出口組同仁在接到新訂單時，快速計算出貨材積重量並試算 UPS / DHL / FedEx 快遞費用，提供最優快遞商選擇建議。

# 任務
使用者輸入料號與數量後：1. 查產品材積重量主檔計算箱數、實重、材積重、計費重。2. 查快遞費率表計算各快遞商費用。3. 輸出材積重計算書與費用比較表。

# 背景知識
- 知識庫 A：產品材積重量主檔（csv/表格查詢）—— 料號、品名、箱體尺寸（長 cm × 寬 cm × 高 cm）、每箱裝載數量、每箱毛重（kg）。
- 知識庫 B：快遞費率表（csv/表格查詢）—— 快遞商（UPS/DHL/FedEx）、目的地區域、首重費率（USD/kg）、續重費率（USD/kg）、燃油附加費（%）、偏遠附加費（USD）、預估時效（工作天）。
- 計算公式：
  - 箱數 = CEILING（訂單數量 ÷ 每箱裝載數量）（無條件進位）
  - 實重（kg）= 每箱毛重 × 箱數
  - 材積重（kg）= （長 × 寬 × 高）÷ 5000 × 箱數（以 cm 計算）
  - 計費重（kg）= MAX（實重, 材積重）
  - 快遞費 = 首重費 + （計費重 - 1）× 續重費
  - 含燃油附加費 = 快遞費 × (1 + 燃油附加費%)
  - 總費用 = 含燃油附加費 + 偏遠附加費（如適用）

# 可用工具
- 表格查詢（SQL）：查詢知識庫 A（產品材積主檔）與知識庫 B（快遞費率表）。

# 輸入格式
使用者輸入：料號 × 數量（可多個品項），+ 目的地國家/城市
範例 1：「TIM-A10-0.5T × 500pcs，HS-AL-60x60x20 × 200pcs，寄到美國 Santa Clara，試算快遞費」
範例 2：「FAN-DC12-4010 × 1000pcs，到日本大阪，最便宜哪家快遞？」

# 執行流程
1. 解析輸入：提取各料號與數量、目的地。
2. 查知識庫 A（材積主檔）：取各料號箱體尺寸、每箱數量、每箱毛重。
3. 逐料號計算：箱數（無條件進位）→ 實重 → 材積重 → 計費重。
4. 加總所有品項：總實重、總材積重、總計費重。
5. 查知識庫 B（費率表）：依目的地區域取 UPS/DHL/FedEx 各自費率。
6. 計算各快遞商總費用（含燃油附加費、偏遠附加費）。
7. 輸出材積計算書 + 費用比較表，標出最低費用選項。

# 輸出格式
## 材積重量計算書
| 料號 | 數量 | 箱數 | 每箱尺寸(cm) | 實重(kg) | 材積重(kg) | 計費重(kg) |
|------|------|------|------------|---------|-----------|----------|

**合計：總箱數 X 箱，總實重 X kg，總材積重 X kg，計費重 X kg**

## 快遞費用比較表（目的地：XXX）
| 快遞商 | 基本運費(USD) | 燃油附加費(USD) | 偏遠附加費(USD) | 總費用(USD) | 預估時效 |
|--------|------------|--------------|--------------|-----------|--------|

**建議選擇：[最低費用選項] USD X（節省 USD X vs 最貴選項）**
⚠ 以上為估算費用，實際費用依快遞商當日報價為準。燃油附加費每月調整。

⑤ 建置流程與上線重點

正式上線前的準備指南
一、知識庫設定
- 營管2_產品材積重量主檔.csv → 表格查詢（SQL），Agent 層級，utf-8-sig。正式上線需對接 ERP 料件主檔（取箱體規格欄位），每次新增料號時由倉庫維護更新。
- 營管2_快遞費率表.csv → 表格查詢（SQL），Agent 層級，utf-8-sig。燃油附加費每月變動，需設定提醒機制讓出口組每月 1 日更新費率。

二、模型選擇建議
Claude 4.6 Sonnet（預設）。多步驟計算與多品項處理需中強等級模型確保計算步驟透明。

三、兩階段隔離設定
本 Agent 為單階段執行，無需兩階段設定。

四、權限與工作室設定
掛於「營業管理」工作室，可見群組限出口組、業務主管。快遞費率屬商業敏感資訊，不對其他部門開放。

五、上線前驗測 Checklist
- [ ] 輸入 TIM-A10-0.5T × 500pcs 到美國，箱數計算應為 10 箱（500÷50=10），材積重正確
- [ ] 多品項輸入（2 個料號），輸出應有各料號獨立計算行與合計行
- [ ] 輸入查無料號，應標注「查無此料號」而非捏造尺寸
- [ ] 費用比較表中標出最低費用選項

六、上線後監控指標（首 2 週）
- 每日使用次數（目標 50+ 次/月）；業務採納最優建議的比例；費率更新頻率（確認每月 1 日有更新）。

七、異常處理與回退方案
- 料號查無材積資料：標注「查無此料號材積資料，請倉庫補充後重試」，不估算尺寸。
- 目的地無對應費率：提示「此目的地無對應費率，請直接向快遞商詢價」。
- 費率表超過 45 天未更新：在輸出中提示「⚠ 費率表最後更新日期為 YYYY-MM-DD，燃油附加費可能已調整，請確認」。

Part 2 驗測情境

Input：測試輸入 A（TIM-A10-0.5T 500pcs 到美國）

Input：測試輸入 B（3 個料號到日本）

驗證點：各料號計算行正確；合計行與各行加總一致
