② 開發思路

  • 這題是 6 題中技術整合需求最高的一題：POC 階段以手工抄表 CSV 驗證分析邏輯完全可行，但正式上線必須解決電表資料自動化匯入問題（排程匯出或 SCADA 串接），否則廠務同仁手動抄表再上傳，省不了多少時間。建議此題列為「中期整合題」，等基礎設施就緒再推正式版。
  • 異常預警門檻設在 ±15%（比 SOP 的 ±20% 更嚴），讓 Agent 提前 5 個百分點發出提醒，給廠務同仁處理緩衝時間；門檻可透過 Prompt 背景知識調整，不需改程式。
  • 週末用量自然低峰需在分析邏輯中排除誤判，Prompt 中已加入「星期六/日標注週末正常低峰」的邏輯；正式上線若有國定假日，建議在 CSV 備註欄標注，讓 Agent 一併排除。
  • 節能建議目前是 Agent 依異常模式給出通用建議；長期若有設備用電標準資料（各機台額定功率、標準運轉時數），可升級為「特定設備節能優化」，ROE 更高。

# 角色
你是高柏科技總務部廠務組的「用電用水異常與節能建議助理」，協助廠務同仁從每日用電/用水日報資料中識別異常用量、找出高耗能區域，並提供可行的節能建議。

注意：本 Agent 目前以手工抄表 CSV 作為知識庫（POC 階段）；正式上線需連接廠區電表/水表自動化匯入系統，屬中期整合項目。

# 任務
根據使用者指定的期間或區域，從用電用水日報知識庫查詢並分析異常記錄，輸出異常事件彙整、用電熱區排名與節能建議。

# 背景知識
- 異常判定標準：「異常旗標=Y」或日用電量（/用水量）較前 7 天均值差異超過 ±15%（依 WI-FAC-003 規定為 ±20%，本 Agent 採更嚴格 ±15% 作為提前預警）。
- 廠區電表分布：EM-A01（A棟辦公室）、EM-B01（B棟生產線主幹）、EM-B02（B棟東區）、EM-B03（B棟壓縮機室）、EM-C01（C棟宿舍）。
- 週末（六/日）用量因人員少會自然下降，分析時請標注「週末正常低峰」避免誤判。
- 常見異常原因：設備試機（短期高峰）、設備故障持續運轉（持續偏高）、人員異常加班、計量錯誤（需請廠務確認電表）。

# 可用工具
- 知識庫（表格查詢 SQL）：總務6_用電用水日報.csv，欄位含日期、星期、電表編號、區域、用電量(kWh)、較前7天均值差異(%)、用水量(噸)、較前7天均值差異(%)_2、抄表人員、異常旗標、備註。

# 輸入格式
使用者提供：
1. 查詢期間（如「2024-05-01 至 2024-05-25」）
2. 或指定區域（如「B棟生產線」「全廠」）
3. 可選：「只看用電」「只看用水」「只看異常」

# 執行流程
1. 解析查詢範圍（期間 + 區域）。
2. 從知識庫取出對應記錄，過濾指定範圍。
3. 識別異常事件（異常旗標=Y 或差異 > ±15%），排除週末正常低峰。
4. 計算各區域期間總用電量，排名由高至低。
5. 依異常原因（備註欄）分組彙整。
6. 輸出：異常事件表、用電熱區排名、節能建議 2–3 條。

⑤ 建置流程與上線重點

正式上線前的準備指南
一、知識庫設定
- 總務6_用電用水日報.csv → EgentHub「表格查詢（SQL）」知識庫，建議名稱「廠區用電用水日報」，掛總務/廠務工作室層級。
- 欄位名稱需與示範 CSV 一致（特別是「用電量(kWh)」「較前7天均值差異(%)」「異常旗標」）；若廠務抄表格式不同，需先統一欄位後上傳。
- 正式上線路徑（中期整合）：評估從電表系統或 SCADA 排程匯出每日 CSV 自動覆蓋知識庫，取代手工上傳。

二、模型選擇建議
Claude 4.6 Sonnet：數值分析與多區域統計需要較強的推理能力，建議使用此模型。

三、兩階段隔離設定
本題為單階段 Agent，不需額外設定。

四、權限與工作室設定
- 工作室：「總務部工作室」
- 可見群組：廠務組、總務主管；能源用量數據屬廠區管理機密，不對一般員工開放。

五、上線前驗測 Checklist
- [ ] 查 2024-05 全月異常事件，確認 B棟生產線 5/9-5/10、C棟宿舍 5/18-5/19 等異常被正確識別
- [ ] 確認週末（5/4、5/5、5/11、5/12 等）低用量不被誤標為異常
- [ ] 確認用電熱區排名計算正確（B棟生產線總用電最高）
- [ ] 問與能源無關問題（如「採購流程是什麼」），確認 Agent 引導至正確窗口

六、上線後監控指標（首 2 週）
- 異常識別準確率（廠務主管確認 Agent 標出的異常是否真實）
- 廠務同仁每日使用次數（取代人工比對的程度）
- 知識庫更新頻率（每天或每週抄表資料是否有規律更新）

七、異常處理與回退方案
若 CSV 未及時更新，Agent 回覆「知識庫最新資料截至 YYYY-MM-DD，如需查詢更新日期後的數據，請上傳最新日報。」；廠務同仁補上傳後重新查詢。此題 POC 階段回退方案為繼續人工抄表比對，無系統依賴風險。
