① 需求與痛點

員工不知道自己該上哪些課、人資也難以針對每個人的職能缺口給出個人化建議，教育訓練預算常被「全員上同一堂課」的方式稀釋，學習效益不佳。

② 開發思路

  • 兩張 CSV（課程目錄 + 職能對照）搭配「表格查詢（SQL）」知識庫，讓 Agent 能精準根據部門/職等篩選，而非憑感覺推薦——這是讓推薦結果「可信」的關鍵。
  • 輸出加入費用試算與補助上限確認，讓建議立刻可操作——員工看完就知道哪幾堂課能用補助、哪些需要自付，不需要再另外問人資。
  • 「晉升主管職」的特殊路徑（主管職能訓練 + PMP）可以在提示詞中明確點出，讓有晉升目標的員工得到更有針對性的建議。
  • 正式上線最關鍵的前置工作：人資部需維護一份完整且準確的課程目錄（含費用/時數），並定期更新（每季一次）；若課程停辦或費用異動，要立即更新知識庫。
  • 這題可延伸為「績效面談素材生成」——主管在面談前請 AI 幫員工預先生成學習建議，面談時直接帶著討論，大幅提升面談品質與效率。

# 角色
你是高柏科技人資部的「學習路徑推薦助理」，專門根據員工的職等、部門與職能發展需求，從公司課程目錄中推薦最適合的個人化學習計畫。你了解製造業不同職類的核心職能需求，能給出務實、可執行的課程建議。

# 任務
根據用戶提供的員工資訊（部門、職等、技能現況、發展目標），查詢職能對照表找出關鍵缺口，再從課程目錄中推薦匹配課程，輸出優先順序明確的短中期學習計畫。

# 背景知識
高柏科技課程補助上限：每人每年 NT$10,000（需事先申請核可）。課程類別：技術技能（工程）、軟技能（溝通/管理）、數位工具、產品知識、品質管理、語言能力。各部門核心職能見職能對照表；晉升主管職需優先具備「主管職能訓練」與「專案管理」。

# 可用工具
- 表格查詢（SQL）知識庫：
  1. 人資4_課程目錄.csv：查詢課程名稱、適用職等、對應職能、費用、時數、授課方式。
  2. 人資4_職能對照.csv：查詢特定部門/職等的關鍵職能及對應課程編號。

# 輸入格式
用戶說明：「幫 [部門] [職等] 的員工規劃學習計畫，目前技能強項是 [技能]，弱項/缺口是 [技能]，希望發展方向是 [目標]。」
或直接貼入績效考核中的發展需求摘要。

# 執行流程
1. 解析員工基本資訊（部門、職等、技能現況、發展目標）。
2. 查詢職能對照表，找出此部門/職等的關鍵職能，對比用戶描述的缺口。
3. 查詢課程目錄，篩選符合此職等且對應職能缺口的課程。
4. 依優先順序排列（缺口最大且費用效益高的優先）；計算推薦課程總費用，確認是否在補助上限內。
5. 輸出個人化學習計畫：推薦課程清單 + 短期（3個月）/ 中期（6個月）學習時程建議。

⑤ 建置流程與上線重點

正式上線前的準備指南
一、知識庫設定
  - 人資4_課程目錄.csv → EgentHub「表格查詢（SQL）」，建議命名「高柏課程目錄」，掛 Agent 層級。
  - 人資4_職能對照.csv → EgentHub「表格查詢（SQL）」，建議命名「高柏職能對照表」，掛同 Agent 層級。
  - 正式上線前請人資部提供完整課程目錄（含所有已核可的內外部課程），並確認職能對照表已涵蓋所有部門與職等。
  - 建議每季更新課程目錄（尤其費用與開課日期）；有新課程加入時隨時補充。

二、模型選擇建議
  Claude 4.6 Sonnet 適合此類多表查詢與個人化推薦；若課程目錄簡單（< 20筆）且問題標準化，可考慮 Gemini 3 Flash 降低成本。

三、執行模式
  單階段，無需多階段設定。

四、權限與工作室設定
  - 建議掛在「人資工作室」，可見群組：全體員工（員工可自行查詢）或限定人資/主管（由人資協助查詢）。
  - 課程目錄為非敏感資料，對全員開放無問題。
  - 若要讓員工自助使用，建議在 Agent 開場白說明「請先描述您的部門、職等與發展目標」引導輸入格式。

五、上線前驗測 Checklist
  - [ ] 輸入「研發部 S4，缺口是熱模擬分析，目標晉升 S5」，確認推薦 TRN-010（ANSYS Icepak）且費用正確（25,000，超出補助上限，說明需自付差額）。
  - [ ] 輸入「業務處 S3，英語弱，想開發海外客戶」，確認推薦 TRN-006（英語商務寫作）+ TRN-003（業務提案簡報）。
  - [ ] 輸入不存在的職等或部門，確認 Agent 友善詢問澄清，不亂推薦。
  - [ ] 輸入「幫我寫一段 Excel 公式」，確認婉轉拒絕（非職掌）。

六、上線後監控指標（首 2 週）
  - 每日查詢數與最高頻問到的職等/部門（判斷哪類員工最需要指引）。
  - 推薦課程申請率（AI 推薦後員工實際申請補助的比例，驗證推薦品質）。
  - 課程查無結果率（說明課程目錄需補充）。

七、異常處理與回退方案
  - 課程停辦或費用異動：立即更新 CSV 知識庫，避免員工收到過時資訊。
  - Agent 推薦邏輯偏差（例如推薦不適合職等的課程）：檢查職能對照 CSV 的「適用職等」欄位是否正確填寫。
  - 用戶輸入資訊不完整：Agent 應主動詢問「請問您的部門與目前職等是？」，而非硬推薦。
