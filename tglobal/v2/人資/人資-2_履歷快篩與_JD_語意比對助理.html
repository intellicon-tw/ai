① 需求與痛點

招募旺季一個職缺收到數十份履歷，人資需逐份閱讀比對 JD 要求，篩選時間長、標準不一致；有時找到的 JD 與履歷格式落差大，比對效率更低。

② 開發思路

  • 兩階段設計是關鍵：第一階段讓 Claude 讀 PDF 並結構化，第二階段才做比對，避免單一對話 Context 過長導致比對不準確。
  • JD 知識庫（CSV）讓人資不需每次貼入整份 JD，只要輸入職缺編號就能自動帶入條件，加快操作流程；POC 階段也接受直接貼 JD 文字。
  • 輸出格式刻意設計成「每項條件單獨標記符合度」，而非只給總分——這讓人資清楚知道哪裡有缺口，決策更有依據。
  • 「電話面試深問方向」是這個 Agent 的亮點功能：不只篩選，還幫人資準備面試問題，直接提升面試品質。
  • 正式上線後需留意：履歷 PDF 格式差異大（掃描圖片 vs 文字 PDF），若遇到掃描圖片版，需先做 OCR 前處理再上傳；建議在上線指引中告知使用者優先收取文字 PDF 版本履歷。

# 角色
你是高柏科技人資部的「履歷快篩助理」，專門協助人資招募人員分析候選人履歷，並與職缺 JD 進行結構化比對，輸出匹配度報告。你熟悉製造業（散熱材料/電子材料/精密製造）的職位要求，能準確識別技術技能、語言能力與管理經驗的符合程度。

# 任務
執行兩階段處理：第一階段解析上傳的履歷 PDF，提取關鍵結構化資訊；第二階段比對指定職缺 JD，產出匹配度評分報告，並給出是否建議電話面試的明確推薦。

# 背景知識
高柏科技主要招募職類：機構工程師（需熟 SolidWorks/ProE）、業務工程師（需英語能力＋散熱產品知識）、製程工程師（需 SPC/MES）、品保工程師（需 ISO 稽核）、人資/財務/MIS 等支援職。評估重點：相關產業年資、核心技術技能符合度、語言能力、前公司規模與業態是否相近。

# 可用工具
- 第一階段：讀取用戶上傳的履歷 PDF（EgentHub 附件解析功能）
- 第二階段：表格查詢知識庫（職缺JD.csv，含職缺必要/加分條件），或接受用戶直接貼入 JD 文字

# 輸入格式
第一階段：「請分析這份履歷」＋上傳 PDF 附件
第二階段：「請比對職缺 [職缺編號 或 職缺名稱]」，或直接貼入 JD 文字

# 執行流程
第一階段——履歷解析：
1. 讀取上傳的履歷 PDF，提取：姓名（若提供）、最高學歷/科系、總工作年資、最近職位/公司/產業、核心技能清單、語言能力、證照/資格、自我介紹重點。
2. 輸出「履歷摘要卡」（結構化，方便下一步比對）。

第二階段——JD 比對：
3. 載入指定職缺 JD（從知識庫查詢或用戶貼入）。
4. 逐項比對 JD 的必要條件與加分條件，每項標記「符合 / 部分符合 / 不符合」。
5. 根據符合比例與重要性加權，給出整體匹配分數（1–10分，附評分說明）。
6. 列出候選人 2–3 個值得在電話面試中深問的問題。
7. 給出「建議電話面試 / 暫不推進」的明確結論與理由。

⑤ 建置流程與上線重點

正式上線前的準備指南
一、知識庫設定
  - 人資2_職缺JD.csv → EgentHub「表格查詢（SQL）」，建議命名「高柏職缺JD」，掛 Agent 層級。欄位含：職缺編號、職缺名稱、必要技能、加分技能、JD摘要。正式上線換成實際所有職缺 JD 資料。
  - 人資2_履歷樣本.csv → 僅作 POC 測試用，正式作業時用戶直接上傳 PDF 履歷，不需要此知識庫。
  - 注意：履歷 PDF 需為文字版（非掃描圖片），掃描版需先 OCR 處理。

二、模型選擇建議
  Claude 4.6 Sonnet — 此題需要閱讀理解 PDF 長文件並進行語意比對，需具備強力閱讀能力的模型，不建議換成輕量模型。

三、兩階段隔離設定
  EgentHub 兩階段設定：第一階段 Prompt 專注「履歷解析 → 輸出摘要卡」；第二階段 Prompt 接收第一階段輸出 + JD 條件，專注「比對 → 評分 → 推薦」。兩個 Agent 之間以對話串連，或在同一 Agent 的執行流程中用步驟分隔。POC 階段建議在同一 Agent 執行，說明清楚兩個指令即可。

四、權限與工作室設定
  - 建議掛在「人資工作室」下，可見群組：人資部（僅限）。
  - 不對製造/業務同仁開放——履歷含候選人個資，僅招募人員可用。
  - 資料敏感度：高（個資），需確保 EgentHub 知識庫與對話記錄設定僅人資可見。

五、上線前驗測 Checklist
  - [ ] 上傳一份文字 PDF 履歷，確認第一階段正確提取學歷/年資/技能。
  - [ ] 輸入職缺編號，確認 JD 從知識庫正確載入必要條件。
  - [ ] 比對結果每項有明確「符合/部分符合/不符合」標記，整體分數 1-10 且有說明。
  - [ ] 上傳空白 PDF 或圖片版 PDF，確認 Agent 友善提示「請上傳文字版 PDF」，而非亂輸出。

六、上線後監控指標（首 2 週）
  - 每週履歷處理數量與平均處理時間（目標：每份 < 3 分鐘）。
  - 人資採用率：AI 推薦「電話面試」候選人，實際進行電話面試的比例（驗證推薦品質）。
  - JD 查無結果率（若職缺編號查不到，表示 JD 知識庫需補充）。

七、異常處理與回退方案
  - PDF 無法解析（掃描圖片）：Agent 提示請轉換為文字版，人資另外使用 Adobe 或 ABBYY 轉換後重試。
  - JD 知識庫查無職缺：人資直接在對話框貼入 JD 文字，跳過知識庫查詢，仍可正常比對。
  - 比對結果明顯偏誤：立即在對話中提出修正，並將 case 記錄供後續優化提示詞參考。
