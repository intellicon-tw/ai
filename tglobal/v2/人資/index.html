部門講評：人資交的盤點表格局完整、有遠見——五個主題（智慧行政、精準招募、留任預警、個人化學習、數據化績效）清楚點出 HR 數位轉型的關鍵方向，每個主題也都附上了 KPI 與導入時程，可以看出整個部門對 AI 的期待是認真的。不過目前內容還停在策略層：主題描述的是「想達到什麼效果」，尚未拆到「一個人、每天或每週、花多少時間、輸入什麼資料、輸出什麼格式」這種任務級。要真正開發，每個主題還需要進一步細化——例如：員工手冊和請假辦法的完整原文電子檔、職缺 JD 與歷史履歷的去識別化樣本、出勤/績效/問卷的系統欄位定義、以及實際的判斷規則（例如出勤低於多少 % 算風險訊號）。以下每一頁以最可落地的 POC 為主，帶大家看從題目到提示詞到驗測的完整開發過程。以下每一頁對應一個需求，點進去可看到：需求與痛點 → 開發思路 → 完整提示詞 → 知識庫設計 → 建置流程 → 驗測情境。

B · 人資-1主題待細化

員工手冊與制度問答 Chatbot

知識庫 RAG 問答型

員工對請假天數、福利申請、規章規定有疑問，反覆找人資詢問，人資每天花大量時間回答重複問題；新人尤其容易問錯人、拿到過期說法。

 B · 人資-2主題待細化

履歷快篩與 JD 語意比對助理

文件生成型（PDF 解析 + 結構化輸出）

招募旺季一個職缺收到數十份履歷，人資需逐份閱讀比對 JD 要求，篩選時間長、標準不一致；有時找到的 JD 與履歷格式落差大，比對效率更低。

 B · 人資-3主題待細化

離職風險訊號彙整助理

資料分析型（多維指標彙整 + 風險清單）

人資難以提前掌握哪些員工有潛在離職風險，往往等到遞辭呈才知道；想分析出勤、績效、問卷等數據找出早期訊號，但資料分散、沒有統一彙整工具。

 B · 人資-4主題待細化

個人化學習路徑推薦助理

知識庫問答型（職能缺口 + 課程推薦）

員工不知道自己該上哪些課、人資也難以針對每個人的職能缺口給出個人化建議，教育訓練預算常被「全員上同一堂課」的方式稀釋，學習效益不佳。

 B · 人資-5主題待細化

績效洞察報告生成助理

報表整理型（績效數據彙整 + 輔導報告生成）

主管每次績效考核前要整理部屬的考核分數、專案成果、KPI 達成情形，再撰寫面談前的輔導摘要，耗時且格式不一；人資也難以快速彙整全部門的績效洞察供高層參考。
