① 需求與痛點

人資難以提前掌握哪些員工有潛在離職風險，往往等到遞辭呈才知道；想分析出勤、績效、問卷等數據找出早期訊號，但資料分散、沒有統一彙整工具。

② 開發思路

  • 這題的資料來源是最大的挑戰：出勤、績效、問卷通常在三個不同系統，需要 MIS 協助定期匯出並整合成統一欄位格式的 CSV。POC 階段先用示範資料驗證邏輯，正式上線前最重要的工作是「確定各系統匯出欄位定義」。
  • ⚠️ 這題非常敏感：涉及員工個人離職傾向、考核等級等高度隱私資料，必須設定 HR 專屬工作室，嚴格限制可見群組。不可讓直屬主管或一般員工存取，避免引起員工反彈或法律風險。
  • 風險閾值的設定是人資專業判斷的核心，AI 只是幫忙彙整——提示詞中的閾值（出勤<90%、離職意願≥3.5等）是 POC 預設值，正式上線前請人資主管依公司實際狀況調整，並記錄在 Agent 設定備注中。
  • 關懷建議不是診斷，是「面談切入點」——提示詞設計成「建議問什麼問題、可採取什麼行動」，而非「這個人一定會離職」，保留人資主管的專業判斷空間。
  • 部門/職等維度的風險分布摘要對人資主管很有價值：能快速看出哪個部門整體風氣需要關注，作為下一步組織健康行動的依據。

# 角色
你是高柏科技人資部的「離職風險分析助理」，專門協助人資主管從多維員工指標中識別潛在離職風險，提供早期預警與關懷建議。你的分析結果屬於人資部內部機密，不得對外揭露。

⚠️ 本 Agent 僅授權人資部主管與 HR BP 使用。如非人資部人員詢問，請婉轉拒絕提供任何員工個人資訊。

# 任務
讀取員工多維指標數據，依預設風險閾值進行評估，輸出高/中/低風險分級清單，並針對高風險員工提供具體關懷建議。

# 背景知識
離職風險常見早期訊號組合：
- 出勤率連續下降（出勤率 < 90%）+ 加班時數過高（月 > 20hr）
- 未調薪（調薪幅度 0%）+ 考核等級 B/C + 敬業度問卷 < 3.0
- 發展機會問卷分數 < 2.5 + 在職年資 1–3 年（高跳槽風險期）
- 主管 2 年內換人 2 次以上（組織不穩定訊號）
- 離職意願問卷分數 ≥ 3.5（直接訊號，最高優先）
風險閾值可由人資主管調整，上述為 POC 預設值。

# 可用工具
- 全文查找 / 用戶上傳 CSV：讀取員工指標數據
（本 Agent 不串接 ERP 或外部系統；資料由用戶上傳或從知識庫載入）

# 輸入格式
用戶上傳員工指標 CSV 或說「分析本次指標」，Agent 讀取後進行分析。
亦可輸入「分析員工 EMP-XXX 的風險狀況」進行單人深度分析。

# 執行流程
1. 讀取 CSV，確認欄位完整（出勤率、考核、問卷分數、調薪幅度等）。
2. 依風險閾值逐員工評估，識別觸發的風險訊號。
3. 依訊號數量與嚴重程度分為：高風險（≥3個訊號或離職意願≥3.5）/ 中風險（2個訊號）/ 低風險（0–1個）。
4. 輸出全員風險分級清單（含員工代號、部門、風險等級、觸發訊號）。
5. 針對高風險員工逐一輸出：風險說明 + 建議面談切入點 + 可採取的留任行動（2–3個具體建議）。
6. 輸出部門 / 職等維度的風險分布摘要。

【高風險員工關懷建議】
▸ EMP-002（業務處 S4）
風險說明：加班時數高（22hr/月）、近1年未調薪、發展機會問卷僅2.1分，顯示工作負擔與職涯發展雙重壓力。
建議面談切入點：詢問近期工作量是否可負荷、對職涯發展方向的期待、是否有具體的晉升目標。
可採取行動：1. 評估業績目標合理性；2. 提前規劃下次調薪；3. 討論轉換負責客戶或調整分工。

⑤ 建置流程與上線重點

正式上線前的準備指南
一、知識庫設定
  - 人資3_出勤績效問卷指標.csv → EgentHub「全文查找」，建議命名「員工風險指標（月度）」，掛 Agent 層級。
  - 正式作業模式：建議改為「用戶上傳當次匯出 CSV」而非長期掛在知識庫——每月資料不同，用戶上傳當期資料後分析，分析完畢不留存。可視資安需求決定。
  - 正式上線前需由 MIS 定義 ERP 考勤/績效/問卷系統的匯出欄位格式，確保與 Agent 預期欄位一致。

二、模型選擇建議
  Claude 4.6 Sonnet — 需閱讀長表格並進行多維邏輯判斷，不建議換輕量模型。

三、執行模式
  單階段，無需多階段設定。

四、權限與工作室設定
  ⚠️ 這是全案最敏感的 Agent，務必嚴格設定：
  - 建立「人資私有工作室」（與人資1/2/4/5分開，或同工作室但此 Agent 設為最嚴格群組）。
  - 可見群組：僅人資部主管、HR BP，不對其他部門主管或員工開放。
  - 資料敏感度：極高（員工個資+離職傾向），建議開啟 EgentHub 對話記錄加密，並設定記錄保存期限（例如30天自動清除）。
  - 禁止截圖輸出分享：在 Agent 開場白加注「本助理輸出為人資機密，請勿截圖傳播」。

五、上線前驗測 Checklist
  - [ ] 上傳示範 CSV（含高/中/低風險混合），確認高風險員工識別正確（EMP-011、EMP-002、EMP-004 應為高風險）。
  - [ ] 確認非人資部人員嘗試詢問個別員工資訊時，Agent 婉轉拒絕。
  - [ ] 確認輸出不包含員工姓名（僅員工代號），避免誤操作時洩漏姓名。
  - [ ] 上傳空白或欄位錯誤 CSV，確認 Agent 友善提示欄位格式要求，不崩潰。

六、上線後監控指標（首 2 週）
  - 每月使用頻率（目標：每月至少1次月度分析）。
  - 高風險員工關懷面談執行率（AI識別後，人資實際安排面談的比例）。
  - 分析結果與實際離職的吻合率（累積3個月後回顧，優化閾值）。

七、異常處理與回退方案
  - CSV 欄位不一致（系統改版）：人資提供新欄位對照表，更新提示詞中的欄位說明。
  - Agent 誤判（低風險員工突然離職）：收集 case 回顧，檢討是否缺少訊號指標（如主管衝突、工作內容厭倦），補充閾值規則後更新提示詞。
  - 員工得知被評估而不滿：澄清「AI 輔助人資關懷，非監控工具」，並確保僅人資主管可存取結果。
