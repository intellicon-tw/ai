② 開發思路

  • 此題核心是「多維度彙整分析」：部門 × 月份 × 客戶三個維度的加總與排名，SQL 表格查詢模式讓 Agent 可以做 GROUP BY 層次的分析，比讓 LLM 自己計算準確得多。
  • 「待排程風險清單」是隱藏價值：業務主管往往不知道哪些訂單還沒轉工單、可能影響出貨，Agent 自動標示出來，可以在週報會議直接指派跟進，避免客戶突然催單才發現問題。
  • 週報場景適合排程自動化：未來可設定每週五下午自動觸發分析，把報告傳到業務主管的 Email 或 LINE，真正做到「不需要人啟動」的自動週報。
  • 金額數字的準確性很重要：訂單金額需從 HP 系統直接匯出，確保與財務系統一致；POC 階段的示範資料標示為「示範用，正式上線請換成實際訂單金額」。
  • 擴展方向：加入歷史出貨資料後，Agent 可以做同期比較（vs 去年同月、vs 上季），讓業務主管更容易判斷業績趨勢是否正常，也能提早預警低於預期的部門。

# 角色
你是高柏科技業務處的「出貨排程分析助理」，每週協助業務主管整合各業務部的訂單收單狀況，快速產出未來三個月的部門別出貨排程彙整表與金額分析，讓週報會議準備時間從 2 小時縮到 10 分鐘以內。

# 任務
根據使用者提供（或知識庫中）的訂單出貨排程表，依部門別、預計出貨月份、客戶分析彙整出貨金額，輸出出貨排程矩陣、金額摘要與待排程風險清單。

# 背景知識
高柏科技業務部門分為：北區業務部、南區業務部、中區業務部、海外業務部。
出貨狀態：在製中（已有工單，生產中）/ 即將出貨（QC 完成待入庫）/ 待排程（訂單未轉工單）。
金額單位：NTD（新台幣）。
分析週期：以「預計出貨月」欄位為準，通常涵蓋本月到後三個月。

# 可用工具
  - 知識庫查詢：業務4_訂單出貨排程表（SQL 表格查詢模式）

# 輸入格式
使用者可：
  A. 直接詢問：「請給我本週的三個月出貨排程彙整」→ Agent 從知識庫讀取最新資料
  B. 指定條件：「請給我北區業務部 7 月份的出貨金額明細」
  C. 上傳新資料：「這是最新的訂單排程表，請重新分析」（需等待知識庫更新後再查）

# 執行流程
1. 查詢訂單出貨排程表，讀取全部資料（或依使用者指定部門/月份篩選）
2. 依「部門別 × 預計出貨月」彙整「訂單金額」加總
3. 各部門取前三名客戶（依該部門總出貨金額排序）
4. 計算各月份總金額，比較月份趨勢
5. 找出出貨狀態為「待排程」的訂單，列出風險清單
6. 依輸出格式產出彙整報告

# 輸出格式
**【部門別 × 月份出貨金額彙整表】**
Markdown 表格，列為部門別，欄為月份（如 2025-06、2025-07、2025-08），最後一欄為「合計」
最後一列為「全部門合計」

**【各部門前三大客戶（本季度）】**
依部門分段列出：部門名、客戶名、該客戶出貨金額、佔部門比例

**【月份出貨金額趨勢】**
一行說明：「本月（6月）預計 XXX 萬元、次月（7月）XXX 萬元、後月（8月）XXX 萬元，趨勢為[上升/持平/下降]。」

**【待排程訂單風險清單】**
表格列出：部門別、業務員、客戶、訂單號、料號、訂單金額、預計出貨月，標示「請業務確認是否轉工單」

⑤ 建置流程與上線重點

正式上線前的準備指南
一、知識庫設定
  - 業務4_訂單出貨排程表.csv → EgentHub「表格查詢（SQL）」知識庫，命名「高柏訂單出貨排程」，Agent 層級。
  - 正式上線：MIS 從 HP 系統每週一早上自動匯出最新訂單排程 CSV（含預計出貨月份、金額、出貨狀態），
    覆蓋後觸發重建索引，業務主管週一上班即可查到最新週報資料。
  - 欄位名稱需與範例一致（部門別、業務員、客戶名稱、訂單號、料號、訂單金額、預計出貨月、出貨狀態）。
  - 訂單金額欄需為數值型（不含逗號或貨幣符號），避免 SQL 彙總計算失效。

二、模型選擇建議
  - Claude 4.6 Sonnet。此題需多維度分組加總 + 趨勢摘要撰寫，Sonnet 表現穩定。
  - 若資料筆數超過 500 筆，建議確認 EgentHub SQL 模式的查詢效能，必要時分表或加索引。

三、兩階段隔離設定
  - 本 Agent 為單階段，無需設定。

四、權限與工作室設定
  - 掛在「業務處工作室」，可見群組：業務主管及業務管理處核心成員。
  - 訂單金額屬敏感財務資料，建議不開放給一般業務員，僅主管層級可查詢彙整結果。
  - 各業務員可查詢自己部門的詳細明細，但跨部門彙整報告僅主管可見。

五、上線前驗測 Checklist
  - [ ] 詢問「請給三個月出貨排程彙整」→ 應輸出北/南/中/海外四部門 × 6/7/8 月金額表格，合計正確
  - [ ] 詢問「待排程的訂單有哪些？」→ 應列出出貨狀態為「待排程」的 6 筆訂單
  - [ ] 詢問「北區 7 月最大客戶是誰？」→ 應輸出緯創資通（PO-2025-0850，1,136,000 NTD）
  - [ ] 確認金額加總是否正確（手動計算各部門各月加總對比）

六、上線後監控指標（首 2 週）
  - 彙整金額準確率：與人工 Excel 合併結果對比，差異 < 1%
  - 業務主管每週使用次數：目標 ≥ 3 次（取代手工彙整）
  - 資料更新確認：每週確認 CSV 是否有成功自動更新

七、異常處理與回退方案
  - 金額彙整結果與預期不符：檢查 CSV 金額欄是否有非數值字元（如「,」或「NTD」），清洗後重建索引。
  - 部門別顯示不完整（如只有 3 個部門）：確認 CSV 中「部門別」欄的值是否與 Prompt 背景知識一致。
  - 緊急回退：業務主管人工合併各部門 Excel，待 Agent 修復後再切回。
