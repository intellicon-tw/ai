# 報價單辨識比對助理 - 正式上線前的準備指南

本文件提供將「報價單辨識比對助理」從 POC 套件正式部署到 EgentHub 平台、並在 採購 日常作業中穩定使用的逐步準備清單。建議由 IT/AI 種子團隊與該 Agent 的業務負責人共同檢核。

## 1. 知識庫設定

本 Agent 為純圖像辨識 + 文字整合，無需設定知識庫。需啟用兩階段隔離。

## 2. 模型選擇建議

- **首選模型**：Claude 4.6 Sonnet（視覺任務必用）
- **測試對照組**：Gemini 3 Flash（適合高頻輕量查詢，成本較低）
- **成本參考**：Claude 4.6 Sonnet 約 USD 0.003–0.008/次查詢（依 token 數）

## 3. 兩階段隔離設定

本 Agent 需啟用兩階段對話隔離。上線前請在 EgentHub Agent 設定頁確認「兩階段模式」已開啟。步驟一完成後，請確認 Agent 正確輸出「初步整理完成，若需進一步處理，請在下一步告訴我。」，步驟二確認以「我將基於上一步的整理結果」開頭。

## 4. 權限與工作室設定

- **工作室**：採購工作室
- **可見群組**：採購部門全員
- **算力點數初期建議**：500 點/月（約 100–200 次查詢，首月觀察後調整）
- **資料敏感度**：中（含內部規格與定價資料，建議限制部門內可見）

## 5. 上線前驗測 Checklist

執行 `3_驗測情境.txt` 中的所有情境，確認以下 pass/fail gate：

- [ ] **情境 #1（標準路徑）**：必須通過，輸出格式正確且無捏造資料
- [ ] **情境 #2（邊界案例）**：Agent 需正確處理不完整輸入，不得直接錯誤輸出
- [ ] **情境 #3（資料缺口）**：Agent 需提示使用者補充資訊，而非猜測填入

## 6. 上線後監控指標（首 2 週）

- 每日使用次數（目標：每天至少 3 次有效查詢，代表部門確實在用）
- 使用者主動回報的錯誤次數（目標：< 2 次/週）
- 異常回覆率（輸出格式錯誤或明顯錯誤）
- 點數消耗速率

## 7. 異常處理與回退方案

- 若 Agent 連續輸出格式異常：進後台查看 Chat Log，確認輸入是否偏離預期格式
- 若知識庫查詢命中率低：重新整理知識庫文件，增加語意多樣性
- 若 Agent 捏造資料：加強 Prompt 中的「勿捏造」指引，考慮加入明確的 fallback 指示
- 暫時下架條件：連續 3 次以上重大錯誤輸出，需下架重測

## 8. 種子團隊接手提示

本 Agent 適合處理格式差異較大的供應商報價單。如果供應商報價單格式固定，未來可考慮升級為串接 ERP API 的自動化流程（第二階段規劃）。
