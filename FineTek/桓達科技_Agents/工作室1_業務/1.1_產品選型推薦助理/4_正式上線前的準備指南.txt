# 產品選型推薦助理 - 正式上線前的準備指南

本文件提供將「產品選型推薦助理」從 POC 套件正式部署到 EgentHub 平台、並在 業務 日常作業中穩定使用的逐步準備清單。建議由 IT/AI 種子團隊與該 Agent 的業務負責人共同檢核。

## 1. 知識庫設定

### 1.1 桓達產品規格主檔（表格查詢知識庫）
- **來源檔案**：`2_知識庫範例資料/桓達產品規格主檔.csv`
- **知識庫類型**：表格查詢知識庫（SQL）
- **建議名稱**：「桓達產品規格主檔」
- **層級**：Agent 層級
- **欄位確認**：產品系列、型號、測量對象、感測原理、量程、輸出訊號、防爆認證、IP等級、適用溫度、適用壓力、連接規格、參考售價
- **正式資料替換**：上線前由業務助理從產品部取得最新型錄資料，轉為 CSV 格式（utf-8-sig），建議每季更新一次

## 2. 模型選擇建議

- **首選模型**：Claude 4.6 Sonnet
- **測試對照組**：Gemini 3 Flash（適合高頻輕量查詢，成本較低）
- **成本參考**：Claude 4.6 Sonnet 約 USD 0.003–0.008/次查詢（依 token 數）

## 3. 兩階段隔離設定

本 Agent 為單階段執行，無須額外設定。

## 4. 權限與工作室設定

- **工作室**：業務工作室
- **可見群組**：業務部門全員
- **算力點數初期建議**：500 點/月（約 100–200 次查詢，首月觀察後調整）
- **資料敏感度**：中（含內部規格與定價資料，建議限制部門內可見）

## 5. 上線前驗測 Checklist

執行 `3_驗測情境.txt` 中的所有情境，確認以下 pass/fail gate：

- [ ] **情境 #1（標準路徑）**：必須通過，輸出格式正確且無捏造資料
- [ ] **情境 #2（邊界案例）**：Agent 需正確處理不完整輸入，不得直接錯誤輸出
- [ ] **情境 #3（資料缺口）**：Agent 需提示使用者補充資訊，而非猜測填入

## 6. 上線後監控指標（首 2 週）

- 每日使用次數（目標：每天至少 3 次有效查詢，代表部門確實在用）
- 使用者主動回報的錯誤次數（目標：< 2 次/週）
- 異常回覆率（輸出格式錯誤或明顯錯誤）
- 點數消耗速率

## 7. 異常處理與回退方案

- 若 Agent 連續輸出格式異常：進後台查看 Chat Log，確認輸入是否偏離預期格式
- 若知識庫查詢命中率低：重新整理知識庫文件，增加語意多樣性
- 若 Agent 捏造資料：加強 Prompt 中的「勿捏造」指引，考慮加入明確的 fallback 指示
- 暫時下架條件：連續 3 次以上重大錯誤輸出，需下架重測

## 8. 種子團隊接手提示

本 Agent 最常見的改善需求是「加入更多篩選維度」（例如加入價格區間篩選）。若客戶產品線擴充，記得同步更新規格主檔 CSV。與「客戶報價草稿助理」有上下游關係，選型結果可直接作為報價草稿的輸入。
