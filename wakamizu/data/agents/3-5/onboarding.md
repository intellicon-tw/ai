# AI內容飛輪引擎 - 正式上線前的準備指南

本指南協助若水金禾相關部門在 EgentHub 上線此 Agent 前,完成資料準備、提示詞校對、驗測與權限設定。

---

## 一、知識庫資料準備

1. **資料流串接規劃**(6-12 個月)
   - YouTube Analytics API
   - IG/FB Graph API
   - 電商後台 API(目前是 Shopify? Shopline? 待確認)
   - UTM 追蹤標準化

2. **歸因模型建立**(3-6 個月)
   - 與資料分析顧問共同設計
   - 至少 6 個月的對照數據

3. **資料治理**
   - 資料權限分級
   - 隱私合規(GDPR、個資法)

【提醒】此 Agent 為 ROF 階段任務,建議在 ROE/ROI 階段成熟後啟動,避免初期資源分散。

## 二、提示詞校對重點

- 歸因模型限制是否清楚標註
- 資料缺口是否誠實提示
- 商業建議是否扣到實際數據

## 三、驗測通過標準

- 至少 6 個月歸因資料對照
- 行銷統籌與品牌經理共同認可建議品質
- 至少 3 個建議經實作後成效驗證

## 四、EgentHub 平台設定建議

- 模型:Claude 4.6 Sonnet
- 知識庫:表格(主)+ 工具串接
- 權限:行銷統籌、品牌經理、執行長
- 算力配額:依資料量擴充

## 五、第一週試運轉觀察重點

- 此 Agent 上線時間預估在 ROE/ROI 完成後 12 個月
- 上線初期僅供管理層使用,逐步開放給行銷團隊

## 六、風險提醒與避坑建議

- **資料隱私**:電商個資與內容互動資料整合需符合個資法
- **歸因偏誤**:模型本身有限制,建議多模型對照
- **過度自動化**:創意決策仍需人類判斷,Agent 為輔助
