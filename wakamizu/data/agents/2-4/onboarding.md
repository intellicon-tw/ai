# 產品製造知識傳承系統 - 正式上線前的準備指南

本指南協助若水金禾相關部門在 EgentHub 上線此 Agent 前,完成資料準備、提示詞校對、驗測與權限設定。

---

## 一、知識庫資料準備

1. **長期訪談計畫**(12-18 個月)
   - 規劃資深師傅(製茶 / 烘豆 / 茶道)系列訪談
   - 每位師傅至少 5-10 次訪談
   - 每次訪談錄音、錄影,並書面結構化整理

2. **製程影片字幕化**
   - 既有製程紀錄影片配字幕,標記關鍵時間軸

3. **配方決策歷史**
   - 每次配方調整的決策邏輯與結果回饋

4. **授權機制**
   - 取得師傅書面同意,訪談內容可用於 AI 知識庫

【提醒】此 Agent 適合在 ROE 與 ROI 階段建立基礎能力後,再啟動長期計畫,初期不建議直接上線。

## 二、提示詞校對重點

- 是否會把「系統推論」誤標為「師傅原話」
- 是否清楚標註知識庫不足的議題
- 配方相關回應是否謹守機密

## 三、驗測通過標準

- 知識庫累積 6 個月以上、至少 30 篇訪談後再上線
- 由資深師傅本人檢視 Agent 回應是否符合其經驗
- 新進人員試用 3 個月,對訓練效果有顯著幫助

## 四、EgentHub 平台設定建議

- 模型:Claude 4.6 Sonnet(需高品質推理)
- 知識庫:Agent 向量(主)
- 權限:Rootopia Brew 研發團隊、品管組、新進人員(限部分內容)
- 算力配額:每月 30,000 點(後期可擴增)

## 五、第一週試運轉觀察重點

- 此 Agent 上線時間預估在 ROE 階段完成 12-18 個月後
- 上線初期僅開放給研發團隊內部使用
- 收集師傅與新進人員的雙向回饋

## 六、風險提醒與避坑建議

- **配方與工藝為核心資產**:Agent 回應僅限內部,絕不對外展示
- **訪談授權**:師傅書面同意是必要前提
- **退化風險**:若過度依賴 AI,新進人員可能失去親近師傅學習的機會,建議 AI 為輔、師傅為主
