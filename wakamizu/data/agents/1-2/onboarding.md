# 茶園履歷與溯源助理 - 正式上線前的準備指南

本指南協助若水金禾相關部門在 EgentHub 上線此 Agent 前,完成資料準備、提示詞校對、驗測與權限設定。

---

## 一、知識庫資料準備

1. **12 座茶園基本資料整理**(2 週)
   - 用 CSV 範本(已附),補充每一筆完整資訊
   - 認證影本、合作合約存檔

2. **茶農訪談**(4-6 週)
   - 每座茶園至少 1 次訪談,30 分鐘以上
   - 整理為結構化文件,經茶農確認後方可入庫

3. **製程紀錄**
   - 每年春摘、夏摘、冬摘批次紀錄
   - 異常事件(如氣候、病蟲害)需特別標記

4. **匯入 EgentHub**:基本資料用表格知識庫;訪談、故事用 Agent 向量知識庫

## 二、提示詞校對重點

- 行銷文案的字數控制是否準確
- 檢查回應是否會誤把「合作起始年」當成「茶園創園年」
- 多茶園比較時欄位是否一致

## 三、驗測通過標準

- 3 種情境全部通過
- Rootopia Brew 品牌經理試用 1 週,可獨立產出 3 款產品包裝故事
- 行銷小編可在 30 分鐘內產出符合品牌的官網產地頁文案

## 四、EgentHub 平台設定建議

- 模型:Claude 4.6 Sonnet
- 知識庫:表格 + Agent 向量(混合檢索)
- 權限:Rootopia Brew 品牌團隊、外銷業務、行銷小編、根本在旅行內容團隊
- 算力配額:每月 80,000 點

## 五、第一週試運轉觀察重點

- 觀察行銷文案是否會誇大(如自封「最高品質」「全台第一」)
- 是否會錯誤連結茶園與產品(如把鹿谷茶園資料用到阿里山產品)
- 收集前 30 篇文案,由 Rootopia Brew 品牌經理校對後再對外

## 六、風險提醒與避坑建議

- **品牌調性失準**:文案若太浮誇會傷品牌,初期所有對外文案都需人工審核
- **資料正確性**:茶園資料若有錯誤,影響溯源可信度,建議每年復核
- **茶農授權**:訪談內容用於行銷需取得茶農書面同意
