# 新品開發知識庫助理 - 正式上線前的準備指南

本指南協助若水金禾相關部門在 EgentHub 上線此 Agent 前,完成資料準備、提示詞校對、驗測與權限設定。

---

## 一、知識庫資料準備

1. **過往配方紀錄整理**(3-4 週)
   - 三年內所有開發案,含未量產配方的失敗原因
   - 配方代號標準化

2. **市場測試報告**
   - 門市試喝、展會、線上問卷的原始資料
   - 結構化為「測試方法、樣本數、結果、學習」

3. **消費者回饋彙整**
   - 從電商評論、客服紀錄、會員問卷整理
   - 標註關鍵詞(風味、價格、包裝、設計)

4. **競品資料**
   - 日本、台灣、美國競品的定價、定位、行銷策略

## 二、提示詞校對重點

- 是否會混淆配方代號
- 評估時是否會過度樂觀
- 競品分析是否引用真實資料

## 三、驗測通過標準

- 3 個情境通過
- Rootopia Brew 研發團隊試用 4 週,3 個新品提案經 Agent 評估後上 RD 排程
- 評估報告經品牌經理認可

## 四、EgentHub 平台設定建議

- 模型:Claude 4.6 Sonnet
- 知識庫:Agent 向量(主)
- 權限:Rootopia Brew 研發團隊、品牌經理、行銷統籌
- 算力配額:每月 30,000 點

## 五、第一週試運轉觀察重點

- 是否會推薦過於小眾、商業可行性低的方向
- 歷史失敗配方是否被誠實引用
- 是否會「政治性地」迎合提案者

## 六、風險提醒與避坑建議

- **配方機密**:配方屬核心資產,Agent 回應限內部研發團隊
- **市場誤判**:Agent 評估僅供參考,實際上市決策需綜合判斷
- **競品資料時效**:競品定價會變動,需定期更新
