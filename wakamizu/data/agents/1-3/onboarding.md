# 生產排程與品管助理 - 正式上線前的準備指南

本指南協助若水金禾相關部門在 EgentHub 上線此 Agent 前,完成資料準備、提示詞校對、驗測與權限設定。

---

## 一、知識庫資料準備

1. **配方表標準化**(2 週)
   - 14 支產品的完整配方,含原料代號與比例
   - 標準工時與單批產能上下限

2. **原料庫存表**
   - 即時更新原料數量(建議每週更新)
   - 設定安全庫存與供應商交期

3. **代工廠資料**
   - 兩家代工廠的產能、聯繫窗口、合約有效期

4. **品管紀錄歷史**(過去 12 個月)
   - 異常事件、處理結果、改善追蹤

## 二、提示詞校對重點

- Agent 是否會誤判「庫存夠用」(注意安全庫存)
- 異常處理建議是否與既定 SOP 一致
- 跨產品共用原料時計算是否正確

## 三、驗測通過標準

- 3 個情境全部通過
- 生產統籌試用 2 週,排程準確率 95% 以上
- 異常處理建議經品管組確認可採用

## 四、EgentHub 平台設定建議

- 模型:Claude 4.6 Sonnet
- 知識庫:表格(配方/庫存/代工)+ 向量(SOP)
- 權限:Rootopia Brew 品牌經理、生產統籌、品管組、代工廠協調窗口
- 算力配額:每月 60,000 點

## 五、第一週試運轉觀察重點

- 觀察是否真實反映原料庫存變動
- 排程建議是否被生產統籌實際採用
- 收集排程實際與建議的差異,持續調校

## 六、風險提醒與避坑建議

- **資料時效性**:原料庫存需即時更新,否則建議失準
- **代工廠關係**:Agent 不應替代與代工廠的人為協商
- **品管權責**:品管異常的最終判定必須由品管組決定
