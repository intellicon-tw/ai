四項設計原則 — 您觀察 Agent 行為時的依據

原則一：流程優先於模型

先定義 GM 業務該怎麼判斷，AI 才介入輔助。提示詞與工具都是流程的延伸，不是流程的核心。

原則二：資料確認是硬閘門

關鍵資料未確認前，AI 不分析、不推估、不下結論。寧可主動回頭要求補資料，也不容許「看似完整但根據不足」的輸出污染決策。

原則三：判斷邏輯與事實資料分離

判斷規則內建於 Agent 行為規範；客戶與產業資料存於知識庫。讓兩者各自演化，避免知識庫變動時必須改提示詞。

原則四：人機協作，不是全自動

AI 的責任是加速、結構化、指出不確定性。最終商機判斷與報價決策仍由業務或業務主管執行。AI 是夥伴，不是取代。

2POC Agent 設計理念

理解這四個 Agent 為什麼這樣拆分、為什麼要有資料閘門、為什麼不做成一個「萬能 Agent」 — 這些設計決策背後都有刻意的考量。

為什麼是 4 個 Agent，而不是 1 個「萬能 Agent」

把工作拆成 4 個專責 Agent，是刻意的設計決策。每個 Agent 都有單一職責、可獨立開發與測試、可獨立優化。這樣做的具體好處有四個：

  • 失敗範圍可控 — 某個 Agent 表現不好不會影響其他 Agent，回歸驗測也只需針對特定 Agent
  • 業務有彈性的進入點 — 從研究素材進來就用 1.1，從詢價單進來就用 1.3，業務不必硬走完整流程
  • 輔助型 + 核心型分工 — 輔助型 Agent（1.1、1.3）處理「結構化、解析、標準化」這類雜事，核心型 Agent（1.2、1.4）專心做最關鍵的判斷與決策
  • 培訓與交接更容易 — 種子團隊可以一個一個 Agent 學會，不必一次掌握整個複雜流程

為什麼有「資料閘門」設計

傳統的 LLM Agent 在資料不完整時，會「強行給看似專業實則根據不足的答案」。對閒聊也許還好，但對商機判斷與報價決策，這種輸出是有害的 — 它會讓業務拿著錯誤的依據去做承諾。

資料閘門設計把「業務判斷的最後一道防線」內建到 Agent 行為中：

  • 每次跑 SOP 時，先評估「資訊充分度」（0-100 分）
  • 分數低於 60 時，Agent 不給結論，只列出資料缺口清單
  • 業務知道要補什麼資料才能繼續，而不是收到一份漂亮但不可信的報告
  • 副作用：稽核 AI 是否「不亂推論」變得可量化（資料閘門退回率）

這個設計直接對應原則二「資料確認是硬閘門」 — 寧可主動回頭要求補資料，也不容許「看似完整但根據不足」的輸出污染決策。

為什麼採用 Phase 1-7 結構化 SOP

把資深業務的判斷邏輯轉成 7 個明確的 Phase 步驟，是為了：

  • 讓判斷邏輯被組織保存 — 不再藏在資深業務腦中，可被新人繼承、可被 AI 放大
  • 每個 Phase 都有明確輸入輸出，方便驗測（例如：Phase 4 的輸出是否引用了 KB 案例編號）
  • 新人也能產出水準一致的初稿，再由資深業務做最後校對
  • 商機分析師（1.2）與報價策略分析師（1.4）的 Phase 結構刻意對稱（都是 Phase 1 需求定義 → Phase 2 資料閘門 → Phase 3 結構化 → Phase 4-6 核心戰場 → Phase 7 輸出），讓組織學習成本降到最低

為什麼採用「鬆耦合」串接（Markdown 中介產物）

1.1 → 1.2、1.3 → 1.4 的串接，刻意採用 Markdown 文字作為中介產物，而不是 API 直連。這樣做的好處：

  • 業務直接複製貼上即可串接，不需特殊框架支援
  • 可中斷可重啟 — 業務跑完 1.1 後可以隔天再用 1.2，輸出文字保留即可
  • 未來可替換訊號層工具 — 例如從 Autonomous Agent 換成其他工具，只要產出格式類似的研究摘要，1.2 商機分析師完全不用改
  • 可審計 — 每個 Agent 的輸入輸出都是純文字，方便 Code Review 與稽核追蹤

為什麼用「條件式兩階段」而非「全部兩階段」

EgentHub 的雙階段隔離設計是為了避免「同時讀圖與調用工具」造成的判斷品質下降。但對 1.1 與 1.3 這類「有時讀 PDF、有時純文字」的 Agent，全部都走兩階段反而拖慢純文字輸入時的回應速度。所以採用條件式設計：

  • 上傳 PDF 或圖片時：先進步驟一（純讀圖、不調 KB），再進步驟二（呼叫 KB、產出結果）
  • 純文字輸入時：直接走步驟二，省掉不必要的階段切換

歷史報價成交資料表　（16 行，含 3 筆進行中）

用途：找相似詢價案件、估算勝率、查歷史落差%。

欄位：詢價單號（RFQ-XXXX-XXXX）、客戶名稱、客戶等級（KA/一般/政府）、產品型號、數量、報價金額NTD、落單金額NTD、勝負、成交價落差(%)、競爭對手、交期(週)、案件年月

內容範圍：2023-2025 共 16 筆歷史報價（10 勝、3 失、3 進行中）。客戶涵蓋上銀、台積電、中油、鴻海、統一、中鋼、永信、台達、南亞、華新麗華、新北汙水廠、台塑、鴻海高雄半導體、彰化離岸風場。競爭對手橫跨東芝、ABB、Siemens、WEG、Delta、SEW、Flender、Mitsubishi、Yaskawa、Howden、Schneider、Vestas。

BOM 標準成本表　（15 行）

用途：取得各標準型號的 BOM 拆解、標準成本、建議底價、標準毛利率，作為三檔報價的成本基礎。

欄位：產品型號、銅(NTD)、矽鋼片(NTD)、其他材料(NTD)、加工費(NTD)、銷管攤銷(NTD)、標準成本(NTD)、建議底價(NTD)、標準毛利率

內容範圍：15 個標準型號的成本結構，含主要原料明細（銅料、矽鋼片、其他材料）。標準毛利率區間 12.5% 至 20.8%（馬達類偏低、變頻器與伺服類偏高）。
