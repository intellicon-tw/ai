原則：能用 Dify 原生／外掛就不自己寫程式。發報／警報＝原生 HTTP 請求節點直接打 LINE／Telegram。記憶＝用已安裝的 google_sheets 外掛讀寫一張 Google Sheet——讀用 google_sheets(Batch Get)、寫用 google_sheets(Batch Update)，免 Apps Script、免 OpenAPI。唯一外接就只剩「那張 Sheet」本身。 Principle: if Dify or a plugin can do it, don't write code. Report / alert = a native HTTP Request node to LINE / Telegram. Memory = the installed google_sheets plugin reading/writing one Google Sheet — read with google_sheets(Batch Get), write with google_sheets(Batch Update); no Apps Script, no OpenAPI. The only external piece is the Sheet itself.

  • 1) 發報／警報（原生，免後端）：用 Dify 原生 HTTP 請求節點 POST 到 LINE Notify／Telegram Bot 的網址即可。1) Report / alert (native, no backend): use Dify's built-in HTTP Request node to POST to a LINE Notify / Telegram Bot URL.
  • 2) 設定 google_sheets 外掛憑證：GCP 建專案、啟用 Google Sheets API、建 service account 下載 JSON 金鑰；把目標 Sheet「共用」給該 service account 的 email；在 Dify 外掛設定貼上 JSON。2) Configure the google_sheets plugin: in GCP create a project, enable the Google Sheets API, create a service account and download its JSON key; share the target Sheet with the service account's email; paste the JSON into the plugin settings in Dify.
  • 3) 開工第一步——讀「昨天的資料」：用 google_sheets（Batch Get） 把昨天那張表整個讀進來，當作今天比價的基準。3) First step each run — read "yesterday's data": use google_sheets (Batch Get) to pull in yesterday's whole sheet as today's comparison baseline.
  • 4) 收工最後一步——存「今天的資料」：用 google_sheets（Batch Update） 把今天抓到的數據寫回那張表、存成新的一列，明天才有得比。4) Last step each run — save "today's data": use google_sheets (Batch Update) to write today's data back to the sheet as a new row, so tomorrow has something to compare against.
結果：外接只剩「一張 Sheet」，且讀寫都用現成外掛、零自寫程式。Result: the only external piece is one Sheet, and even reading/writing uses a ready-made plugin — zero custom code. 搜尋、抓頁、發報、警報、記憶全在 Dify 內完成。換成 Hermes，連這張 Sheet 都省（記憶內建）——這就是兩者的差距。Search, scraping, report, alert and memory all happen inside Dify. On Hermes even this Sheet is unnecessary (memory is built in) — that's the gap between the two.

// 備案 — Google Apps Script 當「記憶」後端：讀昨日基準 / 寫今日數據
// 部署：部署 → 新增部署作業 → 類型「網頁應用程式」→ 存取權限「任何人」→ 取得 /exec 網址
const SHEET_ID = 'YOUR_SHEET_ID';

function doGet(e){            // 讀昨日基準
  const sh = SpreadsheetApp.openById(SHEET_ID).getSheetByName('baseline');
  return json({ ok:true, baseline: sh.getDataRange().getValues() });
}
function doPost(e){           // 寫今日數據
  const p = JSON.parse(e.postData.contents || '{}');
  SpreadsheetApp.openById(SHEET_ID).getSheetByName('snapshot')
    .appendRow([new Date(), JSON.stringify(p.data || {})]);
  return json({ ok:true });
}
function json(o){ return ContentService.createTextOutput(JSON.stringify(o))
  .setMimeType(ContentService.MimeType.JSON); }

近 24 小時討論；東元負面輿情一律列入；補助政策動態一併查。

## 六、鐵則
1. 絕不編造價格、排名、輿情；抓不到就註明「未能取得」。
2. 每筆數據附來源 URL 與擷取時間。
3. 牌價與券後價分開記錄。
4. 事實與判讀分開，判讀以「研判：」開頭。
5. 遇反爬蟲記錄失敗即可，不要規避。
6. 4–9 月空調旺季，加密監控頻率與促銷追蹤深度。
7. 繁體中文，技術術語保留英文。

動手練習Hands-On Exercises

練習 1｜讀懂檢核清單（15 分鐘）Exercise 1 | Read the Checklist (15 min)

打開「Dify」分頁的系統提示詞，找出：哪三個段落是在「補 Dify 平台的洞」？如果把這三段刪掉，Agent 會發生什麼事？Open the system prompt on the "Dify" tab and find: which three sections exist only to "patch Dify's gaps"? What would happen to the Agent if you deleted them?

練習 2｜找出兩版差異（15 分鐘）Exercise 2 | Spot the Differences (15 min)

把兩版 Prompt 並排，圈出 Hermes 版「沒有」的內容。想一想：每一處消失的段落，對應 Hermes 的哪一個內建能力？Put the two prompts side by side and circle what the Hermes version does not have. For each missing section, ask: which built-in Hermes capability does it correspond to?

練習 3｜改成你的題目（30 分鐘）Exercise 3 | Make It Your Own (30 min)

複製任一版 Prompt，把監控對象換成你部門負責的產品線（例如商用空調經銷報價、冷凍冷藏設備標案）。需要改的四個區塊：監控範圍（通路/品牌/品類）、報告格式（你的主管想看什麼）、警報條件（什麼事不能等到明天）、鐵則（你的領域裡 AI 最容易出錯的地方）。改完先在 Dify 手動跑一次驗證。Copy either prompt and swap the monitoring target for the product line your department owns (for example, commercial-HVAC dealer quotes or refrigeration-equipment tenders). Four blocks need editing: monitoring scope (channels / brands / categories), report format (what your manager wants to see), alert conditions (what cannot wait until tomorrow), and the rules (where AI is most likely to go wrong in your field). Once edited, run it manually in Dify once to validate.

建議路徑：Recommended path: 先在 Dify 用兩個通路跑一個月 POC，淬鍊出穩定的 SOP 與提示詞 → 驗證有價值後，把指令改寫成任務書，部署到 Hermes 長駐執行。Prompt 的功力是帶得走的，平台只是宿舍。Start with a one-month POC in Dify covering two channels to refine a stable SOP and prompt. Once it proves valuable, rewrite the instructions as a task brief and deploy it on Hermes for persistent execution. Prompt skill is portable — the platform is just the dormitory.
