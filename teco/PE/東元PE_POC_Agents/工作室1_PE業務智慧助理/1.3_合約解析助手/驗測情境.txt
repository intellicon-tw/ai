# 合約解析助手 - 驗測情境

以下 3 個情境，作為手動或半自動回歸驗測的依據。

## 情境 #7：完整合約結構化

**輸入描述**

> 聯電嘉義廠太陽能 EPC 合約 v2（87 頁、5 個附件齊全）

**預期輸出**

> 文件完整性檢查通過；條款分類為四類，每條註明頁碼；產出合約摘要約 200 字；引導使用者交給合約問答助手

**驗證重點**

> 1) 步驟一不調 KB；2) 條款分類齊全；3) 頁碼標註

## 情境 #8：文件完整性閘門啟動

**輸入描述**

> 南亞塑膠 ESS 合約：附件 C 未上傳

**預期輸出**

> Agent 明確告知「附件 C『售後維護細則』未上傳，無法進行完整解析」；列出缺漏附件清單；要求補上後再進入下一步

**驗證重點**

> Agent 未強行解析殘缺合約，啟動閘門

## 情境 #9：純文字輸入請求上傳

**輸入描述**

> 「我有一份合約想看一下」（未上傳檔案）

**預期輸出**

> Agent 婉轉請求使用者上傳合約 PDF，告知本 Agent 處理 PDF 的能力範圍

**驗證重點**

> Agent 不憑空編造合約內容
