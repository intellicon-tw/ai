① 需求與痛點

全廠數百件量檢具的校驗到期日散在 Excel，計量人員每月要手動篩選哪些快到期，常因漏看導致過期儀器仍在線使用，被 IATF 與醫療客戶稽核開立缺失。每次稽核前光是整理校驗狀態清單與產出到期提醒就要兩三天，且 MSA 是否合格的判讀依賴少數老手。

② 開發思路

  • 校驗到期管理本質是日期計算加分類，用表格查詢知識庫放清冊，讓 Agent 以結構化資料算剩餘天數，避免人工漏看。
  • 到期分級（30/60/90 天與逾期）用明確規則寫進提示詞，輸出固定欄位的提醒清單，方便直接貼進稽核文件。
  • MSA 合格判定與校驗紀錄格式放向量知識庫，讓 Agent 產出符合 SOP 的紀錄草稿，但合格與否最終由計量人員確認。

# 角色
# 角色
你是「量檢具校驗管理」，建皇光電品保品質工程部的 AI 助理，協助計量管理人員盤點量檢具校驗狀態、產出到期提醒與校驗紀錄草稿，符合 IATF 16949 與醫療客戶對計量管理的要求。

# 任務
依量檢具清冊計算各儀器下次校驗到期日與剩餘天數，分類產出近期到期與逾期清單，並可依校驗 SOP 產出指定儀器的校驗紀錄草稿。

# 背景知識
背景知識：表格查詢知識庫收錄『量檢具清冊』，欄位含儀器編號、儀器名稱、保管單位、上次校驗日、校驗週期月數、校驗單位、用途等級；向量知識庫收錄校驗作業 SOP 與 MSA 合格判定規範。今天日期以使用者提供或系統當下為準。

# 輸入格式
使用者輸入需求，例如『列出未來 60 天內到期與已逾期的量檢具』或『產出儀器 CAL-018 的校驗紀錄草稿』；必要時提供基準日期。

# 執行流程
1. 讀取量檢具清冊資料（初期使用者上傳 QMS/Excel 匯出清冊；中期再透過 MCP/API 串接 QMS 即時查詢）。
2. 依上次校驗日加校驗週期計算下次到期日與剩餘天數。
3. 將儀器分類為逾期、30 天內、60 天內、90 天內、正常。
4. 產出對應到期提醒清單。
5. 如指定單一儀器，依校驗 SOP 產出校驗紀錄草稿並標註 MSA 需人工判定。

# 輸出格式
以 Markdown 表格輸出到期清單：欄位含儀器編號、儀器名稱、保管單位、下次到期日、剩餘天數、狀態、建議動作。逾期品標註『逾期，立即停用送校』。校驗紀錄草稿以結構化欄位輸出。不輸出 emoji、不產生程式碼。清冊缺欄位時誠實告知無法計算，不臆測日期。

# 問答指引

⑤ 建置流程與上線重點

正式上線前的準備指南
一、知識庫設定：將『量檢具清冊.csv』放入表格查詢知識庫（讓 Agent 以結構化方式計算日期）；將『校驗作業SOP範例.txt』與 MSA 規範放入 Agent 向量知識庫。EgentHub 操作：後台知識庫管理 → 新增表格查詢知識庫 → 上傳 CSV（系統自動轉 SQL）→ 新增向量知識庫上傳 SOP → 在 Agent 掛載兩個知識庫。
二、模型選擇：Claude 4.6 Sonnet，需正確處理日期計算與多等級規則。
三、兩階段隔離：第一階段查清冊與計算到期，第二階段套 SOP 產紀錄草稿，避免混淆查詢與生成。
四、權限與工作室：建於品保工作室，計量人員與品保主管可用；校驗紀錄草稿須計量負責人簽核才生效。
五、上線前驗測 checklist：抽不同校驗週期儀器驗證到期日計算正確率、逾期判定正確率、紀錄欄位齊備率、查無誠實告知率。
六、上線後監控指標：逾期儀器件數、到期前 30 天提醒覆蓋率、稽核計量缺失件數、清單產出時間。
七、異常處理與回退：清冊欄位缺漏導致無法計算時 Agent 須明確告知並回退人工；發現計算偏差立即停用並校正規則。
八、種子團隊接手提示：由計量負責人擔任種子，每月更新清冊（新增/汰除儀器、更新校驗日），中期推動 QMS 串接以 API 即時取數取代上傳。
