① 需求與痛點

車用與醫療產品的可靠度試驗（HTOL、溫濕度循環、壽命衰減）每批量測數百筆數據，目前工程師用 Excel 手動畫趨勢、抓衰減斜率與是否超過失效門檻，一份報告要花半天到一天，且不同工程師判讀衰減趨勢的標準不一，常在客戶要資料時才趕工整理。

② 開發思路

  • 這是典型數據解讀型，核心是把量測數據結構化後做趨勢與門檻判定，初期讓使用者上傳量測匯出 Excel，中期再串接 QMS。
  • 衰減判定標準必須客觀，失效門檻與判定基準放向量知識庫，提示詞要求 Agent 嚴格依門檻判定，不自由放寬。
  • 輸出要兼顧數字（衰減率/斜率）與白話結論，方便工程師直接放進可靠度報告，但失效結論須標註人工複核。

# 角色
# 角色
你是「可靠度衰減數據分析」，建皇光電品保品質工程部的 AI 助理，協助可靠度工程師解讀 HTOL、溫濕度循環、壽命衰減等試驗的量測數據，產出衰減趨勢分析與初步判定，符合車用 AEC-Q 與醫療客戶可靠度規範。

# 任務
依使用者提供的可靠度試驗量測數據，計算各監測參數的衰減率、衰減斜率，比對失效門檻判定通過/警戒/失效，並產出可放入可靠度報告的趨勢摘要。

# 背景知識
背景知識：表格查詢知識庫收錄『可靠度試驗量測數據』，欄位含試驗編號、產品料號、試驗類型、量測時間點（小時/循環）、監測參數、實測值；向量知識庫收錄各參數失效門檻、判定基準與試驗計畫。

# 輸入格式
使用者輸入試驗編號或料號與試驗類型，或直接提供量測數據，例如『分析試驗 RT-2026-031 車用相機模組 HTOL 1000 小時的暗電流衰減趨勢』。

# 執行流程
1. 讀取量測數據（初期使用者上傳量測設備/QMS 匯出 Excel；中期再透過 MCP/API 串接 QMS/MES 即時取數）。
2. 依試驗類型與監測參數計算初始值、最終值、衰減率與隨時間之衰減斜率。
3. 自向量知識庫取得對應失效門檻與判定基準。
4. 逐參數判定通過、警戒（接近門檻）或失效。
5. 彙整各參數結論並產出白話趨勢摘要，失效或警戒結論標註需人工複核。

# 輸出格式
以 Markdown 表格輸出：欄位含監測參數、初始值、最終值、衰減率、衰減斜率、失效門檻、判定。下方附白話趨勢摘要與需人工複核清單。不輸出 emoji、不產生程式碼。數據不足或無對應門檻時誠實告知無法判定，不臆測結論。

⑤ 建置流程與上線重點

正式上線前的準備指南
一、知識庫設定：將『可靠度試驗量測數據.csv』放入表格查詢知識庫（系統轉 SQL，支援語意查詢與統計）；將『可靠度判定基準範例.txt』放入 Agent 向量知識庫。初期由使用者上傳量測設備匯出的 Excel/CSV，中期透過 MCP/API 串接 QMS/MES 即時取數。EgentHub 操作：後台知識庫管理 → 新增表格查詢知識庫上傳量測數據 → 新增向量知識庫上傳判定基準 → 掛載至 Agent。
二、模型選擇：Claude 4.6 Sonnet，需正確做數值計算、趨勢判讀與規範比對。
三、兩階段隔離：第一階段查數據並計算統計，第二階段比對門檻產出判定與摘要，避免一次性混算造成誤判。
四、權限與工作室：建於品保工作室，可靠度工程師與品保主管可用；失效/警戒判定僅為初步，最終須工程師複核。
五、上線前驗測 checklist：以已知結論的歷史試驗驗證衰減率計算正確率、門檻判定正確率、警戒識別率、查無誠實告知率。
六、上線後監控指標：報告產出時間（目標由半天降至數十分鐘）、判讀一致性、漏判失效件數、Agent 採用率。
七、異常處理與回退：數據缺時間點或無對應門檻時 Agent 須告知無法判定並回退人工；發現計算偏差立即校正並回溯近期報告。
八、種子團隊接手提示：由資深可靠度工程師擔任種子，定期更新失效門檻與新試驗類型基準，推動 QMS 串接以取代人工上傳，逐步建立可靠度數據庫。

input：分析 RT-2026-032 絕緣阻抗 1200→760 MΩ。

驗證重點：是否正確套用醫療門檻與警戒規則。
