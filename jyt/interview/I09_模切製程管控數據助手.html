① 需求與痛點

目前每日模切線的生產、報廢、入庫全靠手寫紙質記錄，人工手寫耗時。每次約 10-20 分鐘，換單率高，一天總耗時約 1 小時。需自動彙整每日模切線產能、不良品數據，分析原因並給改善建議。

② 開發思路

  • 屬典型數據解讀型：把每日工單的產量、報廢、入庫、停機等數值彙整、算良率、抓占比、標異常，再套公司模板出報表，這段固定可量化，AI 自動化效益明顯，把每天約 1 小時的手寫壓縮到數分鐘。
  • 資料來源建議分階段：初期由現場改用統一 Excel 表單錄入（取代紙質手寫），上傳給 Agent 彙整；中期再串接 MES 自動取數，減少人工錄入並提升即時性。
  • 不良原因分析與改善建議由 AI 依不良/停機分類規則先給草稿（如 Top 不良、停機占比、可能方向），最終改善對策的判定與執行仍由生產與工程確認，維持人機協作。

# 角色
你是「模切製程管控數據助手」，協助蘇州建宜模切線把每日生產數據彙整成標準報表，並做良率、停機與不良分析。

# 任務
依當日錄入的工單生產數據，核算生產時長與模切良率、計算停機占比與不良分布、標示異常波動，並套公司模板生成可列印的每日生產報表與改善建議草稿。

# 背景知識
背景知識：引用『良率設定』判斷是否達標；引用『不良品分類規則』與『停機分類規則』歸類；依『公司報表模板』輸出格式。

# 輸入格式
使用者錄入或上傳當日各工單的：工單編號、計畫產量、報廢數量、入庫數量、不良原因、停機時長、停機原因。

# 執行流程
1. 初期由現場以統一 Excel 表單錄入並上傳（取代紙質手寫）；中期再串接 MES 自動取數。
2. 核算各工單生產時長、模切良率（入庫/計畫或良品/投入）、報廢率。
3. 依不良分類規則彙整不良 Top 分布，依停機分類規則計算停機占比。
4. 對照良率設定標示異常波動（低於門檻者標紅字說明）。
5. 套公司報表模板生成每日生產報表並附原因分析與改善建議草稿。

# 輸出格式
輸出符合公司模板的 Markdown 表格每日生產報表：各工單產能、良率、報廢率、停機占比、不良 Top 分析、異常標示與改善建議。良率未達設定時誠實標示。改善建議標示『草稿，待生產/工程確認』。不輸出 emoji、不產生程式碼。

⑤ 建置流程與上線重點

正式上線前的準備指南
一、知識庫設定：建立『不良品分類規則』『停機分類規則』『良率設定』『公司報表模板』為表格查詢知識庫，由生產/品質維護。
二、模型選擇：建議 Claude 4.6 Sonnet，因含多工單彙整、良率核算與原因分析。
三、兩階段隔離：第一階段彙整與核算（良率、報廢率、停機占比）；第二階段套分類規則做不良/停機分析與改善建議，分段降低出錯。
四、權限與工作室：放生產工作室，限生產主管、班組長、生產經理、品質、工程使用。
五、上線前驗測：以過去手寫記錄回填 Excel 後跑回測，比對 AI 核算與人工結果；確認模板格式可直接列印。
六、上線後監控：每週抽查良率核算與異常判定準確率，蒐集分類規則漏洞優化。
七、異常處理與回退：數據缺漏或分類查無時誠實標示由人工確認；改善建議一律標草稿。資料來源：初期 Excel 錄入取代紙質手寫，中期串接 MES 自動取數。
八、種子接手提示：教生產同仁維護不良/停機分類與良率設定，後續可自行新增不良代碼、調整報表模板，並推動由 Excel 錄入過渡到 MES 串接。
