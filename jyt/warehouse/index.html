共 3 款 Agent POC部門 倉管／物料部

倉管部的三個題目都很務實，分別對應呆滯料、出貨品質與盤點差異三大痛點，但目前缺 WMS/ERP 匯出的庫存明細、標籤樣張與盤點差異紀錄。庫存呆滯料與帳料差異屬數據解讀型，初期先用 WMS/ERP 匯出 Excel 跑，中期串接 WMS 後可常態化；出貨包裝標籤檢核是讀圖比對，補幾張真實標籤照片與檢核規則就能驗證。建議先把一份庫存明細、幾張標籤照片與一次盤點差異表整理出來，POC 會很完整。以下每一頁對應一個需求，點進去可看到：需求與痛點 → 開發思路 → 完整提示詞 → 知識庫設計 → 建置流程 → 驗測情境。

#9.1

庫存呆滯料數據解讀

數據解讀型（Text-to-SQL）

倉管每月要盤點呆滯料報給財會與生管，目前從 WMS/ERP 匯出庫存明細，自己用 Excel 算庫齡、篩呆滯，每月花兩三小時，且庫齡分級口徑不一致，常漏掉快變呆滯的料，造成庫存金額與報廢風險難掌握。

 #9.2

出貨包裝標籤檢核

審查比對型（圖面比對）

出貨前作業員要核對包裝標籤上的料號、批號、數量、客戶與出貨地是否與出貨單一致，全靠目視，每天數十箱，趕貨時容易看漏，曾發生貼錯客戶標籤或數量不符被客戶退貨、扣分的情況。

 #9.3

帳料差異比對

數據解讀型（Text-to-SQL）

每次盤點後要把實盤量與帳面量逐料比對，找出差異料號、差異數與差異金額，再分類原因報給財會，目前用 Excel 人工比對，數百個料號花半天到一天，趕結帳時很吃緊，差異原因分類也常前後不一致。
